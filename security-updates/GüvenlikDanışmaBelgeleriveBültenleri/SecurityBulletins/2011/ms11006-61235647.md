Security Bulletin

Microsoft Güvenlik Bülteni MS11-006 - Kritik
============================================

Windows Kabuğu'nda Grafik İşleme Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (2483185)
--------------------------------------------------------------------------------------------------

Yayım Tarihi: 8 Şubat 2011 Salı | Güncelleştirme Tarihi: 14 Şubat 2011 Pazartesi

**Sürüm:** 1.1

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Windows Kabuğu grafik işleyicisindeki genel olarak duyurulan bir güvenlik açığını giderir. Bu güvenlik açığı, bir kullanıcı özel hazırlanmış bir küçük resmi görüntülerse uzaktan kod yürütülmesine olanak verebilir. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açan kullanıcı ile aynı haklara sahip olabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

Bu güvenlik güncelleştirmesi Windows XP, Windows Server 2003, Windows Vista ve Windows Server 2008'in tüm desteklenen sürümleri için Kritik olarak derecelendirilmiştir. Windows 7 ve Windows Server 2008 R2'nin desteklenen sürümleri bu güvenlik açığından etkilenmez. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, Windows Kabuğu grafik işlemcisinin küçük resimleri işleme biçimini düzelterek bu güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

Bu güvenlik güncelleştirmesi ilk olarak [Microsoft Güvenlik Danışma Belgesi 2490606](http://technet.microsoft.com/security/advisory/2490606)'da açıklanan güvenlik açığını da giderir.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi hemen uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** Yok

Windows Kabuğu'nda Grafik İşleme Sırasında Taşma Güvenlik Açığı - CVE-2010-3970  
-------------------------------------------------------------------------------
  
<span></span>
Windows Kabuğu grafik işlemcisinin özel hazırlanmış küçük resimleri işleme biçiminde bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açmış kullanıcının güvenlik bağlamında rasgele kod çalıştırabilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2010-3970](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3970).
  
#### Windows Kabuğu'nda Grafik İşleme Sırasında Taşma Güvenlik Açığı - CVE-2010-3970 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Güvenlik açığından, e-posta yoluyla otomatik olarak yararlanılamaz.  
-   Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açan kullanıcı ile aynı haklara sahip olabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
#### Windows Kabuğu'nda Grafik İşleme Sırasında Taşma Güvenlik Açığı - CVE-2010-3970 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Windows XP ve Windows Server 2003 tabanlı sistemlerde shimgvw.dll dosyasına uygulanan Erişim Denetim Listesi'ni (ACL) değiştirin**
  
    **Not** Bu geçici çözümü etkinleştirmek veya devre dışı bırakmak üzere otomatik **Microsoft Düzelt** çözümünü kullanmak için, bkz: [Microsoft Bilgi Bankası makalesi 2483185](http://support.microsoft.com/kb/2483185).
  
    shimgvw.dll dosyasına uygulanan ACL'yi daha kısıtlayıcı olacak şekilde değiştirmek için, komut isteminde yönetici olarak aşağıdaki komutları çalıştırın:
  
    **Windows XP ve Windows Server 2003'ün 32-bit sürümleri:**
  
    `Echo y| cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /P everyone:N`
  
    **Windows XP ve Windows Server 2003'ün 64-bit sürümleri:**
  
    `Echo y| cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /P everyone:NEcho y| cacls %WINDIR%\SYSWOW64\shimgvw.dll /E /P everyone:N`
  
    **Geçici Çözümün Etkisi.** Windows Kabuğu grafik işlemcisi tarafından işlenen medya dosyaları düzgün görüntülenmez. MS11-006 güvenlik güncelleştirmesini yüklemeden önce, bu geçici çözümü daha önce uyguladığınız sistemlerde geri almanız gerekir.
  
    **Geçici çözüm nasıl geri alınır.**
  
    Komut isteminde aşağıdaki komutları yönetici olarak çalıştırın:
  
    Windows XP ve Windows Server 2003'ün 32-bit sürümleri:
  
    `cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /R everyone`
  
    Windows XP ve Windows Server 2003'ün 64-bit sürümleri:
  
    `cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /R everyonecacls %WINDIR%\SYSWOW64\shimgvw.dll /E /R everyone`
  
-   **Windows Vista ve Windows Server 2008 tabanlı sistemlerde küçük resimlerin Windows Gezgini'nde görüntülenmesini devre dışı bırakın**
  
    Windows Vista ve Windows Server 2008 tabanlı sistemlerde küçük resimlerin Windows Gezgini'nde görüntülenmesini devre dışı bırakmak için aşağıdaki adımları gerçekleştirin:
  
    1.  **Denetim Masası**'nı açın ve **Görünüm ve Kişiselleştirme**'yi seçin.  
    2.  Menüden, **Klasör Seçenekleri**'ni seçin.  
    3.  **Görünüm** sekmesini seçin ve **Her zaman simge göster, küçük resim gösterme** seçeneğinin işaretli olduğundan emin olun. **Tamam**'ı tıklatarak ayarları kaydedin.

#### Windows Kabuğu'nda Grafik İşleme Sırasında Taşma Güvenlik Açığı - CVE-2010-3970 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**    
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, etkilenen sistemin tüm denetimini uzaktan ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Güvenlik açığına ne neden olmaktadır?**    
Güvenlik açığı, Windows Kabuğu grafik işlemcisinin özel hazırlanmış bir küçük resmi düzgün ayrıştırmaması sonucunda yığın arabelleği taşması oluşması nedeniyle ortaya çıkar.
  
**Windows Kabuğu nedir?**    
Windows kullanıcı arabirimi (UI), kullanıcılara uygulamaları yürütmek ve işletim sistemini yönetmek için gerekli olan çok çeşitli nesnelere erişim sağlar. Bunların en bilinen ve çok olanları, bilgisayarın sabit disklerindeki klasörler ve dosyalardır. Aynı zamanda, kullanıcının uzaktaki yazıcılara dosyaları göndermesi veya Geri Dönüşüm Kutusu'na erişmesi gibi görevleri gerçekleştirmesine olanak sağlayan çok sayıda sanal nesne de vardır. Kabuk, bu nesneleri hiyerarşik ad alanlarında düzenler ve kullanıcılarla uygulamalara nesnelere erişmesi ve bunları yönetmesi için tutarlı, etkin bir yöntem sağlar.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**    
Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**    
Bu güvenlik açığı bir kullanıcının özel hazırlanmış bir küçük resmi görüntülemesini gerektirir.
  
Ağ saldırısı senaryosunda, saldırgan UNC veya WebDAV konumu gibi bir ağ paylaşımına özel hazırlanmış bir küçük resim ya da özel hazırlanmış küçük resim içeren bir dosya yerleştirebilir ve kullanıcıyı Windows Gezgini aracılığıyla bu klasöre gözatmaya ikna edebilir. Kullanıcı bu paylaşıma gittiğinde, etkilenen denetim yolu Windows Kabuğu grafik işlemcisi aracılığıyla tetiklenir. Böylece, özel hazırlanmış küçük resim bu güvenlik açığından yararlanabilir ve oturum açmış kullanıcının güvenlik bağlamında kod yürütebilir. Saldırganın, bir kullanıcıyı ağ konumunu, UNC ya da WebDAV konumunu ziyaret etmeye zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisinde bulunan ve ağ konumuna, UNC'ye ya da WebDAV konumuna götüren bir bağlantıyı tıklatmaya ikna ederek paylaşımı ziyaret etmeye ikna etmesi gerekir.
  
**Evrensel Adlandırma Kuralı (UNC) nedir? **  
Evrensel Adlandırma Kuralı (UNC), bir ağdaki paylaşım adları ya da dosyalar gibi kaynakları tanımlama yöntemidir. Tipik bir UNC adı iki ters eğik çizgi ve devamında bir sunucu adı içerir (\\\\sunucu\\paylaşım\\altdizin\\dosyaadı gibi).
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**    
İş istasyonları ve terminal sunucuları birincil derecede risk altındadır. Yöneticiler kullanıcıların sunucularda oturum açmalarına ve program çalıştırmalarına izin verirse sunucular daha fazla etkilenebilir. Ancak, en iyi uygulamalarda buna izin verilmesi kesinlikle önerilmemektedir.
  
**Güncelleştirme ne yapıyor?**    
Güncelleştirme, Windows Kabuğu grafik işlemcisinin küçük resimleri işleme biçimini düzelterek bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**    
Evet. Bu güvenlik açığı genel olarak duyurulmuştur. Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE-2010-3970](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3970) atanmıştır. Bu güvenlik güncelleştirmesi [Microsoft Güvenlik Danışma Belgesi 2490606](http://technet.microsoft.com/security/advisory/2490606)'da açıklanmıştır.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**    
Hayır. Microsoft, bu güvenlik bülteni ilk yayımlandığında, güvenlik açığının kanıtı olabilecek kod örneği yayımlandığını gördü, ancak açığın müşterilere saldırmak amacıyla açıkça kullanıldığına dair doğrudan bildirimler almadı.
  
**Bu güvenlik güncelleştirmesini uygulamak, müşterilerin açıktan yararlanmaya yönelik olarak yayımlanan koddan korunmasına yardımcı olur mu?**    
Evet. Bu güvenlik güncelleştirmesi, kanıtı yayımlanmış kod örneği kullanılarak yararlanılabilecek güvenlik açığını giderir. Güvenlik açığı giderilmiş, ayrıca [CVE-2010-3970](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3970) Ortak Güvenlik Açığı ve Etkilenme numarası atanmıştır.
