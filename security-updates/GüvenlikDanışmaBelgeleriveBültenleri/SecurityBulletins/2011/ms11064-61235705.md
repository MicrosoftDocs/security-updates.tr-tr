Security Bulletin

Microsoft Güvenlik Bülteni MS11-064 - Önemli
============================================

TCP-IP Yığınındaki Güvenlik Açıkları Hizmet Reddine Olanak Verebilir (2563894)
------------------------------------------------------------------------------

Yayım Tarihi: 9 Ağustos 2011 Salı

**Sürüm:** 1.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Microsoft Windows'daki özel olarak bildirilen iki güvenlik açığını giderir. Bu güvenlik açıkları, bir saldırgan hedeflenen sisteme özel hazırlanmış bir dizi Internet Denetim İletisi Protokolü (ICMP) iletisi veya Web içeriğine hizmet veren ve URL tabanlı Hizmet Kalitesi (QoS) özelliği etkinleştirilmiş olan bir sunucuya özel hazırlanmış bir URL isteği gönderirse hizmet reddine olanak verebilir.

Bu güvenlik güncelleştirmesi Windows Server 2008 ve Windows Server 2008 R2'nin tüm desteklenen sürümleri için Önemli, Windows Vista ve Windows 7'nin tüm desteklenen sürümleri için Orta olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, URL tabanlı QoS özelliği etkinleştirilmiş durumdayken Windows TCP/IP yığınının ICMP iletilerini işleme ve bellekteki URL'leri işleme biçimini değiştirerek bu güvenlik açıklarını giderir. Bu güvenlik açıkları hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** Yok

ICMP Hizmet Reddi Güvenlik Açığı - CVE-2011-1871  
------------------------------------------------
  
<span></span>
Windows TCP/IP yığınının özel hazırlanmış bir dizi ICMP iletisini düzgün işlememesi nedeniyle bir hizmet reddi güvenlik açığı bulunmaktadır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, hedeflenen sistemin yanıt vermemesine ve otomatik olarak yeniden başlatılmasına neden olabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2011-1871](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-1871).
  
#### ICMP Hizmet Reddi Güvenlik Açığı - CVE-2011-1871 için Azaltıcı Etkenler
  
Microsoft bu güvenlik açığı için herhangi bir azaltıcı etken belirleyememiştir.
  
#### ICMP Hizmet Reddi Güvenlik Açığı - CVE-2011-1871 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Güvenlik duvarında ICMP'yi engelleyin**
  
    Tüm ICMP iletilerinin ağ sınırlaması güvenlik duvarında veya yönlendiricide bu bağlantı noktalarının engellenmesi, güvenlik duvarının veya yönlendiricinin arkasındaki sistemleri bu açıktan yararlanma girişimlerinden korumaya yardımcı olur. Internet'ten gelen tüm istenmeyen iletişimi engellemeniz önerilir.
  
    **Geçici çözümün etkisi.** Bu geçici çözüm TCP'nin ağ iletişimini en iyi duruma getirmesini engelleyerek performansı olumsuz yönde etkileyebilir. ICMP ağ paketleri, farklı MTU değerleri olan ağları birbirine bağlayan yönlendiricilerde bölünmeyi engelleyebilir. Bölünme, TCP çıkışını azaltır ve ağda oluşan sıkışmayı artırır.
  
#### ICMP Hizmet Reddi Güvenlik Açığı - CVE-2011-1871 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**    
Bu bir hizmet reddi güvenlik açığıdır.
  
**Güvenlik açığına ne neden olmaktadır?**    
Güvenlik açığı, Windows TCP/IP yığınının özel hazırlanmış bir dizi ICMP iletisini düzgün işlememesi nedeniyle ortaya çıkar.
  
**Güvenlik açığından hangi bileşen etkilenmektedir?**    
Bu güvenlik açığından etkilenen bileşen Tcpip.sys dosyasıdır.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**    
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, hedeflenen sistemin yanıt vermemesine ve otomatik olarak yeniden başlatılmasına neden olabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**    
Kimliği doğrulanmamış bir uzak saldırgan, hedeflenen sisteme özel hazırlanmış bir dizi ICMP iletisi göndermek üzere bir program oluşturarak bu güvenlik açığından yararlanabilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**    
Sunucular ve iş istasyonları bu güvenlik açığından etkilenir.
  
**Güncelleştirme ne yapıyor?**    
Güncelleştirme, Windows TCP/IP yığınının ICMP iletilerini işleme biçimini değiştirerek bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**    
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere koordinasyon içinde olduğu sorumlu kişiler aracılığıyla ulaştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**    
Hayır. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının müşterilere saldırmak üzere genel olarak kullanıldığına dair herhangi bir bilgi almamıştı.

------------------------------------------------------
  
<span></span>
URL tabanlı Hizmet Kalitesi (QoS) özelliği etkinleştirilmiş durumdayken Windows TCP/IP yığınının bellekteki URL'leri düzgün işlememesi nedeniyle bir hizmet reddi güvenlik açığı bulunmaktadır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, hedeflenen sistemin yanıt vermemesine ve otomatik olarak yeniden başlatılmasına neden olabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2010-1965](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-1965).
  
#### TCP/IP QOS Hizmet Reddi Güvenlik Açığı - CVE-2011-1965 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   URL tabanlı Hizmet Kalitesi özelliği varsayılan olarak hiçbir Windows işletim sisteminde etkinleştirilmiş durumda değildir. Kullanıcıların bu güvenlik açığından etkilenmeleri için bu özelliği el ile yüklemeleri gerekmektedir.
  
#### TCP/IP QOS Hizmet Reddi Güvenlik Açığı - CVE-2011-1965 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   İlke tabanlı QoS özelliğini devre dışı bırakın. Daha fazla bilgi için, [İlke Tabanlı Hizmet Kalitesi (QoS)](http://technet.microsoft.com/en-us/library/dd919203(ws.10).aspx) adlı Microsoft TechNet makalesine bakın.
  
#### TCP/IP QOS Hizmet Reddi Güvenlik Açığı - CVE-2011-1965 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**    
Bu bir hizmet reddi güvenlik açığıdır.
  
**Güvenlik açığına ne neden olmaktadır?**    
Güvenlik açığı, TCP/IP yığınının bellekteki özel hazırlanmış URL'leri düzgün işlememesi nedeniyle ortaya çıkar. Bu güvenlik açığı yalnızca hedef sistemde IIS gibi bir hizmetle Web içeriği barındırılıyorsa ve URL tabanlı QoS etkinleştirilmiş durumdaysa tetiklenir.
  
**Güvenlik açığından hangi bileşen etkilenmektedir?**    
Bu güvenlik açığından etkilenen bileşen Tcpip.sys dosyasıdır.
  
**URL tabanlı Hizmet Kalitesi (QoS) nedir?**    
Hizmet Kalitesi (QoS), kritik uygulamalarda yüksek kalitede performans sağlamak üzere ağlara yönelik olarak sektör genelinde kullanılan bir dizi standart ve düzenektir. Ağ yöneticileri QoS düzeneklerini kullanarak, giden ağ trafiğinin önceliğini belirleyebilir ve gönderim hızını yönetebilir. Bu düzenekler kullanılarak, kaynakların verimli şekilde kullanılması ve böylece gerekli hizmet düzeyinin sunulması sağlanır.
  
Windows Server 2008 R2, BT uzmanlarının Web trafiği önceliğini URL'ye bağlı olarak belirlemelerine olanak verir. URL tabanlı QoS ile, BT uzmanları önemli Web trafiğinin daha az önemli trafikten önce işlenmesini ve böylece yoğun kullanım olan ağlarda performansın artırılmasını sağlayabilir. Örneğin ağın yoğun olarak kullanıldığı zamanlar için, BT uzmanları kritik şirket içi Web sitelerinin trafiğine dış Web sitelerinden daha yüksek öncelik atayarak performansı en üst düzeye çıkarabilirler. Daha fazla bilgi için, [URL tabanlı QoS](http://technet.microsoft.com/en-us/library/dd637810(ws.10).aspx) adlı Microsoft TechNet makalesine bakın.
  
**URI nedir?**    
Tekdüzen Kaynak Tanımlayıcısı (URI), Internet'teki veya ağ üzerindeki kaynaklara göre davranan veya bunları tanımlayan bir karakter dizisidir. Bir URL, Web sitesi gibi bir kaynağa başvuran tipik bir URI örneğidir. URI'lar hakkında daha fazla bilgi için, [RFC-2396](http://www.ietf.org/rfc/rfc2396.txt)'ya bakın.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**    
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, hedeflenen sistemin yanıt vermemesine ve otomatik olarak yeniden başlatılmasına neden olabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**    
Uzaktan saldırı senaryosunda, kimliği doğrulanmamış bir saldırgan, Web içeriğine hizmet veren ve URL tabanlı QoS hizmetinin etkinleştirilmiş olduğu bir sunucuya özel hazırlanmış bir URL göndererek bu güvenlik açığından yararlanabilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**    
URL tabanlı QoS etkinleştirilmiş olarak Web içeriğine hizmet veren sunucular birincil derecede risk altındadır. URL tabanlı QoS özelliği etkinleştirilmemiş olan veya Web içeriğine hizmet vermeyen sunucular bu açıktan etkilenmezler. Windows 7 tabanlı sistemler gibi Internet Information Services (IIS) çalıştırabilen iş istasyonları, hem IIS hem de URL tabanlı QoS etkinleştirilmiş durumdaysa daha fazla risk altındadır.
  
**Windows 7 tabanlı sistemler IIS çalıştırılan bir sunucu olarak yapılandırılırmışsa daha fazla risk altında mıdır?**    
Evet. Windows 7 tabanlı sistemlere IIS gibi sunucu yazılımları yüklenip yapılandırılabilir. Windows 7 tabanlı sistemler iş istasyonu olarak nitelendirilseler de, bu yapılandırmada sunucu gibi davranırlar ve bu nedenle de bu güvenlik açığından dolayı daha fazla risk altındadırlar.
  
**Güncelleştirme ne yapıyor?**    
Güncelleştirme, URL tabanlı QoS özelliği etkinleştirilmiş durumdayken TCP/IP yığınının bellekteki URL'leri işleme biçimini değiştirerek bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**    
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere koordinasyon içinde olduğu sorumlu kişiler aracılığıyla ulaştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**    
Hayır. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının müşterilere saldırmak üzere genel olarak kullanıldığına dair herhangi bir bilgi almamıştı.

    5.  Son olarak, **Önceki Sürümler** sekmesini tıklatarak dosyanın önceki sürümünün dosya bilgilerini, yeni veya güncelleştirilmiş sürümünün dosya bilgileriyle de karşılaştırabilirsiniz.
