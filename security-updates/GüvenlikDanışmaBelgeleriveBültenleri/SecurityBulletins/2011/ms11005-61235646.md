Security Bulletin

Microsoft Güvenlik Bülteni MS11-005 - Önemli
============================================

Active Directory'deki Güvenlik Açığı Hizmet Reddine Olanak Verebilir (2478953)
------------------------------------------------------------------------------

Yayım Tarihi: 8 Şubat 2011 Salı

**Sürüm:** 1.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Active Directory'deki genel olarak duyurulan bir güvenlik açığını giderir. Güvenlik açığı, etkilenen bir Active Directory sunucusuna özel hazırlanmış bir paket gönderirse hizmet reddine olanak verebilir. Saldırganın bu güvenlik açığından yararlanabilmesi için, etki alanına katılmış bilgisayarda geçerli yerel yönetici ayrıcalıkları olmalıdır.

Bu güvenlik güncelleştirmesi Windows Server 2003'ün tüm desteklenen sürümleri üzerinde Active Directory için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, Active Directory sunucusunun hizmet asıl adı (SPN) güncelleştirme isteklerini işleme biçimini düzelterek bu güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** Yok

Active Directory'de SPN Doğrulaması Güvenlik Açığı - CVE-2011-0040  
------------------------------------------------------------------
  
<span></span>
Microsoft Windows Active Directory uygulamalarında, hizmet asıl adlarının (SPN) düzgün doğrulanmaması sonucunda SPN çarpışmalarına neden olabilecek bir hizmet reddi güvenlik açığı bulunmaktadır. Bu durumda, SPN'yi kullanan hizmetler anlaşma sağlayacak şekilde yapılandırılmışlarsa NT LAN Yöneticisi'ne (NTLM) indirgenirler. Anlaşma sağlayacak şekilde yapılandırılmamış olan hizmetler kullanılamaz ve bu da hizmet reddi koşulunun oluşmasına neden olur. Bu güvenlik açığından yararlanan saldırgan, etkilenen sistemin yanıt vermemesine neden olabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2011-0040](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-0040).
  
#### Active Directory'de SPN Doğrulaması Güvenlik Açığı - CVE-2011-0040 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Saldırganın bu hizmet reddi güvenlik açığından yararlanabilmesi için, etki alanına katılmış bir bilgisayarın yöneticisi olması gerekir.
  
#### Active Directory'de SPN Doğrulaması Güvenlik Açığı - CVE-2011-0040 için Geçici Çözümler
  
Microsoft bu güvenlik açığı için herhangi bir geçici çözüm belirleyememiştir.
  
#### Active Directory'de SPN Doğrulaması Güvenlik Açığı - CVE-2011-0040 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**    
Bu bir kimlik doğrulaması indirgeme ve aynı zamanda bir hizmet reddi güvenlik açığıdır. Bu güvenlik açığından yararlanan bir saldırgan, etkilenen sistemin Kerberos kimlik doğrulamasından NT LAN Yöneticisi'ne (NTLM) indirgenmesine ve en kötü olasılıkla hizmetin yanıt vermemeye başlamasına neden olabilir. Bu güvenlik açığı, bir saldırganın kod yürütmesine veya kullanıcı haklarını yükseltmesine izin vermez.
  
**Güvenlik açığına ne neden olmaktadır?**    
Active Directory, hizmet asıl adını (SPN) güncelleştirmeye yönelik özel hazırlanmış istekleri hatalı şekilde işleyerek etki alanında ad çarpışmalarına neden olabilir.
  
**Active Directory nedir?**    
Active Directory'nin başlıca amacı, Windows tabanlı bilgisayarlar için merkezi kimlik doğrulama ve yetkilendirme hizmetlerini sağlamaktır.
  
**SPN nedir?**    
Hizmet Asıl Adı (SPN), istemcinin bir hizmet örneğini benzersiz şekilde tanımlamasını sağlayan bir addır. Hizmet asıl adlarının kullanımı ile ilgili daha fazla bilgi için, [Hizmet Asıl Adları](http://msdn.microsoft.com/en-us/library/ms677949(vs.85).aspx) adlı MSDN makalesine bakın.
  
**NTLM nedir?**    
Windows Sınama/Yanıt (NT LAN Yöneticisi veya NTLM), Windows işletim sisteminin çalıştırıldığı sistemlerin bulunduğu ağlarda ve tek başına sistemlerde kullanılan kimlik doğrulaması protokolüdür. NTLM kimlik bilgileri, etkileşimli oturum açma işlemi sırasında alınan verilere dayanır ve etki alanı adı, kullanıcı adı ve kullanıcı parolasının tek yönlü karmasından oluşur. NTLM, kullanıcının kimliğini doğrulamak için kullanıcı parolasını ağ üzerinden göndermeden şifreli bir sınama/yanıt protokolü kullanır. Bunun yerine, kimlik doğrulaması isteminde bulunan sistemin güvenli NTLM kimlik bilgilerine erişimi olduğunu doğrulamak üzere bir hesaplama gerçekleştirmesi gerekir.
  
Ağ üzerinden etkileşimli NTLM kimlik doğrulaması genelde iki sistemden oluşur: kullanıcının kimlik doğrulaması isteminde bulunduğu bir istemci sistemi ve kullanıcı parolasıyla ilgili bilgilerin saklandığı bir etki alanı denetleyicisi. Zaten oturum açmış bir kullanıcının sunucu uygulaması gibi bir kaynağa erişmesine izin vermek için gerekebilecek etkileşimli olmayan kimlik doğrulaması genelde üç sistemden oluşur: bir istemci, bir sunucu ve kimlik doğrulaması hesaplamalarını sunucunun yerine yapan bir etki alanı denetleyicisi.
  
**Kerberos nedir?**    
Kerberos, açık ve güvenli olmayan bir ağ üzerinde kullanıcıların ve hizmetlerin karşılıklı kimliklerini doğrulamak için kullanılan bir protokoldür. Hizmetlerin, kullanıcının kimliğini doğrulamalarına gerek kalmadan Kerberos anahtarını doğru şekilde tanımlamalarına olanak verir. Bunun için, paylaşılan gizli anahtarlar kullanılır.
  
Kerberos protokolü, kullanıcıların kimlik bilgilerini şifrelemek ve imzalamak için bu paylaşılan gizli anahtarları kullanır. İstemcinin kimlik doğrulaması, Kerberos Anahtar Dağıtım Merkezi (KDC) tarafından yapılır. Kimlik doğrulamasının ardından, kullanıcı ağ üzerindeki belirli bir hizmete erişmek için bir hizmet anahtarı isteyebilir. Bu anahtar, kullanıcının şifrelenmiş ve doğrulanmış kimliğini içerir.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**    
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, etki alanında ad çarpışmalarına neden olur ve SPN kullanan hizmetlerin kimlik doğrulamasının NTLM'ye indirgenmesine yol açabilir. Hizmetin anlaşma sağlayacak şekilde yapılandırılmamış olduğu yapılandırmalarda hizmet kullanılamayabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**    
Saldırgan, Active Directory sunucusuna özel hazırlanmış paketler göndererek bu güvenlik açığından yararlanmaya çalışabilir. Saldırganın bu güvenlik açığından yararlanabilmesi için, etki alanına katılmış bilgisayarda geçerli yerel yönetici ayrıcalıkları olmalıdır.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**    
Bu güvenlik açığı Windows Server 2003 üzerinde Active Directory çalıştırılan ortamları etkiler.
  
**Güncelleştirme ne yapıyor?**    
Güncelleştirme, Active Directory sunucusunun hizmet asıl adı (SPN) güncelleştirme isteklerini işleme biçimini düzelterek bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**    
Evet. Bu güvenlik açığı genel olarak duyurulmuştur. Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE-2011-0040](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-0040) atanmıştır.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**    
Hayır. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının müşterilere saldırmak üzere genel olarak kullanıldığına dair herhangi bir bilgi almamıştı.

----------------------------------------
  
<span></span>
