Security Bulletin

Microsoft Güvenlik Bülteni MS11-062 - Önemli
============================================

Uzaktan Erişim Hizmeti'nin NDISTAPI Sürücüsündeki Güvenlik Açığı Ayrıcalık Yükselmesine İzin Verebilir (2566454)
----------------------------------------------------------------------------------------------------------------

Yayım Tarihi: 9 Ağustos 2011 Salı

**Sürüm:** 1.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi, Windows XP'nin ve Windows Server 2003'ün tüm desteklenen sürümlerinde özel olarak bildirilen bir güvenlik açığını giderir. Bu güvenlik güncelleştirmesi Windows XP ve Windows Server 2003'ün tüm desteklenen sürümleri için Önemli olarak derecelendirilmiştir. Windows Vista, Windows Server 2008, Windows 7 ve Windows Server 2008 R2 bu güvenlik açığından etkilenmez. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Bu güvenlik açığı, bir saldırgan etkilenen sistemde oturum açar ve güvenlik açığından yararlanacak şekilde özel olarak tasarlanmış bir uygulamayı çalıştırırsa ayrıcalık yükselmesine ve saldırganın etkilenen sistemin tüm denetimini ele geçirmesine izin verebilir. Saldırganın bu güvenlik açığından yararlanabilmesi için geçerli oturum açma kimlik bilgilerine sahip olması ve yerel olarak oturum açabilmesi gerekir.

Güvenlik güncelleştirmesi, NDISTAPI sürücüsünün kullanıcı modu girişini Windows çekirdeğine göndermeden önce doğrulama biçimini düzelterek bu güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** Yok

NDISTAPI Sürücüsünde Ayrıcalık Yükselmesi Güvenlik Açığı - CVE-2011-1974  
------------------------------------------------------------------------
  
<span></span>
Uzaktan Erişim Hizmeti'nin NDISTAPI sürücüsünde bir ayrıcalık yükselmesi güvenlik açığı bulunmaktadır. Güvenlik açığı, veriler kullanıcı modundan Windows çekirdeğine geçirilirken NDISTAPI sürücüsünün kullanıcı tarafından sağlanan girişi düzgün şekilde doğrulamaması nedeniyle ortaya çıkar. Bu güvenlik açığından yerel olarak başarıyla yararlanan bir saldırgan rasgele kod yürütebilir ve etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2011-1974](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-1974).
  
#### NDISTAPI Sürücüsünde Ayrıcalık Yükselmesi Güvenlik Açığı - CVE-2011-1974 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Saldırganın bu güvenlik açığından yararlanabilmesi için geçerli oturum açma kimlik bilgilerine sahip olması ve yerel olarak oturum açabilmesi gerekir.
  
#### NDISTAPI Sürücüsünde Ayrıcalık Yükselmesi Güvenlik Açığı - CVE-2011-1974 için Geçici Çözümler
  
Microsoft bu güvenlik açığı için herhangi bir geçici çözüm belirleyememiştir.
  
#### NDISTAPI Sürücüsünde Ayrıcalık Yükselmesi Güvenlik Açığı - CVE-2011-1974 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir ayrıcalık yükseltmesi güvenlik açığıdır.
  
**Güvenlik açığına ne neden olmaktadır?**  
Güvenlik açığı, veriler kullanıcı modundan Windows çekirdeğine geçirilirken NDISTAPI sürücüsünün kullanıcı tarafından sağlanan girişi düzgün şekilde doğrulamaması nedeniyle ortaya çıkar.
  
**Güvenlik açığından etkilenen bileşen hangisidir?**  
Bu güvenlik açığından etkilenen bileşen NDISTAPI.sys dosyasıdır.
  
**Uzaktan Erişim Hizmeti (RAS) nedir?**  
Uzaktan Erişim Hizmeti (RAS), kullanıcıların telefon hattı, Internet veya başka bir ağ bağlantısı üzerinden bir uzak bilgisayara bağlanarak, sistemleri uzak ağa fiziksel olarak bağlıymış gibi çalışabilmelerini sağlar. RAS kullanıcıların e-posta gönderip alma, belgeleri faks ile gönderme, dosya alma ve belgeleri bir ofis yazıcısında yazdırma gibi etkinlikleri uzaktan gerçekleştirmelerine olanak verir. Daha fazla bilgi için, [RAS Mimarisine Genel Bakış](http://msdn.microsoft.com/en-us/library/ff563285(v=vs.85).aspx) (NDIS 5.1) adlı MSDN makalesine bakın.
  
RAS, Windows XP ve Windows Server 2003'te bulunan bir özgün hizmettir. RAS sonraki işletim sistemlerinde yerini Yönlendirme ve Uzaktan Erişim Hizmeti'ne (RRAS) bırakmıştır. RRAS, bir bilgisayarın ağ yönlendiricisi olarak çalışmasına olanak tanır. RRAS ayrıca Windows için RAS için sonraki nesil sunucu işlevlerini sağlar. RRAS sunucu işlevleri, önceki işletim sistemlerinde bulunan RAS işlevselliği üzerine oluşturulmuştur.
  
**NDISTAPI sürücüsü nedir?**  
NDISTAPI sürücüsü, RAS mimarisinin bir parçasıdır ve NDISWAN için TAPI hizmetlerine arabirim oluşturur. Telefon uygulamaları TAPI hizmetlerini kullanır; telefon hatlarını ve aramaları ayarlamak, izlemek ve sonlandırmak için TAPI isteklerinde bulunabilir. NDISTAPI sürücüsü bu TAPI isteklerini NDISWAN'a geçirir. Daha fazla bilgi için, [NDISTAPI'ye Genel Bakış (NDIS 5.1)](http://msdn.microsoft.com/en-us/library/ff554813(v=vs.85).aspx) adlı MSDN makalesine bakın.
  
**NDISWAN nedir?**  
Ağ Sürücü Arabirimi Standardı Geniş Alan Ağı (NDISWAN), sistem tarafından sağlanan bir NDIS ara sürücüsüdür ve WAN miniport sürücüleri tarafından kullanılan veri sıkıştırma, şifreleme, geri döngü ve basit PPP çerçeveleme gibi işlevleri sağlar. WAN miniport sürücüleri bu nedenle, yalnızca ortama özgü özellikleri uygulamak için gereklidir (örneğin ISDN için Q931 sinyali gereklidir). Daha fazla bilgi için, [NDISWAN'a Genel Bakış](http://msdn.microsoft.com/en-us/library/ff564653(v=vs.85).aspx) adlı MSDN makalesine bakın.
  
**TAPI hizmeti nedir?**  
Telefon Uygulama Programı Arabirimi (TAPI) hizmeti, telefon iletişimi gerçekleştirebilen uygulamalara telefon hizmetleri sağlayan bir program arabirimidir.
  
**NDIS nedir?**  
Ağ Sürücü Arabirimi Standardı (NDIS), Windows Sürücüsü Aygıt Paketi'ni içeren ve Windows sürücülerinin kararlılığını ve güvenilirliğini sınayan, tamamen tümleşik bir sürücü geliştirme sistemi olan Windows Sürücü Paketi'nin bir parçasıdır. NDIS kitaplığı, ağ donanımını ağ sürücülerinden soyutlar. NDIS ayrıca katman oluşturulmuş ağ sürücüleri arasında standart bir arabirim belirleyerek, donanımı yöneten alt düzey sürücüleri ağ aktarımları gerçekleştirenler gibi üst düzey sürücülerden soyutlar. Bunun yanı sıra NDIS, ağ sürücülerinin durum bilgilerini ve işlev işaretçileri, tanıtıcılar, bağlantı parametre blokları gibi parametrelerini ve diğer sistem değerlerini saklar.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, yerel sistemin bağlamında rasgele kod çalıştırabilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırganın bu güvenlik açığından yararlanabilmesi için öncelikle sistemde oturum açması gerekir. Böylece saldırgan, güvenlik açığından yararlanan özel olarak hazırlanmış bir uygulama çalıştırabilir ve etkilenen sistemin tüm denetimini ele geçirebilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
İş istasyonları ve terminal sunucuları birincil derecede risk altındadır. Yöneticiler kullanıcıların sunucularda oturum açmalarına ve program çalıştırmalarına izin verirse sunucular daha fazla etkilenebilir. Ancak, en iyi uygulamalarda buna izin verilmesi kesinlikle önerilmemektedir.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, NDISTAPI sürücüsünün kullanıcı modu girişini Windows çekirdeğine göndermeden önce doğrulama biçimini düzelterek bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere koordinasyon içinde olduğu sorumlu kişiler aracılığıyla ulaştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının müşterilere saldırmak üzere genel olarak kullanıldığına dair herhangi bir bilgi almamıştı.

-   **Dosya Sürümü Doğrulaması**

    Microsoft Windows'un birçok sürümü olduğundan, aşağıdaki adımlar sizin sisteminizde farklı olabilir. Böyle bir durumda, bu adımları tamamlamak için ürün belgelerinize bakın.

    1.  **Başlat**'ı ve sonra **Ara**'yı tıklatın.
    2.  **Arama Sonuçları** bölmesinde, **Arama Yardımcısı**'nın altındaki **Tüm dosya ve klasörler**'i tıklatın.
    3.  **Dosya adının tümü veya bir parçası** kutusuna, uygun dosya bilgileri tablosundan bir dosya adı yazın ve **Ara**'yı tıklatın.
    4.  Dosya listesinde, uygun dosya bilgileri tablosundan bir dosya adını sağ tıklatın ve sonra **Özellikler**'i tıklatın.
        **Not** Sisteminizde yüklü işletim sisteminin veya programların sürümüne bağlı olarak, dosya bilgileri tablosunda listelenen dosyalardan bazıları yüklenmeyebilir.
    5.  **Sürüm** sekmesinde, sisteminize yüklenmiş olan dosyanın sürümünü uygun dosya bilgileri tablosunda belgelenen sürümle karşılaştırarak belirleyin.
