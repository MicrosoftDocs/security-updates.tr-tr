Security Bulletin

Microsoft Güvenlik Bülteni MS12-027 - Kritik
============================================

Windows Ortak Bileşenleri'ndeki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (2664258)
-------------------------------------------------------------------------------------------------

Yayım Tarihi: 10 Nisan 2012 Salı | Güncelleştirme Tarihi: 26 Nisan 2012 Perşembe

**Sürüm:** 2.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Windows ortak denetimlerindeki özel olarak duyurulan bir güvenlik açığını giderir. Güvenlik açığı, bir kullanıcı bu açıktan yararlanacak şekilde özel olarak tasarlanmış içerik bulunan bir web sitesini ziyaret ederse uzaktan kod yürütülmesine izin verebilir. Ancak bu durumların hiçbirinde, saldırganın kullanıcıları bu tür bir web sitesini ziyaret etmeye zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi web sitesine yönlendiren bir bağlantıyı tıklatmalarını sağlayarak onları bu web sitesini ziyaret etmeye ikna etmesi gerekir. Kötü amaçlı dosya bir e-posta eki olarak da gönderilebilir; ancak saldırganın bu güvenlik açığından yararlanabilmesi için, kullanıcıyı bu eki açmaya ikna etmesi gerekir.

Bu güvenlik güncelleştirmesi, varsayılan yüklemelerinde Windows ortak denetimlerini içeren tüm desteklenen Microsoft yazılımları için Kritik olarak derecelendirilmiştir. Microsoft Office 2003, Microsoft Office 2007; Microsoft Office 2010 (x64 tabanlı sürümleri dışında); Microsoft SQL Server 2000 Analysis Services, Microsoft SQL Server 2000 (Itanium tabanlı sürümleri dışında), Microsoft SQL Server 2005 (Microsoft SQL Server 2005 Express Edition dışında, ancak Microsoft SQL Server 2005 Express Edition with Advanced Services dahil), Microsoft SQL Server 2008, Microsoft SQL Server 2008 R2, Microsoft BizTalk Server 2002, Microsoft Commerce Server 2002, Microsoft Commerce Server 2007, Microsoft Commerce Server 2009, Microsoft Commerce Server 2009 R2, Microsoft Visual FoxPro 8.0, Microsoft Visual FoxPro 9.0 ve Visual Basic 6.0 Çalışma Zamanı Modülü'nün tüm desteklenen sürümleri buna dahildir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, Windows ortak denetimlerinin güvenlik açığından etkilenen sürümünü devre dışı bırakarak ve güvenlik açığından etkilenmeyen yeni bir sürümüyle değiştirerek bu güvenlik açığını ortadan kaldırır. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşteriler, [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmetini kullanarak güncelleştirmeleri Microsoft Update sitesinden çevrimiçi olarak denetleyecek biçimde otomatik güncelleştirmeleri yapılandırabilir. Bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için, otomatik güncelleştirmeler özelliğini etkinleştirmiş ve güncelleştirmeleri Microsoft Update sitesinden çevrimiçi olarak denetleyecek biçimde yapılandırmış olan müşterilerin herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri Microsoft Update sitesinden el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Windows XP ve Windows Server 2003'ün desteklenen sürümlerinde otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871). Windows Vista, Windows Server 2008, Windows 7 ve Windows Server 2008 R2'nin desteklenen sürümlerinde otomatik güncelleştirmeler hakkında bilgi için, bkz: [Windows'un otomatik güncelleştirme özelliğini anlama](http://windows.microsoft.com/en-us/windows-vista/understanding-windows-automatic-updating).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi hemen uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.**[Microsoft Bilgi Bankası makalesi 2664258](http://support.microsoft.com/kb/2664258), bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir. Bu makale ayrıca, bu sorunlar için önerilen çözümleri de içermektedir.

Yazılımınızın sürümünün destek ömrünü belirlemek için [Microsoft Destek Ömrü](http://go.microsoft.com/fwlink/?linkid=21742) Web sitesini ziyaret edin.

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)
-----------------------------------------------------------------

<span></span>
**Bu bülten** **26 Nisan 2012'de** **neden yeniden düzenlendi?**
Bu bülten birkaç ekleme ve düzeltme yapmak üzere yeniden düzenlendi:

-   Microsoft SQL Server 2000 Analysis Services Service Pack 4 için QFE ve GDR ayrımı geçerli olmadığından, güncelleştirmenin bu ürünün tüm yüklemelerine uygulandığı Etkilenen Yazılımlar tablosunda açıklığa kavuşturuldu. Bu yalnızca bilgilendirme amaçlı bir değişikliktir. Güvenlik güncelleştirmesi dosyalarında veya algılama mantığında herhangi bir değişiklik yapılmamıştır. Güncelleştirme ilk sürümünden itibaren doğru şekilde sunulmuş olduğu için, bu güncelleştirmeyi daha önce başarıyla yüklemiş olan müşterilerin herhangi bir işlem gerçekleştirmeleri gerekmez.
-   Etkilenen Yazılımlar bölümüne SQL Server 2008 R2'nin Service Pack 1 sürümleri eklendi. Bu yalnızca bilgilendirme amaçlı bir değişikliktir. Güvenlik güncelleştirmesi dosyalarında veya algılama mantığında herhangi bir değişiklik yapılmamıştır. Güncelleştirme ilk sürümünden itibaren doğru şekilde sunulmuş olduğu için, bu güncelleştirmeyi daha önce başarıyla yüklemiş olan müşterilerin herhangi bir işlem gerçekleştirmeleri gerekmez.
-   GDR ya da QFE güncelleştirmelerini gerektiren sürüm aralığına dayalı olarak hangi SQL Server 2000 güncelleştirmesinin kullanılacağını açıklamak üzere Güncelleştirme Hakkında SSS bölümüne bir girdi eklendi. Bu yalnızca bilgilendirme amaçlı bir değişikliktir. Güvenlik güncelleştirmesi dosyalarında veya algılama mantığında herhangi bir değişiklik yapılmamıştır.
-   Dağıtım bölümünde, SQL Server 2000 için güncelleştirme kaldırma bilgileri düzeltildi. Bu yalnızca bilgilendirme amaçlı bir değişikliktir. Güvenlik güncelleştirmesi dosyalarında veya algılama mantığında herhangi bir değişiklik yapılmamıştır.

**SQL Server 2000 için hem GDR hem de QFE güncelleştirmeleri sunuluyor. Hangi güncelleştirmeyi kullanacağımı nasıl anlarım?**
İlk önce SQL Server sürüm numaranızı belirleyin. SQL Server sürüm numaranızı belirleme hakkında daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 321185](http://support.microsoft.com/kb/321185).

İkinci olarak, aşağıdaki tabloda SQL Server sürüm numaranızın bulunduğu aralığı belirleyin. Karşılık gelen güncelleştirmeyi yüklemelisiniz.

**Not** SQL Server sürüm numaranız aşağıdaki tabloda yer alan aralıklardan birinde değilse, SQL Server sürümünüz artık desteklenmemektedir. Bu güvenlik güncelleştirmesini ve sonraki güncelleştirmeleri uygulamak için en son Service Pack veya SQL Server ürününe yükseltin.

**SQL Server 2005 veya SQL Server 2008 çalıştırıyorum, hangi** **güncelleştirmeler** **çalışma ortamıma uygulanabilir?**
Varsayılan olarak, Microsoft SQL Server 2005 ve Microsoft SQL Server 2008 varsayılan yüklemelerinin desteklenen sürümleri Windows ortak denetimlerini içerir. Microsoft Update, Microsoft SQL Server 2005 ve Microsoft SQL Server 2008'in desteklenen sürümlerini otomatik olarak algılar ve Windows ortak denetim paketlerini bu sürümlere dağıtır. SQL Server 2005 ve SQL Server 2008 için olan güncelleştirme paketleri, Microsoft Office için olan paketlerle aynıdır. Daha fazla bilgi için, **Etkilenen Yazılımlar** bölümüne bakın.

**Bu bültende açıklanan güvenlik açığının etkilerini azaltmak içinMicrosoft Office 2003, Microsoft Office** **2007** **ve Microsoft Office 2010'un** **hangi bileşenlerinin** **güncelleştirilmesi gerekmektedir?
**Etkilenen ikili dosya birden çok Microsoft Office ürününde paylaşılan bir bileşen olduğu için, Microsoft Update yüklemenizde Windows ortak denetimlerinin güvenlik açığından etkilenen bir sürümü bulunduğunu algılarsa, bu güncelleştirme Microsoft Office yüklemenize uygulanabilir. Güncelleştirme belirli Microsoft Office bileşenlerini değil, belirli Microsoft Office sürümlerini hedefler.

**Bir üçüncü taraf uygulama geliştiricisiyim ve uygulamamda ActiveX denetimini kullanıyorum. Uygulamam güvenlik açığından etkileniyor mu ve nasıl güncelleştirebilirim?**
ActiveX denetimini yeniden dağıtan geliştiriciler, bu bültende sağlanan güncelleştirmeyi karşıdan yükleyerek uygulamalarıyla birlikte yüklenen ActiveX denetimi sürümünü güncelleştirdiklerinden emin olmalıdırlar. Yeniden dağıtılan bileşenin kullanımı konusundaki en iyi uygulamalar hakkında daha fazla bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 835322](http://support.microsoft.com/kb/835322) ve [Yalıtılmış Uygulamalar ve Yan Yana Derlemeler](http://msdn.microsoft.com/en-us/library/aa375193.aspx).

**Yeniden dağıtılabilen ActiveX denetimini içeren yazılımlar geliştiriyorum. Ne yapmalıyım?**
Geliştirme yazılımınız için bu güvenlik bülteninde sunulan güvenlik güncelleştirmesini yüklemelisiniz. Uygulamanızla birlikte ActiveX denetimini yeniden dağıttıysanız, müşterilerinize uygulamanızın güncelleştirilmiş bir sürümünü yayımlayarak, bu dosyanın geliştirme yazılımınız için bu güvenlik güncelleştirmesinin karşıdan yüklemesinde bulunan güncelleştirilmiş sürümünü edinmelerini sağlamalısınız.

MSCOMCTL.OCX RCE Güvenlik Açığı - CVE-2012-0158
-----------------------------------------------

<span></span>
Windows ortak bileşenlerinde, uzaktan kod yürütülmesine neden olabilecek bir güvenlik açığı bulunmaktadır. Saldırgan, özel hazırlanmış bir web sayfası oluşturarak bu güvenlik açığından yararlanabilir. Bir kullanıcı bu web sayfasını görüntülediğinde, güvenlik açığı uzaktan kod yürütülmesine izin verebilir. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açan kullanıcı ile aynı haklara sahip olabilir.

Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2012-0158](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2012-0158).

#### MSCOMCTL.OCX RCE Güvenlik Açığı - CVE-2012-0158 için Azaltıcı Etkenler

Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:

-   Web tabanlı saldırı senaryosunda, bir saldırganın bu açıktan yararlanmak için kullanılan bir web sayfası içeren bir web sitesi barındırabilir. Ayrıca, kullanıcı tarafından sağlanan içeriği veya reklamları kabul eden ya da barındıran güvenliği aşılmış web sitelerinde, bu güvenlik açığından yararlanılabilecek özel hazırlanmış içerik bulunabilir. Ancak bu durumların hiçbirinde, saldırganın, bu web sitelerini ziyaret etmek için kullanıcıyı zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi web sitesine yönlendiren bir bağlantıyı tıklatmalarını sağlayarak onları bu web sitesini ziyaret etmeye ikna etmesi gerekir.
-   Kötü amaçlı dosya bir e-posta eki olarak gönderilebilir; ancak saldırganın bu güvenlik açığından yararlanabilmesi için, kullanıcıyı bu eki açmaya ikna etmesi gerekir.
-   Bu güvenlik açığından başarıyla yararlanan bir saldırgan, yerel kullanıcı ile aynı haklara sahip olabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

#### MSCOMCTL.OCX RCE Güvenlik Açığı - CVE-2012-0158 için Geçici Çözümler

Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. #### MSCOMCTL.OCX RCE Güvenlik Açığı - CVE-2012-0158 Hakkında SSS

**Bu güvenlik açığının kapsamı nedir?**
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, etkilenen sistemin tüm denetimini geçerli kullanıcının bağlamında ele geçirebilir.

**Güvenlik açığına ne neden olmaktadır?**
Güvenlik açığı, ActiveX denetimlerinin Internet Explorer'da kullanıldıkları sırada sistem durumunu bir saldırganın rasgele kod yürütmesine olanak sağlayacak biçimde bozmaları nedeniyle ortaya çıkar.

**Windows** **ortak denetimleri nedir?**
Windows Ortak Denetimleri, MSCOMCTL.OCX dosyasının içerdiği ActiveX denetimleridir. Güvenlik açığından etkilenen ActiveX denetimleri MSCOMCTL.TreeView, MSCOMCTL.ListView2, MSCOMCTL.TreeView2 ve MSCOMCTL.ListView adlı denetimlerdir.

**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açan kullanıcı ile aynı haklara sahip olabilir. Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

**Bir saldırgan bu güvenlik açığından nasıl yararlanabilir?**
Bir saldırgan, bu güvenlik açığından Internet Explorer yardımıyla yararlanacak ve kullanıcıyı web sitesini görüntülemeye ikna edecek biçimde özel olarak tasarlanmış bir web sitesi barındırabilir. Buna, güvenliği aşılan web siteleri ve kullanıcı tarafından sağlanan içeriği veya reklamları kabul eden ya da barındıran web siteleri de dahil olabilir. Bu tür web sitelerinde, bu güvenlik açığından yararlanacak şekilde özel olarak tasarlanmış içerik bulunabilir. Ancak bu durumların hiçbirinde, saldırganın, bu web sitelerini ziyaret etmek için kullanıcıyı zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger isteğindeki kendi web sitesine götüren bir bağlantıyı tıklatmalarını sağlayarak onları bu web sitesine çekmesi gerekir. Ardından, özel hazırlanmış web içeriği kullanılarak, etkilenen sistemlerde bu güvenlik açığından yararlanılabilir.

**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**
İş istasyonları ve terminal sunucuları birincil derecede risk altındadır. Yöneticiler kullanıcıların sunucularda oturum açmalarına ve program çalıştırmalarına izin verirse sunucular daha fazla etkilenebilir. Ancak, en iyi uygulamalarda buna izin verilmesi kesinlikle önerilmemektedir.

**SQL Server 2005 veya SQL Server 2008 çalıştırıyorum, hangi düzeltme ekleri çalışma ortamıma uygulanabilir?**
Varsayılan olarak, Microsoft SQL Server 2005 ve Microsoft SQL Server 2008 varsayılan yüklemelerinin desteklenen sürümleri Windows ortak denetimlerini içerir. Microsoft Update, Microsoft SQL server 2005 ve Microsoft SQL Server 2008'in desteklenen sürümlerini otomatik olarak algılar ve Windows ortak denetim paketlerini bu sürümlere dağıtır.

**Bu güvenlik açığının etkilerini azaltmak içinMicrosoft Office 2003, Microsoft Office** **2007** **ve Microsoft Office 2010'un hangi bileşenlerinin güncelleştirilmesi gerekmektedir?
**Etkilenen ikili dosya birden çok Microsoft Office ürününde paylaşılan bir bileşen olduğu için, Microsoft Update yüklemenizde MSCOMCTL.OCX dosyasının güvenlik açığından etkilenen bir sürümü bulunduğunu algılarsa, bu güncelleştirme Microsoft Office yüklemenize uygulanabilir. Güncelleştirme belirli Microsoft Office bileşenlerini değil, belirli Microsoft Office sürümlerini hedefler.

**Güncelleştirme ne yapıyor?**
Güncelleştirme, Windows ortak denetimlerinin güvenlik açığından etkilenen sürümünü devre dışı bırakarak bu güvenlik açığını ortadan kaldırır. Bunun için, kitaplık dosyalarında barındırılan aşağıdaki sınıf tanımlayıcılarının kill bitini ayarlar:

`{bdd1f04b-858b-11d1-b16a-00c0f0283628}{996BF5E0-8044-4650-ADEB-0B013914E99C}{C74190B6-8589-11d1-B16A-00C0F0283628}{9181DC5F-E07D-418A-ACA6-8EEA1ECB8E9E}`

Güncelleştirme ayrıca Windows ortak denetimlerinin güvenlik açığından etkilenen sürümünü, güvenlik açığı bulunmayan yeni bir sürümüyle değiştirir.

**Kill biti nedir?**
Microsoft Internet Explorer'da bir ActiveX denetiminin Internet Explorer HTML işleme altyapısı tarafından yüklenmesini engelleyen bir güvenlik özelliğidir. Bu işlem, kill biti adı verilen bir kayıt defteri ayarı belirlenerek gerçekleştirilir. Kill biti ayarlandıktan sonra, denetim tam olarak kurulu olsa da bir daha asla yüklenemez. Kill biti ayarlanarak, güvenlik açığından etkilenen bir bileşenin sisteme yüklendiğinde veya yeniden yüklendiğinde etkisiz ve zararsız kalması sağlanabilir.

Kill bitleri hakkında daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 240797](http://support.microsoft.com/kb/240797): Internet Explorer'da ActiveX denetiminin çalışması nasıl engellenir
