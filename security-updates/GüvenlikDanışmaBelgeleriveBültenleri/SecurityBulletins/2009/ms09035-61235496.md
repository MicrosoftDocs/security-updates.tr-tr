Security Bulletin

Microsoft Güvenlik Bülteni MS09-035 - Orta
==========================================

Visual Studio Etkin Şablon Kütüphanesi'ndeki Güvenlik Açıkları Uzaktan Kod Yürütülmesine İzin Verebilir (969706)
----------------------------------------------------------------------------------------------------------------

Yayım Tarihi: 28 Temmuz 2009 Salı | Güncelleştirme Tarihi: 12 Ocak 2010 Salı

**Sürüm:** 3.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi, Visual Studio ile birlikte gelen Microsoft Etkin Şablon Kütüphanesi'nin (ATL) genel sürümlerinde özel olarak bildirilen birkaç güvenlik açığını giderir. Bu güvenlik güncelleştirmesi özellikle bileşen ve denetim geliştirenlere yöneliktir. ATL kullanarak bileşen ve denetim geliştirenler bu bültende sağlanan güncelleştirmeyi yüklemeli ve bu güvenlik bülteninde açıklanan güvenlik açıklarından etkilenmeyen bileşenler ve denetimler oluşturup müşterilerine dağıtmak için sağlanan yönergeleri uygulamalıdır.

Bu güvenlik bülteninde, bir kullanıcı ATL'nin güvenlik açığından etkilenen bir sürümü kullanılarak oluşturulmuş bir bileşeni veya denetimi yüklerse uzaktan kod yürütülmesine olanak verebilecek güvenlik açıkları anlatılmaktadır.

Microsoft Güvenlik Bültenleri belirli bir ürüne yönelik güvenlik açığı riski açıklanır, ancak bu güvenlik bülteninde ATL kullanılarak geliştirilen ürünlerde bulunabilecek güvenlik açıkları anlatılmaktadır. Bu nedenle, bu güvenlik güncelleştirmesi Microsoft Visual Studio .NET 2003, Microsoft Visual Studio 2005, Microsoft Visual Studio 2008, Microsoft Visual C++ 2005 Yeniden Dağıtılabilir Paketi ve Microsoft Visual C++ 2008 Yeniden Dağıtılabilir Paketi'nin tüm desteklenen sürümleri için Orta olarak derecelendirilmiştir.

Bu güvenlik açıklarından etkilenebilecek denetim ve bileşenlerin etkisi, geçici çözümler ve azaltıcı etkenler hakkında daha fazla bilgi için bkz. [Microsoft Güvenlik Danışma Belgesi (973882)](http://technet.microsoft.com/security/advisory/973882).

Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, ATL üstbilgileri kullanılarak geliştirilen bileşen ve denetimler bir veri akışından güvenli şekilde başlatılacak biçimde ATL üstbilgilerini değiştirerek bu güvenlik açıklarını giderir. Bu güvenlik açıkları hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** ATL kullanarak bileşen ve denetimler geliştirmiş olanların bu güncelleştirmeyi karşıdan yüklemeleri ve şu [MSDN makalesinde](http://go.microsoft.com/?linkid=9674481) açıklanan yönergeleri uygulayarak bileşen ve denetimlerini yeniden derlemeleri gerekmektedir.

Otomatik güncelleştirme özelliğinin etkinleştirilmiş olduğu Visual Studio müşterilerinin büyük çoğunluğu bu güncelleştirmeyi ve güncelleştirilmiş ATL'yi otomatik olarak alacaktır. Ancak önceden de belirtildiği gibi, güvenlik açığı bulunabilecek denetim ve bileşenleri güncelleştirmek için ek adımlar uygulanması gerekmektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi hemen uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** [Microsoft Bilgi Bankası makalesi 969706](http://support.microsoft.com/kb/969706), bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir. Bu makale ayrıca, bu sorunlar için önerilen çözümleri de içermektedir.

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)  
-----------------------------------------------------------------
  
<span></span>
**Bu bülten 12 Ocak 2010'da neden yeniden yayımlanmıştır?**  
Microsoft bu bülteni, etkilenen yazılım olarak Windows Embedded CE 6.0'ı ekleyerek yeniden yayımladı. Windows Embedded CE 6.0 güncelleştirmesi, (KB974616) yalnızca [Microsoft Yükleme Merkezi](http://www.microsoft.com/downloads/details.aspx?familyid=99d114f8-4d95-4075-a0f1-45f498f0ade8)'nden edinilebilen bir güncelleştirme toplamasıdır. Windows Embedded CE 6.0 platformunu kullanan müşteriler bu güncelleştirme toplamasını uygulamalıdır. Bu yeniden yayımlanan sürümden, başka bir güncelleştirme paketi etkilenmemektedir.
  
**Bu bülten 8 Eylül 2009'da neden yeniden düzenlendi?**    
Bu bülten, ilişkili [Microsoft Bilgi Bankası makalesi 969706](http://support.microsoft.com/kb/969706)'daki **Bu güvenlik güncelleştirmesiyle ilgili olduğu bilinen sorunlar** bölümünün yeniden düzenlenerek, KB974223'te açıklanan bilinen sorunu gideren güvenlikle ilgili olmayan bir güncelleştirmenin KB974479'da sunulduğunu duyurmak üzere yeniden düzenlenmiştir. Bu bültende açıklanan güvenlik güncelleştirmesini yüklememiş olan Microsoft Visual Studio 2008 Service Pack 1 kullanıcıları hem güvenlik güncelleştirmesini hem de güvenlikle ilgili olmayan güncelleştirmeyi Microsoft Update aracılığıyla otomatik olarak alacaktır. Güvenlik güncelleştirmesini zaten yüklemiş olan Microsoft Visual Studio 2008 Service Pack 1 kullanıcılarına, güvenlikle ilgili olmayan güncelleştirme Windows Update aracılığıyla otomatik olarak sunulacaktır. Güvenlik güncelleştirmesini zaten yüklemiş olan Microsoft Visual Studio 2008 Service Pack 1 kullanıcılarının bu güvenlik güncelleştirmesini yeniden yüklemeleri gerekmemektedir ve güvenlikle ilgili olmayan güncelleştirmeyi yalnızca KB974479'da açıklanan sorunlarla karşılaşırlarsa yüklemeleri gerekmektedir.
  
**Bu bülten 11 Ağustos 2009'da neden yeniden yayımlandı?**  
Microsoft bu bülteni, aşağıdaki platformlar için yeni güncelleştirmeler sunmak üzere yeniden yayımladı:
  
-   Microsoft Visual Studio 2005 Service Pack 1 (KB973673)  
-   Microsoft Visual Studio 2008 (KB973674)  
-   Microsoft Visual Studio 2008 Service Pack 1 (KB973675)
  
Yeni güvenlik güncelleştirmeleri, Akıllı Aygıtlar için ATL kullanan taşınabilir uygulamalara yönelik olarak Visual Studio kullanarak bileşenler ve denetimler oluşturan geliştiriciler içindir. Visual Studio kullanan tüm geliştiriciler bildirilen sorunlardan etkilenmeyen bileşen ve denetimleri Visual Studio kullanarak oluşturmak için bu yeni güncelleştirmeleri yüklemelidir.
  
Bu bilinen sorun hakkında daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 969706](http://support.microsoft.com/kb/969706).
  
**Bu bülten 4 Ağustos 2009'da neden yeniden düzenlendi?**    
Bu bülten, ilgili [Microsoft Bilgi Bankası makalesi 969706](http://support.microsoft.com/kb/969706)'daki **Bu güvenlik güncelleştirmesiyle ilgili olduğu bilinen sorunlar** bölümünün ve paketin yeniden sunulma sorununu düzeltmek üzere KB973923 ve KB973924 için güncelleştirme algılama mantığının yeniden düzenlendiğini duyurmak üzere yeniden düzenlendi. Özgün algılama mantığı, güncelleştirmenin yüklenmesi gerekip gerekmediğini belirlemek için daha önce yüklenmiş olup olmadığını denetliyordu. Microsoft Yükleme Merkezi'nden edinilebilecek Visual C++ Yeniden Dağıtılabilir paketleri (KB973544, KB973551, KB973552), yalnızca Microsoft Update aracılığıyla edinilebilen karşılık gelen ATL güncelleştirme paketlerinin (KB973923, KB973924) eksiksiz bir üst kümesidir. Visual C++ Yeniden Dağıtılabilir paketinin güncelleştirilmiş sürümünü yüklemiş olan müşterilerin karşılık gelen ATL güncelleştirme paketlerini yüklemeleri gerekmez. Algılama değişiklikleri bunu dikkate alır ve Visual C++ Yeniden Dağıtılabilir güncelleştirmesinin karşılık gelen düzeltilmiş sürümü yüklüyse ATL Güncelleştirme paketlerini yeniden sunmaz. Visual C++ Yeniden Dağıtılabilir paketleri ile diğer Visual Studio güncelleştirmeleri arasındaki farkı açılığa kavuşturmak üzere **Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)** bölümüne yeni bir girdi daha eklenmiştir. Güncelleştirmeleri dağıtmak için Microsoft Update hizmetinin kullanılmadığı ve bu bültende sunulan güncelleştirmelerin başka yollarla dağıtılması gereken kuruluşlarda tam Visual C++ Yeniden Dağıtılabilir paketleri karşıdan yüklenip dağıtılabilir; çünkü bunlar, karşılık gelen ATL güncelleştirme paketlerinin eksiksiz bir üst kümesidir.
  
**Visual C++ Yeniden Dağıtılabilir paketleri ile diğer Visual Studio güncelleştirmeleri arasındaki fark nedir?**    
Visual C++ Yeniden Dağıtılabilir paketleri, bu ürünün başka geliştiriciler tarafından kendi ürünleriyle birlikte yeniden dağıtılabilen tam sürümleridir. Visual C++ Yeniden Dağıtılabilir paketini kullanması gereken müşterilerin bu paketleri edinip kendi kurulumlarına katmaları veya zincir oluşturmaları gerekmektedir. Diğer Visual Studio güncelleştirmeleri, daha önce Visual C++ Yeniden Dağıtılabilir ürününün daha eski bir sürümünü veya Visual Studio ürününü yüklemiş olan müşteriler tarafından kullanılabilir. Bu güncelleştirme paketleri, müşterilere yalnızca Microsoft Update aracılığıyla sunulur ve yalnızca sistemde Visual C++ Yeniden Dağıtılabilir paketinin önceki bir sürümü veya Visual Studio yüklüyse uygulanabilir.
  
**Microsoft Yükleme Merkezi güncelleştirme KB numaraları SMS, SCCM, WSUS ve MU güncelleştirme KB numaralarından neden farklıdır?**    
Düzeltilen Visual C++ 2005 ve 2008 yeniden dağıtılabilir paketlerinin (KB973544, KB973551 ve KB973552) tam sürümleri, bu ürünlerin yeni tam sürümleri olduklarından yalnızca Microsoft Yükleme Merkezi'nde listelenmektedir. SMS, SCCM, WSUS ve MU (KB973923, KB9739234) üzerinde listelenen güncelleştirmeler yalnızca Visual C++ yeniden dağıtılabilir paketlerinin güvenlik açığından etkilenen sürümlerini yüklemiş olan müşteriler içindir. Bu güncelleştirmeler, Yükleme Merkezi'ndeki sürümler değildir. Microsoft, müşterilerin Microsoft Yükleme Merkezi'nden yüklenebilecek tam sürümler (KB973544, KB973551 ve KB973552) dışındaki sürümleri yeniden dağıtmamalarını önerir.
  
**Bu güvenlik güncelleştirmesi neden normal yayın döngüsü dışında yayımlanmıştır?**    
Bu güvenlik güncelleştirmesi, "Microsoft Etkin Şablon Kütüphanesi'ndeki (ATL) Güvenlik Açıkları Uzaktan Kod Yürütülmesine İzin Verebilir" adlı [Microsoft Güvenlik Danışma Belgesi (973882)](http://technet.microsoft.com/security/advisory/973882) kapsamında açıklanan sorunları gidermek için normal yayın döngüsü dışında yayımlanmıştır.
  
**Bu bülten ile Microsoft Güvenlik Danışma Belgesi'nde (973882) açıklanan güvenlik açıkları arasında nasıl bir ilişki var?**  
Bu bültende, Etkin Şablon Kütüphanesi'nin (ATL) genel sürümündeki güvenlik açıkları anlatılmaktadır. ATL'nin özel sürümündeki güvenlik açıkları [Microsoft Güvenlik Danışma Belgesi (973882)](http://technet.microsoft.com/security/advisory/973882) içinde anlatılmaktadır.
  
**Bunlar uzaktan kod yürütme güvenlik açıklarıysa bu bülten neden yalnızca Orta olarak derecelendirilmiş?**    
Bu bültenin Orta olarak derecelendirilmesinin nedeni, Microsoft Visual Studio'nun varsayılan olarak bu güvenlik açıklarından etkilenmemesidir. Güvenlik açıklarından etkilenen ATL sürümleri kullanılarak geliştirilen denetim ve bileşenler risk altındadır ve etkileniyorlarsa, denetimde veya bileşende bulunan güvenlik açığı türüne bağlı olarak Kritik veya Orta olarak derecelendirilir.
  
**Visual Studio kullanıcıları bu güvenlik açıklarından doğrudan etkileniyor mu?**    
Hayır. Visual Studio'nun sisteminizde yüklü olması, bu sorundan etkileneceğiniz anlamına gelmez.
  
Bu bültenle ilişkilendirilmiş güncelleştirme, Visual Studio kullanarak bileşen ve denetim geliştiricilerin bildirilen sorunlardan etkilenmeyen bileşen ve denetimler oluşturabilmeleri için hazırlanmıştır. Yalnızca etkilenen bir ATL sürümüyle geliştirilmiş bileşen ve denetimler bu güvenlik açıklarından etkilenir. ATL'nin etkilenen sürümleriyle bileşen ve denetim geliştirmiş veya yeniden dağıtmış olan geliştiricilerin bu bültende sağlanan güncelleştirmeyi yüklemeleri ve bu bültende açıklanan güvenlik açıklarının bileşen ve denetimlerinde bulunmadığından emin olmak için sağlanan adımları uygulamaları gerekmektedir.
  
**Etkin Şablon Kütüphanesi'nin hangi sürümleri bu güvenlik açıklarından etkilenir?**  
7.0, 7.1, 8.0 ve 9.0 sürümleri etkilenir. Başka hiçbir sürüm desteklenmemektedir.
  
**Visual Studio'nun hangi sürümleri etkilenir?**  
Lütfen bu Microsoft Güvenlik Bülteni'nin **Etkilenen ve Etkilenmeyen Yazılımlar** bölümüne bakın.
  
**Bu güvenlik güncelleştirmesi aynı şekilde normal yayın döngüsü dışında bir güncelleştirme olarak yayımlanan MS09-034 ile ilişkili mi?**    
Evet. "Internet Explorer için Toplu Güvenlik Güncelleştirmesi" adlı Microsoft Güvenlik Bülteni [MS09-034](http://go.microsoft.com/fwlink/?linkid=158199), ilişkili olmayan birkaç güvenlik açığını gidermesinin yanı sıra, ATL'nin güvenlik açığından etkilenen sürümleri kullanılarak geliştirilen bileşen ve denetimlerden Internet Explorer'da yararlanılmasını engelleyen bir azaltıcı etken de içerir. MS09-034'te sunulan yeni kapsamlı korumalar, ActiveX kill bitiyle sağlanan güvenlik özelliğinin atlanmasına yol açabilecek güvenlik açıkları da dahil olmak üzere tüm bilinen genel ve özel ATL güvenlik açıklarını izleyen ve bunlardan yararlanılmasını engelleyen Internet Explorer 5.01, Internet Explorer 6, Internet Explorer 6 Service Pack 1, Internet Explorer 7 ve Internet Explorer 8 güncelleştirmelerini içerir. Bu korumalar, müşterilerin Web tabanlı saldırılara karşı korunmalarına yardımcı olacak biçimde tasarlanmıştır.
  
**MS09-034'te açıklanan güncelleştirmeyi yüklediysem, bu güncelleştirmeyi yine de yüklemem gerekiyor mu?**

ATL'de Başlatılmamış Nesne Güvenlik Açığı - CVE-2009-0901  
---------------------------------------------------------
  
<span></span>
Microsoft Etkin Şablon Kütüphanesi'nde (ATL), ATL üstbilgilerinde bir saldırganın doğru başlatılmamış bir VARIANT için VariantClear işlevini zorlamasına izin verebilecek bir sorun nedeniyle uzaktan kod yürütme güvenlik açığı bulunmaktadır. Bu nedenle saldırgan, bir hatanın işlenmesi sırasında VariantClear işlevi çağrıldığında gerçekleştirilen işlemi bozuk bir akış sağlayarak denetleyebilir. Bu güvenlik açığı yalnızca Visual Studio ATL kullanılarak geliştirilmiş bileşen ve denetimlerin yüklü olduğu sistemleri doğrudan etkiler. Bu sorun, kimliği doğrulanmamış uzak kullanıcıların etkilenen sistemde uzaktan kod yürütebilmesini sağlar. Saldırgan, özel hazırlanmış bir Web sayfası oluşturarak bu güvenlik açığından yararlanabilir. Bir kullanıcı bu Web sayfasını görüntülediğinde, güvenlik açığı uzaktan kod yürütülmesine izin verebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2009-0901](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-0901).
  
#### ATL'de Başlatılmamış Nesne Güvenlik Açığı - CVE-2009-0901 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Visual Studio ürünü varsayılan olarak bu sorundan etkilenmez. Bunun yerine, ATL'nin güvenlik açığı bulunan sürümleri kullanılarak geliştirilmiş bileşen ve denetimler bu güvenlik açığından etkilenebilir.  
-   Güvenlik açığı bulunabilecek bileşen ve denetimlere yönelik Azaltıcı Etkenler ve Geçici Çözümler, [Microsoft Güvenlik Danışma Belgesi (973882)](http://technet.microsoft.com/security/advisory/973882) içinde açıklanmaktadır.
  
#### ATL'de Başlatılmamış Nesne Güvenlik Açığı - CVE-2009-0901 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   Güvenlik açığı bulunabilecek bileşen ve denetimlere yönelik Azaltıcı Etkenler ve Geçici Çözümler, [Microsoft Güvenlik Danışma Belgesi (973882)](http://technet.microsoft.com/security/advisory/973882) içinde açıklanmaktadır.
  
#### ATL'de Başlatılmamış Nesne Güvenlik Açığı - CVE-2009-0901 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**    
Bu güvenlik açığı yalnızca Microsoft ATL kullanılarak geliştirilen ve güvenlik açığından etkilenen bileşen ve denetimlerin yüklü olduğu sistemleri doğrudan etkiler.
  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Örneğin, kullanıcı özel hazırlanmış bir Web sayfasını Internet Explorer kullanarak ziyaret edip bu güvenlik açığından etkilenen bir bileşenin veya denetimin bir örneğini oluşturursa bu güvenlik açığı uzaktan kod yürütülmesine izin verebilir. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açan kullanıcı ile aynı haklara sahip olabilir.
  
Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
**Güvenlik açığına ne neden olmaktadır?**    
Güvenlik açığı, ATL üstbilgilerinde bir saldırganın doğru başlatılmamış bir değişken için VariantClear() işlevini çağırmasına izin verebilecek bir sorun nedeniyle ortaya çıkar. ATL kullanılarak bu şekilde geliştirilen bir bileşen veya denetim, oturum açmış kullanıcının güvenlik bağlamında uzaktan kod yürütülmesine izin verebilir.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**    
Kullanıcının sisteminde bu güvenlik açığından etkilenen bir denetim varsa, saldırgan [Microsoft Güvenlik Danışma Belgesi (973882)](http://technet.microsoft.com/security/advisory/973882) içinde açıklanan azaltıcı etkenleri atladığında etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**    
Bir saldırgan, bu güvenlik açığından etkilenen bir denetimden yararlanmayı deneyip açıktan Internet Explorer aracılığıyla yararlanacak ve kullanıcıyı Web sitesini görüntülemeye ikna edecek biçimde tasarlanan özel hazırlanmış bir Web sitesi barındırabilir. Buna, güvenliği aşılan Web siteleri ve kullanıcı tarafından sağlanan içeriği veya reklamları kabul eden ya da barındıran Web siteleri de dahil olabilir. Bu Web sitelerinde bu güvenlik açığından yararlanan özel hazırlanmış içerik bulunabilir. Ancak bu durumların tümünde, saldırganın, bu güvenlik açığından etkilenen bir denetim bulması ve kullanıcıları bu Web sitelerini ziyaret etmeye zorlaması gerekmektedir. Saldırgan bunu yapmak için, kullanıcıları bir e-posta iletisindeki veya Anlık İleti isteğindeki kendi Web sitesine götüren bir bağlantıyı tıklatmalarını sağlayarak onları bu Web sitesine çekmelidir. Etkilenen sistemlere Web içeriği ulaştırmak için başlık reklamları veya başka yöntemler kullanılarak da özel olarak oluşturulmuş Web içeriği görüntülenebilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**    
İş istasyonları ve terminal sunucuları birincil derecede risk altındadır. Sunucular, yeterli yönetimsel izinlere sahip olmaması gereken kullanıcılara sunucularda oturum açma ve programları çalıştırma olanağı verilirse daha fazla etkilenebilir. Ancak, en iyi uygulamalarda buna izin verilmesi kesinlikle önerilmemektedir.
  
**Güncelleştirme ne yapıyor?**    
Güncelleştirme, VariantClear() işlevinin yalnızca başlatılmış değişkenlerde çağrılabilmesini sağlamak yoluyla bu sorunu giderir ve geliştiricilerin bu güvenlik açığından etkilenebilecek denetimlerde bu sorunu gidermelerine olanak veren güncelleştirilmiş ATL sürümleri sağlar.
