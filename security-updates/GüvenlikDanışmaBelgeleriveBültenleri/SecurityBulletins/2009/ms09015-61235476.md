Security Bulletin

Microsoft Güvenlik Bülteni MS09-015 - Orta
==========================================

SearchPath İşlevindeki Karma Tehdit Güvenlik Açığı Ayrıcalık Yükselmesine İzin Verebilir (959426)
-------------------------------------------------------------------------------------------------

Yayım Tarihi: 14 Nisan 2009 Salı | Güncelleştirme Tarihi: 15 Nisan 2009 Çarşamba

**Sürüm:** 1.1

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi, bir kullanıcı özel hazırlanmış bir dosyayı belirli bir konuma karşıdan yükleyip daha sonra dosyayı belirli koşullarda yükleyebilecek bir uygulamayı açarsa Windows SearchPath işlevinde ayrıcalık yükselmesine neden olabileceği genel olarak duyurulan bir güvenlik açığını giderir.

Güvenlik güncelleştirmesi Windows XP, Windows Server 2003, Windows Vista ve Windows Server 2008'in tüm desteklenen sürümleri için Orta olarak derecelendirilmiştir. Microsoft Windows 2000 için, bu güncelleştirme yalnızca kapsamlı bir koruma değişikliği olarak sınıflandırılmıştır. Ayrıntılı bilgi, **Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)** bölümünde bulunmaktadır. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, Windows'un masaüstündeki dosyaları yükleme biçimini değiştirerek bu güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

Bu güvenlik güncelleştirmesi ilk olarak [Microsoft Güvenlik Danışma Belgesi 953818](http://technet.microsoft.com/security/advisory/953818)'de açıklanan güvenlik açığını da giderir.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 294781](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güvenlik güncelleştirmesini hemen uygulamalarını önerir.

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)
-----------------------------------------------------------------

<span></span>
**Dosya bilgileri ayrıntılarını nerede bulabilirim?**  
Dosya bilgileri ayrıntıları [Microsoft Bilgi Bankası makalesi 959426](http://support.microsoft.com/kb/959426)'da bulunabilir.

**Bu güvenlik güncelleştirmesi MS09-014 ile nasıl ilişkilidir?**  
Bu güvenlik bülteninde açıklanan Karma Tehdit Oluşturabilecek Ayrıcalık Yükselmesi Güvenlik Açığı, [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540), aynı zamanda Internet Explorer'ı da etkiler. Microsoft, kullanıcıların Internet Explorer Toplu Güvenlik Güncelleştirmesi (963027), [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659)'te sağlanan güvenlik güncelleştirmesini de yüklemelerini önerir.

**Bu güncelleştirme, işlevlerde güvenlikle ilgili değişiklikler içeriyor mu?**  
Evet. Bu bültenin **Güvenlik Açığı Bilgileri** bölümünde listelenen değişikliklerin yanı sıra, bu güncelleştirme, SetSearchPathMode adlı yeni bir API'nın kullanıldığı kapsamlı bir koruma değişikliği de içerir.

**SetSearchPathMode kapsamlı koruma değişikliği ne yapıyor?**  
Bu yeni API, dosyaları bulmak üzere SearchPath işlevinin her işlem için ayrı ayrı kullanılabilmesini sağlar. Böylece uygulamaların, uygulama ve sistem konumlarından sonra geçerli dizinin aranmasını zorlayabilmeleri sağlanır.

**Yalnızca bu güvenlik güncelleştirmesini uygularsam CVE-2008-2540'ta açıklanan karma tehdit sorununa karşı tümüyle korunur muyum?**  
Hayır, Internet Explorer Toplu Güvenlik Güncelleştirmesi, [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659)'te açıklanan kapsamlı korumayı da etkinleştirmelisiniz. Ancak, karma tehdit sorununa karşı bu kapsamlı korumayı çalışma ortamınızda etkinleştirmenin uygun olduğunu doğrulamanız gerekmektedir.

Internet Explorer'da kapsamlı korumayı etkinleştirmek için, [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659)'teki Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS) bölümünde açıklanan SearchPath özelliğinin denetim anahtarını etkinleştirin. Karma tehdit sorunu hakkında daha fazla bilgi için, bkz: Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540).

**Etkilenen Yazılımlar tablosunda Microsoft Windows 2000 neden bir güvenlik etkisi ve önem derecesi olmadan listeleniyor?**
Microsoft Windows 2000'in bir güvenlik etkisi ve önem derecesi olmadan listelenmesinin nedeni, bu güncelleştirmenin Microsoft Windows 2000 için diğer desteklenen Windows işletim sistemleriyle aynı düzeltmeyi uygulamamasıdır. Bunun yerine, bu güncelleştirme yalnızca Microsoft Windows 2000'e özgü kapsamlı bir koruma değişikliği içerir.

**Microsoft Windows 2000'deki karma tehdit sorununa yönelik düzeltme neden diğer platformlardan farklıdır? **
SetSearchPathMode işlevini uygulamaya yönelik mimari Microsoft Windows 2000 sisteminde bulunmadığı için, güvenlik açığının Microsoft Windows 2000'de giderilmesi mümkün değildir. Bunun yapılabilmesi için yalnızca etkilenen bileşenin değil, Microsoft Windows 2000 işletim sisteminin büyük bir bölümünün mimarisinin yeniden geliştirilmesi gerekmektedir. Microsoft Windows 2000 üzerinde çalışacak şekilde tasarlanmış uygulamaların yeniden mimarisi oluşturulan sistemde çalışmaya devam edeceği garanti edilemediğinden, bu tür bir çalışmanın sonuçları da büyük olasılıkla Microsoft Windows 2000 ile uyumlu olmayacaktır. Ancak Microsoft, Microsoft Windows 2000 üzerinde bu güvenlik açığını hedef alabilecek geçerli saldırı vektörlerinin bulunduğuna dair henüz bir rapor almamıştır.

Microsoft Windows 2000'deki bu mimari değişiklikleri nedeniyle, HKLM\\System\\CCS\\control\\Session Manager\\SafeDLLSearchMode kayıt defteri anahtarını kullanarak Microsoft Windows 2000'e özgü kapsamlı savunma değişikliğini etkinleştirmeniz gerekmektedir. Yönergeler için, bkz: [Microsoft Bilgi Bankası makalesi 959426](http://support.microsoft.com/kb/959426). Daha fazla bilgi için, [Dinamik Bağlantı Kitaplığı Arama Sırası](http://msdn.microsoft.com/en-us/library/ms682586.aspx) adlı MSDN makalesine bakın.

**SearchPath işlevi nedir?**  
SearchPath işlevi, belirtilen bir yoldaki belirli bir dosyayı aramak için kullanılır. Daha fazla bilgi için, [SearchPath İşlevi](http://msdn.microsoft.com/en-us/library/aa365527.aspx) adlı MSDN makalesine bakın.

**LoadLibrary işlevi nedir?**  
LoadLibrary işlevi, belirtilen modülü, çağrıyı yapan işlemin adres alanına yükler. Belirtilen modül, başka modüllerin yüklenmesine neden olabilir. Daha fazla bilgi için, [LoadLibrary İşlevi](http://msdn.microsoft.com/en-us/library/ms684175(vs.85).aspx) adlı MSDN makalesine bakın.

Karma Tehdit Oluşturabilecek Ayrıcalık Yükselmesi Güvenlik Açığı - CVE-2008-2540  
--------------------------------------------------------------------------------
  
<span></span>
Windows'daki SearchPath işlevinin sistemdeki dosyaları bulma ve açma biçiminde, ayrıcalık yükselmesine olanak veren bir karma tehdit güvenlik açığı bulunmaktadır. Bir saldırgan, kullanıcıyı özel hazırlanmış bir dosyayı belirli bir konuma karşıdan yükleyip daha sonra bu dosyayı belirli koşullarda yükleyebilecek bir uygulamayı açmaya ikna ederek bu güvenlik açığından yararlanabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540).
  
#### Karma Tehdit Oluşturabilecek Ayrıcalık Yükselmesi Güvenlik Açığı - CVE-2008-2540 için Azaltıcı Etkenler
  
Microsoft bu güvenlik açığı için herhangi bir azaltıcı etken belirleyememiştir.
  
#### Karma Tehdit Oluşturabilecek Ayrıcalık Yükselmesi Güvenlik Açığı - CVE-2008-2540 için Geçici Çözümler

#### Karma Tehdit Oluşturabilecek Ayrıcalık Yükselmesi Güvenlik Açığı - CVE-2008-2540 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu, karma tehdit oluşturabilecek bir ayrıcalık yükselmesi güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, etkilenen sistemin tüm denetimini uzaktan ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
Windows çalışan sistemlerde uygulama kitaplıklarının konumu nedeniyle, Internet Explorer, masaüstündeki özel hazırlanmış bir dosyayı açabilir. Kullanıcının sistemine dosyaların sorulmadan karşıdan yüklenebilmesine olanak veren karma tehdit, Internet Explorer'ın bu karşıdan yüklenen dosyayı Windows sistemi yerine istenmeden masaüstünden yüklemesine neden olabilir.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan rasgele kod çalıştırabilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Bir saldırgan özel hazırlanmış bir dosya oluşturabilir ve kullanıcıyı bu dosyayı masaüstüne karşıdan yüklemeye ikna edebilir. Kullanıcının daha sonra bu özel hazırlanmış dosyayı açacak bir uygulama başlatması gerekir. Ancak saldırgan, kullanıcıları bu tür dosyaları karşıdan yükleyip masaüstlerinde konumlandırmaya zorlama yolu yoktur.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
İş istasyonları ve terminal sunucuları gibi, kullanıcıların dosya karşıdan yükledikleri ve uygulama başlattıkları tüm sistemler bu güvenlik açığına karşı risk altındadır.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, geçerli çalışma dizininden önce sistem dizini aranacak şekilde, uygulama kitaplıklarını arama sırasını değiştirir. Güncelleştirme ayrıca ileri düzey kullanıcıların SearchPath işlevi için istedikleri arama sırasını ayarlamalarına olanak veren SetSearchPathMode adlı yeni bir API'nın kullanıldığı kapsamlı bir koruma değişikliği de içerir. Bu kapsamlı koruma değişikliği hakkında daha fazla bilgi için, **Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)** bölümüne de bakın.
  
**Internet Explorer için Toplu Güvenlik Güncelleştirmesi Explorer (963027) başlıklı MS09-014'te bu güvenlik açığına ve Karma Tehdit Oluşturabilecek Uzaktan Kod Yürütme Güvenlik Açığı'na neden aynı CVE numarası (CVE-2008-2540) atanmış?**  
[CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) hem Windows XP Service Pack 2 ve Windows XP Service Pack 3 için Internet Explorer 7'yi ve Windows XP Professional x64 Edition ve Windows XP Professional x64 Edition Service Pack 2 için Internet Explorer 7'yi hem de Windows'un SearchPath işlevini etkiler. Microsoft, kullanıcıların Internet Explorer Toplu Güvenlik Güncelleştirmesi (963027), [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659)'te sağlanan güvenlik güncelleştirmesini de yüklemelerini önerir. Bu güvenlik açığına yönelik olarak bu güncelleştirmedeki kapsamlı savunma değişikliği (SetSearchPathMode adlı API) aynı zamanda [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659)'teki düzeltmeye karışık gelir. **Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)** bölümündeki Bu güncelleştirme, işlevlerde güvenlikle ilgili değişiklikler içeriyor mu?" sorusuna da bakın.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Evet. Bu güvenlik açığı genel olarak duyurulmuştur. Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) atanmıştır. Bu güvenlik güncelleştirmesi ilk olarak [Microsoft Güvenlik Danışma Belgesi 953818](http://technet.microsoft.com/security/advisory/953818)'de açıklanan güvenlik açığını da giderir.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.

#### Dağıtım Bilgileri
  
**Güncelleştirmeyi Yükleme**
  
Bu güvenlik güncelleştirmesini yüklediğinizde, yükleyici, sisteminizde güncelleştirilmekte olan dosyalardan bir veya daha fazlasının daha önce Microsoft tarafından yapılan bir düzeltme ile güncelleştirilmiş olup olmadığını denetler.
  
Bu bültende kullanılan *düzeltme* gibi terimler hakkında daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 824684](http://support.microsoft.com/kb/824684).
  
Bu güvenlik güncelleştirmesi aşağıdaki kurulum anahtarlarını destekler:
  
| Anahtar           | Açıklama                                                                                                                                                  |  
|-------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Desteklenen anahtarlarla ilgili yardım görüntüler.                                                                                                        |  
| **/quiet**        | Durum veya hata iletilerinin görüntülenmesini bastırır.                                                                                                   |  
| **/norestart**    | **/quiet** anahtarıyla birlikte kullanılırsa, yükleme işlemini tamamlamak için yeniden başlatma gerekse de yükleme sonrasında sistem yeniden başlatılmaz. |
  
**Not** Wusa.exe yükleyicisi hakkında daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 934307](http://support.microsoft.com/kb/934307).
  
**Güncelleştirmenin Uygulandığını Doğrulama**
  
-   **Microsoft Baseline Security Analyzer**
  
    Etkilenen bir sisteme güvenlik güncelleştirmesinin uygulandığını doğrulamak için Microsoft Baseline Security Analyzer (MBSA) aracını kullanabilirsiniz. Daha fazla bilgi için, bu bültenin önceki bölümünde yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** alt bölümüne bakın.
  
-   **Dosya Sürümü Doğrulaması**
  
    Microsoft Windows'un birçok sürümü olduğundan, aşağıdaki adımlar sizin sisteminizde farklı olabilir. Böyle bir durumda, bu adımları tamamlamak için ürün belgelerinize bakın.
  
    1.  **Başlat**'ı tıklatın ve **Aramaya Başla** alanına bir güncelleştirme dosyası adı girin.  
    2.  Dosya **Programlar** altında göründüğünde, dosya adını sağ tıklatın ve sonra da **Özellikler**'i tıklatın.  
    3.  **Genel** sekmesi altındaki dosya boyutunu, bültenle ilgili BB makalesinde sağlanan dosya bilgileri tablolarıyla karşılaştırın.  
    4.  Dilerseniz, **Ayrıntılar** sekmesini tıklatarak dosya sürümü ve değiştirilme tarihi gibi bilgileri de bültenle ilgili BB makalesinde sağlanan dosya bilgileri tablolarıyla karşılaştırabilirsiniz.  
    5.  Son olarak, **Önceki Sürümler** sekmesini tıklatarak dosyanın önceki sürümünün dosya bilgilerini, yeni veya güncelleştirilmiş sürümünün dosya bilgileriyle de karşılaştırabilirsiniz.
