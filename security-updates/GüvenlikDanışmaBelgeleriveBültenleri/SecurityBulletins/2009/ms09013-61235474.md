Security Bulletin

Microsoft Güvenlik Bülteni MS09-013 - Kritik
============================================

Windows HTTP Hizmetleri'ndeki Güvenlik Açıkları Uzaktan Kod Yürütülmesine İzin Verebilir (960803)
-------------------------------------------------------------------------------------------------

Yayım Tarihi: 14 Nisan 2009 Salı | Güncelleştirme Tarihi: 29 Nisan 2009 Çarşamba

**Sürüm:** 1.1

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi, Microsoft Windows HTTP Hizmetleri'ndeki (WinHTTP) genel olarak bildirilen bir ve özel olarak bildirilen iki güvenlik açığını giderir. En ciddi güvenlik açığı uzaktan kod yürütülmesine olanak verebilir. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

Güvenlik güncelleştirmesi Microsoft Windows 2000, Windows XP, Windows Server 2003, Windows Vista ve Windows Server 2008'in tüm desteklenen sürümleri için Kritik olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, Windows HTTP Hizmetleri'nin hataları işleme ve sertifikaları doğrulama biçimini değiştirip, NTLM kimlik bilgilerini yansıtma koruması mekanizmalarını doğru şekilde kullanmasını sağlayarak bu güvenlik açıklarını giderir. Bu güvenlik açıkları hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi hemen uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** [Microsoft Bilgi Bankası makalesi 960803](http://support.microsoft.com/kb/960803), bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir. Bu makale ayrıca, bu sorunlar için önerilen çözümleri de içermektedir.

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)
-----------------------------------------------------------------

<span></span>
**Bu bülten 29 Nisan 2009'da neden yeniden düzenlendi?**  
Bu bülten, ilgili [Microsoft Bilgi Bankası makalesi 960803](http://support.microsoft.com/kb/960803)'teki **Güvenlik güncelleştirmesiyle ilgili olduğu bilinen sorunlar** bölümünün, bu güncelleştirme ile Windows Sınama/Yanıt kimlik doğrulaması (NTLM) kullanacak biçimde yapılandırılmış Websense Logon Agent arasındaki bir uygulama uyumluluğu sorunu tanımlanacak biçimde güncelleştirildiğini duyurmak üzere yeniden düzenlendi.

**Dosya bilgileri ayrıntılarını nerede bulabilirim?**  
Dosya bilgileri ayrıntıları [Microsoft Bilgi Bankası makalesi 960803](http://support.microsoft.com/kb/960803)'te bulunabilir.

**Windows Server 2008 Service Pack 2 Beta, Windows Vista Service Pack 2 Beta ve Windows 7 Beta sürümleri bu güvenlik açığından etkileniyor mu?**
Evet. Bu güvenlik açıkları Windows Server 2008 Service Pack 2 Beta, Windows Vista Service Pack 2 Beta ve Windows 7 Beta yayımlandıktan sonra bildirilmiştir. Bu platformları çalıştıran müşterilerin güncelleştirmeyi karşıdan yükleyip sistemlerine uygulamaları önerilir.

Güvenlik güncelleştirmeleri [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) ve [Windows Update](http://go.microsoft.com/fwlink/?linkid=21130) sitelerinden edinilebilir.

**Bu güvenlik güncelleştirmesi MS09-014 ile nasıl ilişkilidir?**  
Bu güvenlik bülteninde açıklanan Windows HTTP Hizmetleri'nde Kimlik Bilgilerini Yansıtma Güvenlik Açığı, [CVE-2009-0550](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-0550) aynı zamanda Internet Explorer'ı ve Windows Internet (WinINet) API'sını da etkiler. Microsoft, Internet Explorer kullanıcılarının Internet Explorer Toplu Güvenlik Güncelleştirmesi (963027), [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659)'te sağlanan güvenlik güncelleştirmesini de yüklemelerini önerir.

**Müşterilerin bu güvenlik güncelleştirmesini yüklerken karşılaşabilecekleri bilinen sorunlar nelerdir?**  
[Microsoft Bilgi Bankası makalesi 960803](http://support.microsoft.com/kb/960803), bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir. Bu makale ayrıca, bu sorunlar için önerilen çözümleri de içermektedir.

**Bu güncelleştirme neden bildirilen birkaç güvenlik açığını ele alıyor?**  
Bu güncelleştirme, bazı güvenlik açıklarını ortadan kaldırmaya yönelik değişikliklerin ilgili dosyalarda bulunması nedeniyle bu sorunlar için destek içermektedir. Müşterilerimiz birbiriyle neredeyse aynı birkaç güncelleştirme yerine, yalnızca bu güncelleştirmeyi yüklemelidir.

**Bu güncelleştirme, işlevlerde güvenlikle ilgili değişiklikler içeriyor mu?**  
Evet; CVE-2009-0089'da açıklanan sorunu gidermek üzere, Microsoft, SOFTWARE\\Microsoft\\Windows\\CurrentVersion\\Internet Settings\\WinHttp altında UseFQDNFallbackForCnCompare adlı yeni bir kayıt defteri anahtarı oluşturmuştur. Bu kayıt defteri anahtarı, bir ana bilgisayarın tam etki alanı adını sertifikanın ayırt edici ad girdisine göre doğrularken WinHTTP'nin davranışını denetlemektedir.

Bu kayıt defteri anahtarı varsayılan olarak bulunmamaktadır; böylece, sertifikanın içerdiği ayırt edici adın, bağlantının başlatıldığı DN ile aynı olduğu WinHTTP tarafından doğrulanır. Bu kayıt defteri anahtarı True olarak ayarlanarak eski davranış yeniden etkinleştirilebilir. Bu durumda, doğrulama işlemi daha esnektir ve sertifika DNS düzeyinde iletim yoluyla bir ad değişikliğiyle eşlenirse kabul edilir. Bu güvenlik güncelleştirmesi yüklenmeden önce daha az güvenli olan varsayılan davranış bu şekildedir.

**Bu güvenlik bülteninde açıklanan yazılımın eski bir sürümünü kullanıyorum. Ne yapmalıyım?**  
Bu bültende listelenen etkilenen yazılımlar, hangi sürümlerinin etkilendiğini belirlemek amacıyla sınanmıştır. Diğer sürümler destek ömrünü tamamlamıştır. Yazılım sürümünüzün destek ömrünü belirlemek için [Microsoft Destek Ömrü](http://go.microsoft.com/fwlink/?linkid=21742) Web sitesini ziyaret edin.

Bu yazılımların eski sürümlerini kullanan müşterilerimiz, ortaya çıkabilecek güvenlik açıklarından etkilenme olasılığını ortadan kaldırmak için desteklenen sürümlere acilen yükseltmelidir. Windows Ürün Destek Ömrü hakkında daha fazla bilgi için [Microsoft Destek Ömrü](http://go.microsoft.com/fwlink/?linkid=21742) Web sitesini ziyaret edin. Bu yazılım sürümleri için genişletilmiş güvenlik güncelleştirmesi destek süreci hakkında daha fazla bilgi için, [Microsoft Ürün Destek Hizmetleri](http://go.microsoft.com/fwlink/?linkid=33328)'ni ziyaret edin.

Yazılımların eski sürümleri için ek özel destek isteyen müşteriler, Microsoft hesap ekibi temsilcilerine, Teknik Hesap Yöneticisi'ne veya özel destek seçenekleri için uygun Microsoft ortağı temsilcisine başvurmalıdır. Alliance, Premier veya Authorized sözleşme sahibi olmayan müşteriler yerel Microsoft satış ofisine de başvurabilir. Başvuru bilgileri için, [Microsoft Worldwide Information](http://go.microsoft.com/fwlink/?linkid=33329) Web sitesini ziyaret edin, bulunduğunuz ülkeyi seçin ve **Go**'yu (Git) tıklatıp telefon numaralarının listesini görüntüleyin. Aradığınızda, yerel Premier Destek satış yöneticisiyle görüşmek istediğinizi söyleyin. Daha fazla bilgi için, bkz: [Windows İşletim Sistemlerinin Ürün Destek Ömrü Hakkında SSS](http://go.microsoft.com/fwlink/?linkid=33330).

#### Windows HTTP Hizmetleri'nde Tamsayı Boşalması Güvenlik Açığı - CVE-2009-0086 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Böylece saldırgan programlar yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; saldırganın Web sunucusuna bağlanmak üzere WinHTTP API'sını çağıran hizmet veya uygulamayla aynı kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.

**Güvenlik açığına ne neden olmaktadır?**  
Güvenlik açığı, uzak Web sunucusu tarafından döndürülen bir parametrenin doğrulanmaması nedeniyle oluşur. Bu durum rasgele kod yürütülmesine yol açabilir.
  
**Windows HTTP Hizmetleri nedir?**  
Windows HTTP Hizmetleri (WinHTTP), geliştiricilerin Web sunucularına HTTP protokolü aracılığıyla istek gönderebilmeleri için bir HTTP istemcisi uygulama programı arabirimi (API) sağlar. WinHTTP hem Microsoft Windows bileşenleri hem de üçüncü taraf uygulamaları tarafından kullanılabilir. WinHTTP ayrıca Evrensel Tak ve Kullan (UPNP) Hizmeti tarafından da kullanılır.
  
**Evrensel Tak ve Kullan nedir?**  
Evrensel Tak ve Kullan (UPnP), ağ üzerindeki aygıtların diğer aygıtları bulmalarına ve onlarla nasıl çalışacaklarını belirlemelerine olanak veren bir özelliktir. UPnP, çoğu Windows kullanıcısının zaten ne olduğunu bildiği tak ve kullan (PnP) özelliğiyle karşılaştırılarak kolayca anlaşılabilir. PnP, bir sisteme yeni donanım eklediğinizde işletim sisteminin bunu algılamasına olanak tanır. Örneğin sisteminize yeni bir fare taktığınızda, PnP özelliği, Windows'un bu aygıtı algılamasını, gerekli sürücüleri yüklemesini ve kullanmaya başlamasını sağlar. UPnP bu kavramı, yerel sistem yerine ağ üzerindeki aygıtları da kapsayacak şekilde genişletir. UPnP, bilgisayarların ağ üzerindeki diğer aygıtlar hakkında bilgi edinebilmelerini ve bunları nasıl kullanacaklarını belirleyebilmelerini sağlar. Örneğin bir bilgisayar, UPnP özelliğini kullanarak, ağ üzerinde kullanabileceği yazıcılar olup olmadığını belirleyebilir ve bunları nasıl kullanacağını öğrenebilir.
  
**Basit Hizmet Algılama Protokolü (SSDP) nedir?**  
Aygıt bulma, UPnP özelliğini destekleyen bilgisayarların kullanabilecekleri aygıtları belirlemeleri ve bu aygıtlardan hizmet isteklerinde nasıl bulunacaklarını öğrenme işlemidir. UPnP özelliğini destekleyen bir bilgisayara önyükleme yapıldığında, ağ üzerinde bu bilgisayarın kullanabileceği aygıtlar olabilir. Bu durumun geçerli olup olmadığını belirlemek için, bilgisayar bir yayın isteği (M-SEARCH yönergesi) göndererek, alt ağ içindeki UPnP özelliğini destekleyen tüm aygıtların doğrudan bu isteği yanıtlamalarını ve aygıtın kullanımıyla ilgili bilgileri sağlamalarını ister.
  
Aynı şekilde, UPnP desteği sağlayan bir aygıta (örneğin, UPnP özelliğini destekleyen bir yazıcıya) önyükleme yapıldığında, ağ üzerinde bu aygıtı kullanmak isteyebilecek UPnP özelliğini destekleyen bilgisayarlar bulunabilir. Bu aygıt alt ağ içindeki tüm bilgisayarlara bir ileti (NOTIFY yönergesi) yayınlayarak ağ üzerindeki varlığını duyurur ve bilgisayarları kendi hizmetlerinden yararlanmaya davet eder.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, kendi Web sunucusuna bağlanmak üzere WinHTTP API'sını çağıran hizmet veya uygulamayla aynı kullanıcı haklarına sahip olabilir. Sistemde daha az kullanıcı hakkına sahip olacak biçimde yapılandırılmış hesaplar altında çalışan uygulamalar, yönetimsel kullanıcı haklarına sahip olan hesaplar altında çalıştırılan uygulamalara göre daha az etkilenirler.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırganın bir programı kendi kötü amaçlı Web sunucusuna bağlanmak için Windows HTTP Hizmetleri'ni kullanmaya zorlaması veya ikna etmesi gerekir. Evrensel Tak ve Kullan (UPnP) hizmetinin WinHTTP kitaplıklarını kullandığının bilinmesi önemlidir. Bu hizmet etkinleştirildiğinde, yerel alt ağdaki kötü amaçlı kullanıcılar SSDP isteklerine yanıt verebilirler ve UPnP hizmetinin bu güvenlik açığından yararlanılabilecek kötü amaçlı bir ana bilgisayara WinHTTP kullanarak bağlanmasını sağlayabilirler. UPNP hizmeti, sistemde kısıtlı ayrıcalıkları olan LocalService hesabı altında çalıştırılır.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
İş istasyonları ve terminal sunucuları birincil derecede risk altındadır. Yöneticiler kullanıcıların sunucularda oturum açmalarına ve program çalıştırmalarına izin verirse sunucular daha fazla etkilenebilir. Ancak, en iyi uygulamalarda buna izin verilmesi kesinlikle önerilmemektedir.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, Windows HTTP Hizmetleri'nin yararlanma durumuna neden olan hatayı işleme biçimini değiştirerek bu güvenlik açığını giderir.

Bir uygulama uzak Web sunucusuna bağlantı oluşturmak üzere Windows HTTP Hizmetleri'ni çağırdığında, WinHTTP yalnızca, URL'nin tam etki alanı adını ana bilgisayarın bağlanmayı denediği özgün Web sitesinin sertifikasına göre doğrular. DNS'de kimlik sahtekarlığının belirli bileşimlerinde, bağlantı, WinHTTP tarafından başlatıldığı Web sayfası yerine, ilgili ana bilgisayar için geçerli bir sertifikası olan farklı bir Web sunucusuna yönlendirilebilir. DNS'de kimlik sahtekarlığı ile bu güvenlik açığı bir arada kullanıldığında, WinHTTP, uzak Web sunucusunun sertifikasını hatalı şekilde kabul edilebilir olarak nitelendirebilir. Windows HTTP Hizmetleri (WinHTTP), geliştiricilerin diğer HTTP sunucularına HTTP protokolü aracılığıyla istek gönderebilmeleri için bir HTTP istemcisi uygulama programı arabirimi (API) sağlar.
  
**Dijital sertifika nedir?**  
Dijital sertifika, kimlik doğrulaması amacıyla yaygın olarak kullanılan bir kimlik bilgisidir. Güvenilen bir kuruluş, sertifikanın konusunu oluşturan kişiye veya varlığa bir sertifika atar. Sertifikayı yayımlayan güvenilen kuruluşa Sertifika Yetkilisi (CA) adı verilir ve "sertifikayı yayımlayan" olarak da bilinir. Güvenilir bir CA, yalnızca sertifikanın konusunu oluşturan kimliği doğruladıktan sonra sertifikayı yayımlar.
  
**DNS'de kimlik sahtekarlığı nedir?**  
Etki Alanı Adı Sistemi (DNS), TCP/IP kullanan endüstri standardı protokol paketlerden biridir. DNS, iki yazılım bileşeni kullanılarak uygulanır: DNS sunucusu ve DNS istemcisi (veya çözümleyici). Her iki bileşen de arka plan hizmet uygulaması olarak çalışır. Ağ kaynakları sayısal IP adreslerine göre tanımlanır, ancak bu IP adreslerini ağ kullanıcılarının anımsamaları zordur. DNS veritabanı, ağ kaynaklarının kullanımı kolay alfasayısal adlarını, bu kaynaklar tarafından iletişim kurmak için kullanılan IP adresleriyle eşleyen kayıtlar içerir. Böylece DNS, belleğe bağlı bir aygıt gibi çalışarak, ağ kaynaklarının ağ kullanıcıları tarafından anımsanmasını kolaylaştırır. Daha fazla bilgi ve DNS'nin diğer Windows teknolojileriyle nasıl çalıştığını gösteren mantıksal çizimler için, [DNS nedir?](http://technet.microsoft.com/en-us/library/cc787921.aspx) adlı TechNet makalesini inceleyin.
  
DNS'de kimlik sahtekarlığı, DNS sistemindeki bir noktada normalde DNS yanıtları vermekten sorumlu olmayan üçüncü taraflar tarafından hatalı DNS yanıtlarının alınmasına neden olan yaygın bir saldırı kategorisidir. Bu tür bir saldırı, bir etki alanı veya ana bilgisayar adının kendi sahipleri tarafından amaçlanmayan bir IP adresine çözümlenmesine neden olabilir. Dolayısıyla, bir ana bilgisayar, üçüncü taraflara ait kötü amaçlı bir sunucuya bağlanabilir.

Saldırgan, güvenli bir Web sitesini taklit edebilir ve Windows HTTP Hizmetleri'ni kullanarak uygulamaya kötü amaçlı içerik sunabilir; uygulama bu içeriğe güvenli bir Web sitesinden gönderilmiş gibi güvenecektir.

Saldırganın sunucuya giden bağlantıyı kendi denetimine yönlendirebilmesi için, bu güvenlik açığından DNS'de kimlik sahtekarlığı veya benzer bir saldırıyla birlikte yararlanması gerekir. #### Windows HTTP Hizmetleri'nde Kimlik Bilgilerini Yansıtma Güvenlik Açığı - CVE-2009-0550 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
Bu sorun ayrıca Internet Explorer'ı ve Windows Internet API'sını etkiler. Microsoft, Internet Explorer kullanıcılarının Internet Explorer Toplu Güvenlik Güncelleştirmesi (963027), [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659)'te sağlanan güvenlik güncelleştirmesini yüklemelerini önerir.
  
**Güvenlik açığına ne neden olmaktadır?**  
Windows HTTP Hizmetleri, kullanıcı kimlik bilgilerinin geri yansıtılmamasını ve kullanıcıya karşı kullanılmamasını sağlamak üzere NTLM kimlik bilgilerini yansıtmaya karşı koruma seçeneğini doğru şekilde kullanmaz.
  
**Microsoft Windows HTTP Hizmetleri nedir?**  
Microsoft Windows HTTP Hizmetleri (WinHTTP), geliştiricilerin diğer HTTP sunucularına HTTP protokolü aracılığıyla istek gönderebilmeleri için bir HTTP istemcisi uygulama programı arabirimi (API) sağlar.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bir saldırgan, oturum açmış kullanıcıyla aynı haklara sahip olabilir ve bu kullanıcının ayrıcalığı olan tüm işlemleri gerçekleştirebilir. Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Bu güvenlik açığı, Windows HTTP Hizmetleri'nin etkilenen bir sürümünün bulunduğu bir kullanıcının kötü amaçlı bir sunucuya erişmesini gerektirir. Saldırgan, kullanıcının NTLM oturum açma kimlik bilgilerini kaydeden özel hazırlanmış bir Web sunucusu barındırmalıdır. Saldırgan bu kimlik bilgilerini daha sonra kullanıcının istemcisinde NTLM kimlik doğrulamasını kabul eden bir hizmet için taklit eder. Saldırganın, bir kullanıcıyı özel hazırlanmış bir sunucuyu ziyaret etmeye zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi Web sitesine götüren bir bağlantıyı tıklatmaya ikna ederek onları bu siteye çekmesi gerekir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
İş istasyonları ve terminal sunucuları birincil derecede risk altındadır. Yöneticiler kullanıcıların sunucularda oturum açmalarına ve program çalıştırmalarına izin verirse sunucular daha fazla etkilenebilir. Ancak, en iyi uygulamalarda buna izin verilmesi kesinlikle önerilmemektedir.
  
**Güncelleştirme ne yapıyor?**  
Güvenlik güncelleştirmesi, Windows HTTP Hizmetleri'nin Windows tarafından sunulan kimlik bilgileri koruma mekanizmalarını doğru şekilde seçmesini sağlayarak bu güvenlik açığını giderir.
  
**Internet Explorer için Toplu Güvenlik Güncelleştirmesi Explorer (963027) başlıklı MS09-014'te bu güvenlik açığına ve WinINet'te Kimlik Bilgilerini Yansıtma Güvenlik Açığı'na neden aynı CVE numarası (CVE-2009-0550) atanmış?**  
CVE-2009-0550 hem Internet Explorer tarafından kullanılan WinINet'i hem de Windows işletim sisteminin bir bileşeni olan Windows HTTP Hizmetleri'ni etkilemektedir. Microsoft, Internet Explorer kullanıcılarının Internet Explorer Toplu Güvenlik Güncelleştirmesi (963027), [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659)'te sağlanan güvenlik güncelleştirmesini de yüklemelerini önerir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Evet. Bu güvenlik açığı genel olarak duyurulmuştur. Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE-2009-0550](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-0550) atanmıştır.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.
