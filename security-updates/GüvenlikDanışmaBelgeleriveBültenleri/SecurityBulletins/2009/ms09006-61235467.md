Security Bulletin

Microsoft Güvenlik Bülteni MS09-006 - Kritik
============================================

Windows Çekirdeğindeki Güvenlik Açıkları Uzaktan Kod Yürütülmesine İzin Verebilir (958690)
------------------------------------------------------------------------------------------

Yayım Tarihi: 10 Mart 2009 Salı

**Sürüm:** 1.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Windows çekirdeğindeki özel olarak bildirilen birkaç güvenlik açığını giderir. Bunların arasında en önemlisi olan güvenlik açığı, bir kullanıcı etkilenen bir sistemdeki özel hazırlanmış bir EMF veya WMF resim dosyasını görüntülerse uzaktan kod yürütülmesine izin verebilir.

Güvenlik güncelleştirmesi Microsoft Windows 2000, Windows XP, Windows Server 2003, Windows Vista ve Windows Server 2008'in tüm desteklenen sürümleri için Kritik olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, GDI'nın çekirdek bileşeni aracılığıyla kullanıcı modundan geçirilen girişi doğrulayıp çekirdeğin tanıtıcıları işleme biçimini düzelterek ve Windows çekirdeğinin özel hazırlanmış olan geçersiz işaretçileri işleme biçimini değiştirerek bu güvenlik açıklarını giderir. Bu güvenlik açıkları hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 294781](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi hemen uygulamalarını önerir.

**Bilinen Sorunlar.** Yok

Windows Çekirdeğinde Giriş Doğrulaması Güvenlik Açığı - CVE-2009-0081  
---------------------------------------------------------------------
  
<span></span>
Windows çekirdeğinde, GDI'nın çekirdek bileşeni aracılığıyla kullanıcı modundan geçirilen girişin düzgün doğrulanmaması nedeniyle bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan çekirdek modunda rasgele kod çalıştırabilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2009-0081](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-0081).
  
#### Windows Çekirdeğinde Giriş Doğrulaması Güvenlik Açığı - CVE-2009-0081 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Web tabanlı saldırı senaryosunda, bir saldırganın bu açıktan yararlanmak için kullanılan bir Web sayfası içeren bir Web sitesi barındırabilir. Ayrıca, kullanıcı tarafından sağlanan içeriği veya reklamları kabul eden ya da barındıran güvenliği aşılmış Web sitelerinde, bu güvenlik açığından yararlanılabilecek özel hazırlanmış içerik bulunabilir. Ancak bu durumların hiçbirinde, saldırganın, bu Web sitelerini ziyaret etmek için kullanıcıyı zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi Web sitesine götüren bir bağlantıyı tıklatmalarını sağlayarak onları bu Web sitesini ziyaret etmeye ikna etmesi gerekir.
  
#### Windows Çekirdeğinde Giriş Doğrulaması Güvenlik Açığı - CVE-2009-0081 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Kayıt defterini değiştirerek meta dosya işlemeyi kapatın**  
    MS07-017 ile ilişkili güvenlik güncelleştirmesini uygulayan veya Windows Vista ya da Windows Server 2008 kullanan müşteriler kayıt defterini değiştirerek meta dosya işlemeyi kapatabilir. Bu ayar, etkilenen sistemin bu güvenlik açığından yararlanma girişimlerine karşı korunmasına yardımcı olur.
  
    Anahtarı değiştirmek için şu adımları izleyin:
  
    **Not** Kayıt Defteri Düzenleyicisi'nin yanlış kullanılması, işletim sisteminizi yeniden yüklemenizi gerektirebilecek ciddi sorunlara neden olabilir. Microsoft, Kayıt Defteri Düzenleyicisi'nin yanlış kullanılmasından kaynaklanan sorunların çözülebileceğini garanti etmemektedir. Kayıt Defteri Düzenleyicisi'ni kullanmak kendi sorumluluğunuzdadır. Kayıt defterini düzenlemeyle ilgili bilgi için, Kayıt Defteri Düzenleyicisi'ndeki (Regedit.exe) "Anahtarları ve Değerleri Değiştirmek " veya Regedt32.exe'deki "Kayıt Defterine Bilgi Eklemek veya Bilgileri Silmek " ve "Kayıt Defteri Verilerini Düzenlemek" adlı Yardım konularına bakın.
  
    1.  **Başlat**'ı tıklatın, **Çalıştır**'ı tıklatın, **Aç** kutusuna **Regedit** yazın ve **Tamam**'ı tıklatın.  
    2.  Aşağıdaki kayıt defteri alt anahtarını bulun ve tıklatın:  
        HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Microsoft\\Windows NT\\CurrentVersion\\GRE\_Initialize  
    3.  **Düzen** menüsünde, **Yeni**'nin üzerine gelin ve **DWORD**'ü tıklatın.  
    4.  **DisableMetaFiles** yazın ve ENTER tuşuna basın.  
    5.  **Düzen** menüsünde, DisableMetaFiles kayıt defteri girdisini değiştirmek üzere **Değiştir**'i tıklatın.  
    6.  **Değer verisi** kutusuna **1** yazın ve **Tamam**'ı tıklatın.  
    7.  Kayıt Defteri Düzenleyicisi'nden çıkın.  
    8.  Bilgisayarı yeniden başlatın.
  
    **Geçici çözümün etkisi.** Meta dosyaların işlenmesini kapatmak, yazılım veya sistem bileşenlerindeki çıktının görünüm kalitesinin düşmesine neden olabilir. Meta dosyaların işlenmesini kapatmak ayrıca yazılım veya sistem bileşenlerinin hiç çalışmamasına neden olabilir. Bu geçici çözümün işlevselliğe önemli bir etkisi olabileceği belirlenmiştir ve uygulanabilirliğini belirlemek üzere dikkatlice değerlendirilmesi ve sınanması gerekmektedir.
  
    Aşağıdaki örnekler verilebilir:
  
    -   Bilgisayarda yazdıramazsınız.  
    -   Bilgisayardaki bazı uygulamalar Küçük Resimleri görüntüleyemeyebilir.  
    -   OLE oluşturmayla ilgili bazı senaryolar başarısız olabilir. Bu durumla özellikle de nesne sunucusu etkin değilse karşılaşılır.
  
    Bu ayar hakkında daha fazla bilgi için, [Microsoft Bilgi Bankası makalesi 941835](http://support.microsoft.com/kb/941835)'i okuyun.
  
    **Geçici çözüm nasıl geri alınır.**
  
    1.  **Başlat**'ı tıklatın, **Çalıştır**'ı tıklatın, **Aç** kutusuna **Regedit** yazın ve **Tamam**'ı tıklatın.  
    2.  Aşağıdaki kayıt defteri alt anahtarını bulun ve tıklatın:  
        HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Microsoft\\Windows NT\\CurrentVersion\\GRE\_Initialize  
    3.  **Düzen** menüsünde, DisableMetaFiles kayıt defteri girdisini değiştirmek üzere **Değiştir**'i tıklatın.  
    4.  **Değer verisi** kutusuna **0** yazın ve **Tamam**'ı tıklatın.  
    5.  Kayıt Defteri Düzenleyicisi'nden çıkın.  
    6.  Bilgisayarı yeniden başlatın.
  
    Bu değişikliğin etkinleştirilmesi için bilgisayarınızı yeniden başlatmanız gerekebilir.
  
#### Windows Çekirdeğinde Giriş Doğrulaması Güvenlik Açığı - CVE-2009-0081 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
Windows çekirdeği GDI'nın çekirdek bileşeni aracılığıyla kullanıcı modundan geçirilen girişi düzgün doğrulamaz.
  
**Windows çekirdeği nedir?**  
Windows çekirdeği işletim sisteminin merkezidir. Aygıt ve bellek yönetimi gibi sistem düzeyi hizmetler sağlar, işlemlere işlemci zamanı ayırır ve hata işlemeyi yönetir.
  
**GDI nedir?**  
Microsoft Windows grafik aygıt arabirimi (GDI), uygulamaların video görüntüsünde ve yazıcıda grafikler ve biçimlendirilmiş metin kullanabilmesini sağlar. Windows tabanlı uygulamalar grafik donanımına doğrudan erişmez. Bunun yerine, GDI, uygulamalar adına aygıt sürücüleriyle etkileşim kurar. GDI hakkında daha fazla bilgi için, [Windows GDI Başlangıç Sayfası](http://msdn.microsoft.com/en-us/library/dd145203(vs.85).aspx)'nı ziyaret edin.
  
**Windows Meta Dosyası (WMF) ve Gelişmiş Meta Dosyası (EMF) resim biçimleri nelerdir?**  
WMF resmi, vektör bilgilerini ve bit eşlem bilgilerini bir arada bulunduran 16-bit bir meta dosyası biçimidir. Windows işletim sistemi için en iyi duruma getirilmiştir.
  
EMF resmi, hem vektör bilgileri hem de bit eşlem bilgileri içerebilen 32-bit bir meta dosyası biçimidir. Bu biçim, Windows Meta Dosya biçiminin geliştirilmiş halidir ve genişletilmiş özellikler içerir.
  
Resim türleri ve biçimleri hakkında daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası Makalesi 320314](http://support.microsoft.com/kb/320314). Bu dosya biçimleri hakkında ek bilgi [MSDN Library Web sitesinde](http://msdn2.microsoft.com/en-us/library/ms536391.aspx) de bulunmaktadır.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan rasgele kod çalıştırabilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Uzaktan kod yürütülebilmesi için, bu güvenlik açığı bir kullanıcının özel hazırlanmış bir resim dosyasını görüntülemesini gerektirir. Bu güvenlik açığından, yerel sistemde oturum açmış bir saldırgan da yararlanabilir. Böylece saldırgan, güvenlik açığından yararlanan özel olarak hazırlanmış bir uygulama çalıştırabilir ve etkilenen sistemin tüm denetimini ele geçirebilir.
  
E-posta tabanlı saldırı senaryosunda, saldırgan kullanıcıya özel hazırlanmış bir EMF veya WMF resim dosyasının katıştırılmış olduğu bir e-posta gönderip kullanıcıyı bu e-postayı görüntülemeye veya önizleme yapmaya ikna ederek bu güvenlik açığından yararlanabilir. Saldırgan ayrıca Office belgesine özel hazırlanmış bir EMF veya WMF resim dosyası katıştırıp kullanıcıyı bu dosyayı açmaya ikna edebilir.
  
Web tabanlı saldırı senaryosunda, saldırganın, bu açıktan yararlanma girişimi için kullanılan EMF veya WMF resim dosyasını içeren bir Web sitesini barındırması gerekir. Ayrıca, kullanıcı tarafından sağlanan içeriği kabul eden ya da barındıran güvenliği aşılmış Web sitelerinde, bu güvenlik açığından yararlanılabilecek özel hazırlanmış içerik bulunabilir. Saldırganın, bir kullanıcıyı özel hazırlanmış bir Web sitesini ziyaret etmeye zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi Web sitesine götüren bir bağlantıyı tıklatmaya ikna ederek onları bu siteye çekmesi gerekir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
İş istasyonları ve terminal sunucuları birincil derecede risk altındadır. Yöneticiler kullanıcıların sunucularda oturum açmalarına ve program çalıştırmalarına izin verirse sunucular daha fazla etkilenebilir. Ancak, en iyi uygulamalarda buna izin verilmesi kesinlikle önerilmemektedir.

Bu güncelleştirme, GDI'nın çekirdek bileşeni aracılığıyla kullanıcı modundan geçirilen girişi doğrulayarak güvenlik açığını giderir.
