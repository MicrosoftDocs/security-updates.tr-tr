Security Bulletin

Microsoft Güvenlik Bülteni MS08-059 - Kritik
============================================

Host Integration Server RPC Hizmeti'ndeki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (956695)
----------------------------------------------------------------------------------------------------------

Yayım Tarihi: 14 Ekim 2008 Salı | Güncelleştirme Tarihi: 29 Ekim 2008 Çarşamba

**Sürüm:** 1.2

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Microsoft Host Integration Server'daki özel olarak bildirilen bir güvenlik açığını giderir. Bu güvenlik açığı, bir saldırgan etkilenen bir sisteme özel hazırlanmış bir Uzaktan Yordam Çağrısı (RPC) isteği gönderirse uzaktan kod yürütülmesine izin verebilir. En iyi uygulamaları kullanan ve SNA RPC hizmet hesabını sistem üzerinde daha az kullanıcı hakkı olacak şekilde yapılandıran müşteriler, SNA RPC hizmet hesabını yönetimsel kullanıcı haklarıyla yapılandıran müşterilere göre daha az etkilenebilir.

Bu güvenlik güncelleştirmesi Microsoft Host Integration Server 2000, Microsoft Host Integration Server 2004 ve Microsoft Host Integration Server 2006'nın tüm desteklenen sürümleri için Kritik olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, RPC isteklerini doğrulayarak güvenlik açığını ortadan kaldırır. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Microsoft, müşterilerinin güncelleştirmeyi hemen uygulamasını önerir.

**Bilinen Sorunlar.** [Microsoft Bilgi Bankası makalesi 956695](http://support.microsoft.com/kb/956695) müşterilerin bu güvenlik güncelleştirmesini yüklerken karşılaşabileceği bilinen sorunları ve önerilen çözümleri belgelemektedir. Bilinen sorunlar ve önerilen çözümler bu yazılımın yalnızca belirli sürümlerine yöneliktir, ancak bu makalede başka makalelere bağlantılar da sağlanır.

HIS'de Komut Yürütme Güvenlik Açığı - CVE- 2008-3466  
----------------------------------------------------
  
<span></span>
Host Integration Server için SNA Uzaktan Yordam Çağrısı (RPC) hizmetinde bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Saldırgan, özel hazırlanmış bir RPC isteği oluşturarak bu güvenlik açığından yararlanabilir. Bu güvenlik açığı, uzaktan kod yürütülmesine olanak verebilir. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2008-3466](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3466).
  
#### HIS'de Komut Yürütme Güvenlik Açığı - CVE- 2008-3466 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Bu güvenlik açığından başarıyla yararlanan bir saldırgan, Host Integration Server yüklenirken belirtilen SNA hizmet hesabıyla aynı kullanıcı haklarına sahip olabilir. En iyi uygulamaları kullanan ve SNA RPC hizmet hesabını sistem üzerinde daha az kullanıcı hakkı olacak şekilde yapılandıran müşteriler, SNA hizmet hesabını yönetimsel kullanıcı haklarıyla yapılandıran müşterilere göre daha az etkilenebilir.  
-   Microsoft Host Integration Server 2000 Administrator Client veya Microsoft Host Integration Server 2004 Client'ın yüklü olduğu sistemlerde, varsayılan yükleme güvenlik açığının bulunduğu SNA Uzaktan Yordam Çağrısı (RPC) hizmetini oluşturmaz veya başlatmaz. Ancak dosyalar varsayılan olarak yüklendiği ve başlatılabileceği için, bu ürünün güvenlik açığından etkilenen bir sürümünü kullanan müşterilerin bu güncelleştirmeyi uygulamaları önerilir.
  
#### HIS'de Komut Yürütme Güvenlik Açığı - CVE- 2008-3466 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   Host Integration Server 2004 ve Host Integration Server 2006 için, HIS/SNA hizmetini bir Yönetici Hesabı ile çalıştırılacak biçimde yapılandırmayın.
  
    -   Host Integration Server ilk kez yüklenirken, yöneticilerden SNA RPC hizmetinin çalıştırılacağı bir hesap istenir. Müşteriler en iyi uygulamaları kullanmalı ve yönetici ayrıcalıklarına sahip olmayan bir hesabı seçmelidir.  
    -   SNA hizmeti, Başlat menüsündeki Host Integration Server klasöründe bulunan Yapılandırma Aracı kullanılarak, düşük ayrıcalıklı bir hesap kullanacak biçimde yapılandırılabilir.
  
    **Not** Bu geçici çözüm Host Integration Server 2000 için kullanılamaz.
  
    **Geçici çözümün etkisi.** Yoktur. En iyi uygulamalar kullanılarak, SNA hizmeti, yönetici ayrıcalıkları olmayan bir hesapla çalıştırılmalıdır.
  
-   Host Integration Server 2000, Host Integration Server 2004 ve Host Integration Server 2006 için, SNA RPC Hizmeti'ni devre dışı bırakın.
  
    1.  Etkilenen hizmeti devre dışı bırakmak için, SnaRpcService hizmetini devre dışı bırakın. Bu hizmet, performans sayaçları eklemek ve bir sunucuya bağlantı hizmetleri ya da başka SNA Server hizmetleri eklerken bağları yenilemek için kullanılır.  
    2.  Bu yapılandırma görevlerini tamamladıktan sonra SNA RPC hizmetini devre dışı bırakın.
  
    **Geçici çözümün etkisi.** Normal çalışma zamanı işlemleri ve bağlantı ekleme gibi diğer yapılandırma değişiklikleri SNA RPC hizmetine bağımlı değildir. Ancak bu hizmetin devre dışı olduğu sırada bağlantı hizmetleri ya da düğümler eklenirse, ilişkili performans sayaçları eklenmez ve yapılandırma değişikliklerinin SNA Yöneticisi'ne yansıtılması için SNA MngAgent hizmetinin geri dönüştürülmesi gerekir. SNA RPC hizmeti devre dışı bırakıldığında performans verileri etkilenmez.
  
#### HIS'de Komut Yürütme Güvenlik Açığı - CVE- 2008-3466 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**    
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Güvenlik açığına ne neden olmaktadır?**    
Özel hazırlanmış RPC istekleri, kimliği doğrulanmamış uzak kullanıcıların SNA RPC hizmeti içindeki kimlik doğrulamasını atlamasına olanak verir.
  
**RPC nedir?**    
Uzak Yordam Çağrısı (RPC), bir programın ağdaki başka bir bilgisayarda bulunan bir programdan hizmet isteğinde bulunmak için kullanabileceği bir protokoldür. RPC kullanan programın, iletişimi destekleyen ağ protokollerini anlaması gerekmediği için, RPC, birlikte çalışabilirliğe yardımcı olur. RPC hizmetinde, istekte bulunan program istemci ve hizmeti sağlayan program da sunucu olarak çalışır.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**    
Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**    
Saldırgan, özel hazırlanmış bir RPC iletisi oluşturup bu iletiyi Host Integration Server tarafından dinamik olarak atanmış RPC TCP/UDP bağlantı noktası üzerinden etkilenen sisteme göndermek yoluyla bu açıktan yararlanmaya çalışabilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**    
Etkilenen Microsoft Host Integration Server'ın yüklü olduğu tüm sistemler bu güvenlik açığından etkilenir.
  
**Güncelleştirme ne yapıyor?**    
Bu güncelleştirme, RPC tarafından gerçekleştirilen yönetim işlevlerine erişimi kısıtlayarak güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**    
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının genel olarak duyurulduğuna dair herhangi bir bilgi almamıştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**    
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.
