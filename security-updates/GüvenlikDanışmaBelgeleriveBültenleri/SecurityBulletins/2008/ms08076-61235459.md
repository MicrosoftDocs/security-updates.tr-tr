Security Bulletin

Microsoft Güvenlik Bülteni MS08-076 - Önemli
============================================

Windows Media Bileşenlerindeki Güvenlik Açıkları Uzaktan Kod Yürütülmesine İzin Verebilir (959807)
--------------------------------------------------------------------------------------------------

Yayım Tarihi: 9 Aralık 2008 Salı | Güncelleştirme Tarihi: 24 Kasım 2009 Salı

**Sürüm:** 5.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi şu Windows Media bileşenlerindeki özel olarak bildirilen iki güvenlik açığını giderir: Windows Media Player, Windows Media Format Çalışma Zamanı Modülü ve Windows Media Hizmetleri. En ciddi güvenlik açığı uzaktan kod yürütülmesine olanak verebilir. Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

Bu güvenlik güncelleştirmesi Windows Media Player 6.4, Windows Media Format Çalışma Zamanı Modülü 7.1, Windows Media Format Çalışma Zamanı Modülü 9.0, Windows Media Format Çalışma Zamanı Modülü 9.5, Windows Media Format Çalışma Zamanı Modülü 11, Windows Media Hizmetleri 4.1, Windows Media Hizmetleri 9 Series ve Windows Media Hizmetleri 2008 için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, Windows Media kimlik doğrulaması yanıtlarının doğrulanma biçimini değiştirerek ilk güvenlik açığını ortadan kaldırır. Güvenlik güncelleştirmesi Windows Media istemcilerinin ISATAP adresleri kullanılan sunucuları dış kaynak olarak nitelendirmelerini sağlayarak ikinci güvenlik açığını giderir. Bu güvenlik açıkları hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Microsoft, müşterilerin güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

**Bilinen Sorunlar.** Yok

#### Etkilenen ve Etkilenmeyen Yazılımlar

Yazılımınızın sürümünün destek ömrünü belirlemek için [Microsoft Destek Ömrü](http://go.microsoft.com/fwlink/?linkid=21742) Web sitesini ziyaret edin.

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)
-----------------------------------------------------------------

<span></span>
**Bu bülten 24 Kasım 2009'da neden yeniden yayımlandı?**  
Bu bültende yer alan Orijinal Windows XP güncelleştirme paketlerindeki algılama ve yükleme sorunlarını gidermek için Windows XP Embedded sistemlerine özgü yeni güncelleştirme paketini sunmak amacıyla Microsoft bu bülteni yeniden yayımladı.

Microsoft ayrıca Windows XP Service Pack 2 (KB952069) ve Windows XP Service Pack 3 (KB952069) için varolan güncelleştirme paketlerinin Windows XP Embedded sistemlerine önerilmesini önlemek üzere bunları yeniden düzenledi. Bu güvenlik güncelleştirmesini önceden yüklemiş olan müşteriler korunmaktadır ve herhangi bir işlem yapmaları gerekmez.

Windows XP Embedded işletim sistemini kullanan müşteriler bu güncelleştirmeyi ilk fırsatta yüklemelidir. Müşterilerin ayrıca [Windows XP Embedded Service Pack 2](http://www.microsoft.com/downloads/details.aspx?familyid=c8f59b79-7d09-4976-a678-9aef5975c186) (KB972187) veya [Windows XP Embedded Service Pack 3](http://www.microsoft.com/downloads/details.aspx?familyid=502dcf76-aec3-48bf-930d-0c02b1604cd1) (KB972187) güncelleştirmesini karşıdan yüklemesi de mümkündür. Bu yeniden yayımlamadan etkilenen başka bir güncelleştirme yoktur.

**Windows XP Embedded işletim sistemi nedir?**  
Microsoft Windows XP Embedded, Windows XP Professional işletim sisteminin bileşenlere ayrılmış bir sürümüdür ve Windows XP Professional'ın zengin özellik kümesinden katıştırılmış aygıtların da yararlanmasını sağlar. Windows XP Embedded, yerleşik ağ özellikleri ve standartları temel alan tak ve kullan desteğiyle perakende satış için en uygun duruma getirilmiştir. Windows XP Embedded, çeşitli perakende satış veya karşılama aygıtı çevre birimlerini tek bir hizmet noktası (POS) sistemine bağlar. Windows XP Embedded işletim sistemleri hakkında daha fazla bilgi için [Microsoft Windows Embedded Geliştirme Merkezi](http://msdn.microsoft.com/en-us/windowsembedded/default.aspx)'ni ziyaret edin.

**Bu bülten 13 Ocak 2009'da neden yeniden yayımlandı?**  
Windows XP Service Pack 2 (KB952069) ve Windows XP Service Pack 3 (KB952069) üzerinde Windows Media Format Çalışma Zamanı 9.5 için yeni güncelleştirme paketlerini sunmak amacıyla Microsoft bu bülteni yeniden yayımladı. Yeni güncelleştirme paketi, Windows Media Format Çalışma Zamanı 9.5 çalıştıran ve müşterinin KB952069'u yüklemeden önce ayrı bir güncelleştirme olan KB944110'u yüklediği Windows XP sistemlerde yüklemenin eksik olmasına neden olan bir yükleme sorununu giderir. Bu gibi müşteriler güvenlik açığına karşı hala korumasızdır ve yeniden yayımlamam güncelleştirme paketlerini karşıdan yükleyip kurmaları gerekir. Bu yeniden yayımlanan etkilenen başka güncelleştirme paketi yoktur. Windows Media Player, Windows Media Format Çalışma Zamanı ve Windows Media Hizmetleri'nin desteklenen ve etkilenen diğer tüm sürümlerini çalıştıran ve özgün güvenlik güncelleştirmesi paketlerini uygulamış olan müşterilerin başka bir işlem yapmaları gerekmez. SPN Güvenlik Açığı - CVE-2008-3009
----------------------------------

<span></span>
Windows Media bileşenlerinde, bir saldırganın yerel kullanıcı ile aynı haklara sahip olarak veya Windows Media Hizmetleri dağıtım kimlik bilgilerini kullanarak kod yürütmesine olanak verebilecek bir kimlik bilgilerini yansıtma güvenlik açığı bulunmaktadır. Bu güvenlik açığı, Windows Media bileşenleri için Hizmet Asıl Adı (SPN) uygulamalarındaki zayıf noktalar nedeniyle bulunmaktadır.

Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2008-3009](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3009).

#### SPN Güvenlik Açığı - CVE-2008-3009 için Azaltıcı Etkenler

Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:

-   Windows Media Hizmetleri, tüm Windows Server platformlarında isteğe bağlı bir bileşendir ve varsayılan olarak yüklenmez.
-   Web tabanlı saldırı senaryosunda, bir saldırganın bu açıktan yararlanmak için kullanılan bir Web sayfası içeren bir Web sitesi barındırması gerekir. Bir saldırganın bir kullanıcıyı kötü amaçlı bir Web sitesini ziyaret etmeye zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları kendi Web sitesine götüren bir bağlantıyı tıklatmalarını sağlayarak onları bu Web sitesine çekmesi gerekir. Bağlantıyı tıklatmalarından sonra birkaç eylem gerçekleştirmeleri istenir. Yalnızca bu eylemleri gerçekleştirmelerinden sonra bir saldırı gerçekleşebilir.
-   Windows Server 2008 ve x64 tabanlı sistemler için Windows Server 2008'de, Windows Media Format Çalışma Zamanı Modülü 11 yalnızca Masaüstü Deneyimi yüklüyse sistemde bulunur.

#### SPN Güvenlik Açığı - CVE-2008-3009 için Geçici Çözümler

Microsoft bu güvenlik açığı için herhangi bir geçici çözüm belirleyememiştir.

#### SPN Güvenlik Açığı - CVE-2008-3009 Hakkında SSS

**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açmış kullanıcıyla aynı haklara sahip olarak kod yürütebilir. Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.  Bu güvenlik açığından başarıyla yararlanan bir saldırgan aynı zamanda Windows Media Hizmetleri dağıtım kimlik bilgileriyle kod yürütebilir. Bu durumda saldırgan, NETWORK SERVICE hesabıyla aynı haklara sahip olarak kod yürütebilir.

**Güvenlik açığına ne neden olmaktadır?**  
Windows Media bileşenleri, kullanıcı kimlik bilgilerinin kullanıcının sistemine geri yansıtılmamasını ve kullanıcıyı taklit etmek için kullanılmamasını sağlamak üzere NTLM kimlik bilgilerini yansıtmaya karşı koruma seçeneğini doğru şekilde kullanmaz.

**Hizmet Asıl Adları (SPN) nedir?**  
Hizmet Asıl Adı (SPN), istemcinin bir hizmet örneğini benzersiz şekilde tanımlamasını sağlayan bir addır. SPN, hizmetin istemcide kimlik doğrulamasını yapmak için bir hizmet kimlik türü sağlar. İstemci bir hizmete bağlanmak istediğinde hizmetin bir örneğini bulur, bu örnek için bir SPN oluşturur, hizmete bağlanır ve hizmetin kimlik doğrulaması için SPN'yi sunar. SPN konusunda ek bilgi için, bkz. [Hizmet Kimliği ve Kimlik Doğrulaması](http://msdn.microsoft.com/en-us/library/ms733130.aspx). SPN'lerin nasıl biçimlendirilmeleri gerektiği konusunda ek bilgi için, bkz. [Benzersiz SPN'ler için Ad Biçimleri](http://msdn.microsoft.com/en-us/library/ms677601(vs.85).aspx).

**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açan kullanıcı ile aynı haklara sahip olabilir. Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Bu güvenlik açığı, Windows Media bileşenlerinin etkilenen bir sürümünün bulunduğu bir kullanıcının kötü amaçlı bir sunucuya erişmesini gerektirir. Saldırgan, kimlik doğrulaması gerektiren kötü amaçlı bir sunucu veya Web sitesi barındırabilir. İstemci bu kötü amaçlı sunucuya bağlanıp kimliğini doğruladığında, saldırgan, istemci tarafından sağlanan kimlik bilgilerini kullanarak istemciye geri bağlanabilir veya bu kimlik bilgilerini istemciyi taklit ederek başka sunuculara bağlanmak için kullanabilir. Saldırganın, kullanıcıları özel hazırlanmış bir sunucu paylaşımını veya Web sitesini ziyaret etmeye zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi Web sitesine ya da sunucu paylaşımına götüren bir bağlantıyı tıklatmaya ikna ederek onları kendi sitesine çekmesi gerekir.

**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Windows Media Player, Windows Media Format Çalışma Zamanı Modülü veya Windows Media Hizmetleri'nin etkilenen sürümlerinin kullanıldığı tüm sistemler bu güvenlik açığından etkilenir. Bu bileşenlerin sıkça kullanıldığı iş istasyonları veya terminal sunucuları gibi sistemler bu açık karşısında en fazla risk altındadır.

**Güncelleştirme ne yapıyor?**  
Güvenlik güncelleştirmesi, kimlik bilgilerinin yeniden kullanılmasını önlemek üzere Windows Media bileşeni kimlik doğrulaması yanıtlarının doğrulanma biçimini değiştirerek bu güvenlik açığını ortadan kaldırır.

**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**

ISATAP Güvenlik Açığı - CVE-2008-3010
-------------------------------------

<span></span>
Windows Media bileşenlerinin desteklenen sürümlerinde, NTLM kimlik bilgilerinin açıklanmasına neden olabilecek bir bilginin açığa çıkması güvenlik açığı bulunmaktadır. ISATAP adresi kullanılan bir URL'ye erişen herhangi bir Windows Media bileşeni, bu URL'yi barındıran sunucuya kullanıcının NTLM kimlik bilgilerini sızdırabilir. Böylece, intranet bölgesinin dışında bulunan bir saldırgan, kuruluş ortamının NTLM kimlik bilgilerini edinebilir.

Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2008-3010](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3010).

#### ISATAP Güvenlik Açığı - CVE-2008-3010 için Azaltıcı Etkenler

Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:

-   En iyi güvenlik duvarı uygulamaları ve standart varsayılan güvenlik duvarı yapılandırmaları, ağ kaynaklarının kuruluş dışından gerçekleştirilen saldırılardan korunmasına yardımcı olabilir. En iyi uygulamalar, Internet'e bağlı sistemlerde olabildiğince az sayıda bağlantı noktasının açılmasını ve protokolün etkinleştirilmesini önerir.
-   Web tabanlı saldırı senaryosunda, bir saldırganın bu açıktan yararlanmak için kullanılan bir Web sayfası içeren bir Web sitesi barındırması gerekir. Bir saldırganın bir kullanıcıyı kötü amaçlı bir Web sitesini ziyaret etmeye zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları kendi Web sitesine götüren bir bağlantıyı tıklatmalarını sağlayarak onları bu Web sitesine çekmesi gerekir. Bağlantıyı tıklatmalarından sonra birkaç eylem gerçekleştirmeleri istenir. Yalnızca bu eylemleri gerçekleştirmelerinden sonra bir saldırı gerçekleşebilir.
-   Windows Server 2008 ve x64 tabanlı sistemler için Windows Server 2008'de, Windows Media Format Çalışma Zamanı Modülü 11 yalnızca Masaüstü Deneyimi yüklüyse sistemde bulunur.

#### ISATAP Güvenlik Açığı - CVE-2008-3010 için Geçici Çözümler

Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:

-   **Güvenlik duvarında IP Protokol Türü 41'i (ISATAP) engelleyin**

    ISATAP protokolü, [RFC4214: Site İçi Otomatik Tünel Adres Protokolü (ISATAP)](http://www.ietf.org/rfc/rfc4214.txt) tarafından 41 numaralı protokol türü olarak tanımlanmıştır. Güvenlik duvarında bu protokol türünün engellenmesi, güvenlik duvarının arkasındaki sistemleri bu açıktan yararlanma girişimlerinden korumaya yardımcı olur. Microsoft, diğer bağlantı noktalarını veya protokolleri kullanabilecek saldırılardan korunmaya yardımcı olmak amacıyla, Internet'ten gelen tüm istenmeyen trafiğin engellenmesini önerir.

#### ISATAP Güvenlik Açığı - CVE-2008-3010 Hakkında SSS

**Bu güvenlik açığının kapsamı nedir?**  
Bu, bir bilginin açığa çıkması güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açmış kullanıcının NTLM kimlik bilgilerini edinebilir. Bu kimlik bilgileri daha sonra kullanıcının sistemine geri yansıtılabilir ve oturum açmış kullanıcı ile aynı haklara sahip olarak kod yürütülebilir.

**Güvenlik açığına ne neden olmaktadır?**  
Etkilenen Windows Media bileşenleri, ISATAP adresi kullanılan bir sunucuya bağlanırken Internet kaynaklarını hatalı şekilde Yerel Intranet bölgesinin bir parçası olarak nitelendirir.

**ISATAP nedir?**  
Site İçi Otomatik Tünel Adres Protokolü (ISATAP), IPv4 Intranet'i içinde IPv6 bağlantısı sağlar. ISATAP için RFC, Internet Engineering Task Force (IETF) tarafından barındırılır. ISATAP hakkında bilgi için, bkz. [RFC4214: Site İçi Otomatik Tünel Adres Protokolü (ISATAP)](http://www.ietf.org/rfc/rfc4214.txt) tarafından 41 numaralı protokol türü olarak tanımlanmıştır.

**NTLM nedir?**  
NTLM, sınama/yanıt mekanizmasına dayalı bir kimlik doğrulama protokolüdür.

**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan kullanıcının NTLM kimlik bilgilerini edinebilir.  Bu kimlik bilgileri daha sonra kullanıcının sistemine geri yansıtılabilir ve oturum açmış kullanıcı ile aynı haklara sahip olarak kod yürütülebilir.

**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Bu güvenlik açığı, Windows Media bileşenlerinin etkilenen bir sürümünün bulunduğu bir kullanıcının kötü amaçlı bir sunucuya erişmesini gerektirir. Saldırgan, kimlik doğrulaması gerektiren kötü amaçlı bir sunucu veya Web sitesi barındırabilir. İstemci bu kötü amaçlı sunucuya bağlanıp kimliğini doğruladığında, saldırgan, istemci tarafından sağlanan kimlik bilgilerini kullanarak istemciye geri bağlanabilir veya bu kimlik bilgilerini istemciyi taklit ederek başka sunuculara bağlanmak için kullanabilir. Saldırganın, kullanıcıları özel hazırlanmış bir sunucu paylaşımını veya Web sitesini ziyaret etmeye zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi Web sitesine ya da sunucu paylaşımına götüren bir bağlantıyı tıklatmaya ikna ederek onları kendi sitesine çekmesi gerekir.

**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Windows Media Player, Windows Media Format Çalışma Zamanı Modülü veya Windows Media Hizmetleri'nin etkilenen sürümlerinin kullanıldığı tüm sistemler bu güvenlik açığından etkilenir. Bu bileşenlerin sıkça kullanıldığı iş istasyonları veya terminal sunucuları gibi sistemler bu açık karşısında en fazla risk altındadır.

**Güncelleştirme ne yapıyor?**  
Güncelleştirme, Windows Media bileşeni istemcilerinin ISATAP adresleri kullanılan sunucuları dış kaynak olarak nitelendirmelerini ve kullanıcı kimlik bilgilerini geçirmemelerini sağlayarak bu güvenlik açığını giderir.

**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının genel olarak duyurulduğuna dair herhangi bir bilgi almamıştı.

Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.
