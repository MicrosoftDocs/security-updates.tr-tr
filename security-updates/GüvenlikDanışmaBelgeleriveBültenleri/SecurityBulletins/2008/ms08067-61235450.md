Security Bulletin

Microsoft Güvenlik Bülteni MS08-067 - Kritik
============================================

Sunucu Hizmeti'ndeki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (958644)
-------------------------------------------------------------------------------------

Yayım Tarihi: 23 Ekim 2008 Perşembe

**Sürüm:** 1.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Sunucu hizmetindeki özel olarak bildirilen bir güvenlik açığını giderir. Bu güvenlik açığı, bir kullanıcı etkilenen bir sistemde özel hazırlanmış bir RPC isteği alırsa uzaktan kod yürütülmesine izin verebilir. Microsoft Windows 2000, Windows XP ve Windows Server 2003 çalışan sistemlerde, bir saldırgan rasgele kod çalıştırmak için kimliğini doğrulamasına gerek kalmadan bu güvenlik açığından yararlanabilir. Bu güvenlik açığından solucan saldırıları düzenlemek için yararlanılabilir. En iyi güvenlik duvarı uygulamaları ve standart varsayılan güvenlik duvarı yapılandırmaları, ağ kaynaklarının kuruluş dışından gerçekleştirilen saldırılardan korunmasına yardımcı olabilir.

Bu güvenlik güncelleştirmesi Microsoft Windows 2000, Windows XP ve Windows Server 2003'ün tüm desteklenen sürümleri için Kritik ve Windows Vista, Windows Server 2008 ve Windows 7 Beta'nın tüm desteklenen sürümleri için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, Sunucu hizmetinin RPC isteklerini işleme biçimini düzelterek güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Microsoft, müşterilerinin güncelleştirmeyi hemen uygulamasını önerir.

**Bilinen Sorunlar.** Yok

Sunucu Hizmeti Güvenlik Açığı - CVE-2008-4250  
---------------------------------------------
  
<span></span>
Windows tabanlı sistemlerdeki Sunucu hizmetinde bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Güvenlik açığı, hizmetin özel hazırlanmış RPC isteklerini düzgün işlememesi nedeniyle oluşur. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2008-4250](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-4250).
  
#### Sunucu Hizmeti Güvenlik Açığı - CVE-2008-4250 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   En iyi güvenlik duvarı uygulamaları ve standart varsayılan güvenlik duvarı yapılandırmaları, ağ kaynaklarının kuruluş dışından gerçekleştirilen saldırılardan korunmasına yardımcı olabilir. En iyi uygulamalar, Internet'e bağlı sistemlerde olabildiğince az sayıda bağlantı noktasının açılmasını önerir.  
-   Windows Vista, Windows Server 2008 ve Windows 7 Beta'da, güvenlik açığı bulunan kod yoluna yalnızca kimliği doğrulanmış kullanıcılar tarafından erişilebilir. Saldırganın kimliği doğrulanmamışsa bu güvenlik açığı tetiklenmez.
  
#### Sunucu Hizmeti Güvenlik Açığı - CVE-2008-4250 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Sunucu ve Bilgisayar Tarayıcı hizmetlerini devre dışı bırakın**
  
    Etkilenen sistemlerde Bilgisayar Tarayıcı ve Sunucu hizmetlerinin devre dışı bırakılması, sistemlerin bu güvenlik açığından uzaktan yararlanma girişimlerinin engellenmesini sağlar.
  
    Aşağıdaki adımları kullanarak bu hizmetleri devre dışı bırakabilirsiniz:
  
    1.  **Başlat**'ı ve sonra **Denetim Masası**'nı tıklatın (veya **Ayarlar**'ın üzerine gidip **Denetim Masası**'nı tıklatın).  
    2.  **Yönetimsel Araçlar**'ı çift tıklatın.  
    3.  **Hizmetler**'i çift tıklatın.  
    4.  **Bilgisayar Tarayıcı Hizmeti**'ni çift tıklatın.  
    5.  Başlangıç türü listesinde, **Devre Dışı** seçeneğini tıklatın.  
    6.  **Durdur**'u, ardından **Tamam**'ı tıklatın.  
    7.  4-6 arası adımları **Sunucu** hizmeti için de yineleyin
  
    **Geçici Çözümün Etkisi**. Bilgisayar Tarayıcı hizmeti devre dışı bırakılırsa, Bilgisayar Tarayıcı hizmetini kullanan hizmetler sistem olay günlüğüne bir hata iletisi kaydedebilir. Bilgisayar Tarayıcı hizmeti hakkında daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 188001](http://support.microsoft.com/kb/188001). Sunucu hizmeti devre dışı bırakılırsa, dosyaları veya yazıcıları bilgisayarınızdan paylaşamazsınız. Ancak diğer sistemlerdeki dosya paylaşımlarını ve yazıcı kaynaklarını görmeye devam edebilirsiniz.
  
    **Geçici çözüm nasıl geri alınır.** Aşağıdaki adımları kullanarak bu hizmetleri etkinleştirebilirsiniz:
  
    1.  **Başlat**'ı ve sonra **Denetim Masası**'nı tıklatın (veya **Ayarlar**'ın üzerine gidip **Denetim Masası**'nı tıklatın).  
    2.  **Yönetimsel Araçlar**'ı çift tıklatın.  
    3.  **Hizmetler**'i çift tıklatın.  
    4.  **Bilgisayar Tarayıcı Hizmeti**'ni çift tıklatın.  
    5.  Başlangıç türü listesinde, **Otomatik** seçeneğini tıklatın.  
    6.  **Başlat**'ı ve sonra **Tamam**'ı tıklatın.  
    7.  4-6 arası adımları **Sunucu** hizmeti için de yineleyin
  
-   **Windows** **Vista, Windows Server 2008 ve Windows 7 Beta'da, etkilenen RPC tanımlayıcısına filtre uygulayın**
  
    Windows Vista, Windows Server 2008 ve Windows 7 Beta sürümlerinde Windows Güvenlik Duvarı'nı kullanarak bağlantı noktalarını engellemenin yanı sıra, RPC Evrensel Benzersiz Tanımlayıcıları'na (UUID) seçmeli olarak filtre uygulayabilirsiniz. Bu güvenlik açığını engellemek için, UUID değeri 4b324fc8-1670-01d3-1278-5a47bf6ee188 değerine eşit olan tüm RPC isteklerini engelleyen bir kural oluşturun. Bu işlem ağ kabuğu aracılığıyla gerçekleştirilir. Ağ kabuğuna erişmek için, ayrıcalığı yükseltilmiş bir komut isteminden şu komutu çalıştırın:
  
    `netsh`
  
    Netsh ortamına geçtikten sonra aşağıdaki komutları girin:

#### Sunucu Hizmeti Güvenlik Açığı - CVE-2008-4250 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**    
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, etkilenen sistemin tüm denetimini uzaktan ele geçirebilir. Microsoft Windows 2000, Windows XP ve Windows Server 2003 çalışan sistemlerde, bir saldırgan rasgele kod çalıştırmak için kimliğini doğrulamasına gerek kalmadan bu güvenlik açığından RPC üzerinden yararlanabilir. Bu güvenlik açığından solucan saldırıları düzenlemek için yararlanılabilir. Saldırgan bu güvenlik açığından başarıyla yararlanırsa program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Güvenlik açığına ne neden olmaktadır?**    
Güvenlik açığı, Windows Sunucu hizmetinin özel hazırlanmış RPC isteklerini düzgün işlememesi nedeniyle oluşur.
  
**Sunucu hizmeti nedir?**    
Sunucu hizmeti, RPC desteğinin yanı sıra ağ üzerinden dosya ve yazıcı desteği ile adlandırılmış yöneltme paylaşımı sağlar. Sunucu hizmeti, yerel kaynaklarınızın (diskler ve yazıcılar gibi) paylaşılmasına olanak tanıyarak ağdaki diğer kullanıcıların onlara erişebilmesini sağlar. Başka bilgisayarlarda ve bilgisayarınızda çalışan uygulamalar arasında RPC amacıyla kullanılan adlandırılmış yöneltme iletişimine de olanak tanır.
  
**RPC nedir?**    
Uzak Yordam Çağrısı (RPC), bir programın ağdaki başka bir bilgisayarda bulunan bir programdan hizmet isteğinde bulunmak için kullanabileceği bir protokoldür. RPC kullanan programın, iletişimi destekleyen ağ protokollerini anlaması gerekmediği için, RPC, birlikte çalışabilirliğe yardımcı olur. RPC hizmetinde, istekte bulunan program istemci ve hizmeti sağlayan program da sunucu olarak çalışır.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**    
Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**    
Saldırgan, özel olarak hazırlanmış bir iletiyi etkilenen sisteme göndermek yoluyla bu açıktan yararlanmaya çalışabilir. Microsoft Windows 2000, Windows XP ve Windows Server 2003 çalışan sistemlerde, hedef ağa erişimi olan herhangi bir anonim kullanıcı etkilenen sisteme özel hazırlanmış bir ağ paketi teslim ederek bu güvenlik açığından yararlanabilir. Windows Vista, Windows Server 2008 ve Windows 7 Beta çalışan sistemlerde ise yalnızca hedef ağa erişimi olan kimliği doğrulanmış bir kullanıcı etkilenen sisteme özel hazırlanmış bir ağ paketi teslim edebilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**    
Tüm iş istasyonları ve sunucular bu sorundan etkilenmekle birlikte Windows 2000, Windows XP veya Windows Server 2003 çalışan sistemler güvenlik açığının benzersiz özellikleri ve etkilenen kod yolu nedeniyle birincil dereceden risk altındadır.
  
**Güncelleştirme ne yapıyor?**    
Bu güncelleştirme, Sunucu hizmetinin RPC isteklerini işleme biçimini düzelterek güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**    
Evet. Microsoft bu güvenlik açığından yararlanmaya çalışan sınırlı sayıda hedefe yönelik saldırı olduğunu bilmektedir. Ancak güvenlik bülteni yayımlandığında, Microsoft, açığın kanıtı olabilecek kod örnekleri yayımlandığını görmemiştir.
  
**Bu güvenlik güncelleştirmesini uygulamak, müşterilerin açıktan yararlanmaya yönelik olarak yayımlanan koddan korunmasına yardımcı olur mu?**    
Evet. Bu güvenlik güncelleştirmesi, şu anda yararlanılmakta olan güvenlik açığını ortadan kaldırır. Güvenlik açığı giderilmiş, ayrıca [CVE-2008-4250](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-4250) Ortak Güvenlik Açığı ve Etkilenme numarası atanmıştır.
