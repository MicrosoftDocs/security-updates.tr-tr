Security Bulletin

Microsoft Güvenlik Bülteni MS08-030 - Kritik
============================================

Bluetooth Yığınındaki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (951376)
--------------------------------------------------------------------------------------

Yayım Tarihi: 10 Haziran 2008 Salı | Güncelleştirme Tarihi: 19 Haziran 2008 Perşembe

**Sürüm:** 2.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Windows'daki Bluetooth yığınında uzaktan kod yürütülmesine izin verebileceği özel olarak bildirilen bir güvenlik açığını giderir. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.

Bu güvenlik güncelleştirmesi Windows XP ve Windows Vista'nın tüm desteklenen sürümleri için Kritik olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, Bluetooth yığınının çok sayıda hizmet açıklaması isteğini işleme biçimini değiştirerek güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Microsoft, müşterilerinin güncelleştirmeyi hemen uygulamasını önerir.

**Bilinen Sorunlar.** Yok

Bluetooth Güvenlik Açığı - CVE-2008-1453  
----------------------------------------
  
<span></span>
Microsoft Windows'daki Bluetooth yığınında, çok sayıda hizmet açıklaması isteğinin doğru şekilde işlenememesi nedeniyle bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Güvenlik açığı bir saldırganın yükseltilmiş ayrıcalıklarla kod çalıştırılmasına izin verebilir. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; tarihleri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2008-1453](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1453).
  
#### Bluetooth Güvenlik Açığı - CVE-2008-1453 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Bu güvenlik açığı yalnızca Bluetooth özelliğini destekleyen sistemleri etkiler.
  
#### Bluetooth Güvenlik Açığı - CVE-2008-1453 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir.
  
-   **Bilgisayarın yeni Bluetooth bağlantıları kabul etmesini engelleme**
  
**Etkileşimli Yöntem**
  
1.  **Denetim Masası**'nda, **Klasik Görünüm**'ü seçin.  
2.  **Bluetooth Aygıtları**'nı seçin.  
3.  **Seçenekler** sekmesini tıklatın.  
4.  **Bluetooth aygıtlarının bu bilgisayarı bulmasına izin ver** onay kutusunu temizleyin.  
5.  **Tamam**'ı tıklatın.
  
**Geçici çözüm nasıl geri alınır**
  
1.  **Denetim Masası**'nda, **Klasik Görünüm**'ü seçin.  
2.  **Bluetooth Aygıtları**'nı seçin.  
3.  **Seçenekler** sekmesini tıklatın.  
4.  **Bluetooth aygıtlarının bu bilgisayarı bulmasına izin ver** onay kutusunu işaretleyin.  
5.  **Tamam**'ı tıklatın.
  
**PowerShell Komut Dosyası Yöntemi**
  
Hakları genişletilmiş yönetici olarak aşağıdaki komutları çalıştırın:
  
$BthDriverKeys = Get-ChildItem "HKLM:\\System\\CurrentControlSet\\Enum\\usb" -recurse -ErrorAction SilentlyContinue | where {$\_.GetValue("Service") -eq "BTHUSB"}  
foreach($Key in $BthDriverKeys) { Set-ItemProperty -Path $Key.OpenSubKey("Device Parameters").Name.Replace("HKEY\_LOCAL\_MACHINE", "HKLM:") -Name "Write Scan Enable" -Value 0 }
  
**Geçici çözüm nasıl geri alınır**
  
Hakları genişletilmiş yönetici olarak aşağıdaki komutları çalıştırın:
  
$BthDriverKeys = Get-ChildItem "HKLM:\\System\\CurrentControlSet\\Enum\\usb" -recurse -ErrorAction SilentlyContinue | where {$\_.GetValue("Service") -eq "BTHUSB"}  
foreach($Key in $BthDriverKeys) { Set-ItemProperty -Path $Key.OpenSubKey("Device Parameters").Name.Replace("HKEY\_LOCAL\_MACHINE", "HKLM:") -Name "Write Scan Enable" -Value 3 }
  
-   **Bluetooth sürücüsünü kapatma ve devre dışı bırakma**
  
Komut satırında aşağıdaki komutu kullanarak sürücüyü durdurabilir ve devre dışı bırakabilirsiniz (Windows XP ve sonrasında ve Windows 2000 Kaynak Seti'nde kullanılabilir):
  
sc config bthport start= disabled
  
**Geçici çözüm nasıl geri alınır**
  
Komut satırında aşağıdaki komutu kullanarak Bluetooth sürücüsünü yeniden etkinleştirebilirsiniz (Windows XP ve sonrasında ve Windows 2000 Kaynak Seti'nde kullanılabilir):
  
sc config bthport start= demand
  
#### Bluetooth Güvenlik Açığı - CVE-2008-1453 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**    
Microsoft Windows'daki Bluetooth yığınında uzaktan kod yürütülmesine neden olabilecek bir güvenlik açığı bulunmaktadır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan yükseltilmiş ayrıcalıklarla kod çalıştırabilir. Böylece saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir.
  
**Güvenlik açığına ne neden olmaktadır?**    
Windows Bluetooth Yığını çok sayıda hizmet açıklaması isteğini doğru şekilde işleyemez.
  
**Bluetooth nedir?**    
Bluetooth, bilgisayarlar, el aygıtları, cep telefonları ve diğer aygıtlar için kablosuz bağlantı sağlayan ve bir bilişim sektörü standardı olan protokoldür. Daha fazla bilgi için [MSDN Bluetooth makalesine](http://msdn.microsoft.com/en-us/library/aa362932(vs.85).aspx) bakın.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**    
Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**    
Bir saldırganın bu güvenlik açığından yararlanabilmesi için, etkilenen bir sisteme özel hazırlanmış çok sayıda Hizmet Algılama Protokolü (SDP) paketi göndermesi gerekir. Sistemin bu paketleri işleme biçimi, saldırganın, etkilenen sistemin tüm denetimini ele geçirmesine olanak verebilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**    
Bluetooth özelliği etkinleştirilmiş olan tüm sistemler bu güvenlik açığı nedeniyle risk altındadır.
  
**Güncelleştirme ne yapıyor?**    
Bu güncelleştirme, Bluetooth yığınının çok sayıda SDP paketini işleme biçimini geliştirerek güvenlik açığını ortadan kaldırır.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**    
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**    
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.
