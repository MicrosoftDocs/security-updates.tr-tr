Security Bulletin

Microsoft Güvenlik Bülteni MS08-062 - Önemli
============================================

Windows Internet Yazdırma Hizmeti'ndeki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (953155)
--------------------------------------------------------------------------------------------------------

Yayım Tarihi: 14 Ekim 2008 Salı | Güncelleştirme Tarihi: 29 Ekim 2008 Çarşamba

**Sürüm:** 2.2

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güncelleştirme Windows Internet Yazdırma Hizmeti'nde uzaktan kod yürütülmesine izin verebileceği özel olarak bildirilen bir güvenlik açığını giderir. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Böylece saldırgan program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir ya da yeni hesaplar oluşturabilir.

Bu güncelleştirme Microsoft Windows 2000, Windows XP, Windows Server 2003 ve Windows Server 2008'in tüm desteklenen sürümleri için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, belleğin Internet Yazdırma Protokolü (IPP) hizmeti içinde ayrılma biçimini değiştirerek bu güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Microsoft, müşterilerin güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

**Bilinen Sorunlar.** Yok

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)
-----------------------------------------------------------------

<span></span>
**Bu bülten 29 Ekim 2008'de neden yeniden düzenlendi?**  
Microsoft bu bülteni, Itanium tabanlı sistemler için Windows Server 2008'e ve Windows Vista'nın tüm desteklenen sürümlerine yönelik güncelleştirme paketlerinin Windows Update, Microsoft Update, Windows Software Update Services (WSUS), Systems Management Server ve System Center Configuration Manager aracılığıyla kullanıma sunulduğu konusunda müşterileri bilgilendirmek üzere yeniden düzenledi. Bu bülten ilk yayımlandığında, bu platformlara yönelik güncelleştirmeler yalnızca Microsoft Yükleme Merkezi'nden edinilebiliyordu.

**Bu bülten 16 Ekim 2008'de neden yeniden düzenlendi?**
Microsoft bu bülteni, Windows Internet Yazdırma hizmetinin sistem ayrıcalıklarıyla çalışan Biriktirici hizmeti bağlamında çalıştırıldığını açıklığa kavuşturmak üzere yeniden düzenledi. Bu güvenlik bülteninde açıklanan güvenlik açığından başarıyla yararlanan bir saldırgan, sistem ayrıcalıklarıyla kod çalıştırabilir ve etkilenen sistemin denetimini tümüyle ele geçirebilir.

**Bu bülten 15 Ekim 2008'de neden yeniden düzenlendi?**
Microsoft bu bülteni, Itanium tabanlı sistemler için Windows Server 2008 önem derecesini kaldırarak yeniden düzenledi. Microsoft ayrıca Itanium tabanlı sistemler için Windows Server 2008 güncelleştirmesinin Microsoft Yükleme Merkezi aracılığıyla edinilebileceğini ve diğer dağıtım kanalları aracılığıyla da en kısa sürede yayımlanacağını açıklığa kavuşturmak üzere bu bülteni güncelleştirdi.

**Itanium tabanlı sistemler için Windows Server 2008 ve Windows** **Vista** **önem dereceleri belirlenmemişse bu sürümler neden Etkilenen Yazılımlar tablosunda listeleniyor?**
Itanium tabanlı sistemler için Windows Server 2008 ve Windows Vista'nın tüm desteklenen sürümleri, bu güvenlik güncelleştirmesi tarafından güncelleştirilen paylaşılan yazdırma biriktiricisi bileşenini içermektedir. Ancak Internet Yazdırması hizmeti Itanium tabanlı sistemler için Windows Server 2008'de veya Windows Vista'da desteklenmediğinden, güvenlik açığı bulunan ve bu güvenlik bülteninde açıklanan saldırı vektörü kullanılamaz. Gelecekte tanımlanabilecek olası saldırı sektörlerine karşı kapsamlı bir savunma önlemi olarak, Itanium tabanlı sistemler için Windows Server 2008 ve Windows Vista müşterilerinin bu güvenlik güncelleştirmesini uygulamaları önerilmektedir.

**Dosya bilgileri ayrıntılarını nerede bulabilirim?**  
Dosya bilgileri ayrıntıları [Microsoft Bilgi Bankası makalesi 953155](http://support.microsoft.com/kb/953155)'te bulunabilir.

**Etkilenen Yazılımlar tablosunda listelenen platformlardan birini çalıştırıyorum. Bu güncelleştirme neden sunuluyor?**
Bu güncelleştirme yalnızca etkilenen bileşenin yüklü olduğu sistemlere sunulur. Microsoft Internet Information Services'ın (IIS) yüklü olmadığı Microsoft Windows 2000 ve Windows XP tabanlı sistemler bu açıktan etkilenmez ve bu güncelleştirmeyi almaz. Hem IIS'nin hem de Internet yazdırmasının yüklü olmadığı Windows Server 2003 ve Windows Server 2008 tabanlı sistemler bu açıktan etkilenmez ve bu güncelleştirmeyi almaz.

IPP Hizmeti'nde Tamsayı Taşması Güvenlik Açığı - CVE-2008-1446  
--------------------------------------------------------------
  
<span></span>
Windows Internet Yazdırma hizmeti etkinleştirilmiş olarak Microsoft Internet Information Services (IIS) çalışan Windows tabanlı sistemlerde bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Bu sorun, kimliği doğrulanmış bir uzak saldırganın etkilenen sistemde uzaktan kod yürütmesine olanak sağlayabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2008-1446](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1446).
  
#### IPP Hizmeti'nde Tamsayı Taşması Güvenlik Açığı - CVE-2008-1446 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Internet Yazdırma Protokolü (IPP) varsayılan olarak Microsoft Windows 2000 ve Windows XP için IIS'de etkinleştirilmiştir. Ancak tüm platformlar için varsayılan dosya izinleri, IPP'ye yalnızca "Kimliği Doğrulanmış" gruptaki kullanıcılar tarafından erişilebilecek şekilde ayarlanmıştır. Bu kısıtlama nedeniyle, kimliği doğrulanmamış kullanıcılar varsayılan olarak bu güvenlik açığından uzaktan yararlanmak için gerekli klasör izinlerine sahip olmayacaktır.  
-   Windows Server 2003 üzerindeki IIS yüklemeleri varsayılan yapılandırmalarında bu güvenlik açığından etkilenmez. Bu açıktan yalnızca IIS içindeki Internet Yazdırma Protokolü adlı ISAPI filtresinin etkinleştirilmiş olduğu Windows Server 2003 yüklemeleri etkilenir. Web tabanlı yazdırma yönetimini ve Internet'ten yazdırmayı etkinleştirmek için hem Internet Yazdırması hem de Active Server Pages bileşenleri gerekmektedir. Active Server Pages bileşeni "İzin Verildi" olarak ve Internet Yazdırması bileşeni "Yasaklandı" olarak ayarlanmışsa, yalnızca Web tabanlı yazıcı yönetimi etkinleştirilir. Kullanıcılar paylaşılan yazıcılara Internet Yazdırması aracılığıyla bağlanamaz.  
-   Windows Server 2008 üzerindeki IIS yüklemeleri varsayılan olarak bu güvenlik açığından etkilenir. Windows Server 2008 çalıştırılan bir bilgisayar, Internet yazdırması kullanan bir istemci bilgisayar gibi davranabilir. Ancak, yazdırma isteklerinde bulunan bilgisayarın kullanıcılarının, sunucuya bağlı olan yazıcılardan birini kullanabilmeleri için yazdırma sunucusu tarafından kimlik doğrulaması yapılmalıdır. Windows Server 2008 çalıştırılan bir bilgisayarın Internet yazdırmasını destekleyen bir sunucu gibi davranmasını sağlamak için Web Sunucusu (IIS) rolü, Yazdırma Hizmetleri rolü ve Yazdırma Hizmetleri rolüne Internet Yazdırması rol hizmeti yüklenmelidir.  
-   Bu güvenlik açığından yararlanılabilmesi için, IIS sunucusunun kötü amaçlı bir bilgisayara 445 numaralı bağlantı noktası üzerinden giden trafik oluşturması gerekir. IIS sunucularından Internet'e giden SMB bağlantılarını engelleyen yöneticiler, bu güvenlik açığından güvenilir olmayan sistemler aracılığıyla yararlanılmasını engelleyebilir.
  
#### IPP Hizmeti'nde Tamsayı Taşması Güvenlik Açığı - CVE-2008-1446 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **IPP Hizmeti'ni devre dışı bırakma**
  
    IIS 5.0 ve IIS 5.1'de IPP Hizmeti'ni devre dışı bırakmak için
  
    1.  **Internet Hizmetleri Yöneticisi**'ni açın.  
    2.  **Varsayılan Web Sitesi**'ni sağ tıklatın ve **Özellikler**'i seçin.  
    3.  **Giriş Dizini** sekmesine gidin ve **Yapılandırma**'yı seçin.  
    4.  Alta doğru kaydırma yapıp **.printer** uzantısını ve **msw3prt.dll** dosyasına karşılık gelen yolu bulun.  
    5.  **Kaldır**'ı tıklatın.  
    6.  **Tamam**'ı tıklatın.  
    7.  IIS Hizmeti'ni yeniden başlatın.
  
    **Not** Internet Yazdırması, grup ilkesi aracılığıyla ya da Internet Hizmetleri Yöneticisi aracılığıyla yapılandırılabilir. Grup ilkesi ayarlarıyla Internet Hizmetleri Yöneticisi ayarları arasında bir çakışma oluşursa, grup ilkesi ayarları kullanılır. Internet Hizmetleri Yöneticisi'ni kullanarak Internet Yazdırması'nı kaldırırsanız, yerel veya etki alanı grup ilkeleri tarafından yeniden etkinleştirilmeyeceğinden emin olun. (Varsayılan grup ilkesi Internet Yazdırması'nı ne etkinleştirir ne de devre dışı bırakır.) MMC Grup İlkesi ek bileşeninde, Bilgisayar Yapılandırması | Yönetim Şablonları | Yazdırma | Web Tabanlı Yazdırma öğesini işaretleyin.
  
    IIS 6.0'da IPP Hizmeti'ni devre dışı bırakmak için
  
    1.  **IIS Yöneticisi**'ni veya **IIS ek bileşenini** başlatın.

#### IPP Hizmeti'nde Tamsayı Taşması Güvenlik Açığı - CVE-2008-1446 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Böylece saldırgan program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir ya da yeni hesaplar oluşturabilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
Internet Yazdırma Protokolü (IPP) işlevi, IIS için bir ISAPI uzantısı olarak uygulanır. IIS çalıştıran Windows tabanlı sunucularda Microsoft'un IPP uygulamasında kimliği doğrulanmış bir uzak güvenlik açığı bulunmaktadır. IPP uygulamasında, kimliği doğrulanmış bir saldırganın etkilenen IIS sunucularında uzaktan kod yürütmesine olanak verebilecek bir tamsayı taşması bulunmaktadır.
  
**ISAPI nedir?**  
ISAPI (Internet Hizmetleri Uygulama Programlama Arabirimi), Web geliştiricilerinin bir Web sunucusu için yeni hizmetler sağlayan özel kod yazarak Web sunucularının işlevselliğini genişletmelerine olanak veren bir teknolojidir. Bu özel kod, yeni işlevsellik bir alt düzey hizmet sağlıyorsa bir ISAPI filtresinde veya bir üst düzey hizmet sağlıyorsa bir ISAPI uzantısında uygulanabilir. Bu durumda, etkilenen kod bir ISAPI uzantısıdır.
  
**Bu soruna hangi ISAPI uzantısı neden olmaktadır?**  
Etkilenen ISAPI uzantısı, RFC 2910 ve 2911'de tanımlanmış bir endüstri standardı olan Internet Yazdırma Protokolü'nü (IPP) kullanır. IPP, yazdırma hizmetleri isteminde bulunmanın ve HTTP aracılığıyla Internet üzerinden yazdırma işlerinin durumunu öğrenmenin bir yolunu sağlar. Örneğin, seyahat halindeki bir kullanıcı IPP kullanarak, şirket ağındaki bir yazıcıda yazdırılmak üzere Internet üzerinden bir yazdırma işi gönderebilir. Kullanıcı aynı zamanda yazdırma isteğinin hatasız olarak tamamlanıp tamamlanmadığını öğrenebilir.
  
Microsoft Windows 2000'de Internet Yazdırması için özgün destek sağlanmaya başlanmıştır. Microsoft Windows 2000 uygulamasında kullanıcıların doğrudan bir URL'ye yazdırabilmeleri ve yazdırma işleriyle ilgili bilgileri tarayıcılarında görüntüleyebilmeleri sağlanmaktadır. Microsoft Windows 2000 Server sürümlerinde Internet Yazdırması desteği varsayılan olarak etkinleştirilmiştir. Microsoft Windows 2000 Professional, Windows XP, Windows Server 2003 ve Windows Server 2008 üzerine isteğe bağlı bir bileşen olarak da yüklenebilir.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan kullanıcının sisteminde sistem ayrıcalıklarıyla rasgele kod çalıştırabilir. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Böylece saldırgan program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir ya da yeni hesaplar oluşturabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırgan güvenlik açığından etkilenen bir Web sunucusuna özel hazırlanmış bir HTTP POST isteği göndererek, sunucunun saldırgan tarafından denetlenen ve IPP kullanan bir yazıcı gibi davranan bir makineye bağlanmasına neden olabilir. Saldırgan daha sonra özel hazırlanmış yanıtları kullanarak hedef sunucuda uzaktan kod yürütebilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Internet Yazdırma Protokolü (IPP) hizmeti etkinleştirilmiş olarak IIS çalıştıran iş istasyonları ve sunucular bu güvenlik açığından birincil derecede etkilenir.
  
**Güncelleştirme ne yapıyor?**  
Bu güvenlik güncelleştirmesi, Internet Yazdırma Protokolü (IPP) hizmeti aracılığıyla uzak makinelerden alınan verileri doğrulayarak bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının genel olarak duyurulduğuna dair herhangi bir bilgi almamıştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Evet, Microsoft bu güvenlik açığından yararlanmaya çalışan saldırılar olduğunu bilmektedir. Ancak bu güvenlik bülteni ilk yayımlandığında, Microsoft, açığın kanıtı olabilecek kod örnekleri yayımlandığını görmemiştir.

-   **Microsoft Baseline Security Analyzer**

    Etkilenen bir sisteme güvenlik güncelleştirmesinin uygulandığını doğrulamak için Microsoft Baseline Security Analyzer (MBSA) aracını kullanabilirsiniz. Daha fazla bilgi için, bu bültenin önceki bölümünde yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** alt bölümüne bakın.

-   **Dosya Sürümü Doğrulaması**

    Microsoft Windows'un birçok sürümü olduğundan, aşağıdaki adımlar sizin sisteminizde farklı olabilir. Böyle bir durumda, bu adımları tamamlamak için ürün belgelerinize bakın.

    1.  **Başlat**'ı ve sonra **Ara**'yı tıklatın.
    2.  **Arama Sonuçları** bölmesinde, **Arama Yardımcısı**'nın altındaki **Tüm dosya ve klasörler**'i tıklatın.
    3.  **Dosya adının tümü veya bir parçası** kutusuna, uygun dosya bilgileri tablosundan bir dosya adı yazın ve **Ara**'yı tıklatın.
    4.  Dosya listesinde, uygun dosya bilgileri tablosundan bir dosya adını sağ tıklatın ve sonra **Özellikler**'i tıklatın.
        **Not** Sisteminizde yüklü işletim sisteminin veya programların sürümüne bağlı olarak, dosya bilgileri tablosunda listelenen dosyalardan bazıları yüklenmeyebilir.
    5.  **Sürüm** sekmesinde, sisteminize yüklenmiş olan dosyanın sürümünü uygun dosya bilgileri tablosunda belgelenen sürümle karşılaştırarak belirleyin.
        **Not** Dosya sürümü dışındaki öznitelikler yükleme sırasında değişebilir. Diğer dosya özniteliklerini dosya bilgileri tablosundaki bilgilerle karşılaştırmak, güncelleştirmenin uygulandığını doğrulama için desteklenen bir yöntem değildir. Ayrıca, belirli durumlarda, dosyalar kurulum sırasında yeniden adlandırılabilir. Dosya veya sürüm bilgileri yoksa, güncelleştirme kurulumunu kullanılabilen diğer yöntemlerden biriyle doğrulayın.
