Security Bulletin

Microsoft Güvenlik Bülteni MS08-006 - Önemli
============================================

Internet Information Services'taki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (942830)
---------------------------------------------------------------------------------------------------

Yayım Tarihi: 12 Şubat 2008 Salı | Güncelleştirme Tarihi: 20 Şubat 2008 Çarşamba

**Sürüm:** 1.1

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu önemli güncelleştirme Internet Information Services'taki (IIS) özel olarak bildirilen bir güvenlik açığını giderir. IIS'nin ASP Web sayfalarına girişleri işleme biçiminde uzaktan kod yürütülmesine neden olabilecek bir güvenlik açığı bulunmaktadır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, IIS sunucusunda Çalışan İşlem Kimliği (WPI) ile aynı haklarla işlem gerçekleştirebilir. WPI varsayılan olarak Network Service hesabı ayrıcalıklarıyla yapılandırılır. Uygulama havuzları yönetimsel ayrıcalıklara sahip bir hesap kullanılan WPI ile yapılandırılmış olan ASP sayfaları çalıştıran IIS sunucular, uygulama havuzu varsayılan WPI ayarlarıyla yapılandırılmış IIS sunuculardan daha fazla etkilenebilir.

Güvenlik güncelleştirmesi, Windows XP ve Windows Server 2003'ün tüm desteklenen sürümleri üzerinde Microsoft Internet Information Services için önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Bu güvenlik açıkları hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Microsoft, müşterilerin güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

**Bilinen Sorunlar.** Yok

ASP Güvenlik Açığı - CVE-2008-0075  
----------------------------------
  
<span></span>
Internet Information Services'ın ASP Web sayfalarına girişleri işleme biçiminde uzaktan kod yürütülmesine neden olabilecek bir güvenlik açığı bulunmaktadır. Saldırgan, bir Web sitesinin ASP sayfasına kötü amaçlı giriş verileri aktararak güvenlik açığından yararlanabilir. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, IIS Server'da varsayılan olarak Network Service hesabı ayrıcalıklarıyla yapılandırılmış olan Çalışan İşlem Kimliği (WPI) ile aynı haklarla işlem gerçekleştirebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2008-0075](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0075).
  
#### ASP Güvenlik Açığı - CVE-2008-0075 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   IIS 5.1, Windows XP Professional Service Pack 2'nin varsayılan yüklemesinin parçası değildir.  
-   Windows Server 2003'ün desteklenen sürümlerinde, IIS varsayılan olarak yüklü değildir veya etkinleştirilmemiştir.  
-   Windows Server 2003'ün desteklenen sürümlerinde, IIS etkinse, klasik ASP varsayılan olarak yüklenmez veya etkinleştirilmez.  
-   Windows Server 2003'ün desteklenen sürümlerinde, IIS etkinse ve klasik ASP kullanılıyorsa, bu güvenlik açığından başarıyla yararlanan bir saldırgan varsayılan olarak yalnızca Network Service hesabı ayrıcalıklarını edinebilir. Network Service hesabı ayrıcalıkları varsayılan olarak kimliği doğrulanmış bir kullanıcıyla aynı kullanıcı haklarına sahiptir.  
-   ASP.NET bu güvenlik açığından etkilenmez. Yalnızca ASP.NET yüklemiş olan ve ASP yüklemeyen müşteriler bu güvenlik açığından etkilenmez.
  
#### ASP Güvenlik Açığı - CVE-2008-0075 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Windows Server 2003'te klasik ASP'yi devre dışı bırakma**
  
    Klasik ASP'yi devre dışı bırakmak için şu adımları izleyin:
  
    1. **Başlat**'ı tıklatın, **Yönetimsel Araçlar**'ı tıklatın ve sonra da **Internet Information Services (IIS) Yöneticisi**'ni tıklatın.
  
    2. Sunucu adının yanındaki **artı işaretini** (+) ve ardından **Web Hizmeti Uzantıları** klasörünü tıklatın.
  
    3. Sağdaki bölmede **Active Server Pages** öğesini tıklatıp vurgulayın, sonra da **Yasakla**'yı tıklatın.
  
    **Geçici çözümün etkisi.**Kullanıcılar klasik ASP sayfalarını kullanamaz.
  
    **Geçici çözüm nasıl geri alınır.**Geçici çözümü geri almak için, yukarıdaki 1. ile 3. adımlar arasını yineleyin ve **Active Server Pages**'ı vurguladığınızda **İzin Ver**'i tıklatın.
  
#### ASP Güvenlik Açığı - CVE-2008-0075 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, IIS sunucusunda varsayılan olarak Network Service hesabı ayrıcalıklarıyla yapılandırılmış olan Çalışan İşlem Kimliği (WPI) ile aynı haklarla işlem gerçekleştirebilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
Güvenlik açığı, Internet Information Services'ın ASP sayfalarına gönderilen giriş verilerini hatalı biçimde işlemesi nedeniyle oluşur.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, IIS Server'da varsayılan olarak Network Service hesabı ayrıcalıklarıyla yapılandırılmış olan Çalışan İşlem Kimliği (WPI) ile aynı haklarla işlem gerçekleştirebilir. Network Service hesap ayrıcalıklarıyla yapılandırılmış hizmetler yönetimsel düzeyde erişim değil, kimliği doğrulanmış kullanıcı düzeyinde erişim sağlar. Daha fazla bilgi için, bkz: [Network Service Hesabı](http://msdn2.microsoft.com/en-us/library/ms684272(vs.85).aspx). Uygulama havuzu yönetimsel ayrıcalıklara sahip bir hesap kullanılan WPI ile yapılandırılmış olan IIS sunucular, uygulama havuzu varsayılan WPI ayarlarıyla yapılandırılmış IIS sunuculardan daha fazla etkilenebilir.
  
Network Service hesabı uzaktan kimlik doğrulama yaparken bilgisayarın kimlik bilgilerini kullanır, ancak sunucuda önemli oranda azaltılmış bir ayrıcalık düzeyine sahiptir ve bu nedenle de yerel yönetici ayrıcalıkları yoktur. Daha fazla bilgi için bkz: [Network Service Hesabı](http://msdn2.microsoft.com/en-us/library/ms684272(vs.85).aspx), [IIS ve Yerleşik Hesaplar (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/3648346f-e4f5-474b-86c7-5a86e85fa1ff.mspx), [Çalışan İşlem Kimliklerini Yapılandırma (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/76be66e2-1db1-432e-95be-14ff59c2ec75.mspx) ve [IIS 6.0'da Uygulama Havuzlarını Yapılandırma (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/2a231dcb-d786-4b6d-b3ba-bda04061a5e7.mspx).
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırgan bir ASP sayfasına özel hazırlanmış giriş verileri gönderebilir. ASP sayfası zaten var olabilir veya barındırılan ortama bir kullanıcı tarafından yüklenebilir.
  
**IIS için çalışan işlemleri nedir?**  
Çalışan işlemleri, bir Web sunucusu için sunucu tarafında işleme görevleri gerçekleştiren işlemlerdir**.** Çalışan işlemleri ve IIS 6.0'da nasıl uygulandıkları hakkında bilgi için, bkz: [Çalışan İşlemleri (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/29f53968-0115-451f-b26d-5ad58d87b5d1.mspx). Çalışan işlemleri ve Web tabanlı uygulamaları yalıtılmış olarak nasıl yürüttükleri hakkında ek bilgi için, bkz: [Çalışan İşlemi Yalıtma Modu (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/333f3410-b566-49bc-8399-57bcc404db8c.mspx).
  
**ASP nedir?**  
Microsoft Active Server Pages (ASP), dinamik ve etkileşimli Web uygulamaları oluşturmak için kullanılabilen bir sunucu tarafı komut dosyası çalıştırma teknolojisidir. ASP sayfası, kullanıcının tarayıcısına gönderilmeden önce Web sunucusu tarafından işlenen sunucu tarafı komut dosyalarının bulunduğu bir HTML sayfasıdır.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
IIS ve ASP etkinleştirilmişse Windows Server 2003 ve Windows XP Professional çalışan Web sunucuları risk altındadır. Müşterilerinin barındırdıkları Web sitelerine ASP sayfaları yüklemelerine izin veren Web barındırma hizmeti sağlayıcıları daha fazla risk altındadır.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, Internet Information Services'ın ASP sayfalarına gönderilen giriş verilerini işleme biçimini değiştirerek güvenlik açığını giderir.

Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının genel olarak duyurulduğuna dair herhangi bir bilgi almamıştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.

Ayrıca, belirli durumlarda, dosyalar kurulum sırasında yeniden adlandırılabilir. Dosya veya sürüm bilgileri yoksa, güncelleştirme kurulumunu kullanılabilen diğer yöntemlerden biriyle doğrulayın.

    Bu güvenlik güncelleştirmesinde açıklanan dosyaların yüklenmiş olduğunu, bu bölümdeki **Başvuru Tablosu**'nda listelenen kayıt defteri anahtarlarını gözden geçirerek de doğrulayabilirsiniz.
