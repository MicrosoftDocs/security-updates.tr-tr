Security Bulletin

Microsoft Güvenlik Bülteni MS10-022 - Önemli
============================================

VBScript Komut Dosyası Çalıştırma Altyapısı'ndaki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (981169)
------------------------------------------------------------------------------------------------------------------

Yayım Tarihi: 13 Nisan 2010 Salı | Güncelleştirme Tarihi: 5 Mayıs 2010 Çarşamba

**Sürüm:** 1.1

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi, Microsoft Windows'daki VBScript'te uzaktan kod yürütülmesine izin verebileceği genel olarak duyurulan bir güvenlik açığını giderir. Bu güvenlik güncelleştirmesi Microsoft Windows 2000, Windows XP ve Windows Server 2003 için Önemli olarak derecelendirilmiştir. Windows Server 2008, Windows Vista, Windows 7 ve Windows Server 2008 R2'de, güvenlik açığından etkilenen koddan yararlanılamaz; ancak kod bulunduğu için, bu güncelleştirme kapsamlı bir koruma önlemi olarak sağlanmaktadır ve önem derecesi belirlenmemiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik açığı, kötü amaçlı bir Web sitesindeki Web sayfasında özel hazırlanmış bir iletişim kutusu görüntülenirse ve kullanıcı F1 tuşuna basarak Windows Yardım Sistemi'nin saldırgan tarafından sağlanan bir Windows Yardım Dosyası ile başlatılmasına yol açarsa, uzaktan kod yürütülmesine olanak verebilir. Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir.

Güvenlik güncelleştirmesi, VBScript altyapısının yardım dosyalarını korumalı modda işleme biçimini değiştirerek güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

Bu güvenlik güncelleştirmesi ilk olarak [Microsoft Güvenlik Danışma Belgesi 981169](http://technet.microsoft.com/security/advisory/981169)'da açıklanan güvenlik açığını giderir.

**Öneri.**  Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871/tr).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya Microsoft Update hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** [Microsoft Bilgi Bankası makalesi 981169](http://support.microsoft.com/kb/981169/tr), bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir. Bu makale ayrıca, bu sorunlar için önerilen çözümleri de içermektedir.

Yazılımınızın sürümünün destek ömrünü belirlemek için [Microsoft Destek Ömrü](http://go.microsoft.com/fwlink/?linkid=21742) Web sitesini ziyaret edin.

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)
-----------------------------------------------------------------

<span></span>
**Sistemimde VBScript'in hangi sürümü yüklü?**  
Sistemdeki VBScript sürümü, sistemde olan Internet Explorer, Windows işletim sistemi veya Windows Komut Dosyası Sistemi ile birlikte gelen en son sürüme göre belirlenir ve Microsoft Internet Explorer, Windows Hizmet Paketi veya Windows Komut Dosyası Sistemi'nin daha yeni bir sürümü yüklenirse değişebilir. Daha fazla bilgi için, [VBScript Sürümü Bilgileri](http://msdn.microsoft.com/en-us/library/4y5y7bh5(vs.85).aspx) adlı MSDN makalesine bakın.

Sisteminizdeki VBScript sürümünü belirlemek için şu adımları gerçekleştirin:

1.  Windows Gezgini'ni açın.
2.  **%systemroot%\\system32** dizinine gidin.
3.  **vbscript.dll** dosyasını sağ tıklatın, **Özellikler**'i seçin ve sonra da **Ayrıntılar** sekmesini tıklatın.

Sürüm numarası, **Dosya Sürümü** alanında listelenir. Örneğin dosya sürümünüz 5.8 ile başlıyorsa (örneğin, 5.8.7600.16385 ise), sisteminizde VBScript 5.8 yüklüdür.

**VBScript 5.1 kullanıyorum. Sistemim bu açıktan etkilenir mi?**  
Evet; sisteminize güvenlik güncelleştirmesi KB981350 uygulanabilir ve VBScript sürümünüzü VBScript 5.6'ya yükseltir. Bu güncelleştirme hakkında daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 981350](http://support.microsoft.com/kb/981350/tr).

**Not** VBScript 5.1 yalnızca henüz Internet Explorer 6 yüklenmemiş olan yeni Microsoft Windows 2000 Service Pack 4 yüklemelerinde bulunur. Bu tür sistemlere bu güvenlik güncelleştirmesi yüklendiğinde VBScript sürümünüz VBScript 5.6'ya yükseltilir.

**Windows** **Vista, Windows Server 2008, Windows 7 veya Windows Server 2008 R2 kullanıyorum. Bu güncelleştirmeyi neden alıyorum?**  
Güvenlik açığı Windows Vista, Windows Server 2008, Windows 7 ve Windows Server 2008 R2'nin desteklenen sürümlerinde yer alan vbscript.dll adlı bir Windows dosyasında bulunur. Microsoft, bu işletim sistemlerinin kullanıldığı bilgisayarlarda bu güvenlik açığından yararlanmanın herhangi bir yolunu belirleyememiştir. Kapsamlı bir savunma ölçütü olarak, bu güvenlik güncelleştirmesi Windows Vista, Windows Server 2008, Windows 7 ve Windows Server 2008 R2'nin desteklenen sürümlerini kullanan tüm müşterilere sunulmuştur.

**981169 numaralı Danışma Belgesi'nde sağlanan geçici çözümleri uyguladıysam, bu güncelleştirmeyi uygulamadan önce ilgili geçici çözümleri geri almam gerekiyor mu?**  
[Microsoft Güvenlik Danışma Belgesi 981169](http://technet.microsoft.com/security/advisory/981169)'da sağlanan geçici çözümleri uyguladıysanız, bu güncelleştirmeyi yüklemeden önce geçici çözümleri geri almanız gerekmez. Geçici çözümleri geri almak istiyorsanız, güncelleştirmeyi yükledikten sonra bunu yapmanız güvenlidir. Danışma belgesi, bu bültende önerilen geçici çözümleri kullanır. Geçici çözümü kullanmaya devam ederseniz ne olacağı hakkında daha fazla bilgi edinmek için, karşılık gelen **Geçici Çözümün Etkisi** bilgilerine bakın.

**Dosya bilgileri ayrıntılarını nerede bulabilirim?**  
Dosya bilgileri ayrıntılarının konumu için **Güvenlik Güncelleştirmesi Dağıtımı** bölümündeki başvuru tablolarına bakın.

VBScript Yardımı'nda Tuşa Basma Güvenlik Açığı - CVE-2010-0483
--------------------------------------------------------------

<span></span>
Internet Explorer kullanılırken VBScript'in Windows Yardım dosyalarıyla etkileşim kurma biçiminde bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Kötü amaçlı bir Web sitesinde özel hazırlanmış bir iletişim kutusu görüntülenirse ve kullanıcı F1 tuşuna basarsa, Windows Yardım Sistemi saldırgan tarafından sağlanan bir Windows Yardım Dosyası ile başlatılır. Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir. Windows Server 2003 çalıştırılan sistemlerde, bu sorunun etkilerinin azaltılmasına yardımcı olan Internet Explorer [Artırılmış Güvenlik Yapılandırması](http://msdn.microsoft.com/en-us/library/ms537180(vs.85).aspx) varsayılan olarak etkinleştirilmiştir.

Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2010-0483](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0483).

#### VBScript Yardımı'nda Tuşa Basma Güvenlik Açığı - CVE-2010-0483 için Azaltıcı Etkenler

Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:

-   Bu güvenlik açığından Windows Vista, Windows Server 2008, Windows 7 veya Windows Server 2008 R2'de yararlanılamaz.
-   Bu güvenlik açığı, kullanıcı etkileşimi gerektirir. Kullanıcı kötü amaçlı bir Web sitesini ziyaret etse de, kullanıcı etkileşimi olmadan bu güvenlik açığından yararlanılamaz. Bunun yerine saldırganın, Web sitesinde komut dosyası çalıştıran bir iletişim kutusu görüntülenirken kullanıcıyı F1 tuşuna basmaya ikna etmesi gerekir.
-   Web tabanlı saldırı senaryosunda, bir saldırganın bu açıktan yararlanmak için kullanılan bir Web sayfası içeren bir Web sitesi barındırabilir. Ayrıca, kullanıcı tarafından sağlanan içeriği veya reklamları kabul eden ya da barındıran güvenliği aşılmış Web sitelerinde, bu güvenlik açığından yararlanılabilecek özel hazırlanmış içerik bulunabilir. Ancak bu durumların hiçbirinde, saldırganın, bu Web sitelerini ziyaret etmek için kullanıcıyı zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi Web sitesine götüren bir bağlantıyı tıklatmalarını sağlayarak onları bu Web sitesini ziyaret etmeye ikna etmesi gerekir.
-   Microsoft Outlook, Microsoft Outlook Express ve Windows Mail'in tüm desteklenen sürümleri varsayılan olarak HTML e-posta iletilerini Yasak siteler bölgesinde açarak, saldırganın bu güvenlik açığından yararlanarak kötü amaçlı kod yürütebilmesi riskini ortadan kaldırır. Yasak siteler bölgesi, HTML e-posta iletileri okunurken Active Komut Dizileri ve ActiveX denetimlerinin kullanılmasını engelleyerek, bu güvenlik açığından yararlanmaya çalışacak saldırıların etkilerinin azaltılmasına yardımcı olur. Bununla birlikte, kullanıcı bir e-posta iletisindeki bağlantıyı tıklatırsa, Web tabanlı saldırı senaryosu aracılığıyla bu güvenlik açığından etkilenebilir. Outlook 2007 ayrıca HTML e-postalarını işlemek için farklı bir bileşen kullanarak bu açıktan yararlanılma riskini ortadan kaldırır.
-   Internet Explorer, Windows Server 2003'te varsayılan olarak [Artırılmış Güvenlik Yapılandırması](http://go.microsoft.com/fwlink/?linkid=92039) adı verilen kısıtlı bir modda çalışır. Bu mod, Internet bölgesi için güvenlik düzeyini Yüksek olarak ayarlar. Bu, Internet Explorer'ın Güvenilen siteler bölgesine eklenmemiş Web siteleri için bir azaltıcı etkendir.
-   Bu güvenlik açığından başarıyla yararlanan bir saldırgan, yerel kullanıcı ile aynı haklara sahip olabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

#### VBScript Yardımı'nda Tuşa Basma Güvenlik Açığı - CVE-2010-0483 için Geçici Çözümler

Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:

-   **Web sitesi tarafından istendiğinde F1 tuşuna basmayın**

    Bu güvenlik açığından başarıyla yararlanılabilmesi için, kullanıcının klavyesindeki F1 tuşuna basarak yararlanma girişimine yardım etmesi gerekir. İncelemelerimiz, kullanıcılar klavyelerindeki F1 tuşuna basmazsa güvenlik açığından yararlanılamadığını göstermektedir.

    Bu nedenle, kötü amaçlı Web siteleri, kullanıcıları F1 tuşuna basmaya ikna etmeye çalışabilir. Bu tür bir Web sitesi sonsuz bir iletişim kutusu döngüsü oluşturup kullanıcıya bu döngüyü sonlandırmak için F1 tuşuna basması gerektiğini bildirebilir veya ücretlendirme bilgilerinin ya da yardım içeriğinin F1 tuşu aracılığıyla sağlanacağı gibi bilgiler sunabilir.

    Kullanıcılara Web sayfaları veya diğer Internet içeriği tarafından istendiğinde F1 tuşuna basmamaları önerilir. Kullanıcıyı F1 tuşuna basmaya ikna etmek üzere sürekli olarak bir iletişim kutusu görüntüleniyorsa, kullanıcılar Internet Explorer işlemini sonlandırmak üzere sistem oturumunu kapatabilir veya Görev Yöneticisi'ni kullanabilirler.

-   **Windows Yardım Sistemi'ne erişimi kısıtlayın**

    Bu güvenlik açığından başarıyla yararlanılabilmesi için Windows Yardım Sistemi'nin çalışır durumda olması gerekir. Windows 2000, Windows XP ve Windows Server 2003'te yönetimsel komut satırından aşağıdaki komutu çalıştırıp winhlp32.exe aracının ACL'sini daha kısıtlayıcı olacak şekilde değiştirerek bu güvenlik açığına karşı korunabilirsiniz:

    echo Y | cacls "%windir%\\winhlp32.exe" /E /P everyone:N

    **Not** Bu geçici çözümü tamamlamak için yönetici olarak oturum açmış veya yönetici kimlik bilgilerine sahip olmanız gerekir.

    **Geçici çözümün etkisi.** Windows Yardım Sistemi kullanılamaz ve kullanıcılar uygulamaların yardım işlevini çağıramaz. Uygulamalarda Yardım işlevi açılmaya çalışıldığında bir hata iletisi görüntülenebilir.

    echo Y | cacls "%windir%\\winhlp32.exe" /E /P everyone

-   **Internet ve Yerel intranet bölgelerinde ActiveX Denetimleri ve Active Komut Dizileri'ni engellemek için bu güvenlik bölgelerinin ayarlarını "Yüksek" olarak ayarlayın**

    Internet güvenlik bölgesi ayarlarınızı, ActiveX denetimlerini ve Active Komut Dizilerini engelleyecek biçimde değiştirerek, bu güvenlik açığına karşı korunabilirsiniz. Bunun için, tarayıcı güvenliğinizi Yüksek olarak ayarlamalısınız.

    Internet Explorer'da tarayıcı güvenlik düzeyini artırmak için şu adımları izleyin:

    1.  Internet Explorer'ın **Araçlar** menüsünde, **Internet Seçenekleri**'ni tıklatın.
    2.  **Internet Seçenekleri** iletişim kutusunda, **Güvenlik** sekmesini ve sonra **Internet** simgesini tıklatın.
    3.  **Bölge güvenlik düzeyi** altında, kaydırıcıyı **Yüksek** düzeyine taşıyın. Ziyaret ettiğiniz tüm Web sitelerinin güvenlik düzeyi Yüksek olarak ayarlanır.

    **Not** Kaydırıcı görünmüyorsa, **Varsayılan Düzey**'i tıklatın ve sonra kaydırıcıyı **Yüksek**'e getirin.

    **Not** Düzeyin Yüksek olarak ayarlanması bazı Web sitelerinin doğru çalışmamasına neden olabilir. Bu ayarı değiştirdikten sonra bir Web sitesini kullanırken sorunlar yaşarsanız ve sitenin güvenli olduğundan eminseniz, bu siteyi güvenilen siteler listenize ekleyebilirsiniz. Böylece, sitenin Yüksek güvenlik ayarında da düzgün çalışması sağlanır.

    **Geçici çözümün etkisi.** ActiveX Denetimlerini ve Active Komut Dizilerini engellemenin yan etkileri vardır. Internet'te veya bir intranette bulunan birçok Web sitesi, ek işlevsellik sağlamak için ActiveX veya Active Komut Dizileri kullanır. Örneğin, bir çevrimiçi e-ticaret sitesi veya bankacılık sitesi, menüleri, sipariş formlarını, hatta hesap ekstrelerini sağlamak için ActiveX Denetimlerini kullanabilir. ActiveX Denetimleri veya Active Komut Dizilerini engellemek, tüm Internet ve intranet sitelerini etkileyen genel bir ayardır. Bu tür siteler için ActiveX Denetimlerini veya Active Komut Dizilerini engellemek istemiyorsanız "Güvendiğiniz siteleri Internet Explorer'ın Güvenilen siteler bölgesine ekleyin" bölümünde özetlenen adımları kullanın.

    **Güvendiğiniz siteleri Internet Explorer'ın Güvenilen siteler bölgesine ekleyin**

    Internet Explorer'ı, Internet bölgesinde ve Yerel intranet bölgesinde ActiveX denetimlerini ve Active Komut Dizilerini engelleyecek şekilde ayarladıktan sonra, güvendiğiniz siteleri Internet Explorer'ın Güvenilen siteler bölgesine ekleyebilirsiniz. Bu, güvenilen Web sitelerini tam olarak bugünkü gibi kullanmaya devam etmenizi sağlarken, güvenilmeyen sitelerde sizi bu saldırıya karşı korumaya yardımcı olur. Güvenilen siteler bölgesine yalnızca güvendiğiniz siteleri eklemeniz önerilir.

    Bunu yapmak için şu adımları izleyin:

    1.  Internet Explorer'da, **Araçlar**'ı tıklatın, **Internet Seçenekleri**'ni tıklatın, sonra da **Güvenlik** sekmesini tıklatın.
    2.  **Güvenlik ayarlarını belirlemek için bir Web içerik bölgesi seçin** kutusunda, **Güvenilen Siteler**'i ve ardından **Siteler**'i tıklatın.
    3.  Şifreli kanal gerektirmeyen siteler eklemek isterseniz, **Bu bölgedeki tüm sitelerden sunucu doğrulaması (https:) iste** onay kutusunu tıklatarak işaretini kaldırın.
    4.  **Aşağıdaki Web sitesini bölgeye ekle** kutusuna, güvendiğiniz bir sitenin URL'sini yazıp, sonra **Ekle**'yi tıklatın.
    5.  Bölgeye eklemek istediğiniz her site için bu adımları yineleyin.
    6.  Değişiklikleri kabul edip Internet Explorer'a dönmek için iki kez **Tamam**'ı tıklatın.

    **Not** Sisteminizde zararlı bir eylemi olmayacağına güvendiğiniz siteleri ekleyin. Özellikle şu ikisini eklemek isteyebilirsiniz: **\*.windowsupdate.microsoft.com** ve **\*.update.microsoft.com**. Bunlar, güncelleştirmeyi barındıran sitelerdir ve güncelleştirmeyi yüklemek için bir ActiveX denetimi gerektirir.

#### VBScript Yardımı'nda Tuşa Basma Güvenlik Açığı - CVE-2010-0483 Hakkında SSS

**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açan kullanıcı ile aynı haklara sahip olabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

**Güvenlik açığına ne neden olmaktadır?**  
Güvenlik açığı, Internet Explorer kullanılırken VBScript'in Windows Yardım dosyalarıyla etkileşim kurma biçimi nedeniyle oluşur.. Windows Yardım Dosyaları temelde güvenilir olmayan dosyalar olmakla birlikte, etkilenen sistemlerde VBScript uygulamaları, bir Web sayfasında iletişim kutusu görüntülenirken saldırganın bir Yardım dosyası konumu belirtmesine olanak verir. Bu tür bir iletişim kutusu görüntülenirken ziyaretçi F1 tuşuna basarsa, Windows Yardım Sistemi bu özel hazırlanmış Yardım dosyasını yükleyerek, oturum açmış kullanıcının güvenlik bağlamında rasgele kod çalıştırılmasına olanak verebilir.

**VBScript nedir?**  
VBScript (Visual Basic Script) genelde Web sitelerini daha esnek ve etkileşimli yapmak için kullanılan, yorumlanan, nesne tabanlı bir komut dosyası çalıştırma dilidir. VBScript komut dosyaları yalnızca Active Server Pages (ASP), Internet Explorer veya Windows Komut Dosyası Sistemi gibi bir yorumlayıcı veya ana bilgisayar varsa çalıştırılabilir. Daha fazla bilgi için, [VBScript Temelleri](http://msdn.microsoft.com/en-us/library/0ad0dkea(vs.85).aspx) adlı MSDN makalesine bakın.

**Güvenli olmayan türler nelerdir?**  
Microsoft "güvenli olmayan dosya türleri" kavramını, kullanıcıların bir saldırgan tarafından başka öğeler katıştırılabilecek yürütülebilir içeriğe karşı korunmalarına yardımcı olmak için kullanır. Daha fazla bilgi için, Microsoft Ürünlerindeki Yürütülebilir İçeriği Anlama adlı teknik incelemeye ve [Microsoft Bilgi Bankası makalesi 291369](http://support.microsoft.com/kb/291369/tr)'a bakın.

**F1 tuşu nedir?**  
F1, standart bir klavyedeki 12 işlev tuşundan biridir (F1 – F12) ve genelde en üstteki tuş sırasında bulunur. İşlev tuşları kullanıcıların bir uygulamadaki belirli işlevlere klavyedeki tek bir tuşa basarak erişebilmelerini sağlar. Çoğu uygulamada F1 tuşu Yardım işlevini çalıştırır ve çoğu kullanıcı "Yardım için F1 tuşuna basın" gibi yönergeleri bilmektedir.

**Kapsamlı savunma nedir?**  
Bilgi güvenliğinde kapsamlı savunma, bir ağın veya sistemin güvenliğinin saldırganlar tarafından aşılmasını önlemeye yardımcı olmak üzere birden çok savunma katmanı kullanılan bir yaklaşım anlamına gelir.

**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan oturum açan kullanıcının bağlamında rasgele kod çalıştırabilir. Kullanıcı yönetimsel haklarla oturum açtıysa, saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir.

**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Bu güvenlik açığından başarıyla yararlanılabilmesi için, saldırganın kullanıcıyı bir iletişim kutusu görüntülenen özel hazırlanmış bir Web sayfasını ziyaret etmeye ve bu iletişim kutusunun görüntülendiği sırada klavyesindeki F1 tuşuna basmaya ikna etmesi gerekir.

**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Web sayfalarına gözatmak için Internet Explorer'ın kullanıldığı istemci, iş istasyonu veya terminal sunucusu sistemleri birincil derecede risk altındadır.

**Windows Server 2003 veya Windows Server 2008 üzerinde Internet Explorer çalıştırıyorum. Bu durum, bu güvenlik açığının etkisini azaltır mı?**  
Internet Explorer, Windows Server 2003 ve Windows Server 2008'de varsayılan olarak bu güvenlik açığının etkilerini azaltan [Artırılmış Güvenlik Yapılandırması](http://go.microsoft.com/fwlink/?linkid=92039) adlı kısıtlı bir modda çalışır. Artırılmış Güvenlik Yapılandırması, bir kullanıcının veya yöneticinin bir sunucudaki özel hazırlanmış Web içeriğini yükleme ve çalıştırma olasılığını azaltan, bir grup önceden yapılandırılmış Internet Explorer ayarıdır. Bu, Internet Explorer'ın Güvenilen siteler bölgesine eklenmemiş Web siteleri için bir azaltıcı etkendir. [Internet Explorer Artırılmış Güvenlik Yapılandırmasını Yönetme](http://go.microsoft.com/fwlink/?linkid=92055) kılavuzuna da bakın.

**Internet bölgesi güvenlik ayarını Yüksek olarak ayarlamak bu güvenlik açığından beni nasıl korur?**  
Internet bölgesi güvenlik ayarının Yüksek olarak ayarlanması, bu güvenlik açığından yararlanmak için gerekli olan Active Komut Dizileri'ni devre dışı bırakarak bu güvenlik açığına karşı koruma sağlar.

**Bu güvenlik açığından Windows** **Vista, Windows Server 2008, Windows 7 veya Windows Server 2008 R2'de neden yararlanılamaz?**  
Windows Vista, Windows Server 2008, Windows 7 ve Windows Server 2008 R2, Internet Explorer'ın Windows işletim sistemi ve VBScript ile etkileşim kurma şekline yönelik mimari geliştirmeler içerir. Bu durum, kullanıcıların Windows Yardım Sistemi'ni yüklemiş oldukları sistemlerde güvenlik açığından etkilenen kod bulunsa da bu açıktan yararlanılmasını engeller.

**Güncelleştirme ne yapıyor?**  
Bu güvenlik güncelleştirmesi, VBScript altyapısının etkilenen Windows işletim sistemi sürümlerinde yardım dosyalarını korumalı modda işleme biçimini değiştirerek güvenlik açığını giderir. Windows Vista, Windows Server 2008, Windows 7 ve Windows Server 2008 R2'de bu güvenlik açığından yararlanmak için bilinen bir saldırı vektörü bulunmamaktadır, ancak bir kapsamlı savunma önlemi olarak bu güncelleştirme yüklenebilir.

**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Evet. Bu güvenlik açığı genel olarak duyurulmuştur. Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE-2010-0483](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0483) atanmıştır. Bu güvenlik güncelleştirmesi [Microsoft Güvenlik Danışma Belgesi 981169](http://technet.microsoft.com/security/advisory/981169)'da açıklanmıştır.

**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının müşterilere saldırmak üzere genel olarak kullanıldığına dair herhangi bir bilgi almamıştı.

    Microsoft Windows'un birçok sürümü olduğundan, aşağıdaki adımlar sizin sisteminizde farklı olabilir. Böyle bir durumda, bu adımları tamamlamak için ürün belgelerinize bakın.

    1.  **Başlat**'ı ve sonra **Ara**'yı tıklatın.
    2.  **Arama Sonuçları** bölmesinde, **Arama Yardımcısı**'nın altındaki **Tüm dosya ve klasörler**'i tıklatın.
    3.  **Dosya adının tümü veya bir parçası** kutusuna, uygun dosya bilgileri tablosundan bir dosya adı yazın ve **Ara**'yı tıklatın.
    4.  Dosya listesinde, uygun dosya bilgileri tablosundan bir dosya adını sağ tıklatın ve sonra **Özellikler**'i tıklatın.
        **Not** Sisteminizde yüklü işletim sisteminin veya programların sürümüne bağlı olarak, dosya bilgileri tablosunda listelenen dosyalardan bazıları yüklenmeyebilir.
    5.  **Sürüm** sekmesinde, sisteminize yüklenmiş olan dosyanın sürümünü uygun dosya bilgileri tablosunda belgelenen sürümle karşılaştırarak belirleyin.
        **Not** Dosya sürümü dışındaki öznitelikler yükleme sırasında değişebilir. Diğer dosya özniteliklerini dosya bilgileri tablosundaki bilgilerle karşılaştırmak, güncelleştirmenin uygulandığını doğrulama için desteklenen bir yöntem değildir. Ayrıca, belirli durumlarda, dosyalar kurulum sırasında yeniden adlandırılabilir. Dosya veya sürüm bilgileri yoksa, güncelleştirme kurulumunu kullanılabilen diğer yöntemlerden biriyle doğrulayın.

-   **Kayıt Defteri Anahtarı Doğrulaması**

    Bu güvenlik güncelleştirmesinde açıklanan dosyaların yüklenmiş olduğunu, bu bölümdeki **Başvuru Tablosu**'nda listelenen kayıt defteri anahtarlarını gözden geçirerek de doğrulayabilirsiniz.
