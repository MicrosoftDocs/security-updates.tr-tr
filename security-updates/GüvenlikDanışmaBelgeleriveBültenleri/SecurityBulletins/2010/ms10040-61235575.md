Security Bulletin

Microsoft Güvenlik Bülteni MS10-040 - Önemli
============================================

Internet Information Services'taki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (982666)
---------------------------------------------------------------------------------------------------

Yayım Tarihi: 8 Haziran 2010 Salı | Güncelleştirme Tarihi: 30 Haziran 2010 Çarşamba

**Sürüm:** 1.1

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi, Internet Information Services'taki (IIS) özel olarak bildirilen bir güvenlik açığını giderir. Bu güvenlik açığı, bir kullanıcı özel hazırlanmış bir HTTP isteği alırsa uzaktan kod yürütülmesine izin verebilir. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir.

Bu güvenlik güncelleştirmesi IIS 6.0, IIS 7.0 ve IIS 7.5 için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, kimlik doğrulama doğrulamasını düzelterek bu güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.**  [Microsoft Knowledge Base 982666 Makalesi](http://support.microsoft.com/kb/982666/tr) dosyaları güvenlik güncellemeleri ile müşterimizin tecrübe kazanmasıonı sağlıyor. Makalemiz çözüm öneren konuları ele alıyor.

-----------------------------------------------------------------

<span></span>
**Hangi sistemler bu güvenlik açığından etkileniyor?**  
Bu güvenlik açığı Windows 7 ve Windows Server 2008 R2 tabanlı sistemleri etkiler. Bunun yanı sıra, Kimlik Doğrulaması için Genişletilmiş Koruma'yı kullanan güvenlikle ilgili olmayan bir güncelleştirme olan [KB973917](http://support.microsoft.com/kb/973917)'nin yüklü olduğu Windows Server 2003, Windows Vista ve Windows Server 2008 tabanlı sistemleri de etkiler.

**KB973917'yi yüklediysem bilgisayarım bu güvenlik açığından etkilenir mi?**  
KB973917'nin yüklü olmadığı desteklenen Windows Server 2003, Windows Vista ve Windows Server 2008 sürümleri bu güvenlik açığından etkilenmez. Windows 7 ve Windows Server 2008 R2'nin desteklenen sürümlerinin çalıştırıldığı sistemler bu açıktan etkilenir.

**KB973917 yüklü değilse bu güvenlik güncelleştirmesi sunulacak mı?**  
KB973917'deki güncelleştirmenin yüklü olmadığı sistemler bu güvenlik açığından etkilenmediği için bu güncelleştirme sunulmayacaktır.

**Dosya bilgileri ayrıntılarını nerede bulabilirim?**

IIS Kimlik Doğrulaması Sırasında Bellek Bozulması Güvenlik Açığı - CVE-2010-1256  
--------------------------------------------------------------------------------
  
<span></span>
Internet Information Services'ta (IIS) bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Güvenlik açığı, kimlik doğrulama bilgilerinin düzgün ayrıştırılmaması nedeniyle ortaya çıkar. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, Çalışan İşlem Kimliği (WPI) bağlamında kod yürütebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz. [CVE-2010-1256](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-1256).
  
#### IIS Kimlik Doğrulaması Sırasında Bellek Bozulması Güvenlik Açığı - CVE-2010-1256 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   KB973917'nin yüklü olmadığı Windows Server 2003, Windows Vista ve Windows Server 2008 tabanlı sistemlerde Kimlik Doğrulaması için Genişletilmiş Koruma özelliği bulunmaz ve bu sistemler bu güvenlik açığından etkilenmezler.  
-   Bir sistemde KB973917 yüklü olsa da, Kimlik Doğrulaması için Genişletilmiş Koruma etkilenen hiçbir platformda varsayılan olarak etkinleştirilmiş durumda değildir. Sistemler yalnızca bu özellik etkinleştirildiğinde bu açıktan etkilenirler.
  
#### IIS Kimlik Doğrulaması Sırasında Bellek Bozulması Güvenlik Açığı - CVE-2010-1256 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Simge Denetimi'ni gerekli kılın**
  
    Internet Information Services'ın [Microsoft Bilgi Bankası makalesi 973917](http://support.microsoft.com/kb/973917)'de açıklandığı şekilde Genişletilmiş Koruma özelliğinin simge denetimini 'Gerekli Kıl' olarak kullanacak biçimde yapılandırılmış olduğu sistemler bu güvenlik açığından etkilenmezler.
  
    **Geçici çözümün etkisi.** Bu yapılandırma değişikliği, Genişletilmiş Koruma özelliğini kullanmayan istemcilerin kimlik doğrulaması yapmalarına izin vermez.
  
#### IIS Kimlik Doğrulaması Sırasında Bellek Bozulması Güvenlik Açığı - CVE-2010-1256 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**    
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan kimliği doğrulanmış bir uzak saldırgan, IIS sunucusunda Çalışan İşlem Kimliği (WPI) ile aynı haklarla işlem gerçekleştirebilir. Bu kimlik varsayılan olarak Windows Server 2003, Windows Vista ve Windows Server 2008'de Ağ Hizmeti hesabı olacak şekilde yapılandırılmıştır. Bu kimlik Windows 7 ve Windows Server 2008 R2'de bir sanal uygulama havuzu kimliği olacak şekilde yapılandırılmıştır.
  
**Güvenlik açığına ne neden olmaktadır?**    
Bu güvenlik açığı, Internet Information Services Web sunucusunun istemciden alınan belirli kimlik doğrulama bilgilerini ayrıştırırken belleği doğru şekilde ayırmaması nedeniyle ortaya çıkar.
  
**Güvenlik açığından hangi bileşen etkilenmektedir?**    
Bu güvenlik açığı, yalnızca Kimlik Doğrulaması için Genişletilmiş Koruma kullanılarak gerçekleştirilen kimlik doğrulaması isteklerini destekleyecek şekilde yapılandırılmış Internet Information Services Web sunucularını etkiler. Bu bileşenin güvenlik açığından etkilenmesi için, Genişletilmiş Koruma özelliğinin simge denetimine izin verecek şekilde varsayılan dışında bir durumda yapılandırılmış olması gerekir.
  
**Kimlik Doğrulaması için Genişletilmiş Koruma nedir?**    
Kimlik Doğrulaması için Genişletilmiş Koruma, bir saldırganın istemci kimlik bilgilerini ele geçirip bir sunucuya iletmesi anlamına gelen ortadaki adam (MITM) saldırılarına karşı koruma sağlar. Kimlik Doğrulaması için Genişletilmiş Koruma, yöneticilerin ve kullanıcıların bu ek korumayı sağlamak üzere yüklemeleri gereken güvenlikle ilgili olmayan güncelleştirmeler olarak dağıtılmaktadır.
  
Genişletilmiş Koruma; Windows Server 2003, Windows Vista ve Windows Server 2008 için güvenlikle ilgili olmayan bir güncelleştirme olarak edinilebilir (bkz: [Microsoft Bilgi Bankası makalesi 973917](http://support.microsoft.com/kb/973917)). Windows 7 ve Windows Server 2008 R2'de bu özellik isteğe bağlı bir bileşen olarak bulunmaktadır ve Internet Information Services (IIS) Web sunucusu için Windows Kimlik Doğrulaması rol hizmetini yüklediğinizde etkinleştirilir.
  
**Kimlik Doğrulaması için Genişletilmiş Koruma'nın etkinleştirilmiş olup olmadığını nasıl doğrularım?**    
Kimlik Doğrulaması için Genişletilmiş Koruma'yı yapılandırmanın birkaç yolu vardır. Bu özelliğin bulunup bulunmadığını doğrulamak için de kullanılabilecek yapılandırma bilgileri için, bkz: [Microsoft Bilgi Bankası makalesi 973917](http://support.microsoft.com/kb/973917).
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**    
Bu güvenlik açığından başarıyla yararlanan kimliği doğrulanmış bir uzak saldırgan, IIS sunucusunda Çalışan İşlem Kimliği (WPI) ile aynı haklarla işlem gerçekleştirebilir. WPI varsayılan olarak Network Service hesabı ayrıcalıklarıyla yapılandırılır. Uygulama havuzları yönetimsel ayrıcalıklara sahip bir hesap kullanılan WPI ile yapılandırılmış olan ASP sayfaları çalıştıran IIS sunucular, uygulama havuzu varsayılan WPI ayarlarıyla yapılandırılmış IIS sunuculardan daha fazla etkilenebilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**    
Kimlik Doğrulaması için Genişletilmiş Koruma özelliği istemcide etkinleştirilmemişse ama sunucuda etkinleştirilmiş durumdaysa IIS sunucusunun kimlik doğrulama girişimlerini işleme biçiminde uzaktan kod yürütülmesine yönelik bir güvenlik açığı bulunmaktadır. Saldırgan, özel hazırlanmış bir HTTP isteği oluşturarak bu güvenlik açığından yararlanabilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**    
Kimlik doğrulaması gerektiren ve Kimlik Doğrulaması için Genişletilmiş Koruma özelliğini destekleyen içerik barındırılan Web sunucuları bu güvenlik açığından en fazla etkilenen sistemlerdir. Yalnızca [Microsoft Bilgi Bankası makalesi 973917](http://support.microsoft.com/kb/973917)'de açıklanan Kimlik Doğrulaması için Genişletilmiş Koruma güncelleştirmesinin yüklü olduğu sistemler bu açıktan etkilenir.
  
**Güncelleştirme ne yapıyor?**    
Bu güncelleştirme IIS'nin istemci tarafından geçirilen belirli kimlik doğrulaması bilgilerini doğrulama biçimini düzelterek bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**    
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**    
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.
