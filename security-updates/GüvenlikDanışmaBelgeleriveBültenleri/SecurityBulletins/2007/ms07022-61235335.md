Security Bulletin

Microsoft Güvenlik Bülteni MS07-022 - Önemli
============================================

Windows Çekirdeğindeki Güvenlik Açığı Ayrıcalık Yükselmesine İzin Verebilir (931784)
------------------------------------------------------------------------------------

Yayım Tarihi: 10 Nisan 2007 Salı | Güncelleştirme Tarihi: 26 Haziran 2007 Salı

**Sürüm:** 2.0

#### Özet

**Bu Belge Kimler İçindir:** Microsoft Windows kullanan müşteriler

**Güvenlik Açığının Etkisi:** Ayrıcalık Yükselmesi

**En Yüksek Önem Derecesi:** Önemli

**Öneriler:** Müşteriler güncelleştirmeyi ilk fırsatta uygulamalıdır

**Değiştirilen Güvenlik Güncelleştirmeleri:** Bu bülten önceki bir güvenlik güncelleştirmesinin yerini alır. Daha ayrıntılı bilgi için bu bültenin Sık Sorulan Sorular (SSS) bölümüne bakın.

**Sınanan Yazılımlar ve Güvenlik Güncelleştirmesi Yükleme Konumları:**

**Etkilenen Yazılımlar:**

-   Microsoft Windows XP Service Pack 2 — [Güncelleştirmeyi karşıdan yükle](http://www.microsoft.com/downloads/details.aspx?familyid=eeaee4a7-4858-4b6b-9c6d-a9f1eae19b51)
-   Microsoft Windows Server 2003, Microsoft Windows Server 2003 Service Pack 1 ve Microsoft 2003 Service Pack 2 — [Güncelleştirmeyi karşıdan yükle](http://www.microsoft.com/downloads/details.aspx?familyid=5e94a29c-7ec0-4459-92eb-d43b524fd6fd)

**Etkilenmeyen Yazılımlar:**

-   Microsoft Windows Server 2003 x64 Edition ve Microsoft Windows Server 2003 x64 Edition Service Pack 2
-   Itanium tabanlı sistemler için Microsoft Windows Server 2003 ve Itanium tabanlı sistemler için Microsoft Windows Server 2003 SP1
-   Windows Vista

Yürütmeyle İlgili Özet
----------------------

<span></span>
**Yürütmeyle İlgili Özet:**

Bu güncelleştirme, yeni keşfedilen ve özel olarak bildirilen bir güvenlik açığını giderir. Güvenlik açığı, bu bültendeki Güvenlik Açığı Ayrıntıları bölümünde kendi alt bölümünde belgelenmektedir.

Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.

Müşterilerin güncelleştirmeyi ilk fırsatta uygulamalarını öneririz.

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)
-----------------------------------------------------------------

<span></span>
**Microsoft bu bülteni 26 Haziran 2007'de neden yeniden yayımladı?**
Microsoft bu bülteni, NEC 98 sistemlerine Windows 2000 Service Pack 4 yükleyen müşterilere güvenlik güncelleştirmesinin yeniden düzenlenen bir sürümünün kullanılabilir olduğunu bildirmek amacıyla güncelleştirdi. Bu sistemlerde, güvenlik güncelleştirmesinin NEC 98 sistemler için hatalı bir derleme sürümü nedeniyle, Windows Update hizmetine bağlanmaya çalışırken sorunlar oluşabilir. Özgün güvenlik güncelleştirmesi, bu güvenlik bülteninde açıklanan güvenlik açıklarını başarıyla gidermiştir.

Microsoft, Windows 2000 Service Pack 4'ün yüklü olduğu NEC 98 kullanan müşterilerin, Windows Update hizmetinin gelecekteki güncelleştirmeleri doğru biçimde sunmasını sağlamak için yeniden düzenlenen güvenlik güncelleştirmesini Microsoft Yükleme Merkezi Web sitesinden karşıdan yüklemelerini önerir. Windows 2000 Service Pack 4 çalıştıran NEC 98 müşterileri bu güvenlik güncelleştirmesini kaldırarak, yeniden düzenlenen güvenlik güncelleştirmesini Windows Update hizmetinden otomatik olarak da edinebilir.

Bu sorunla ilgili daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 931784](http://support.microsoft.com/kb/931784/). [Microsoft Bilgi Bankası makalesi 931784](http://support.microsoft.com/kb/931784/), yeniden düzenlenen bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir.

Güvenlik Açığı Ayrıntıları
--------------------------

<span></span>
#### Çekirdekte Yerel Olarak Ayrıcalık Yükselmesi Güvenlik Açığı - CVE-2007-1206:

Windows Çekirdeği'nde, eşlenen bir bellek kesimindeki hatalı izinler nedeniyle bir ayrıcalık yükselmesi güvenlik açığı bulunmaktadır. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir.

#### Çekirdekte Yerel Olarak Ayrıcalık Yükselmesi Güvenlik Açığı - CVE-2007-1206 için Azaltıcı Etkenler:

Saldırganın bu güvenlik açığından yararlanabilmesi için geçerli oturum açma kimlik bilgilerine sahip olması ve yerel olarak oturum açabilmesi gerekir. Bu güvenlik açığı uzaktan veya anonim kullanıcılar tarafından kullanılamaz.

#### Çekirdekte Yerel Olarak Ayrıcalık Yükselmesi Güvenlik Açığı - CVE-2007-1206 için Geçici Çözümler:

Bu güvenlik açığı için herhangi bir geçici çözüm belirlenememiştir.

#### Çekirdekte Yerel Olarak Ayrıcalık Yükselmesi Güvenlik Açığı - CVE-2007-1206 Hakkında SSS:

**Bu bülten neden yeniden sunuluyor? **
NEC 98 makinelerde Windows 2000 çalıştıran müşteriler güvenlik güncelleştirmesi 931784'ü doğru almamış olabilir. Microsoft bu sorunu gidermek için bu bülteni yeniden yayımlamıştır. Yalnızca NEC 98 kullanan müşteriler bu yeniden yayımlanan sürümden etkilenir, diğer müşteriler etkilenmez.

**Bu güvenlik açığının kapsamı nedir? **
Windows Çekirdeği'nde, eşlenen bir bellek kesimindeki hatalı izinler nedeniyle bir ayrıcalık yükselmesi güvenlik açığı bulunmaktadır. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Bir saldırganın bu güvenlik açığından yararlanmaya çalışması için, sistemde yerel olarak oturum açabilmesi gerekir.

**Güvenlik açığına ne neden olmaktadır? **
Windows Çekirdeği, bir bellek kesimi eşlenirken yanlış izinlerin kullanılmasına izin verir.

**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir? **
Bu güvenlik açığından başarıyla yararlanan bir saldırgan kullanıcının sisteminde yükseltilmiş ayrıcalıklarla rasgele kod çalıştırabilir. Bu, saldırganın etkilenen sistemin tüm denetimini ele geçirmesine izin verebilir.

**Bir saldırgan güvenlik açığından nasıl yararlanabilir? **
Saldırganın bu güvenlik açığından yararlanabilmesi için öncelikle sistemde oturum açması gerekir. Böylece saldırgan, güvenlik açığından yararlanan özel olarak hazırlanmış bir uygulama çalıştırabilir ve etkilenen sistemin tüm denetimini ele geçirebilir.

**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor? **
İş istasyonları ve terminal sunucuları birincil derecede risk altındadır. Yöneticiler kullanıcıların sunucularda oturum açmalarına ve program çalıştırmalarına izin verirse sunucular daha fazla etkilenebilir. Ancak, en iyi uygulamalarda buna izin verilmesi kesinlikle önerilmemektedir.

**Bu güvenlik açığı Internet üzerinden kullanılabilir mi? **
Hayır. Saldırının gerçekleşebilmesi için, saldırganın hedeflenen sistemde oturum açabilmesi gerekir. Saldırgan, bu güvenlik açığından yararlanarak bir programı uzaktan yükleyemez ve çalıştıramaz.

**Güncelleştirme ne yapıyor? **
Güncelleştirme, Windows Çekirdeği'nin bellek eşleme sırasındada izinleri doğrulama biçimini değiştirerek bu güvenlik açığını ortadan kaldırır.

**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu? **
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının genel olarak duyurulduğuna dair herhangi bir bilgi almamıştı.

**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı? **
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.
