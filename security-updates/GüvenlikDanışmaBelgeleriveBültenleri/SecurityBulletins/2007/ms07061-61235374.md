Security Bulletin

Microsoft Güvenlik Bülteni MS07-061 - Kritik
============================================

Windows URI İşlemedeki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (943460)
---------------------------------------------------------------------------------------

Yayım Tarihi: 13 Kasım 2007 Salı | Güncelleştirme Tarihi: 16 Ocak 2008 Çarşamba

**Sürüm:** 1.2

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güncelleştirme, genel olarak bildirilen bir güvenlik açığını giderir. Windows kabuğunun kendisine iletilen özel hazırlanmış URI'ları işleme biçiminde bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Windows kabuğu bu URI'ları yeterli düzeyde doğrulamamışsa, bir saldırgan bu güvenlik açığından yararlanarak rasgele kod yürütebilir. Microsoft yalnızca Internet Explorer 7 kullanılan sistemlerde bu güvenlik açığından yararlanmanın yollarını belirlemiştir. Ancak güvenlik açığı, Windows XP ve Windows Server 2003'ün tüm desteklenen sürümlerinde bulunan Shell32.dll adlı bir Windows dosyasında yer almaktadır.

Bu, Windows XP ve Windows Server 2003'ün tüm desteklenen sürümleri için kritik bir güvenlik güncelleştirmesidir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Bu güvenlik güncelleştirmesi, Windows kabuğunun geçersiz URI'ları işleme biçimini değiştirerek güvenlik açığını ortadan kaldırır. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

Bu güvenlik güncelleştirmesi, ilk olarak [Microsoft Güvenlik Danışma Belgesi 943521](http://technet.microsoft.com/security/advisory/943521)'de açıklanan güvenlik açığını da ortadan kaldırır. Daha fazla bilgi için Güvenlik Açığı Bilgileri altındaki belirli bir güvenlik açığı girdisine yönelik Sık Sorulan Sorular alt bölümüne bakın.

**Öneri.** Microsoft, müşterilerinin güncelleştirmeyi hemen uygulamasını önerir.

**Bilinen Sorunlar.** [Microsoft Bilgi Bankası makalesi 943460](http://support.microsoft.com/kb/943460), müşterilerin bu güvenlik güncelleştirmesini yüklerken karşılaşabileceği bilinen sorunları belgelemektedir. Bu makale ayrıca, bu sorunlar için önerilen çözümleri de içermektedir.

Windows URI İşleme Güvenlik Açığı - CVE-2007-3896
-------------------------------------------------

<span></span>
Windows kabuğunun kendisine iletilen özel hazırlanmış URI'ları işleme biçiminde bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Saldırgan, uzaktan kod yürütülmesine izin verebilecek özel hazırlanmış bir URI'yı bir uygulamaya veya eke iliştirerek bu güvenlik açığından yararlanabilir.

Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2007-3896](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3896).

#### Windows URI İşleme Güvenlik Açığı - CVE-2007-3896 için Azaltıcı Etkenler

Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:

-   Windows 2000 Service Pack 4 etkilenmez
-   Windows Vista etkilenmez
-   Windows Vista x64 Edition etkilenmez
-   Microsoft, Internet Explorer sürüm 6 çalışan Windows işletim sistemlerinde bu güvenlik açığından yararlanmanın herhangi bir yolunu belirleyememiştir
-   Bu güvenlik açığından başarıyla yararlanan bir saldırgan, yerel kullanıcı ile aynı haklara sahip olabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
-   Bu saldırının e-posta kullanılarak yapılan türünde, e-postaları düz metin olarak okuyan müşteriler bu güvenlik açığına karşı daha az risk altındadır. Kullanıcıların bu güvenlik açığına karşı daha fazla risk altında olmaları için, kötü amaçlı bir Web sitesine götüren bir bağlantıyı tıklatmaları veya bir eki açmaları gerekir.

#### Windows URI İşleme Güvenlik Açığı - CVE-2007-3896 için Geçici Çözümler

Microsoft bu güvenlik açığı için herhangi bir geçici çözüm belirleyememiştir.

#### Windows URI İşleme Güvenlik Açığı - CVE-2007-3896 Hakkında SSS

**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açan kullanıcı ile aynı haklara sahip olabilir.

Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

**Güvenlik açığına ne neden olmaktadır?**  
Windows kabuğu geçersiz URI'ları yeterli düzeyde işlemez.

**URI nedir?**
Tekdüzen Kaynak Tanımlayıcısı (URI), Internet'teki veya ağ üzerindeki kaynaklara göre davranan veya bunları tanımlayan bir karakter dizisidir. Bir URL, Web sitesi gibi bir kaynağa başvuran tipik bir URI örneğidir. URI'lar hakkında daha fazla bilgi için, [RFC 2396](http://www.ietf.org/rfc/rfc2396.txt)'ya bakın.

**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Kullanıcıyı bir postanın ekini açmaya veya saldırganın Web sitesine yönlendiren bir bağlantıyı izlemeye ikna eden bir saldırgan, oturum açan kullanıcının bağlamında rasgele kod çalıştırabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırganın özel hazırlanmış bir URI oluşturması ve bu URI'yı etkilenen sistemdeki bir uygulamaya girdi olarak sağlayarak, URI tarafından başvurulan kaynağa erişmeye çalışması gerekir. URI'ları e-posta ekleri ya da belgeler gibi güvenilir olmayan kaynaklardan veya güvenilir olduğu varsayılarak ağ verilerinden alan uygulamalar bu güvenlik açığından etkilenir. Belirli durumlarda, özel hazırlanmış URI'ların işlenmesi rasgele kod yürütülmesine izin verebilir. Saldırganın bu güvenlik açığından yararlanabilmesi için, kullanıcıyı ilgili eki veya uygulamayı açmaya ya da Web sitesini ziyaret etmeye ikna etmesi gerekir. Bir saldırgan, bu güvenlik açığından Internet Explorer yardımıyla yararlanacak ve kullanıcıyı Web sitesini görüntülemeye ikna edecek biçimde tasarlanmış özel hazırlanmış bir Web sitesi barındırabilir. Bu güvenlik açığından güvenliği aşılan Web siteleri ve kullanıcı tarafından sağlanan içeriği veya reklamları kabul eden ya da barındıran Web siteleri aracılığıyla da yararlanılabilir. Bu Web sitelerinde bu güvenlik açığından yararlanan özel hazırlanmış URI'lar bulunabilir. Etkilenen sistemlere Web içeriği ulaştırmak için başlık reklamları veya başka yöntemler kullanılarak da özel olarak oluşturulmuş URI'lar görüntülenebilir.

**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Internet Explorer 7 yüklü olarak Windows XP ve Windows Server 2003'ün desteklenen sürümlerini çalıştıran sistemler bu güvenlik açığından etkilenir.

**Güncelleştirme ne yapıyor? **
Güncelleştirme, Windows kabuğunun geçersiz URI'ları işleme biçimini değiştirerek güvenlik açığını ortadan kaldırır.

**Windows Internet Explorer 7 yüklü değil. Bu güncelleştirmeyi neden alıyorum? **
Güvenlik açığı, Windows XP ve Windows Server 2003'ün tüm desteklenen sürümlerinde bulunan Shell32.dll adlı bir Windows dosyasında yer almaktadır. Microsoft, bu işletim sistemlerinde bulunan tarayıcı olan Internet Explorer 6'nın kullanıldığı sistemlerde bu güvenlik açığından yararlanmanın herhangi bir yolunu belirleyememiştir. Kapsamlı bir savunma ölçütü olarak, bu güvenlik güncelleştirmesi hangi Internet Explorer sürümünün yüklü olduğuna bakılmaksızın Windows XP ve Windows Server 2003'ün desteklenen sürümlerini kullanan tüm müşterilere sunulmuştur.

**Windows** **Vista** **kullanıyorum. Bu güvenlik açığından hala etkilenir miyim? **
Hayır. Windows Vista bu güvenlik açığından etkilenmez. Windows Internet Explorer 7, Windows Vista ile birlikte gelir; ancak Windows Vista'daki Windows kabuğu bu güvenlik açığından etkilenmez.

**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu? **
Evet. Bu güvenlik bülteni ilk yayımlandığında, bu güvenlik açığı genel olarak duyurulmuştu. Ortak Güvenlik Açığı ve [CVE-2007-3896](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3896) Etkilenme Numarası atanmıştır. Bu güvenlik açığı ilk olarak [Microsoft Güvenlik Danışma Belgesi 943521](http://technet.microsoft.com/security/advisory/943521)'de açıklanmıştır.

**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı? **
Evet. Güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığı bilgisini almıştı.

**Bu güvenlik güncelleştirmesini uygulamak, müşterilerin açıktan yararlanmaya yönelik olarak yayımlanan koddan korunmasına yardımcı olur mu? **
Evet. Bu güvenlik güncelleştirmesi, şu anda yararlanılmakta olan güvenlik açığını ortadan kaldırır. Güvenlik açığı giderilmiş, ayrıca Ortak Güvenlik Açığı ve [CVE-2007-3896](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3896) Etkilenme numarası atanmıştır.

    Microsoft Windows'un birkaç sürümü olduğundan, aşağıdaki adımlar sizin bilgisayarınızda farklı olabilir. Böyle bir durumda, bu adımları tamamlamak için ürün belgelerinize bakın.

    1.  **Başlat**'ı ve sonra **Ara**'yı tıklatın.
    2.  **Arama Sonuçları** bölmesinde, **Arama Yardımcısı**'nın altındaki **Tüm dosya ve klasörler**'i tıklatın.
    3.  **Dosya adının tümü veya bir parçası** kutusuna, uygun dosya bilgileri tablosundan bir dosya adı yazın ve **Ara**'yı tıklatın.
    4.  Dosya listesinde, uygun dosya bilgileri tablosundan bir dosya adını sağ tıklatın ve sonra **Özellikler**'i tıklatın.

Ayrıca, belirli durumlarda, dosyalar kurulum sırasında yeniden adlandırılabilir. Dosya veya sürüm bilgileri yoksa, güncelleştirme kurulumunu kullanılabilen diğer yöntemlerden biriyle doğrulayın.

-   **Kayıt Defteri Anahtarı Doğrulaması**

    Bu güvenlik güncelleştirmesinde açıklanan dosyaların yüklenmiş olduğunu, bu bölümdeki **Başvuru Tablosu**'nda listelenen kayıt defteri anahtarlarını gözden geçirerek de doğrulayabilirsiniz.

    Bu kayıt defteri anahtarları yüklü dosyaların tam bir listesini içermeyebilir. Ayrıca bir yönetici veya OEM, bu güvenlik güncelleştirmesini Windows kurulum kaynak dosyalarıyla tümleştirdiğinde veya eklediğinde, bu kayıt defteri anahtarları doğru oluşturulmayabilir.
