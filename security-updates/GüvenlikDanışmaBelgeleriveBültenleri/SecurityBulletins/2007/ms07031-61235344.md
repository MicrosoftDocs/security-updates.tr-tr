Security Bulletin

Microsoft Güvenlik Bülteni MS07-031 - Kritik
============================================

Windows Schannel Güvenlik Paketindeki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (935840)
------------------------------------------------------------------------------------------------------

Yayım Tarihi: 12 Haziran 2007 Salı

**Sürüm:** 1.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu kritik güvenlik güncelleştirmesi, Windows'daki Güvenli Kanal (Schannel) güvenlik paketindeki özel olarak bildirilen bir güvenlik açığını giderir. Schannel güvenlik paketi, Güvenli Yuva Katmanı (SSL) ve Aktarım Katmanı Güvenliği (TLS) Internet standartları kimlik doğrulama protokollerini kullanır. Bu güvenlik açığı, bir kullanıcı bir Internet web tarayıcısı kullanarak özel hazırlanmış bir Web sayfasını görüntülerse veya SSL/TLS kullanan bir uygulama çalıştırırsa uzaktan kod yürütülmesine olanak verebilir. Ancak bu güvenlik açığından yararlanma girişimleri büyük bir olasılıkla Internet Web tarayıcısından veya uygulamadan çıkılmasına neden olacaktır. Sistem yeniden başlatılıncaya kadar Web sitelerine veya kaynaklara SSL veya TLS kullanarak bağlanamaz.

Bu, Windows XP'nin desteklenen sürümleri için kritik, Windows 2003 sürümleri için önemli ve Windows 2000 sürümleri için orta düzeyde bir güvenlik güncelleştirmesidir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Bu güvenlik güncelleştirmesi, istemcinin sunucudan gönderilen sunucu anahtarı değişim verilerini ayrıştırma biçimini değiştirerek güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Microsoft, müşterilerinin güncelleştirmeyi hemen uygulamasını önerir.

**Bilinen Sorunlar.** Yok

**Yerine Geçtiği Güncelleştirme.** Bu güvenlik güncelleştirmesi önceki bir güvenlik güncelleştirmesinin yerini almaz.

Windows Schannel Güvenlik Paketindeki Güvenlik Açığı - CVE-2007-2218  
--------------------------------------------------------------------
  
<span></span>
Bir istemci bilgisayardaki Windows Schannel güvenlik paketinin sunucu tarafından gönderilen dijital imzaları doğrulama biçiminde bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Bir saldırgan, bu güvenlik açıklarından Internet Web tarayıcısı üzerinden yararlanacak biçimde tasarlanmış özel hazırlanmış bir Web sitesi barındırabilir ve kullanıcıyı bu Web sitesini görüntülemeye ikna edebilir. Ancak bu durumların hiçbirinde, saldırganın, bu Web sitelerini ziyaret etmek için kullanıcıyı zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi Web sitesine götüren bir bağlantıyı tıklatmalarını sağlayarak onları bu Web sitesine çekmesi gerekir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2007-2218](http://cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-2218)
  
#### Windows Schannel Güvenlik Paketindeki Güvenlik Açığı - CVE-2007-2218 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Web tabanlı saldırı senaryosunda, bir saldırganın bu açıktan yararlanmak için kullanılan bir Web sayfası içeren bir Web sitesi barındırabilir. Ayrıca, kullanıcı tarafından sağlanan içeriği veya reklamları kabul eden ya da barındıran Web sitelerinde, bu güvenlik açığından yararlanılabilecek özel hazırlanmış içerik bulunabilir. Ancak bu durumların hiçbirinde, saldırganın, bu Web sitelerini ziyaret etmek için kullanıcıyı zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi Web sitesine götüren bir bağlantıyı tıklatmalarını sağlayarak onları bu Web sitesini ziyaret etmeye ikna etmesi gerekir.  
-   Bu güvenlik açığından yararlanma girişimleri, büyük bir olasılıkla Uzaktan Kod Yürütme yerine, etkilenen sistemin beklenmedik biçimde yeniden başlatılması nedeniyle oluşan bir Hizmet Reddi durumunun oluşmasına neden olabilir. Windows 2000'de, hizmet reddi durumu varolan ve yeni SSL ve TLS bağlantılarıyla sınırlıdır ve bunun dışındaki durumlarda sistem kullanılmaya devam edilebilir. Windows Server 2003'te, başarılı bir saldırı sistemin yeniden başlatılmasına neden olabilir.
  
#### Windows Schannel Güvenlik Paketindeki Güvenlik Açığı - CVE-2007-2218 için Geçici Çözümler
  
Microsoft bu güvenlik açığı için herhangi bir geçici çözüm belirleyememiştir.
  
#### Windows Schannel Güvenlik Paketindeki Güvenlik Açığı - CVE-2007-2218 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Windows XP'de bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, etkilenen sistemin tüm denetimini uzaktan ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Bu güvenlik açığından yararlanma girişimleri, büyük bir olasılıkla uzaktan kod yürütülmesi yerine, bir hizmet reddi durumuna neden olacaktır.
  
Windows 2000'de, bu güvenlik açığı etkilenen sistemin SSL ve TLS bağlantılarını kabul etmemeye başlamasına neden olabilir. Sistem yeniden başlatılıncaya kadar Web sitelerine veya kaynaklara SSL veya TLS kullanarak bağlanamaz.
  
Windows 2003'te, bu güvenlik açığı etkilenen sistemin yeniden başlatılmasına neden olabilir.
  
Windows 2000 ve Windows 2003'te, bu güvenlik açığı bir saldırganın doğrudan kod yürütmesine veya kullanıcı haklarını yükseltmesine izin vermez.
  
**Güvenlik açığına ne neden olmaktadır?**  
Schannel güvenlik paketi SSL el sıkışması sırasında sunucu tarafından gönderilen özel hazırlanmış dijital imzalar için yeterli denetim gerçekleştirmez.
  
**Schannel nedir?**  
Güvenli Kanal (Schannel) güvenlik paketi, Güvenli Yuva Katmanı (SSL) ve Aktarım Katmanı Güvenliği (TLS) Internet standartları kimlik doğrulama protokollerini kullanan bir Güvenlik Destek Sağlayıcısı'dır (SSP). Schannel, istemci ile sunucu arasında güvenli iletişim için kimlik doğrulama hizmeti sağlayan güvenlik paketinin bir parçasıdır. Schannel hakkında daha fazla bilgi için şu [MSDN Web sitesini](http://msdn2.microsoft.com/en-us/library/ms678421.aspx) ziyaret edin.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Windows XP sürümlerinde, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin tüm denetimini ele geçirebilir. Ancak bu güvenlik açığından yararlanma girişimleri büyük bir olasılıkla Internet Web tarayıcısından veya SSL/TLS kullanan uygulamadan çıkılmasına neden olacaktır. Sistem yeniden başlatılıncaya kadar Web sitelerine veya kaynaklara SSL veya TLS kullanarak bağlanamaz. Windows 2000 ve Windows 2003 sürümlerinde, bu güvenlik açığından yararlanma girişimleri bir hizmet reddi durumunun oluşmasına neden olur.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Bir saldırgan, bu güvenlik açığından Internet Web tarayıcısı üzerinden yararlanacak biçimde tasarlanmış özel hazırlanmış bir Web sitesi barındırabilir ve kullanıcıyı bu Web sitesini görüntülemeye ikna edebilir. Sunucu tarafından gönderilen bu özel hazırlanmış dijital imzanın işlenmesi, bu API'yi kullanan tüm uygulamaların sonlandırılmasına neden olur ve sistem yeniden başlatılıncaya kadar SSL protokolünü devre dışı bırakır. Ancak bu durumların hiçbirinde, saldırganın, bu Web sitelerini ziyaret etmek için kullanıcıyı zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Anlık İleti isteğindeki kendi Web sitesine götüren bir bağlantıyı tıklatmalarını sağlayarak onları bu Web sitesine çekmesi gerekir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Bu güvenlik açığı, kötü amaçlı bir eylemin gerçekleşmesi için bir kullanıcının oturum açmasını ve bir Web sitesini ziyaret etmesini gerektirir. Bu nedenle, bir Internet Web tarayıcısının veya iş istasyonları veya terminal sunucuları gibi SSL ya da TLS kullanan uygulamaların sıkça kullanıldığı sistemler bu açık karşısında en fazla risk altındadır.
  
**Güncelleştirme ne yapıyor?**  
Bu güncelleştirme, istemcinin sunucudan gönderilen sunucu anahtarı değişim verilerini ayrıştırma biçimini değiştirerek güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.

-   **Dosya Sürümü Doğrulaması**

    Microsoft Windows'un birçok sürümü olduğundan, aşağıdaki adımlar sizin sisteminizde farklı olabilir. Böyle bir durumda, bu adımları tamamlamak için ürün belgelerinize bakın.

    1.  **Başlat**'ı ve sonra **Ara**'yı tıklatın.
    2.  **Arama Sonuçları** bölmesinde, **Arama Yardımcısı**'nın altındaki **Tüm dosya ve klasörler**'i tıklatın.
    3.  **Dosya adının tümü veya bir parçası** kutusuna, uygun dosya bilgileri tablosundan bir dosya adı yazın ve **Ara**'yı tıklatın.
    4.  Dosya listesinde, uygun dosya bilgileri tablosundan bir dosya adını sağ tıklatın ve sonra **Özellikler**'i tıklatın.
        **Not** Yüklü işletim sisteminin veya programların sürümüne bağlı olarak, dosya bilgileri tablosunda listelenen dosyalardan bazıları yüklenmeyebilir.
    5.  **Sürüm** sekmesinde, sisteminize yüklenmiş olan dosyanın sürümünü uygun dosya bilgileri tablosunda belgelenen sürümle karşılaştırarak belirleyin.
        **Not** Dosya sürümü dışındaki öznitelikler yükleme sırasında değişebilir. Diğer dosya özniteliklerini dosya bilgileri tablosundaki bilgilerle karşılaştırmak, güncelleştirmenin uygulandığını doğrulama için desteklenen bir yöntem değildir. Ayrıca, belirli durumlarda, dosyalar kurulum sırasında yeniden adlandırılabilir. Dosya veya sürüm bilgileri yoksa, güncelleştirme kurulumunu kullanılabilen diğer yöntemlerden biriyle doğrulayın.

-   **Kayıt Defteri Anahtarı Doğrulaması**

    Bu güvenlik güncelleştirmesinde açıklanan dosyaların yüklenmiş olduğunu, bu bölümdeki **Başvuru Tablosu**'nda listelenen kayıt defteri anahtarlarını gözden geçirerek de doğrulayabilirsiniz.
