Security Bulletin

Microsoft Güvenlik Bülteni MS07-065 - Önemli
============================================

Message Queuing Hizmetindeki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (937894)
---------------------------------------------------------------------------------------------

Yayım Tarihi: 11 Aralık 2007 Salı | Güncelleştirme Tarihi: 16 Ocak 2008 Çarşamba

**Sürüm:** 1.3

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu önemli güvenlik güncelleştirmesi, Message Queuing Hizmeti'nde (MSMQ) özel olarak bildirilen ve Microsoft Windows 2000'deki uygulamalarında uzaktan kod yürütmeye veya Microsoft Windows XP'deki uygulamalarında ayrıcalık yükselmesine olanak verebilecek bir güvenlik açığını giderir. Windows XP'de bir saldırganın ayrıcalık yükselmesi güvenlik açığından yararlanabilmesi için geçerli oturum açma kimlik bilgilerine sahip olması gerekir. Böylece saldırgan program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir ya da yeni hesaplar oluşturabilir.

Bu, desteklenen Microsoft Windows 2000 sürümleri için önemli, desteklenen Windows XP sürümleri için de orta düzeyde bir güvenlik güncelleştirmesidir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Bu güvenlik güncelleştirmesi, giriş dizesini ara belleğe kopyalamadan önce doğrulamak suretiyle güvenlik açığını giderir. Bu güvenlik açıkları hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.**  Microsoft, müşterilerin güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

**Bilinen Sorunlar.**  Yok

Message Queuing Hizmeti Uzaktan Kod Yürütme Güvenlik Açığı – CVE-2007-3039  
--------------------------------------------------------------------------
  
<span></span>
Message Queuing Hizmeti'nde, giriş dizelerini arabelleğe geçirmeden önce yanlış şekilde doğrulamasından kaynaklanan bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Bir saldırgan, Microsoft Windows 2000'de uzaktan saldırı senaryosunda uzaktan kod yürütmeye, Windows XP'de yerel saldırı senaryosunda da ayrıcalık yükselmesine olanak veren özel olarak hazırlanmış bir MSMQ iletisi oluşturarak bu güvenlik açığından yararlanabilir. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2007-3039](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3039).
  
#### Message Queuing Hizmeti Uzaktan Kod Yürütme Güvenlik Açığı – CVE-2007-3039 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Windows XP'de saldırganın bu güvenlik açığından yararlanabilmesi için geçerli oturum açma kimlik bilgileri olması gerekir.  
-   Message Queuing bileşeni varsayılan olarak etkilenen işletim sistemi sürümlerinde hiçbirinde yüklü değildir ve yalnızca yönetici ayrıcalıkları olan bir kullanıcı tarafından etkinleştirilebilir. Yalnızca Message Queuing bileşenini el ile yükleyen müşteriler bu sorundan etkilenebilir.  
-   Message Queuing bileşenini kullanması gereken müşteriler için, güvenlik duvarlarına yönelik en iyi uygulamalar ve standart olarak kullanılan varsayılan güvenlik duvarı yapılandırmaları, ağ yapılarının işletme dışından gerçekleştirilen saldırılardan korunmasına yardımcı olabilir. En iyi uygulamalar, Internet'e bağlı sistemlerde olabildiğince az sayıda bağlantı noktasının açılmasını önerir.
  
#### Message Queuing Service Hizmeti Uzaktan Kod Yürütme Güvenlik Açığı – CVE-2007-3039 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Güvenlik duvarında aşağıdakileri engelleyin:**  
    -   1024'ten sonraki bağlantı noktalarında istenmeyen tüm gelen trafik  
    -   Özel olarak yapılandırılmış tüm diğer RPC bağlantı noktaları
  
        Bu bağlantı noktaları, RPC ile bağlantı başlatmak için kullanılır. Güvenlik duvarında bu bağlantı noktalarının engellenmesi, güvenlik duvarının arkasındaki sistemleri bu açıktan yararlanma girişimlerinden engellemeye yardımcı olur. Ayrıca, uzak sistemde özel olarak yapılandırılmış diğer tüm RPC bağlantı noktalarını engellediğinizden emin olun. Microsoft, diğer bağlantı noktalarını kullanabilecek saldırılardan korunmaya yardımcı olmak amacıyla, Internet'ten gelen tüm istenmeyen trafiğin engellenmesini önerir. RPC tarafından kullanılan bağlantı noktaları hakkında daha fazla bilgi için, bkz: [TCP ve UDP Bağlantı Noktası Atamaları](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/cnet/cnfc_por_gdqc.mspx?mfr=true).
  
-   **Message Queuing hizmetini devre dışı bırakma:**
  
    -   Etkileşimli olarak:
  
        Message Queuing hizmetini devre dışı bırakmak, etkilenen sistemin bu güvenlik açığından yararlanma girişimlerine karşı korunmasına yardımcı olur. Message Queuing hizmetini devre dışı bırakmak için şu adımları izleyin:
  
        1.  **Başlat**'ı ve sonra **Denetim Masası**'nı tıklatın. Alternatif olarak, **Ayarlar**'ın üzerine gelin ve **Denetim Masası**'nı tıklatın.  
        2.  **Yönetimsel Araçlar**'ı çift tıklatın. Alternatif olarak, **Klasik Görünüm'e geçiş yap**'ı ve ardından **Yönetimsel Araçlar**'ı tıklatın.  
        3.  **Hizmetler**'i çift tıklatın.  
        4.  **Message Queuing** öğesini çift tıklatın.  
        5.  **Başlangıç türü** listesinde, **Devre Dışı** seçeneğini tıklatın.  
        6.  **Durdur**'u, ardından **Tamam**'ı tıklatın.
  
        Ayrıca, komut satırında aşağıdaki komutu kullanarak MSMQ hizmetini durdurabilir ve devre dış bırakabilirsiniz (Windows XP ve Microsoft Windows 2000 Kaynak Seti'nde bulunmaktadır):
  
        `sc stop MSMQ & sc config MSMQ start= disabled`
  
    -   Grup İlkesi ile:
  
        Grup İlkesi ayarlarını kullanarak Message Queuing hizmetini devre dışı bırakın. Bu hizmetin başlatılmasını, Microsoft Windows 2000 veya Windows Server 2003 etki alanı ortamlarındaki Grup İlkesi nesne işlevselliğini kullanarak yerel, site, etki alanı veya kuruluş birimi düzeyinde devre dışı bırakabilirsiniz.
  
        **Not:** Windows Server 2003 Güvenlik Kılavuzu'nu da inceleyebilirsiniz. Bu kılavuz, hizmetlerin devre dışı bırakılması hakkında bilgi içerir.
  
        Grup İlkesi hakkında daha fazla bilgi için, aşağıdaki Microsoft Web sitelerini ziyaret edin:
  
        -   [Grup İlkesi Özelliği Ayarını Anlayabilmek için Kapsamlı Kılavuz](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/technologies/directory/activedirectory/stepbystep/gpfeat.mspx)  
        -   [Windows 2000 Grup İlkesi](http://technet.microsoft.com/en-us/library/bb742376.aspx)  
        -   [Windows Server 2003 Grup İlkesi](http://www.microsoft.com/windowsserver2003/technologies/management/grouppolicy/default.mspx)
  
    **Geçici çözüm nasıl geri alınır:** Yukarıdaki adımları kullanarak Başlangıç Türü'nü **Otomatik** olarak ayarlayın ve hizmeti başlatın.
  
#### Message Queuing Service Hizmeti Uzaktan Kod Yürütme Güvenlik Açığı – CVE-2007-3039 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir? **  
Bu, Microsoft Windows 2000 için bir uzaktan kod yürütme güvenlik açığı ve Windows XP için yerel olarak ayrıcalık yükselmesi güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, etkilenen sistemin tüm denetimini Microsoft Windows 2000'de uzaktan ve Windows XP'de yerel olarak ele geçirebilir. Böylece saldırgan program yükleyebilir ya da verileri görüntüleyebilir, değiştirebilir veya silebilir.
  
**Güvenlik açığına ne neden olmaktadır? **  
MSMQ hizmeti giriş dizelerini arabelleğe geçirmeden önce düzgün denetlemez. Sonuç olarak, hizmet kodunda denetlenmeyen bir arabellek taşması oluşarak saldırganın LOCAL SYSTEM bağlamında rasgele kod yürütmesi mümkün olmaktadır.
  
**Message Queuing nedir? **  
Microsoft Message Queuing teknolojisi, farklı zamanlarda çalışan uygulamaların, farklı yapılardaki ağlar ve geçici olarak çevrimdışı olan sistemler üzerinden iletişim kurmasına olanak sağlar. Uygulamalar, iletileri sıralara gönderir ve sıralardaki iletileri okur. Message Queuing, ileti tesliminin garanti edilmesini ve yönlendirme, güvenlik ve öncelik tabanlı iletilerin verimli şekilde gönderilip alınmasını sağlar. Zaman uyumlu olan ve olmayan ileti senaryoları için çözümler geliştirmek amacıyla kullanılabilir. Message Queuing hakkında daha fazla bilgi için [Message Queuing ürün belgelerine](http://www.microsoft.com/windowsserver2003/technologies/msmq/default.mspx) bakın.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir? **  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, Microsoft Windows 2000'deki uygulamalarında uzaktan kod yürütülmesine ve Windows XP'deki uygulamalarında ayrıcalık yükselmesine neden olabilecek yerel sistem hakları edinebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir? **  
Saldırganın bu güvenlik açığından Windows XP'de yerel olarak yararlanabilmesi için öncelikle sistemde oturum açması gerekir. Microsoft Windows 2000'de saldırganın bu güvenlik açığından uzaktan yararlanabilmesi için, MSMQ hizmetine güvenlik açığından yararlanan özel hazırlanmış paketler göndermesi ve etkilenen sistemin tüm denetimini ele geçirmesi gerekir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor? **  
MSMQ hizmetinin etkinleştirilmiş olduğu Microsoft Windows 2000 ve Windows XP sistemler birincil dereceden risk altındadır.
  
**Güncelleştirme ne yapıyor? **  
Güncelleştirme, MSMQ hizmetinin giriş dizelerini ayrılmış arabelleğe geçirmeden önce doğrulamak için kullandığı yöntemi değiştirerek bu güvenlik açığını ortadan kaldırır.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu? **  
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı? **  
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.
