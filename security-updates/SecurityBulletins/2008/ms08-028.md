Security Bulletin

Microsoft Güvenlik Bülteni MS08-028 - Kritik
============================================

Microsoft Jet Veritabanı Altyapısındaki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (950749)
--------------------------------------------------------------------------------------------------------

Yayım Tarihi: 13 Mayıs 2008 Salı | Güncelleştirme Tarihi: 16 Temmuz 2008 Çarşamba

**Sürüm:** 1.3

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Windows'da bulunan Microsoft Jet Veritabanı Altyapısı'ndaki (Jet) bir güvenlik açığını giderir. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

Bu güvenlik güncelleştirmesi Microsoft Jet 4.0 Veritabanı Altyapısı için Kritik olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Bu güvenlik güncelleştirmesi, Microsoft Jet Veritabanı Altyapısı'nın bir veritabanındaki verileri ayrıştırma biçimini değiştirerek güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

Bu güvenlik güncelleştirmesi ilk olarak [Microsoft Güvenlik Danışma Belgesi 950627](http://technet.microsoft.com/security/advisory/950627)'de açıklanan güvenlik açığını da giderir. Microsoft Word kullanan müşterilerin bu tür saldırılar için saldırı vektörüne karşı en güncel koruma sağlamak üzere bu güncelleştirmenin yanı sıra, [Microsoft Güvenlik Bülteni MS08-26: Microsoft Word'deki Güvenlik Açıkları Uzaktan Kod Yürütülmesine İzin Verebilir (951207)](http://go.microsoft.com/fwlink/?linkid=117295) tarafından sağlanan güncelleştirmeleri de yüklemeleri önerilir.

**Öneri.** Microsoft, müşterilerinin güncelleştirmeyi hemen uygulamasını önerir.

**Bilinen Sorunlar.** Yok

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)
-----------------------------------------------------------------

<span></span>
**CVE-2005-0944, Microsoft Jet Veritabanı Altyapısı'ndaki bir güvenlik açığını da açıklar. CVE-2005-0944 bu güncelleştirme ile nasıl ilişki kurar?**    
CVE-2005-0944 tarafından belirtilen güvenlik açığı bu güncelleştirme tarafından da giderilmiştir. Etkilenen veya etkilenmeye yazılım, azaltıcı etkenler ve geçici çözümler dahil, CVE-2005-0944 güvenlik açığı ayrıntıları CVE-2007-6026'nınkilerle aynı kalır. Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2005-0944](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2005-0944).

**Bu güncelleştirme neden Kritik olarak derecelendirilmiştir?**    
.mdb dosyaları güvenli olmayan dosya türleri listesinde bulunmakla birlikte, güvenli olmayan dosya türleri için sağlanan azaltıcı etkenleri atlayabilen saldırı senaryoları bulunmaktadır. Özellikle de Microsoft Office kullanan müşteriler e-posta veya doğrudan karşıdan yükleme saldırı senaryolarına karşı risk altında olabilir. Örneğin bir saldırgan, kullanıcıya özel hazırlanmış bir .mdb dosyasının katıştırılmış olduğu bir Word dosyası gönderip kullanıcıyı bu belgeyi açmaya veya özel hazırlanmış dosyayı içeren e-postayı görüntülemeye ikna ederek bu güvenlik açığından yararlanabilir. Outlook 2003 ve Outlook 2007 için, belge Outlook 2003 veya Outlook 2007 önizleme bölmesinde HTML biçiminde görüntülendiğinde saldırı gerçekleştirilebilir. Bu müşterilerin güncelleştirmeyi hemen uygulaması önerilir.

**Bu güncelleştirme diğer uygulamalar aracılığıyla gerçekleştirilen saldırılara karşı koruma sağlıyor mu?**  
Evet. Bu güvenlik açığı için saldırı vektörü olarak kullanılabilecek uygulamalar yüklü olabilir. Örneğin bir Jet veritabanı dosyası, Word belgesinden veya e-posta içinden ya da kötü amaçlı bir Web sitesi ziyaret edilerek açılabilir. Bu güncelleştirme uygulandığında bu tür saldırılar engellenir. Ancak müşterilerin yeni keşfedilen saldırı vektörlerini tümüyle engellemek için [Microsoft Güvenlik Bülteni MS08-026: Microsoft Word'deki Güvenlik Açıkları Uzaktan Kod Yürütülmesine İzin Verebilir (951207)](http://go.microsoft.com/fwlink/?linkid=117295) tarafından sağlanan güncelleştirmeleri de uygulamaları önerilir.

**Önceden JET Blue olarak bilinen Genişletilebilir Depolama Altyapısı'nın (ESE) kullanıldığı uygulamalar ve hizmetler bu güvenlik açığından etkileniyor mu?**  
Hayır. Bu güvenlik açığı yalnızca Microsoft Jet 4.0 Veritabanı Altyapısı'nda bulunur. Genişletilebilir Depolama Altyapısı'nı kullanan uygulamalar ve hizmetler bu açıktan etkilenmez.

Microsoft Jet Veritabanı Altyapısı'nda MDB Dosyasını Ayrıştırma Sırasında Yığın Taşması Güvenlik Açığı - CVE-2007-6026  
----------------------------------------------------------------------------------------------------------------------
  
<span></span>
Microsoft Jet Veritabanı Altyapısı'nda (Jet), etkilenen bir sistemde uzaktan kod yürütülmesine olanak veren bir arabellek taşması güvenlik açığı bulunmaktadır. Saldırgan, özel olarak hazırlanmış bir veritabanı sorgusu oluşturup bu sorguyu etkilenen bir sistemdeki Jet kullanan bir uygulama aracılığıyla göndermek yoluyla bu güvenlik açığından yararlanabilir. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2007-6026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-6026).
  
#### Microsoft Jet Veritabanı Altyapısı'nda MDB Dosyasını Ayrıştırma Sırasında Yığın Taşması Güvenlik Açığı - CVE-2007-6026 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Bu güvenlik açığından başarıyla yararlanan bir saldırgan, yerel kullanıcı ile aynı haklara sahip olabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.  
-   Web tabanlı saldırı senaryosunda, saldırganın, bu açıktan yararlanma girişimi için kullanılan özel hazırlanmış Word dosyasını içeren bir Web sitesini barındırması gerekir. Ayrıca, kullanıcı tarafından sağlanan içeriği kabul eden ya da barındıran güvenliği aşılmış Web sitelerinde, bu güvenlik açığından yararlanılabilecek özel hazırlanmış içerik bulunabilir. Saldırganın, bu Web sitelerini ziyaret etmek için kullanıcıyı zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi sitesine götüren bir bağlantıyı tıklatmalarını sağlayarak onları bu Web sitesine çekmesi gerekir.  
-   E-posta istemcisi olarak Microsoft Outlook 2003 ve 2007 kullanılan sistemler postayı yalnızca düz metin olarak okunacak biçimde yapılandırılarak Outlook 2007 için HTML e-postası saldırı vektörünün etkileri azaltılabilir.  
-   Windows XP Service Pack 3, Windows Server 2003 Service Pack 2, Windows Vista ve Windows Server 2008'in desteklenen sürümlerinin çalıştırıldığı sistemler bu güvenlik açığından etkilenmez.
  
#### Microsoft Jet Veritabanı Altyapısı'nda MDB Dosyasını Ayrıştırma Sırasında Yığın Taşması Güvenlik Açığı - CVE-2007-6026 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Microsoft Jet Veritabanı Altyapısı'nın hiçbir uygulama için çalışmamasını sağlayın**
  
    Geçici çözümü uygulamak için, komut istemine aşağıdaki komutu yazın:
  
    `echo y| cacls "%SystemRoot%\system32\msjet40.dll" /E /P everyone:N`
  
    **Geçici çözümün etkisi.** Veri erişim çağrıları yapmak için Microsoft Jet Veritabanı Altyapısı kullanılmasını gerektiren uygulamalar çalışmaz.
  
    **Geçici çözüm nasıl geri alınır.** Komut istemine aşağıdaki komutu girin
  
    `echo y| cacls "%SystemRoot%\system32\msjet40.dll" /E /R everyone`
  
-   **Grup İlkesi'ni kullanarak Microsoft Jet Veritabanı Altyapısı'nın hiçbir uygulama için çalışmamasını sağlayın**
  
    Geçici çözümü uygulamak için aşağıdaki adımları gerçekleştirin:
  
    1.  Gösterim amacıyla JetCacls.cmd adlı aşağıdaki komut dosyasını oluşturun:  
        @echo off  
        if exist %systemdrive%\\Cacls.log goto end  
        cacls "%SystemRoot%\\system32\\msjet40.dll" /E /P everyone:N &gt; nul 2&gt;&1  
        echo %date% %time%: Msjet Cacls updated &gt; %systemdrive%\\Cacls.log  
        :end  
        exit  
    2.  JetCacls.cmd dosyasını çalıştırılacağı etki alanı denetleyicisindeki Netlogon paylaşılan klasörüne veya başka bir paylaşılan klasöre kopyalayın.  
    3.  JetCacls.cmd dosyasını kurun. **Active Directory Kullanıcı ve Bilgisayarları MMC** ek bileşeninde, etki alanı adını sağ tıklatın ve ardından **Özellikler**'i tıklatın.  
    4.  **Grup İlkesi** sekmesini tıklatın.  
    5.  Yeni bir Grup İlkesi nesnesi (GPO) oluşturmak üzere **Yeni**'yi tıklatın ve ilke adı olarak JetCacls girin.  
    6.  Yeni ilkesi ve ardından **Düzen**'i tıklatın.  
    7.  **Bilgisayar Yapılandırması için Windows Ayarları**'nı genişletin ve sonra da **Komut Dosyaları**'nı tıklatın.  
    8.  **Oturum Açma**'yı çift tıklatın ve ardından **Ekle**'yi tıklatın. Komut Dosyası Ekle iletişim kutusu görüntülenir.  
    9.  Komut Dosyası Adı kutusuna \\\\*sunucuadı*\\*paylaşımadı*\\JetCacls.cmd yazın.  
    10. **Tamam**'ı ve sonra da **Uygula**'yı tıklatın.  
    11. Ardından, bu etki alanının üyesi olan istemci bilgisayarları yeniden başlatın.
  
    **Geçici çözümün etkisi.** Veri erişim çağrıları yapmak için Microsoft Jet Veritabanı Altyapısı kullanılmasını gerektiren uygulamalar çalışmaz. Bu kısıtlama yalnızca etki alanındaki istemci bilgisayarlarda çalışan uygulamalar için geçerlidir.
  
-   **MDB dosyalarının posta altyapınız aracılığıyla işlenmesini engelleyin**
  
    **Not** Genel kullanıcılar için tüm Jet veritabanı dosyaları güvenli olmayan dosya türleri olarak değerlendirilmelidir ve Microsoft, e-postayla aktarılan veritabanı dosyalarının şüpheli olarak değerlendirilmesini önerir.
  
    Bu geçici çözümü uygulamak için, posta ortamınızın e-posta iletisi içinde belirli bir dosya yapısını (yalnızca dosya uzantısını değil) içeren ekleri arayabilmesi ve daha sonra bu ek üzerinde silme, karantinaya alma, bildirme ve algılanan dosyayı raporlama gibi eylemleri gerçekleştirebilmesi gerekir.
  
    Başka bir dosya türü olarak yeniden adlandırılmış olabilecek Jet dosyalarını algılamak için, 0x4 konumunda (tırnak işaretleri olmadan) aşağıdaki 15 baytlık imzalardan birine sahip dosyaları arayın:
  
    `"Jet System DB  ""Standard Jet DB""Temp Jet DB    "`
  
    Forefront (önceden Antigen olarak biliniyordu) teknolojilerini kullanan Microsoft Exchange müşterilerine özgü yapılandırmalar konusunda daha fazla bilgi için, bkz: [Forefront Security Server: Dosya Filtreleme](http://technet.microsoft.com/en-us/library/bb795068.aspx).
  
    Bu bilgiler [Microsoft Güvenlik Yanıt Birliği](http://www.microsoft.com/security/msra/default.mspx) üyeleriyle paylaşılmıştır. MDB dosyalarını algılamak üzere MSRA araçlarını kullanmak için, MSRA giriş sayfasında listelenen sağlayıcılara başvurun.
  
    **Geçici Çözümün Etkisi.** Bu yapılandırma tarafından algılanan dosyaların, bir kuruluşun e-posta sistemi aracılığıyla işlenmesi engellenir.
  
-   Outlook 2007'de postalar düz metin olarak okunacak biçimde yapılandırın.  
-   Güvenilmeyen kaynaklardan gelen ya da güvenilen kaynaklardan beklemediğiniz halde gelen Jet veya Microsoft Word dosyalarını açmayın ve kaydetmeyin. Kullanıcı özel hazırlanmış bir dosyayı açtığında bu güvenlik açığından yararlanılabilir.
  
#### Microsoft Jet Veritabanı Altyapısı'nda MDB Dosyasını Ayrıştırma Sırasında Yığın Taşması Güvenlik Açığı - CVE-2007-6026 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından uzaktan başarıyla yararlanan anonim bir saldırgan, yerel kullanıcının bağlamında rasgele kod çalıştırabilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
Microsoft Jet Veritabanı Altyapısı (Jet), bir veri yapısı için yeterli doğrulama gerçekleştirmez. Saldırgan, özel olarak hazırlanmış bir veritabanı sorgusu oluşturup bu sorguyu etkilenen bir sistemdeki Jet kullanan bir uygulama aracılığıyla göndermek yoluyla bu güvenlik açığından yararlanabilir.
  
**Microsoft Jet Veritabanı Altyapısı (Jet) nedir?**  
Microsoft Jet Veritabanı Altyapısı, Microsoft Access, Microsoft Visual Basic ve üçüncü taraf uygulamalar gibi birçok uygulamaya veri erişimi sağlar. Jet, veritabanı işlevleri gerektiren Internet uygulamaları tarafından da kullanılabilir.
  
**.mdb dosyaları güvenli olmayan dosya türleri listesinde yer almıyor mu? Öyleyse, bu güvenlik açığı neden Kritik olarak derecelendirilmiş?**  
.mdb dosyaları güvenli olmayan dosya türleri listesinde bulunmakla birlikte, Jet veritabanı dosyasının bir Word belgesinden açılabilmesi nedeniyle oluşan yeni bir saldırı vektörü keşfedilmiştir. Bu güncelleştirme geçerli saldırıları engellemesinin yanı sıra, Jet Veritabanı Altyapısı için ek düzeltmeler de sağlar. Ancak müşterilerin yeni keşfedilen saldırı vektörlerini tümüyle engellemek için [Microsoft Güvenlik Bülteni MS08-026: Microsoft Word'deki Güvenlik Açıkları Uzaktan Kod Yürütülmesine İzin Verebilir (951207)](http://go.microsoft.com/fwlink/?linkid=117295) tarafından sağlanan güncelleştirmeleri de uygulamaları önerilir.
  
**.mdb dosyalarının güvenli olmadığı mı kabul edilmelidir?**  
Evet. Bu güncelleştirme etkin saldırıları engeller, ancak .mdb veritabanı dosyaları tasarım gereği güvenli olmayan işlevler gerçekleştirir. Müşteriler bu dosyaları kullanırken dikkatli olmalıdır. Ayrıca, desteklenen Microsoft e-posta uygulamaları .mdb dosyalarının açılmasını engellemeye devam eder.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açan kullanıcı ile aynı haklara sahip olabilir. Kullanıcı yönetimsel haklarla oturum açtıysa, saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir? **  
Bir saldırgan, kullanıcıyı özel hazırlanmış bir veritabanı dosyasını Jet kullanarak açmaya ikna ederek bu güvenlik açığından yararlanabilir. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, yerel kullanıcı ile aynı haklara sahip olabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
E-posta tabanlı saldırı senaryosunda, saldırgan kullanıcıya özel hazırlanmış bir .mdb dosyasının katıştırılmış olduğu bir Word dosyası gönderip kullanıcıyı bu belgeyi açmaya veya özel hazırlanmış dosyayı içeren e-postayı görüntülemeye ikna ederek bu güvenlik açığından yararlanabilir. Outlook 2003 ve Outlook 2007 için, belge Outlook 2003 veya Outlook 2007 önizleme bölmesinde HTML biçiminde görüntülendiğinde saldırı gerçekleştirilebilir.
  
Web tabanlı saldırı senaryosunda, saldırganın, bu açıktan yararlanma girişimi için kullanılan Jet dosyasını veya özel hazırlanmış Jet dosyasının katıştırılmış olduğu Word dosyasını içeren bir Web sitesini barındırması gerekir. Ayrıca, kullanıcı tarafından sağlanan içeriği kabul eden ya da barındıran güvenliği aşılmış Web sitelerinde, bu güvenlik açığından yararlanılabilecek özel hazırlanmış içerik bulunabilir. Saldırganın, bir kullanıcıyı özel hazırlanmış bir Web sitesini ziyaret etmeye zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi Web sitesine götüren bir bağlantıyı tıklatmaya ikna ederek onları bu siteye çekmesi gerekir.
  
**HTML e-postası saldırı riskinin etkilerini azaltmaya yardımcı olacak bir yordam var mı?**  
E-posta istemcisi olarak Microsoft Outlook kullanılan sistemler postayı yalnızca düz metin olarak okunacak biçimde yapılandırılarak Outlook 2003 ve Outlook 2007 için HTML e-postası saldırı vektörünün etkileri azaltılabilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Microsoft Windows 2000, Windows XP Service Pack 2 ve Windows Server 2003 Service Pack 1 üzerinde Microsoft Jet 4.0 Veritabanı Altyapısı'nın güvenlik açığından etkilenen bir sürümünün yüklü olduğu sistemler birincil derecede risk altındadır. **Genel Bilgiler** bölümündeki Etkilenen Yazılımlar tablosunda da listelenen bu işletim sistemleri, Jet'in güvenlik açığından etkilenen sürümüyle birlikte gelir. Bu nedenle, sisteminizde güvenlik açığından etkilenen bir Jet sürümünün bulunup bulunmadığını nasıl belirleyeceğiniz ile ilgili olan sonraki SSS'yi inceleyin.
  
**Güvenlik açığından etkilenen bir Jet sürümünün bulunup bulunmadığını nasıl anlarım?**  
Msjet40.dll sürümü 4.0.9505.0 sürümünden daha küçükse, sisteminizde güvenlik açığından etkilenen bir Jet sürümü bulunuyor demektir. Msjet40.dll dosyasının 4.0.9505.0 sürümü ve sonrası bu arabellek taşması sorunundan etkilenmez.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, Jet içindeki veri yapılarının doğrulanma biçimini geliştirerek güvenlik açığını ortadan kaldırır.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Evet. Bu güvenlik açığı genel olarak duyurulmuştur ve “Microsoft Jet Veritabanı Altyapısı'nda MDB Dosyasını Ayrıştırma Sırasında Yığın Taşması Güvenlik Açığı” adı verilerek Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE 2007-6026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-6026) atanmıştır.
  
**Bu güvenlik güncelleştirmesi Microsoft Güvenlik Danışma Belgesi 950627'de tanımlanan sorunu gideriyor mu?**  
Evet. Bu güvenlik güncelleştirmesi, [Microsoft Güvenlik Danışma Belgesi 950627](http://technet.microsoft.com/security/advisory/950627)'de CVE 2008-1092 olarak tanımlanan güvenlik sorununu gidermektedir. Microsoft bu sorunu araştırdıktan sonra [CVE-2007-6026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-6026) ile aynı sorun olduğunu belirlemiş ve bu nedenle bu güvenlik bülteni için yeniden sınıflandırmıştır.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Evet. Güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığı bilgisini almıştı.
  
**Bu güvenlik güncelleştirmesini uygulamak, müşterilerin açıktan yararlanmaya yönelik olarak yayımlanan koddan korunmasına yardımcı olur mu?**  
Evet. Bu güvenlik güncelleştirmesi, şu anda yararlanılmakta olan güvenlik açığını ortadan kaldırır. Güvenlik açığı giderilmiş, ayrıca [CVE-2007-6026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-6026) Ortak Güvenlik Açığı ve Etkilenme numarası atanmıştır.

    3.  **Dosya adının tümü veya bir parçası** kutusuna, uygun dosya bilgileri tablosundan bir dosya adı yazın ve **Ara**'yı tıklatın.
