Security Bulletin

Microsoft Güvenlik Bülteni MS08-037 - Önemli
============================================

DNS'deki Güvenlik Açıkları Kimlik Sahtekarlığına Neden Olabilir (953230)
------------------------------------------------------------------------

Yayım Tarihi: 8 Temmuz 2008 Salı | Güncelleştirme Tarihi: 9 Aralık 2009 Çarşamba

**Sürüm:** 3.1

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi, Windows Etki Alanı Adı Sistemi'nde (DNS) kimlik sahtekarlığına olanak verebileceği özel olarak bildirilen iki güvenlik açığını giderir. Bu güvenlik açıkları hem DNS istemcisinde hem de DNS sunucusunda bulunur ve bir uzak saldırganın Internet üzerindeki sistemlere gönderilen trafiği kendi sistemlerine yönlendirmesine olanak verebilir.

Güvenlik güncelleştirmesi Microsoft Windows 2000, Windows XP, Windows Server 2003 ve Windows Server 2008'in tüm desteklenen sürümleri için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, rasgeleliği yüksek DNS işlem kimlikleri kullanarak, UDP sorguları için rasgele yuvalar kullanarak ve DNS önbelleğini yönetmek için kullanılan mantığı güncelleştirerek bu güvenlik açıklarını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Microsoft, müşterilerin güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

**Bilinen Sorunlar.** [Microsoft Bilgi Bankası makalesi 953230](http://support.microsoft.com/kb/953230) bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir.

#### Etkilenen ve Etkilenmeyen Yazılımlar

Aşağıdaki yazılımlar, hangi sürümlerinin etkilendiğini belirlemek amacıyla sınanmıştır. Diğer sürümler destek ömrünü tamamlamıştır veya bu açıktan etkilenmemektedir. Yazılımınızın sürümünün destek ömrünü belirlemek için [Microsoft Destek Ömrü](http://go.microsoft.com/fwlink/?linkid=21742) Web sitesini ziyaret edin.

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)  
-----------------------------------------------------------------
  
<span></span>
**Bu bülten 8 Aralık 2009'da neden yeniden yayımlandı?**  
Microsoft bu bülteni, rasgeleliği yüksek DNS işlem kimliklerine yönelik ek bir kod yolu sağlamak üzere Microsoft Windows 2000 Service Pack 4 üzerinde DNS istemcisi güncelleştirmesini (KB951748) yeniden sunmak için tekrar yayımladı. Diğer Windows platformlarından farklı olarak Microsoft Windows 2000'de, DNS işlemleri için iki kod yolu bulunmaktadır. Önceki güncelleştirme bu kod yollarından yalnızca biri için işlem kimliği rasgeleliği sağlıyordu. Bu güncelleştirmenin yeniden yayımlanan sürümü, aynı işlem kimliği rasgeleliğini Microsoft Windows 2000'deki diğer kod yolu için de sağlamaktadır. Microsoft Windows 2000 Service Pack 4 üzerinde DNS istemcisi güncelleştirmesini (KB951748) daha önce yüklemiş olan müşterilerin otomatik olarak yeniden sunulan güncelleştirmeyi yüklemeleri gerekmektedir. Bu yeniden yayımlanan sürümden başka bir güncelleştirme etkilenmemektedir.
  
**Bu güvenlik bülteni 13 Ocak 2009'da neden yeniden düzenlendi?**  
Microsoft, bu bülten tarafından sunulan güncelleştirmenin Windows XP Service Pack 3 çalıştıran tüm sistemlere doğru şekilde sunulmamış olabileceğini bildirmek için bu bülteni yeniden düzenledi. Algılama dağıtım sorunu düzeltilmiştir ve bu bültendeki güncelleştirmeyi uygulamamış olan Windows XP Service Pack 3 sistemi sahibi müşterilere güncelleştirme şimdi doğru şekilde sunulacaktır. Bu güncelleştirmede ikili dosyalarda değişiklik yapılmamıştır. Sistemlerini başarıyla güncelleştirmiş olan müşterilerin bu güncelleştirmeyi yeniden yüklemeleri gerekmez.
  
**Bu güvenlik bülteni 25 Temmuz 2008'de neden yeniden düzenlendi?**  
Microsoft bu güvenlik bültenini, **Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)** bölümüne bilinen sorun olarak üç yeni girdi ekleyerek yeniden düzenledi. Bu SSS'ler şunlardır:
  
-   **Müşterilerin bu güncelleştirmeyi yükledikten sonra karşılaşabilecekleri bağlantı noktası çakışma sorunları nelerdir?**  
    DNS sunucusunun yuva havuzu için ayırdığı bağlantı noktası aralığındaki statik UDP bağlantı noktalarına gereksinim duyan ek hizmetleri gerçekleştiren DNS sunucularında bağlantı noktası çakışmaları oluşabilir ve dolayısıyla hizmetler başarısız olabilir. Ek bilgi için, [Microsoft Bilgi Bankası makalesi 953230](http://support.microsoft.com/kb/953230)'da bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunlar belgelenmektedir.  
-   **SBS müşterilerinin bu güncelleştirmeyi yükledikten sonra karşılaşabilecekleri sorunlar nelerdir?**  
    Windows Small Business Server (SBS) 2003'ün desteklenen sürümleri Windows Server 2003 Service Pack 1 ve Windows Server 2003 Service Pack 2 ile aynı etkilenen kodu içerir. Ancak bu bültende sunulan DNS sunucusu güvenlik güncelleştirmesi yüklendikten sonra, SBS sistemlerinde hizmetler başarısız olabilir veya düzgün çalışmayabilir. Ek bilgi için, [Microsoft Bilgi Bankası makalesi 953230](http://support.microsoft.com/kb/953230)'da bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunlar belgelenmektedir.  
-   **Bu güncelleştirme yüklendikten sonra çevre güvenlik duvarlarında ve çevre NAT aygıtlarında hangi sorunlarla karşılaşılabilir?**  
    Bu bültende sunulan güvenlik güncelleştirmesi Windows tabanlı bir bilgisayara yüklendikten sonra, bu ana bilgisayardan güvenlik duvarında yapılan DNS sorgularında artık rasgele kaynak bağlantı noktaları kullanılmaz. Ek bilgi için, [Microsoft Bilgi Bankası makalesi 953230](http://support.microsoft.com/kb/953230)'da bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunlar belgelenmektedir.
  
**Bu güvenlik bülteni 23 Temmuz 2008'de neden yeniden düzenlendi?**  
Microsoft bu güvenlik bültenini, bu güncelleştirme tarafından değiştirilen bültenler olarak MS06-064, MS07-062 ve MS08-001 listelenecek biçimde yeniden düzenledi.
  
**Bu güvenlik bülteni 23 Temmuz 2008'de neden yeniden düzenlendi?**  
Microsoft, [Check Point Software Technologies Ltd.](http://www.checkpoint.com/) tarafından geliştirilen ZoneAlarm ve Check Point Endpoint Security (önceden Check Point Integrity olarak biliniyordu) yazılımlarının kullanıcılarını aşağıdaki SSS'de ayrıntılı olarak açıklanan bir Internet bağlantısı sorunu konusunda bilgilendirmek üzere bu güvenlik bültenini yeniden düzenledi. Bu düzenlemede, bu bültende açıklanan güvenlik güncelleştirmesi dosyaları değişmedi. Güncelleştirmeyi zaten uygulamış olan müşterilerin yeniden uygulamaları gerekmez. Ancak, Check Point tarafından geliştirilen yukarıdaki yazılımları kullananlar, daha ayrıntılı yönergeler için aşağıdaki SSS'yi okumalıdır.
  
**ZoneAlarm ve Check Point Endpoint Security kullanıcıları bu güncelleştirmeyi yükledikten sonra hangi sorunla karşılaşabilir?**  
Microsoft, Check Point Software Technologies Ltd. tarafından geliştirilen ZoneAlarm ve Check Point Endpoint Security (önceden Check Point Integrity olarak biliniyordu) kullanıcılarının güvenlik bülteni MS08-037'de sunulan güvenlik güncelleştirmelerini uyguladıktan sonra bir Internet bağlantısı sorunu yaşadığına yönelik bildirimlerin farkındadır. Microsoft bu sorunu Check Point ile birlikte araştırmaya devam etmektedir. Microsoft, [ZoneAlarm](http://download.zonealarm.com/bin/free/pressreleases/2008/lossofinternetaccessissue.html) ve Check Point Endpoint Security kullanıcılarının en son yönergeler veya yazılım güncelleştirmeleri için ilgili [Check Point Web sitesini](https://supportcenter.checkpoint.com/supportcenter/index.jsp) ve bu güvenlik bültenini gözden geçirmelerini önerir.
  
Microsoft, üçüncü taraf iletişim bilgilerini teknik destek bulmanıza yardımcı olmak için sağlamaktadır. Bu iletişim bilgileri önceden haber verilmeksizin değiştirilebilir. Microsoft hiç bir şekilde bu üçüncü taraf iletişim bilgilerinin doğruluğunu garanti etmez. Bu makalede açıklanan üçüncü taraf ürünleri Microsoft'tan bağımsız şirketler tarafından üretilmektedir. Microsoft bu ürünlerin performansı veya güvenilirliğiyle ilgili, örtük veya başka bir biçimde, garanti vermez.
  
**Müşterilerin bu güvenlik güncelleştirmelerini yükledikten sonra karşılaşabilecekleri kaldırma sorunları nelerdir?**  
Microsoft Windows 2000 ve Windows 2003'ün desteklenen sürümlerini çalıştıran sistemler, DNS istemci güncelleştirmelerinin yanı sıra DNS sunucusu güncelleştirmesini de alır. DNS sunucusu güncelleştirmesi ve DNS istemcisi güncelleştirmeleri aynı ikili dosyaları paylaşır ve paylaşılan ikili dosyaların önceki sürümlerine döndürülmesini engellemek için bu güncelleştirmelerin yüklendikleri sıranın tersi şeklinde kaldırılmaları gerekir. [Microsoft Bilgi Bankası makalesi 823836](http://support.microsoft.com/kb/823836) bu güvenlik güncelleştirmelerini kaldıran müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir.
  
**Müşterilerin bu güvenlik güncelleştirmesini yüklerken karşılaşabilecekleri rasgele yuva bağlantısı sorunları nelerdir?**  
Bu güvenlik bülteninde sunulan DNS güncelleştirmeleri varsayılan olarak çok sayıda kullanılabilir yuvadan yararlanır ve daha fazla entropiyi sunar. Ancak kullanıcının kayıt defterinde tanımlı bağlantı noktası aralıkları varsa, güncelleştirmeler, kullanıcı tanımlı ayarları kullanır ve yalnızca tanımlanmış yuvaları ayırır.
  
Yuva aralıkları aşağıdaki kayıt defteri konumunda tanımlanabilir:  
**HKLM\\System\\CurrentControlSet\\Services\\DNS\\Parameters Reg key Name: SocketPoolSize**  
**Not** Bu değişikliklerin uygulanabilmesi için DNS hizmetinin yeniden başlatılması gerekir.
  
**Yuva havuz aralığının tanımlanması ne işe yarar?**  
İletişim için aynı yuva havuzuna gereksinim duyan diğer uygulamalarla veya hizmetlerle çakışma oluşmasını önlemek için, DNS'nin seçim yapabileceği bir yuva aralığı tanımlanması gerekebilir. Bu kayıt defteri anahtarları hakkında daha ayrıntılı bilgi için, bkz. [MaxUserPort](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/regentry/58791.mspx?mfr=true) ve [Microsoft Bilgi Bankası makalesi 812873](http://support.microsoft.com/kb/812873).
  
**Yuva havuz aralıkları tanımlanırken işletim sistemleri arasındaki farklılıklar nelerdir?**  
Windows Vista ve Windows Server 2008'de kullanılan MaxUserPort kayıt defteri anahtarının anlamları, Microsoft Windows Server 2000 ve Windows Server 2003'tekinden farklıdır. [Microsoft Bilgi Bankası makalesi 929851](http://support.microsoft.com/kb/929851)'de, Windows Vista ve Windows Server 2008'deki davranış değişikliği ayrıntılı olarak açıklanmaktadır.
  
Microsoft Windows Server 2000 ve Windows Server 2003'te MaxUserPort anahtarı ayarlandığında, dinamik bağlantı noktası aralığının bitiş noktası tanımlanır. Bu aralık 1024'ten başlar ve MaxUserPort kayıt defteri anahtarı ayarında kullanımlı tarafından tanımlanan değere kadar devam eder. Bu güvenlik bülteninde sunulan güncelleştirmeler yüklendikten sonra, Microsoft Windows Server 2000 ve Windows Server 2003'teki varsayılan davranış olarak, yuvalar 49152 ile 65535 aralığındaki bağlantı noktalarından rasgele olarak ayrılır. MaxUserPort aralığı tanımlanmışsa, bağlantı noktaları 1024'ten başlayarak MaxUserPort kayıt defteri anahtarı ayarındaki tanımlı değere kadar rasgele ayrılır. Microsoft Windows 2000 Server ve Windows Server 2003'te bağlantı noktası aralıklarını ayırma konusunda daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 812873](http://support.microsoft.com/kb/812873).
  
Windows Vista ve Windows Server 2008'de MaxUserPort anahtarı ayarlandığında, dinamik bağlantı noktası aralığının başlangıç noktası tanımlanır. Windows Vista ve Windows Server 2008'de bu aralık varsayılan olarak 49152 - 65535 aralığıdır.
  
**Dosya bilgileri ayrıntılarını nerede bulabilirim?**  
Dosya bilgileri ayrıntıları [Microsoft Bilgi Bankası makalesi 953230](http://support.microsoft.com/kb/953230)'da bulunabilir.

DNS'de Yetersiz Yuva Entropisi Güvenlik Açığı - CVE-2008-1447  
-------------------------------------------------------------
  
<span></span>
Windows DNS istemcisinde ve Windows DNS sunucusunda bir kimlik sahtekarlığı güvenlik açığı bulunmaktadır. Bu güvenlik açığı, kimliği doğrulanmamış bir uzak saldırganın yanıtları hızla ve güvenilir şekilde sızdırmasına ve DNS sunucusunun veya istemcisinin önbelleğine kayıtlar ekleyerek Internet trafiğini yeniden yönlendirmesine neden olabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2008-1447](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1447).
  
#### DNS'de Yetersiz Yuva Entropisi Güvenlik Açığı - CVE-2008-1447 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir.
  
-   IPsec veya SSL/TLS gibi TCP ve IP katmanlarının üzerinde çalışan şifreleme protokolleri, bir saldırganın yeniden yönlendirilen trafiği izlemesini veya kesintiye uğratmasını engelleyebilir.
  
#### DNS'de Yetersiz Yuva Entropisi Güvenlik Açığı - CVE-2008-1447 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir.
  
-   Microsoft bu güvenlik açığı için herhangi bir geçici çözüm belirleyememiştir.
  
#### DNS'de Yetersiz Yuva Entropisi Güvenlik Açığı - CVE-2008-1447 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Windows DNS istemcisinde ve DNS sunucusunda bir kimlik sahtekarlığı güvenlik açığı bulunmaktadır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan DNS önbelleğine rasgele adresler ekleyebilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
Windows DNS istemcisindeki Windows DNS hizmeti ve DNS sunucusu, DNS sorguları gerçekleştirirken yeterli entropi sağlamaz.
  
**Etki Alanı Adı Sistemi (DNS) nedir?**  
Etki Alanı Adı Sistemi (DNS), TCP/IP kullanan endüstri standardı protokol paketlerden biridir. DNS, iki yazılım bileşeni kullanılarak uygulanır: DNS sunucusu ve DNS istemcisi (veya çözümleyici). Her iki bileşen de arka plan hizmet uygulaması olarak çalışır. Ağ kaynakları sayısal IP adreslerine göre tanımlanır, ancak bu IP adreslerini ağ kullanıcılarının anımsamaları zordur. DNS veritabanı, www.microsoft.com gibi ağ kaynaklarının kullanımı kolay alfasayısal adlarını, bu kaynaklar tarafından iletişim kurmak için kullanılan IP adresleriyle eşleyen kayıtlar içerir. Böylece DNS, belleğe bağlı bir aygıt gibi çalışarak, ağ kaynaklarının ağ kullanıcıları tarafından anımsanmasını kolaylaştırır. Daha fazla bilgi ve DNS'nin diğer Windows teknolojileriyle nasıl çalıştığını gösteren mantıksal çizimler için, [DNS nedir](http://technet2.microsoft.com/windowsserver/en/library/ff937311-03ce-4d04-b72c-b39c4d51cb361033.mspx) adlı makaleyi inceleyin.
  
**DNS Önbelleği nedir?**  
Etki Alanı Adı Sistemi (DNS) önbellek çözme hizmeti, DNS sunucusunun aynı bilgiler için art arda sorgulanmaması için DNS sorgularının yanıtlarını kaydeden bir hizmettir. Daha fazla bilgi için, [DNS Önbelleği](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/regentry/30643.mspx?mfr=true) veya [DNS Önbellek Çözme Hizmeti](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/cnet/cnbc_imp_qxht.mspx?mfr=true) adlı Technet makalesine bakın. DNS önbelleğini kirletme konusunda daha fazla bilgi için [Saldırı algılama](http://www.microsoft.com/technet/isa/2004/help/fw_alertattack.mspx?mfr=true) adlı TechNet makalesine bakın.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, DNS önbelleğine rasgele adresler ekleyerek DNS önbelleğini kirletme olarak da bilinen bir işlem gerçekleştirebilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir? **  
Saldırgan, güvenlik açığından etkilenen bir DNS sunucusuna veya istemcisine belirli sorgular gönderebilir ve aynı zamanda yanlış ya da yanıltıcı DNS verileri eklemesine olanak verecek şekilde sorguları yanıtlayabilir. Böylece saldırgan, yasal konumlardan gelen Internet trafiğini kendi belirlediği bir adrese yeniden yönlendirebilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Ağa veya Internet'e bağlı olan Windows sistemleri risk altındadır. Windows Vista ve Windows Server 2008 bu güvenlik açığından etkilenmez.
  
**Güncelleştirme ne yapıyor? **  
Güncelleştirme, uzak sorgular için rasgeleliği yüksek DNS işlem kimliği değerleri ve rasgele UDP yuvaları kullanarak bu güvenlik açığını ortadan kaldırır.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.

**Güncelleştirme Uyumluluğu Değerlendiricisi ve Uygulama Uyumluluğu Araç Takımı**
  
Güncelleştirmeler genelde uygulamalarınızın çalışması için gerekli olan aynı dosyalara ve kayıt defteri ayarlarına yazar. Bu durum da uyumsuzlukları tetikleyebilir ve güvenlik güncelleştirmelerinin dağıtılması için gereken zamanı artırabilir. [Uygulama Uyumluluğu Araç Takımı 5.0](http://www.microsoft.com/downloads/details.aspx?familyid=24da89e9-b581-47b0-b45e-492dd6da2971&displaylang=en) ile birlikte gelen [Güncelleştirme Uyumluluğu Değerlendiricisi](http://technet2.microsoft.com/windowsvista/en/library/4279e239-37a4-44aa-aec5-4e70fe39f9de1033.mspx?mfr=true) bileşenlerini kullanarak, Windows güncelleştirmelerinin yüklü uygulamalara göre sınanması ve doğrulanması sürecini hızlandırabilirsiniz.
  
Uygulama Uyumluluğu Araç Takımı (ACT), çalışma ortamınıza Microsoft Windows Vista'yı, bir Microsoft Güvenlik Güncelleştirmesi'ni ya da Windows Internet Explorer'ın yeni bir sürümünü dağıtmadan önce uygulama uyumluluğu sorunlarını değerlendirmek ve etkilerini azaltmak için kullanılabilecek gerekli araçları ve belgeleri içerir.

**Güncelleştirmeyi Yükleme**

Bu güvenlik güncelleştirmesini yüklediğinizde, yükleyici, sisteminizde güncelleştirilmekte olan dosyalardan bir veya daha fazlasının daha önce Microsoft tarafından yapılan bir düzeltme ile güncelleştirilmiş olup olmadığını denetler.

Bu dosyalardan birini güncelleştirmek için daha önce bir düzeltme yüklediyseniz, yükleyici RTMQFE, SP1QFE veya SP2QFE dosyalarını sisteminize kopyalar. Bu davranış hakkında daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 824994](http://support.microsoft.com/kb/824994).

Yükleyici hakkında daha fazla bilgi için, [Microsoft TechNet Web sitesini](http://go.microsoft.com/fwlink/?linkid=38951) ziyaret edin.

Bu bültende kullanılan *düzeltme* gibi terimler hakkında daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 824684](http://support.microsoft.com/kb/824684).
