Security Bulletin

Microsoft Güvenlik Bülteni MS08-068 - Önemli
============================================

SMB'deki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (957097)
-------------------------------------------------------------------------

Yayım Tarihi: 11 Kasım 2008 Salı | Güncelleştirme Tarihi: 10 Aralık 2008 Çarşamba

**Sürüm:** 1.2

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Microsoft Sunucu İleti Bloğu'ndaki (SMB) genel olarak duyurulan bir güvenlik açığını giderir. Güvenlik açığı, etkilenen sistemlerde uzaktan kod uzaktan kod yürütülmesine olanak verebilir. Bu güvenlik açığından başarıyla yararlanan bir saldırgan program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

Bu güvenlik güncelleştirmesi Microsoft Windows 2000, Windows XP ve Windows Server 2003'ün tüm desteklenen sürümleri için Önemli ve Windows Vista ve Windows Server 2008'in tüm desteklenen sürümleri için Orta olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, kimlik bilgilerinin yeniden kullanılmasını önlemek üzere SMB kimlik doğrulaması yanıtlarının doğrulanma biçimini değiştirerek bu güvenlik açığını ortadan kaldırır. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Microsoft, müşterilerin güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

**Bilinen Sorunlar.** [Microsoft Bilgi Bankası makalesi 957097](http://support.microsoft.com/kb/957097), bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir. Bu makale ayrıca, bu sorunlar için önerilen çözümleri de içermektedir.

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)
-----------------------------------------------------------------

<span></span>
**Müşterilerin bu güvenlik güncelleştirmesini yüklerken karşılaşabilecekleri bilinen sorunlar nelerdir??**   
Bu güncelleştirmeyi yükledikten sonra, NTLM'nin farklı adlandırma kurallarına yerel varlıklar yerine uzak varlıklar gibi davranması nedeniyle, SQL veya IIS gibi uygulamalar yerel NTLM kimlik doğrulama isteklerinde bulunurken başarısız olabilir. NTLM, sağlanan kimlik bilgilerinin orijinalliğini belirlemek üzere sınama/yanıt mekanizmasına dayalı bir kimlik doğrulama protokolüdür. İstemci bir NTLM sınamasının (sunucu tarafından gönderilen) doğru yanıtını hesaplayıp yanıtı sunucuya geri göndermeden önce yerel 'lsass' belleğinde önbelleğe aldığında yerel bir kimlik doğrulama hatası oluşabilir. NTLM ile ilgili sunucu kodu alınan yanıtın yerel 'lsass' önbelleğinde bulunduğunu belirlediğinde, isteği bir yeniden gönderme saldırısı olarak değerlendirir ve kimlik doğrulama isteğini reddeder. Bu da yerel kimlik doğrulama hatasına neden olur. Yansıma korumasının devre dışı bırakılması bu sistemlerin başarıyla kimlik doğrulaması için gereklidir. Kimlik doğrulama sorunları ve adlandırma kuralları hakkında ek bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 896861](http://support.microsoft.com/kb/896861), [Microsoft Bilgi Bankası makalesi 887993](http://support.microsoft.com/kb/887993) ve [Microsoft Bilgi Bankası makalesi 926642](http://support.microsoft.com/kb/926642). Bu yükleme sorunu hakkında, yansıma korumasını devre dışı bırakmayla ilgili ayrıntılı adımları da içeren ek bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 957097](http://support.microsoft.com/kb/957097).

**Dosya bilgileri ayrıntılarını nerede bulabilirim?**  
Dosya bilgileri ayrıntıları [Microsoft Bilgi Bankası makalesi 957097](http://support.microsoft.com/kb/957097)'de bulunabilir.

SMB'de Kimlik Bilgilerini Yansıtma Güvenlik Açığı - CVE-2008-4037  
-----------------------------------------------------------------
  
<span></span>
Bir kullanıcı saldırganın SMB sunucusuna bağlandığında Microsoft Sunucu İleti Bloğu (SMB) Protokolü'nün NTLM kimlik bilgilerini işleme biçiminde bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Bu güvenlik açığı, kullanıcının kimlik bilgilerinin saldırgan tarafından taklit edilmesine ve oturum açmış kullanıcının bağlamında kod yürütülmesine olanak verebilir. Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2008-4037](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-4037).
  
#### SMB'de Kimlik Bilgilerini Yansıtma Güvenlik Açığı - CVE-2008-4037 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   En iyi güvenlik duvarı uygulamaları ve standart varsayılan güvenlik duvarı yapılandırmaları, ağ yapılarının kuruluş dışından gerçekleştirilen saldırılardan korunmasına yardımcı olabilir. En iyi uygulamalar, Internet'e bağlı sistemlerde olabildiğince az sayıda bağlantı noktasının açılmasını önerir. Bu durumda, SMB bağlantı noktasına Internet'ten erişim engellenmelidir.  
-   Bu güvenlik açığından başarıyla yararlanan bir saldırgan, yerel kullanıcı ile aynı haklara sahip olabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.  
-   Windows Vista'nın ve Windows Server 2008'in tüm desteklenen sürümlerinde dosya paylaşımı varsayılan olarak devre dışı bırakılmış olduğundan, bu platformlarda bu güvenlik açığından yararlanma olasılığı düşüktür. Önceki platformlardan farklı olan bu varsayılan yapılandırma, Windows Vista ve Windows Server 2008 için saldırı yüzeyinin azaltılmasını ve bu platformlar için bu güvenlik açığının önem derecesinin Orta olarak yeniden sınıflandırılmasını sağlar.
  
#### SMB'de Kimlik Bilgilerini Yansıtma Güvenlik Açığı - CVE-2008-4037 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Güvenlik duvarında 139 ve 445 numaralı TCP bağlantı noktalarını engelleyin**
  
    Bu bağlantı noktaları, etkilenen bileşenle bağlantı başlatmak için kullanılır. Güvenlik duvarında 139 ve 445 numaralı TCP bağlantı noktalarının engellenmesi, güvenlik duvarının arkasındaki sistemleri bu açıktan yararlanma girişimlerinden korumaya yardımcı olur. Microsoft, diğer bağlantı noktalarını kullanabilecek saldırılardan korunmaya yardımcı olmak amacıyla, Internet'ten gelen tüm istenmeyen trafiğin engellenmesini önerir. Bağlantı noktaları hakkında daha fazla bilgi için, bkz. [TCP ve UDP Bağlantı Noktası Atamaları](http://go.microsoft.com/fwlink/?linkid=21312).
  
    **Geçici çözümün etkisi.** Birkaç Windows hizmeti etkilenen bağlantı noktalarını kullanır. Bağlantı noktalarına bağlanabilirliği engellemek, çeşitli uygulama ve hizmetlerin çalışmamasına neden olabilir. Etkilenebilecek uygulama ve hizmetlerden bazıları aşağıda listelenmektedir:
  
    -   SMB (CIFS) kullanan uygulamalar  
    -   İleti yuvaları veya adlandırılmış yöneltmeler kullanan uygulamalar (SMB üzerinden RPC)  
    -   Sunucu (Dosya ve Yazıcı Paylaşımı)  
    -   Grup İlkesi  
    -   Net Logon  
    -   Dağıtılmış Dosya Sistemi (DFS)  
    -   Terminal Server Lisansı  
    -   Yazdırma Biriktiricisi  
    -   Bilgisayar Tarayıcısı  
    -   Uzaktan Yordam Çağrısı Konumlandırıcı  
    -   Faks Hizmeti  
    -   Dizin Oluşturma Hizmeti  
    -   Performans Günlükleri ve Uyarılar  
    -   Systems Management Server  
    -   Lisans Günlüğü Hizmeti
  
-   **SMB imzalamayı etkinleştirme**
  
    SMB imzalama etkinleştirildiğinde, saldırgan, oturum açmış kullanıcının bağlamında kod yürütemez. Microsoft, Grup İlkesi'ni kullanarak SMB imzalamayı yapılandırmanızı önerir. Microsoft Windows 2000, Windows XP ve Windows Server 2003 için Grup İlkesi'ni kullanarak SMB imzalamayı etkinleştirme ve devre dışı bırakma konusunda ayrıntılı yönergeler için, bkz. [Microsoft Bilgi Bankası makalesi 887429](http://support.microsoft.com/kb/887429). Windows XP ve Windows Server 2003 için [Microsoft Bilgi Bankası makalesi 887429](http://support.microsoft.com/kb/887429)'daki yönergeler Windows Vista ve Windows Server 2008 için de geçerlidir.
  
    **Geçici çözümün etkisi.** SMB paket imzaları kullanmak, dosya hizmeti işlemlerinin performansında düşüşe neden olabilir. Bu ilkenin ayarlanmış olduğu bilgisayarlar, istemci tarafı paket imzalamanın etkinleştirilmemiş olduğu bilgisayarlarla iletişim kuramaz. SMB imzalama ve olası etkileri hakkında daha fazla bilgi için, bkz. [Microsoft ağ sunucusu: İletişimleri dijital olarak imzalama (her zaman)](http://technet.microsoft.com/en-us/library/cc786681.aspx).
  
#### SMB'de Kimlik Bilgilerini Yansıtma Güvenlik Açığı - CVE-2008-4037 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
SMB protokolü, kullanıcı kimlik bilgilerinin geri yansıtılmamasını ve kullanıcıya karşı kullanılmamasını sağlamak üzere NTLM kimlik bilgilerini yansıtmaya karşı koruma seçeneğini doğru şekilde kullanmaz.
  
**Microsoft Sunucu İleti Bloğu (SMB) Protokolü nedir?**  
Microsoft Sunucu İleti Bloğu (SMB) Protokolü, Microsoft Windows'da kullanılan bir Microsoft ağı dosya paylaşım protokolüdür. SMB hakkında daha fazla bilgi için, bkz. [Microsoft SMB Protokolüne ve CIFS Protokolüne Genel Bakış](http://msdn.microsoft.com/en-us/library/aa365233(vs.85).aspx).
  
**NTLM nedir?**  
NTLM, sağlanan kimlik bilgilerinin orijinalliğini belirlemek üzere sınama/yanıt mekanizmasına dayalı bir kimlik doğrulama protokolüdür.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bir saldırgan, oturum açmış kullanıcıyla aynı haklara sahip olabilir ve bu kullanıcının ayrıcalığı olan tüm işlemleri gerçekleştirebilir. Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Bu güvenlik açığı, SMB'nin etkilenen bir sürümünün bulunduğu bir kullanıcının kötü amaçlı bir sunucuya erişmesini gerektirir. Saldırganın özel hazırlanmış bir sunucu paylaşımı veya Web sitesi barındırması gerekir. Saldırganın, kullanıcıları özel hazırlanmış bir sunucu paylaşımını veya Web sitesini ziyaret etmeye zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi Web sitesine ya da sunucu paylaşımına götüren bir bağlantıyı tıklatmaya ikna ederek onları kendi sitesine çekmesi gerekir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Windows sistemlerinin günümüzde desteklenen tüm sürümleri risk altındadır.
  
**Güncelleştirme ne yapıyor?**  
Güvenlik güncelleştirmesi, kimlik bilgilerinin yeniden kullanılmasını önlemek üzere SMB kimlik doğrulaması yanıtlarının doğrulanma biçimini değiştirerek bu güvenlik açığını ortadan kaldırır.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Evet. Bu güvenlik açığı genel olarak duyurulmuştur. Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE-2008-4037](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-4037) atanmıştır.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Microsoft, bu güvenlik bülteni ilk yayımlandığında güvenlik açığının kanıtı olabilecek kod örneklerinin yayımlandığını gördü; ancak açığın müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı.
