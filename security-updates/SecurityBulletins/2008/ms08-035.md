Security Bulletin

Microsoft Güvenlik Bülteni MS08-035 - Önemli
============================================

Active Directory'deki Güvenlik Açığı Hizmet Reddine Olanak Verebilir (953235)
-----------------------------------------------------------------------------

Yayım Tarihi: 10 Haziran 2008 Salı | Güncelleştirme Tarihi: 11 Haziran 2008 Çarşamba

**Sürüm:** 1.1

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Microsoft Windows 2000 Server, Windows Server 2003 ve Windows Server 2008 üzerinde Active Directory; Windows XP Professional ve Windows Server 2003 üzerine yüklendiğinde Active Directory Uygulama Modu (ADAM) ve Windows Server 2008 üzerine yüklendiğinde Active Directory Basit Dizin Hizmeti (AD LDS) uygulamalarında özel olarak bildirilen bir güvenlik açığını giderir. Güvenlik açığından yararlanan bir saldırgan, hizmet reddi durumunun oluşmasına neden olabilir. Windows XP Professional, Windows Server 2003 ve Windows Server 2008'de bir saldırganın bu açıktan yararlanabilmesi için, geçerli oturum açma kimlik bilgilerine sahip olması gerekir. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, sistemin yanıt vermemesine veya otomatik olarak yeniden başlatılmasına neden olabilir.

Bu güvenlik güncelleştirmesi, Microsoft Windows 2000 Server'ın tüm desteklenen sürümleri için Önemli ve Windows XP Professional, Windows Server 2003 ve Windows Server 2008'in belirli sürümleri için Orta olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, istemci LDAP isteklerini doğrulayarak güvenlik açığını ortadan kaldırır. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Microsoft, müşterilerin güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

**Bilinen Sorunlar.** Yok

#### Etkilenen ve Etkilenmeyen Yazılımlar

Yazılımınızın sürümünün destek ömrünü belirlemek için [Microsoft Destek Ömrü](http://go.microsoft.com/fwlink/?linkid=21742) Web sitesini ziyaret edin.

Active Directory Güvenlik Açığı - CVE-2008-1445  
-----------------------------------------------
  
<span></span>
Microsoft Windows 2000 Server, Windows Server 2003 ve Windows Server 2008 üzerinde Active Directory uygulamalarında bir hizmet reddi güvenlik açığı bulunmaktadır. Güvenlik açığı, Windows XP ve Windows Server 2003 üzerine yüklendiğinde Active Directory Uygulama Modu (ADAM) ve Windows Server 2008 üzerine yüklendiğinde Active Directory Basit Dizin Hizmeti (AD LDS) uygulamalarında da bulunmaktadır. Bu güvenlik açığı, özel hazırlanmış LDAP isteklerinin yetersiz doğrulanması nedeniyle oluşur. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, bilgisayarın yanıt vermemesine ve yeniden başlatılmasına neden olabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2008-1445](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1445).
  
#### Active Directory Güvenlik Açığı - CVE-2008-1445 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
Microsoft bu güvenlik açığı için herhangi bir azaltıcı etken belirleyememiştir.
  
#### Active Directory Güvenlik Açığı - CVE-2008-1445 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Çevre güvenlik duvarında 389, 636, 3268 ve 3269 numaralı TCP bağlantı noktalarını engelleyin**
  
    Bu bağlantı noktaları, etkilenen bileşenle bağlantı başlatmak için kullanılır. Kuruluşun güvenlik duvarında bu bağlantı noktalarının gelen ve giden trafik için engellenmesi, güvenlik duvarının arkasındaki sistemlerin bu açıktan yararlanma girişimlerini engellemeye yardımcı olur. Diğer bağlantı noktalarını kullanabilecek saldırılardan korunmaya yardımcı olmak amacıyla, Internet'ten gelen tüm istenmeyen trafiğin engellenmesini öneririz. Bağlantı noktaları hakkında daha fazla bilgi için, bkz. [TCP ve UDP Bağlantı Noktası Atamaları](http://go.microsoft.com/fwlink/?linkid=21312).
  
#### Active Directory Güvenlik Açığı - CVE-2008-1445 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir hizmet reddi güvenlik açığıdır. Bu güvenlik açığından yararlanan saldırgan, etkilenen sistemin yanıt vermemesine ve yeniden başlatılmasına neden olabilir. Hizmet reddi güvenlik açığının, bir saldırganın kod yürütmesine veya kullanıcı haklarını yükseltmesine izin vermeyeceğini, ancak etkilenen sistemin istekleri kabul etmeyi durdurmasına neden olabileceğini unutmayın.
  
**Güvenlik açığına ne neden olmaktadır?**  
LDAP hizmeti özel hazırlanmış LDAP istekleri için yeterli denetim gerçekleştirmez.
  
**LDAP nedir?**  
Basit Dizin Erişimi Protokolü (LDAP), dağıtılmış dizinlere erişim sağlamak üzere tasarlanmış bir açık ağ protokolü standardıdır.
  
**Active Directory nedir?**  
Active Directory, Windows tabanlı bilgisayarlar için merkezi kimlik doğrulama ve yetkilendirme hizmetlerini sağlayan bir Windows bileşenidir.
  
**Active Directory Uygulama Modu (ADAM) nedir? **  
[ADAM](http://technet2.microsoft.com/windowsserver/en/library/29fb059e-544c-4577-bf7c-ba4b08df48431033.mspx?mfr=true), Windows XP ve Windows Server 2003 için bir sistem hizmeti olarak değil, bir kullanıcı hizmeti olarak çalışan Basit Dizin Erişim Protokolü (LDAP) dizin hizmetidir.
  
**Active Directory Basit Dizin Hizmeti (AD LDS) nedir? **  
[AD LDS](http://technet2.microsoft.com/windowsserver2008/en/library/6a3bedf7-9c5b-4ada-9a51-6b794adc9ab81033.mspx?mfr=true), Windows Server 2008 için bir sistem hizmeti olarak değil, bir kullanıcı hizmeti olarak çalışan Basit Dizin Erişim Protokolü (LDAP) dizin hizmetidir.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, bir kullanıcının sisteminin yanıt vermemesine ve yeniden başlatılmasına neden olabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırgan, ADAM, Active Directory veya AD LDS sunucusuna özel hazırlanmış bir LDAP paketi göndererek bu güvenlik açığından yararlanmaya çalışabilir. Windows 2000 Server için, hedef ağa erişimi olan herhangi bir anonim kullanıcı etkilenen sisteme özel hazırlanmış bir ağ paketi teslim ederek bu güvenlik açığından yararlanabilir. Windows Server 2003'te veya ADAM yüklenmiş olan sistemlerde, saldırganın bu güvenlik açığından yararlanabilmesi için geçerli kimlik doğrulama bilgileri olması gerekir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Basit Dizin Erişim Protokolü (LDAP), Active Directory ve ADAM için gerekmektedir. Bu nedenle, ADAM, Active Directory veya AD LDS sunucusu olarak çalışan tüm sistemler bu güvenlik açığından etkilenir.
  
**Güncelleştirme ne yapıyor? **  
Güncelleştirme, istemci tarafından gönderilen özel hazırlanmış LDAP isteklerini doğrulayarak güvenlik açığını giderir.

**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı? **  
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.
