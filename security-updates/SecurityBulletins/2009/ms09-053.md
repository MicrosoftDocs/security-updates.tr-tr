Security Bulletin

Microsoft Güvenlik Bülteni MS09-053 - Önemli
============================================

Internet Information Services için FTP Hizmeti'ndeki Güvenlik Açıkları Uzaktan Kod Yürütülmesine İzin Verebilir (975254)
------------------------------------------------------------------------------------------------------------------------

Yayım Tarihi: 13 Ekim 2009 Salı | Güncelleştirme Tarihi: 19 Ekim 2009 Pazartesi

**Sürüm:** 1.1

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Microsoft Internet Information Services (IIS) 5.0, Microsoft Internet Information Services (IIS) 5.1, Microsoft Internet Information Services (IIS) 6.0 ve Microsoft Internet Information Services (IIS) 7.0'daki FTP Hizmeti'nde genel olarak duyurulan iki güvenlik açığını giderir. IIS 7.0'da yalnızca FTP Hizmeti 6.0 etkilenir. Bu güvenlik açıkları IIS 5.0 üzerinde FTP Hizmeti çalışan sistemlerde uzaktan kod yürütülmesine (RCE) ya da IIS 5.0, IIS 5.1, IIS 6.0 veya IIS 7.0 üzerinde FTP Hizmeti çalışan sistemlerde hizmet reddine (DoS) neden olabilir.

Bu güvenlik güncelleştirmesi IIS 5.0, IIS 5.1, IIS 6.0 ve IIS 7.0 üzerinde FTP Hizmeti 6.0 için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, FTP Hizmeti'nin liste işlemlerini işleme biçimini değiştirerek bu güvenlik açıklarını giderir. Bu güvenlik açıkları hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

Bu güvenlik güncelleştirmesi ilk olarak [Microsoft Güvenlik Danışma Belgesi 975191](http://technet.microsoft.com/security/advisory/975191)'de açıklanan güvenlik açığını da giderir.

**Öneri.**  Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** [Microsoft Bilgi Bankası makalesi 975254](http://support.microsoft.com/kb/975254), bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir. Bu makale ayrıca, bu sorunlar için önerilen çözümleri de içermektedir.

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)
-----------------------------------------------------------------

<span></span>
**Dosya bilgileri ayrıntılarını nerede bulabilirim?**  
Dosya bilgileri ayrıntılarının konumu için **Güvenlik Güncelleştirmesi Dağıtımı** bölümündeki başvuru tablolarına bakın.

**Bu güncelleştirme neden bildirilen birkaç güvenlik açığını ele alıyor?**  
Bu güncelleştirme, bazı güvenlik açıklarını ortadan kaldırmaya yönelik değişikliklerin ilgili dosyalarda bulunması nedeniyle bu sorunlar için destek içermektedir. Müşterilerimiz birbiriyle neredeyse aynı birkaç güncelleştirme yerine, yalnızca bu güncelleştirmeyi yüklemelidir.

**Windows Server 2008 Sunucu Yöneticisi'ni kullanarak FTP hizmetini yükledim. Sistemim bu açıktan etkilenir mi?**  
Windows Server 2008 Sunucu Yöneticisi'ni kullanarak FTP hizmetini yüklediğinizde, açıktan etkilenen FTP Hizmeti 6.0 yüklenir. Microsoft, bu bültenden uygun güvenlik güncelleştirmesini yüklemenizi veya FTP Hizmeti 7.5 sürümüne yükseltmenizi önerir.

**Windows** **Vista** **veya Windows Server 2008 için FTP Hizmeti 7.5 bu güvenlik açığından etkileniyor mu?**  
Hayır. Windows Vista ve Windows Server 2008'in 32-bit ve x64 tabanlı sürümleri için kullanıma hazır karşıdan yükleme olarak edinilebilen FTP Hizmeti 7.5 bu güvenlik açığından etkilenmez. Yükleme Merkezi'ndeki aşağıdaki sitelere bakın:

-   [IIS 7.0 için Microsoft FTP Hizmeti 7.5 (x86)](http://www.microsoft.com/downloads/details.aspx?displaylang=en&familyid=b7f5b652-8c5c-447a-88b8-8cfc5c13f571)
-   [IIS 7.0 için Microsoft FTP Hizmeti 7.5 (x64)](http://www.microsoft.com/downloads/details.aspx?displaylang=en&familyid=ffb7c167-279e-48d3-8169-dea85784c4d1)

**Bu güvenlik bülteninde açıklanan yazılımın eski bir sürümünü kullanıyorum. Ne yapmalıyım?**  
Bu bültende listelenen etkilenen yazılımlar, hangi sürümlerinin etkilendiğini belirlemek amacıyla sınanmıştır. Diğer sürümler destek ömrünü tamamlamıştır. Yazılım sürümünüzün destek ömrünü belirlemek için [Microsoft Destek Ömrü](http://go.microsoft.com/fwlink/?linkid=21742) Web sitesini ziyaret edin.

Bu yazılımların eski sürümlerini kullanan müşterilerimiz, ortaya çıkabilecek güvenlik açıklarından etkilenme olasılığını ortadan kaldırmak için desteklenen sürümlere acilen yükseltmelidir. Windows Ürün Destek Ömrü hakkında daha fazla bilgi için [Microsoft Destek Ömrü](http://go.microsoft.com/fwlink/?linkid=21742) Web sitesini ziyaret edin. Bu yazılım sürümleri için genişletilmiş güvenlik güncelleştirmesi destek süreci hakkında daha fazla bilgi için, [Microsoft Ürün Destek Hizmetleri](http://go.microsoft.com/fwlink/?linkid=33328)'ni ziyaret edin.

Yazılımların eski sürümleri için ek özel destek isteyen müşteriler, Microsoft hesap ekibi temsilcilerine, Teknik Hesap Yöneticisi'ne veya özel destek seçenekleri için uygun Microsoft ortağı temsilcisine başvurmalıdır. Alliance, Premier veya Authorized sözleşme sahibi olmayan müşteriler yerel Microsoft satış ofisine de başvurabilir. Başvuru bilgileri için, [Microsoft Worldwide Information](http://go.microsoft.com/fwlink/?linkid=33329) Web sitesini ziyaret edin, bulunduğunuz ülkeyi seçin ve **Go**'yu (Git) tıklatıp telefon numaralarının listesini görüntüleyin. Aradığınızda, yerel Premier Destek satış yöneticisiyle görüşmek istediğinizi söyleyin. Daha fazla bilgi için, bkz: [Windows İşletim Sistemlerinin Ürün Destek Ömrü Hakkında SSS](http://go.microsoft.com/fwlink/?linkid=33330).

IIS FTP Hizmeti'nde DoS Güvenlik Açığı - CVE-2009-2521
------------------------------------------------------

<span></span>
Microsoft Internet Information Services (IIS) 5.0, Microsoft Internet Information Services (IIS) 5.1, Microsoft Internet Information Services (IIS) 6.0 ve Microsoft Internet Information Services (IIS) 7.0'daki FTP Hizmeti'nde bir güvenlik açığı bulunmaktadır. Bu güvenlik açığı hizmet reddine (DoS) neden olabilir.

Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz. [CVE-2009-2521](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2521).

#### IIS FTP Hizmeti'nde DoS Güvenlik Açığı - CVE-2009-2521 için Azaltıcı Etkenler

Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:

-   FTP Hizmeti varsayılan olarak Microsoft Windows 2000, Windows XP, Windows Vista, Windows Server 2003 Windows Server 2008 ve Windows 7'nin desteklenen hiçbir sürümünde yüklü değildir.

#### IIS FTP Hizmeti'nde DoS Güvenlik Açığı - CVE-2009-2521 için Geçici Çözümler

Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:

-   **FTP hizmetini devre dışı bırakın**

    Aşağıdaki yordamlardan birini kullanarak FTP hizmetini devre dışı bırakın:

    -   Komut istemine aşağıdaki komutu girin:
        **net stop msftpsvc**
    -   Internet Information Services bölmesinde aşağıdaki adımları gerçekleştirin:
        1. Sunucunuzu genişletin ve FTP sitesini sağ tıklatın.
        2. **Durdur**'u tıklatın.

    **Geçici çözümün etkisi.** FTP hizmetini kullanarak dosya aktarımı gerçekleştiremezsiniz.

    **Geçici çözüm nasıl geri alınır.**

    Aşağıdaki yordamlardan birini kullanarak FTP hizmetini yeniden etkinleştirin:

    -   Komut istemine aşağıdaki komutu girin:
        **net start msftpsvc**
    -   Internet Information Services bölmesinde aşağıdaki adımları gerçekleştirin:
        1. Sunucunuzu genişletin ve FTP sitesini sağ tıklatın.
        2. **Başlat**'ı tıklatın.

-   **Anonim kullanıcıların FTP'ye yazma erişimini engelleyin**

    Bu geçici çözüm, anonim kullanıcıların gerçekleştirdiği hizmet reddi saldırılarını engeller.

    Bu geçici çözüm bilinen yararlanma vektörlerini engeller ve bu saldırı vektörünün tüm türevlerinin etkisini en aza indirir. Ancak geçici çözüm, güvenlik açığından hiçbir şekilde yararlanılamamasını sağlayamaz.

    Varsayılan olarak, anonim kullanıcıların FTP'ye yazma erişimine izin verilmez. FTP sunucusunda anonim yazma erişimi atanmışsa, yönetici, anonim yazma erişimini engellemek üzere IIS izinlerini değiştirebilir. Bu geçici çözüm kullanıldığında, anonim kullanıcılar uzaktan kod yürütülmesine yol açamaz.

    IIS izinlerini değiştirerek anonim kullanıcıların FTP'ye yazma erişimini engellemek için aşağıdaki adımları gerçekleştirin:

    1.  IIS Yöneticisi'ni başlatın.
    2.  **Varsayılan FTP Sitesi**'ni sağ tıklatın ve **Özellikler**'in üzerine gelin.
    3.  **Giriş Dizini** sekmesini tıklatın.
    4.  **Yazma** izninin seçili olmadığını doğrulayın.

    **Geçici çözümün etkisi:** Anonim kullanıcılar FTP Hizmeti'ni kullanarak sunucuda içerik yayımlayamaz.

    **Geçici çözüm nasıl geri alınır.**

    1.  IIS Yöneticisi'ni başlatın.
    2.  **Varsayılan FTP Sitesi**'ni sağ tıklatın ve **Özellikler**'in üzerine gelin.
    3.  **Giriş Dizini** sekmesini tıklatın.
    4.  **Yazma** izninin seçili olduğunu doğrulayın.

-   **Anonim kullanıcıların FTP'ye erişimini engelleyin**

    Bu geçici çözüm, geçerli bir kullanıcı hesabı bulunmayan kimliği doğrulanmamış kullanıcıların gerçekleştirdiği hizmet reddi saldırılarını engeller.

    Bu geçici çözüm bilinen yararlanma vektörlerini engeller ve bu saldırı vektörünün tüm türevlerinin etkisini en aza indirir. Ancak geçici çözüm, güvenlik açığından hiçbir şekilde yararlanılamamasını sağlayamaz.

    Varsayılan olarak, anonim kullanıcıların FTP'ye yazma erişimine izin verilmez. FTP sunucusunda anonim yazma erişimi atanmışsa, yönetici, anonim yazma erişimini engellemek üzere IIS izinlerini değiştirebilir. Bu geçici çözüm kullanıldığında, anonim kullanıcılar güvenlik açığından yararlanamaz.

    IIS izinlerini değiştirerek anonim kullanıcıların FTP'ye yazma erişimini engellemek için aşağıdaki adımları gerçekleştirin:

    1.  IIS Yöneticisi'ni başlatın.
    2.  **Varsayılan FTP Sitesi**'ni sağ tıklatın ve **Özellikler**'i seçin.
    3.  **Güvenlik Hesapları** sekmesini tıklatın.
    4.  **Anonim bağlantılara izin ver** seçeneğinin işaretli olmadığından emin olun.

    **Geçici Çözümün Etkisi:** Anonim kullanıcılar FTP Sitesi'ne erişemez.

    **Geçici çözüm nasıl geri alınır.**

    1.  IIS Yöneticisi'ni başlatın.
    2.  **Varsayılan FTP Sitesi**'ni sağ tıklatın ve **Özellikler**'i seçin.
    3.  **Güvenlik Hesapları** sekmesini tıklatın.
    4.  **Anonim bağlantılara izin ver** seçeneğinin işaretli olduğundan emin olun.

-   **NTFS dosya sistemi izinlerini değiştirerek FTP kullanıcılarının dizin oluşturmasını engelleyin**

    Bu geçici çözüm, kimliği doğrulanmamış kullanıcıların gerçekleştirdiği hizmet reddi saldırılarını engeller.

    Bu geçici çözüm bilinen yararlanma vektörlerini engeller ve bu saldırı vektörünün tüm türevlerinin etkisini en aza indirir. Ancak geçici çözüm, güvenlik açığından hiçbir şekilde yararlanılamamasını sağlayamaz.

    Bir yönetici, sunucuda barındırılan FTP sitelerinin kök dizinleri için NTFS dosya sistemi izinlerini değiştirerek, FTP kullanıcılarının dizin oluşturmasını engelleyebilir. Bu değişiklik sonrasında, FTP kullanıcıları varolan dizinlere dosya yüklemeye devam edebilirler. Bu geçici çözüm kullanıldığında, kimliği doğrulanmış ve doğrulanmamış olan kullanıcılar uzaktan kod yürütülmesine yol açamaz.

    Yönetici olarak, Users grubundan dizin oluşturma ayrıcalıklarını kaldırmak üzere aşağıdaki adımları gerçekleştirin. FTP kullanıcılarınızı yönetmek üzere yapılandırılmış bir FTP kullanıcınız veya özel grubunuz varsa, aşağıdaki Adım 5'te yer alan Users grubunun yerine bu özel kimlikleri kullanın.

    1.  FTP sitenizin kök dizinine gidin. Bu dizin, varsayılan olarak %systemroot%\\inetpub\\ftproot konumundadır.
    2.  Dizini sağ tıklatın ve **Özellikler**'i seçin.
    3.  **Güvenlik** sekmesini ve ardından **Gelişmiş**'i tıklatın.
    4.  **Değiştirme İzinleri**'ni tıklatın.
    5.  **Users** grubunu seçin **Düzen**'i tıklatın.
    6.  **Klasörler Oluştur/Veriyi Arkasına Ekle** izninin seçimini kaldırın.

    **Geçici Çözümün Etkisi:** FTP kullanıcıları, FTP Hizmeti'ni kullanarak dizin oluşturamaz. Ancak FTP kullanıcıları FTP Hizmeti'ni kullanarak varolan dizinlere dosya yüklemeye devam edebilir.

-   **FTP Hizmeti 7.5'e yükseltin**

    FTP Hizmeti 7.5, Internet Information Services 7.0 üzerinde kullanılabilir. FTP Hizmeti'nin bu sürümü, bu bültende açıklanan güvenlik açıklarından etkilenmemektedir. Yükleme Merkezi'ndeki aşağıdaki sitelere bakın:

    -   [IIS 7.0 için Microsoft FTP Hizmeti 7.5 (x86)](http://www.microsoft.com/downloads/details.aspx?displaylang=en&familyid=b7f5b652-8c5c-447a-88b8-8cfc5c13f571)
    -   [IIS 7.0 için Microsoft FTP Hizmeti 7.5 (x64)](http://www.microsoft.com/downloads/details.aspx?displaylang=en&familyid=ffb7c167-279e-48d3-8169-dea85784c4d1)

    **Geçici çözümün etkisi:** FTP sitelerinin FTP Hizmeti 6.0'dan FTP Hizmeti 7.5'e yükseltilmeleri gerekir.

#### IIS FTP Hizmeti'nde DoS Güvenlik Açığı - CVE-2009-2521 Hakkında SSS

**Bu güvenlik açığının kapsamı nedir?**  
Bu bir hizmet reddi güvenlik açığıdır. Bu güvenlik açığından yararlanan saldırgan, etkilenen hizmetin yanıt vermemeye başlamasına ve yeniden başlatılmasına neden olabilir. Hizmet reddi güvenlik açığının, bir saldırganın kod yürütmesine veya kullanıcı haklarını yükseltmesine izin vermeyeceğini, ancak etkilenen sistemin istekleri kabul etmeyi durdurmasına neden olabileceğini unutmayın.

**Güvenlik açığına ne neden olmaktadır?**  
Bu güvenlik açığı IIS'deki Microsoft FTP hizmetinin liste komutlarını işleme biçimi nedeniyle ortaya çıkar.

**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, bir kullanıcının sisteminin yanıt vermemesine ve yeniden başlatılmasına neden olabilir.

**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırgan, sunucuya özel hazırlanmış liste komutları göndererek bu güvenlik açığından yararlanabilir.

**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Microsoft Dosya Aktarım Protokolü Hizmeti (FTP) çalıştırılan sunucu sistemleri.

**Güncelleştirme ne yapıyor?**  
Bu güncelleştirme, Microsoft FTP sunucusunun liste komutu işlemlerini gerçekleştirme biçimini değiştirir.

**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Evet. Bu güvenlik açığı genel olarak duyurulmuştur. Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE-2009-2521](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2521) atanmıştır.

**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Evet. Microsoft bu güvenlik açığından yararlanmaya çalışan sınırlı sayıda hedefe yönelik saldırı olduğunu bilmektedir.

**Bu güvenlik güncelleştirmesini uygulamak, müşterilerin açıktan yararlanmaya yönelik olarak yayımlanan koddan korunmasına yardımcı olur mu?**  
Evet. Bu güvenlik güncelleştirmesi, şu anda yararlanılmakta olan güvenlik açığını ortadan kaldırır. Güvenlik açığı giderilmiş, ayrıca [CVE-2009-2521](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2521) Ortak Güvenlik Açığı ve Etkilenme numarası atanmıştır.

    **Geçici Çözümün Etkisi:** Anonim kullanıcılar FTP Sitesi'ne erişemez.

    **Geçici çözüm nasıl geri alınır.**

    1.  IIS Yöneticisi'ni başlatın.
    2.  **Varsayılan FTP Sitesi**'ni sağ tıklatın ve **Özellikler**'i seçin.
    3.  **Güvenlik Hesapları** sekmesini tıklatın.
    4.  **Anonim bağlantılara izin ver** seçeneğinin işaretli olduğundan emin olun.

-   **NTFS dosya sistemi izinlerini değiştirerek FTP kullanıcılarının dizin oluşturmasını engelleyin**

    Bu geçici çözüm, kimliği doğrulanmış kullanıcıların gerçekleştirdiği uzaktan kod yürütme saldırılarını engeller. Bu geçici çözüm kullanıldığında, IIS sunucusunun tüm etkilenen sürümleri, kimliği doğrulanmış kullanıcılar tarafından gerçekleştirilen hizmet reddi saldırılarından etkilenmeye devam edecektir.

    Bu geçici çözüm bilinen yararlanma vektörlerini engeller ve bu saldırı vektörünün tüm türevlerinin etkisini en aza indirir. Ancak geçici çözüm, güvenlik açığından hiçbir şekilde yararlanılamamasını sağlayamaz.

    Bir yönetici, sunucuda barındırılan FTP sitelerinin kök dizinleri için NTFS dosya sistemi izinlerini değiştirerek, FTP kullanıcılarının dizin oluşturmasını engelleyebilir. Bu değişiklik sonrasında, FTP kullanıcıları varolan dizinlere dosya yüklemeye devam edebilirler. Bu geçici çözüm kullanıldığında, kimliği doğrulanmış ve doğrulanmamış olan kullanıcılar uzaktan kod yürütülmesine yol açamaz.

    Yönetici olarak, Users grubundan dizin oluşturma ayrıcalıklarını kaldırmak üzere aşağıdaki adımları gerçekleştirin. FTP kullanıcılarınızı yönetmek üzere yapılandırılmış bir FTP kullanıcınız veya özel grubunuz varsa, aşağıdaki Adım 5'te yer alan Users grubunun yerine bu özel kimlikleri kullanın.

    1.  FTP sitenizin kök dizinine gidin. Bu dizin, varsayılan olarak %systemroot%\\inetpub\\ftproot konumundadır.
    2.  Dizini sağ tıklatın ve **Özellikler**'i seçin.
    3.  **Güvenlik** sekmesini ve ardından **Gelişmiş**'i tıklatın.
    4.  **Değiştirme İzinleri**'ni tıklatın.
    5.  **Users** grubunu seçin **Düzen**'i tıklatın.
    6.  **Klasörler Oluştur/Veriyi Arkasına Ekle** izninin seçimini kaldırın.

    **Geçici Çözümün Etkisi:** FTP kullanıcıları, FTP Hizmeti'ni kullanarak dizin oluşturamaz. Ancak FTP kullanıcıları FTP Hizmeti'ni kullanarak varolan dizinlere dosya yüklemeye devam edebilir.

    **Geçici çözüm nasıl geri alınır.**

    1.  FTP sitenizin kök dizinine gidin. Bu dizin, varsayılan olarak %systemroot%\\inetpub\\ftproot konumundadır.
    2.  Dizini sağ tıklatın ve **Özellikler**'i seçin.
    3.  **Güvenlik** sekmesini ve ardından **Gelişmiş**'i tıklatın.
    4.  **Değiştirme İzinleri**'ni tıklatın.
    5.  **Users** grubunu seçin **Düzen**'i tıklatın.
    6.  **Klasörler Oluştur/Veriyi Arkasına Ekle** iznini seçin.

#### IIS FTP Hizmeti'nde RCE ve DoS Güvenlik Açığı - CVE-2009-3023 Hakkında SSS

**Bu güvenlik açığının kapsamı nedir?**  
Bu bir hizmet reddi güvenlik açığıdır. Bu güvenlik açığından yararlanan saldırgan, etkilenen hizmetin yanıt vermemeye başlamasına ve yeniden başlatılmasına neden olabilir. Hizmet reddi güvenlik açığının, bir saldırganın kod yürütmesine veya kullanıcı haklarını yükseltmesine izin vermeyeceğini, ancak etkilenen sistemin istekleri kabul etmeyi durdurmasına neden olabileceğini unutmayın.

Bu, Microsoft Windows 2000 Service Pack 4 çalıştırılan sistemlerde uzaktan kod yürütme güvenlik açığı ve etkilenen diğer tüm platformlarda hizmet reddi güvenlik açığıdır. Microsoft Windows 2000 Service Pack 4 üzerinde IIS 5.0'daki bu güvenlik açığından başarıyla yararlanan bir saldırgan LocalSystem bağlamında uzaktan kod yürütülmesine neden olabilir. IIS 5.1 ve IIS 6.0'daki bu güvenlik açığından başarıyla yararlanan bir saldırgan IIS FTP hizmetinin durdurulmasına ve yanıt vermemeye başlamasına neden olabilir.

**Güvenlik açığına ne neden olmaktadır?**  
Bu güvenlik açığı IIS'deki Microsoft FTP hizmetinin liste komutlarını işleme biçimi nedeniyle ortaya çıkar.

**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
FTP Hizmeti'ne erişimi olan bir saldırgan bu güvenlik açığından yararlanarak yığın tabanlı bir taşmaya neden olabilir ve IIS 5.0 çalışan sistemlerde LocalSystem hesabının bağlamında rasgele kod yürütülmesine ya da IIS 5.1 veya IIS 6.0 çalıştırılan etkilenen sistemlerde hizmet reddine olanak verebilir. Anonim kimlik doğrulamasına izin verilen FTP Hizmeti yapılandırmalarında, bu güvenlik açıklarından yararlanılabilmesi için saldırganın kimliğinin doğrulanması gerekmez. LocalSystem hesabı hakkında daha fazla bilgi için, [LocalSystem Hesabı](http://msdn.microsoft.com/en-us/library/ms684190(vs.85).aspx) adlı MSDN makalesine bakın.

**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
FTP hizmetine erişimi olan bir anonim kullanıcı veya kimliği doğrulanmış kullanıcı, FTP NLST komutları göndererek Microsoft Windows 2000 üzerinde IIS 5.0 çalıştırılan sistemlerde belleğin bozulmasına ve uzaktan kod yürütülmesine, diğer platformlar üzerinde IIS 5.1 veya IIS 6.0 çalıştırılan sistemlerde hizmet reddine neden olabilir.

**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Microsoft Dosya Aktarım Protokolü Hizmeti (FTP) çalıştırılan sunucu sistemleri.

**Güncelleştirme ne yapıyor?**  
Bu güncelleştirme, Microsoft FTP sunucusunun liste komutu işlemlerini gerçekleştirme biçimini değiştirir.

**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Evet. Bu güvenlik açığı genel olarak duyurulmuştur. Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE-2009-3023](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-3023) atanmıştır.

**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Evet. Microsoft bu güvenlik açığından yararlanmaya çalışan sınırlı sayıda hedefe yönelik saldırı olduğunu bilmektedir.

**Bu güvenlik güncelleştirmesini uygulamak, müşterilerin açıktan yararlanmaya yönelik olarak yayımlanan koddan korunmasına yardımcı olur mu?**  
Evet. Bu güvenlik güncelleştirmesi, şu anda yararlanılmakta olan güvenlik açığını ortadan kaldırır. Güvenlik açığı giderilmiş, ayrıca [CVE-2009-3023](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-3023) Ortak Güvenlik Açığı ve Etkilenme numarası atanmıştır.
