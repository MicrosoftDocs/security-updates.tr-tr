Security Bulletin

Microsoft Güvenlik Bülteni MS09-070 - Önemli
============================================

Active Directory Federasyon Hizmetleri'ndeki Güvenlik Açıkları Uzaktan Kod Yürütülmesine İzin Verebilir (971726)
----------------------------------------------------------------------------------------------------------------

Yayım Tarihi: 8 Aralık 2009 Salı | Güncelleştirme Tarihi: 9 Aralık 2009 Çarşamba

**Sürüm:** 1.1

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Microsoft Windows'daki özel olarak bildirilen iki güvenlik açığını giderir. Bu güvenlik açıklarından daha önemli olanı, bir kullanıcı ADFS etkin bir Web sunucusuna özel hazırlanmış bir HTTP isteği gönderirse uzaktan kod yürütülmesine olanak verebilir. Saldırganın bu güvenlik açıklarından birinden yararlanabilmesi için kimliği doğrulanmış bir kullanıcı olması gerekir.

Bu güvenlik güncelleştirmesi etkilenen Windows Server 2003, Windows Server 2003 x64 Edition, Windows Server 2008 ve Windows Server 2008 x64 Edition sürümleri için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, ADFS etkin Web sunucularının bir Web istemcisi tarafından gönderilen istek üstbilgilerine uyguladığı doğrulama biçimini düzelterek bu güvenlik açıklarını giderir. Bu güvenlik açıkları hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.**  Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** [Microsoft Bilgi Bankası makalesi 971726](http://support.microsoft.com/kb/971726), bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir. Bu makale ayrıca, bu sorunlar için önerilen çözümleri de içermektedir.

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)
-----------------------------------------------------------------

<span></span>
**Dosya bilgileri ayrıntılarını nerede bulabilirim?**  
Dosya bilgileri ayrıntılarının konumu için **Güvenlik Güncelleştirmesi Dağıtımı** bölümündeki başvuru tablolarına bakın.

**Müşterilerin bu güvenlik güncelleştirmesini yüklerken karşılaşabilecekleri bilinen sorunlar nelerdir?**  
[Microsoft Bilgi Bankası makalesi 971726](http://support.microsoft.com/kb/971726), bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir. Bu makale ayrıca, bu sorunlar için önerilen çözümleri de içermektedir.

**Bu güncelleştirme neden bildirilen birkaç güvenlik açığını ele alıyor?**  
Bu güncelleştirme, bazı güvenlik açıklarını ortadan kaldırmaya yönelik değişikliklerin ilgili dosyalarda bulunması nedeniyle bu sorunlar için destek içermektedir. Müşterilerimiz birbiriyle neredeyse aynı birkaç güncelleştirme yerine, yalnızca bu güncelleştirmeyi yüklemelidir.

**Active Directory Federasyon Hizmetleri nedir?**  
Active Directory Federasyon Hizmetleri (ADFS), yeni kullanılmaya başlanan, sektör tarafından desteklenen ve WS-\* belirtimlerinde tanımlanan Web Hizmetleri Mimarisi'ne dayalıdır. ADFS, tek bir çevrimiçi oturum sırasında birden çok ilgili Web uygulaması için kullanıcıların kimliğini doğrulamak üzere çoklu oturum açma (SSO) kullanmanıza yardımcı olur. ADFS bunu, dijital kimlik ve yetkilendirme haklarını güvenlik ve kuruluş sınırları içinde güvenli şekilde paylaşarak gerçekleştirir.

**ADFS'nin sunucumda yüklü olup olmadığını nasıl doğrulayabilirim? **
ADFS'nin sunucunuzda yüklü olduğunu doğrulamak için, sırasıyla **Başlat**'ı, **Denetim Masası**'nı ve **Program Ekle/Kaldır**'ı tıklatın, **Windows Bileşenleri**'ni çift tıklatın, ardından **Federasyon Hizmetleri** bileşeninin yüklü olup olmadığına bakın.

**Windows Server 2003 R2 nedir?**  
Windows Server 2003 R2, varolan Windows Server 2003 çalışma ortamınızla kolayca tümleştirilebilmesinin yanı sıra, Windows Server 2003 işletim sistemini genişleterek yerel ve uzak kaynaklara erişimi daha etkin şekilde yönetebilmenizi ve denetleyebilmenizi sağlar. Windows Server 2003 R2 ölçeklenebilir, güvenliği artırılmış bir Web platformu ve UNIX tabanlı sistemlerle sorunsuz birlikte çalışabilirlik sağlar; basitleştirilmiş sunucu dalı yönetimi, geliştirilmiş kimlik ve erişim yönetimi, daha etkin bir depolama yönetimi de dahil yeni senaryolara olanak verir. Windows Server 2003 R2 Datacenter Edition ve Windows Server 2003 R2 Enterprise Edition sürümleri ayrıca müşterilerin sunucu sanallaştırmasından daha fazla değer elde etmelerine olanak veren yeni ve dinamik bir lisanslama sunar.

Windows Server 2003 R2 hakkında daha fazla bilgi için, [Windows Server 2003 R2](http://www.microsoft.com/windowsserver2003/default.mspx) sitesine bakın.

**WS-\* Web belirtimleri nedir?**  

Basit Nesne Erişim Protokolü (SOAP), XML, Web Hizmeti Tanımlama Dili (WSDL) ve Evrensel Bulma, Açıklama ve Tümleştirme (UDDI) gibi endüstri standartlarına dayanan Web Hizmetleri Mimarisi belirtimleridir. WS-\* federe kimliği ve güvenliği yönetme becerisi de dahil olmak üzere, genişletilmiş kuruluşlar için eksiksiz ve birlikte çalışabilir iş çözümleri sunmaya yönelik bir temel sağlar.

Web hizmetleri modeli, kurumsal sistemlerin çok çeşitli türde aygıtlarda çalıştırılan ve bu aygıtlardan erişilen farklı programlama modelleriyle farklı dillerde yazılmış olduğu fikrine dayanır. Web hizmetleri, hangi dil ile yazılmış olduklarına veya hangi platformda çalıştırıldıklarına bakılmaksızın Internet üzerinden birbiriyle kolayca ve etkin şekilde bağlantı oluşturabilen ve etkileşim kurabilen dağıtılmış sistemler oluşturmanın bir yoludur.

ADFS'de Çoklu Oturum Açma Sırasında Kimlik Sahtekarlığı Güvenlik Açığı - CVE-2009-2508  
--------------------------------------------------------------------------------------
  
<span></span>
Active Directory Federasyon Hizmetleri'ndeki kimlik sahtekarlığı güvenlik açığı, saldırgan bir iş istasyonuna ve çoklu oturum açma özelliğini sunan bir Web sitesine erişmek üzere hedeflenen kullanıcı tarafından yakın zamanda kullanılmış olan Web tarayıcısına erişebiliyorsa, saldırganın kimliği doğrulanmış bir kullanıcıyı taklit etmesine olanak verebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2009-2508](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2508).
  
#### ADFS'de Çoklu Oturum Açma Sırasında Kimlik Sahtekarlığı Güvenlik Açığı - CVE-2009-2508 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Saldırgan taklit edeceği kullanıcıyı seçemez. Saldırgan yalnızca kimlik doğrulama anahtarını örneğin bir Internet bilgi noktasından edinebildiği bir kullanıcıyı taklit edebilir.  
-   Saldırı yalnızca sunucudaki oturum zaman aşımına uğramadan önce gerçekleştirilebilir. Oturumlar varsayılan olarak 600 dakika boyunca sunucuda açık kalır. Bu zaman aşımı süresi dolduktan sonra saldırgan bir kurbanın kimlik doğrulama anahtarını yeniden kullanmayı denerse saldırı başarısız olur.  
-   Saldırganın ilgili terminalde bir önceki kullanıcı tarafından kullanılan kimlik doğrulama anahtarına erişimi olmalıdır. HTTPS etkin sunucularda, bu kimlik doğrulama anahtarı iletim sırasında korunuyor olacaktır ve saldırganın bu güvenlik açığından yararlanabilmesi için kurbanın bilgisayarına (örneğin, bir Internet bilgi noktasına) erişimi olması gerekecektir. Internet bilgi noktasında, kullanıcı bir ADFS Çoklu Oturum Açma (SSO) Web uygulamasında oturum açıp daha sonra uygulamadaki oturumunu kapatabilir. Saldırgan daha sonra bu bilgisayarı kullanarak, önceki kullanıcı oturumunu kapatmış olsa da bu kullanıcıyla aynı Web uygulamasına erişim sağlayabilir.
  
#### ADFS'de Çoklu Oturum Açma Sırasında Kimlik Sahtekarlığı Güvenlik Açığı - CVE-2009-2508 için Geçici Çözümler
  
Microsoft bu güvenlik açığı için herhangi bir geçici çözüm belirleyememiştir.
  
#### ADFS'de Çoklu Oturum Açma Sırasında Kimlik Sahtekarlığı Güvenlik Açığı - CVE-2009-2508 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu, bir sızdırma güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan bir başka kullanıcıyı taklit ederek, Active Directory Federasyon Hizmetleri'nin (ADFS) çoklu oturum açma işlevselliğini kullanan bir Web sitesine hedeflenen kullanıcıymış gibi erişim sağlayabilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
Bu güvenlik açığı, ADFS'de yeterli düzeyde oturum yönetimi doğrulaması gerçekleştirilmemesi nedeniyle ortaya çıkar.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Internet bilgi noktaları gibi paylaşılan erişim sağlanan sistemlerde, bu güvenlik açığından başarıyla yararlanan bir saldırgan, daha önce kimliği doğrulanmış bir kullanıcıya ait kaynaklara erişim sağlayabilir. Bu saldırı yalnızca ADFS yöneticisi tarafından yapılandırılan ve varsayılan olarak 600 dakika olan oturum zaman aşımından önce gerçekleştirilebilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırgan yalnızca tarayıcının önbelleğinden belirli verilere erişip bu verileri ADFS çoklu oturum açma (SSO) özelliğini kullanan Web uygulamasında kimlik doğrulaması için yeniden kullanmak yoluyla bu güvenlik açığından yararlanabilir. Sunucudaki oturum zaman aşımına uğradıktan sonra, ADFS, sunucu tarafındaki kimliği doğrulanmış oturumu devre dışı bırakır ve saldırganın bu kimlik bilgilerini yeniden kullanmasına izin vermez.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Çoklu oturum açma amacıyla ADFS kullanan Web sunucuları bu güvenlik açığından etkilenir. Bu güvenlik açığı istemci sistemleri doğrudan etkilemez.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, kullanıcı ağdaki oturumunu kapattığında kimlik doğrulama bilgilerini düzgün şekilde yok sayarak bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının müşterilere saldırmak üzere genel olarak kullanıldığına dair herhangi bir bilgi almamıştı.
  
ADFS'de Uzaktan Kod Yürütme Güvenlik Açığı - CVE-2009-2509  
----------------------------------------------------------
  
<span></span>
Active Directory Federasyon Hizmetleri (ADFS) uygulamalarında bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Bu güvenlik açığı, kimliği doğrulanmış bir kullanıcı ADFS etkin bir Web sunucusuna bağlanırken istek üstbilgilerinin hatalı doğrulanması nedeniyle oluşur. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2009-2509](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2509).
  
#### ADFS'de Uzaktan Kod Yürütme Güvenlik Açığı - CVE-2009-2509 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Saldırganın bu güvenlik açığından etkilenen sunucuda geçerli oturum açma kimlik bilgileri bulunmalıdır. Saldırgan yalnızca ADFS'yi destekleyen sunucuda başarıyla kimlik doğrulaması yaptıktan sonra işlem gerçekleştirebilir.
  
#### ADFS'de Uzaktan Kod Yürütme Güvenlik Açığı - CVE-2009-2509 için Geçici Çözümler
  
Microsoft bu güvenlik açığı için herhangi bir geçici çözüm belirleyememiştir.
  
#### ADFS'de Uzaktan Kod Yürütme Güvenlik Açığı - CVE-2009-2509 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, IIS sunucusunda varsayılan olarak Network Service hesabı ayrıcalıklarıyla yapılandırılmış olan Çalışan İşlem Kimliği (WPI) ile aynı haklarla işlem gerçekleştirebilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
Bu güvenlik açığı, ADFS'nin kimliği doğrulanmış istemciler tarafından gönderilen istek üstbilgilerini düzgün şekilde doğrulamaması nedeniyle oluşur.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, IIS sunucusunda Çalışan İşlem Kimliği (WPI) ile aynı haklarla işlem gerçekleştirebilir. WPI varsayılan olarak Network Service hesabı ayrıcalıklarıyla yapılandırılır. Uygulama havuzları yönetimsel ayrıcalıklara sahip bir hesap kullanılan WPI ile yapılandırılmış olan ASP sayfaları çalıştıran IIS sunucular, uygulama havuzu varsayılan WPI ayarlarıyla yapılandırılmış IIS sunuculardan daha fazla etkilenebilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırgan, ADFS etkin Web sunucusuna özel hazırlanmış bir HTTP isteği göndermek yoluyla bu açıktan yararlanmaya çalışabilir. Saldırganın bu güvenlik açığından yararlanabilmesi için kimliği doğrulanmış bir kullanıcı olması gerekir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
ADFS etkin Web sunucuları risk altındadır. İstemci sistemler bu güvenlik açığından etkilenmez.
  
**Güncelleştirme ne yapıyor?**  
Bu güncelleştirme, ADFS etkin Web sunucularının bir Web istemcisi tarafından gönderilen istek üstbilgilerine uyguladıkları doğrulamayı düzeltir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının müşterilere saldırmak üzere genel olarak kullanıldığına dair herhangi bir bilgi almamıştı.
