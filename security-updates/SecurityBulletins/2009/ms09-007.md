Security Bulletin

Microsoft Güvenlik Bülteni MS09-007 - Önemli
============================================

SChannel'deki Güvenlik Açığı Kimlik Sahtekarlığına Neden Olabilir (960225)
--------------------------------------------------------------------------

Yayım Tarihi: 10 Mart 2009 Salı

**Sürüm:** 1.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi, Windows'daki Güvenli Kanal (SChannel) güvenlik paketindeki özel olarak bildirilen bir güvenlik açığını giderir. Güvenlik açığı, bir saldırgan kimlik doğrulama için son kullanıcı tarafından kullanılan sertifikaya erişim sağlarsa kimlik sahtekarlığına olanak verebilir. Müşteriler yalnızca kimlik doğrulaması için kullanılan ortak anahtar bileşeni başka yollarla saldırganın eline geçmişse bu güvenlik açığından etkilenir.

Güvenlik güncelleştirmesi Microsoft Windows 2000, Windows XP, Windows Server 2003, Windows Vista ve Windows Server 2008'in tüm desteklenen sürümleri için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, sunucunun TLS el sıkışma sırasında anahtar değişim verilerini ayrıştırma biçimini değiştirerek güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 294781](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

**Bilinen Sorunlar.** Yok.

SChannel'de Kimlik Sahtekarlığı Güvenlik Açığı - CVE-2009-0085  
--------------------------------------------------------------
  
<span></span>
Microsoft Windows SChannel kimlik doğrulama bileşeninde, sertifika tabanlı kimlik doğrulaması kullanılırken bir kimlik sahtekarlığı güvenlik açığı bulunmaktadır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, yalnızca yetkili kullanıcının dijital sertifikasını kullanarak ve ilişkili özel anahtar olmadan sunucuda kimliğini doğrulayabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2009-0085](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-0085).
  
#### SChannel'de Kimlik Sahtekarlığı Güvenlik Açığı - CVE-2009-0085 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Müşteriler yalnızca kimlik doğrulaması için kullanılan sertifika başka yollarla saldırganın eline geçmişse bu güvenlik açığından etkilenir. IIS Web sunucusu için, sertifikalar kimlik doğrulaması işlemi sırasında şifresiz metin olarak aktarılmaz.  
-   Yalnızca yerel olarak yapılandırılmış kullanıcı hesapları bu güvenlik açığından etkilenir. Uygulamalarının sertifika yoluyla kimliği doğrulanmış kullanıcılarını bir Active Directory etki alanıyla eşleyen müşteriler bu güvenlik açığından etkilenmez.
  
#### SChannel'de Kimlik Sahtekarlığı Güvenlik Açığı - CVE-2009-0085 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   Müşteriler, bu güvenlik açığından etkilenmeyen Active Directory sertifika eşlemesini kullanabilirler. Active Directory sertifika eşlemesi, güvenilen ortak anahtarı bulunan bir kullanıcının, kullanıcı adını ve parolayı yazmasına gerek kalmadan etki alanı kaynaklarına erişebilmesini sağlar. İstemci sertifikası doğrudan bir Active Directory kullanıcı hesabıyla eşlenebilir.  
    Sertifika eşlemelerini yapılandırmanın farklı yolları hakkında daha fazla bilgi için, bkz: [Sertifikaları kullanıcı hesaplarıyla eşleme](http://technet.microsoft.com/en-us/library/cc736706.aspx).
  
#### SChannel'de Kimlik Sahtekarlığı Güvenlik Açığı - CVE-2009-0085 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Microsoft Windows SChannel kimlik doğrulama bileşeninde, sertifika tabanlı kimlik doğrulaması kullanılırken bir kimlik sahtekarlığı güvenlik açığı bulunmaktadır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, başka bir kullanıcıyı taklit edebilir ve kullanıcının kimlik doğrulama bilgilerinin ortak bileşenini kullanarak korumalı bir sunucuda kimliğini doğrulayabilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
SChannel kimlik doğrulama bileşeni, istemcinin kimlik doğrulaması için kullanılan sertifikaya bağlantılı özel anahtara erişimi olmasını sağlamak üzere belirli Aktarım Katmanı Güvenliği (TLS) el sıkışması iletilerini yeterli düzeyde doğrulamaz.
  
**Güvenli Kanal (SChannel) güvenlik paketi nedir?**  
Güvenli Kanal (SChannel) güvenlik paketi, Güvenli Yuva Katmanı (SSL) ve Aktarım Katmanı Güvenliği (TLS) Internet standartları kimlik doğrulama protokollerini kullanan bir Güvenlik Destek Sağlayıcısı'dır (SSP). Bu bileşenler, Web'de gezinme gibi birkaç genel Internet ve ağ uygulamasını desteklemek için güvenli iletişim sağlamak üzere kullanılır. SChannel, istemci ile sunucu arasında güvenli iletişim için kimlik doğrulama hizmeti sağlanmasına yardımcı olan güvenlik paketinin bir parçasıdır. SChannel hakkında daha fazla bilgi için MSDN'deki [SChannel](http://msdn.microsoft.com/en-us/library/ms678421.aspx) Web sitesini ziyaret edin.
  
**x.509 dijital sertifikası nedir?**  
Ortak ve özel anahtarlar kullanılan şifreleme teknikleri, ortak anahtarların dağıtımını ve sertifika sağlanmasını destekleyen bir ortak anahtar altyapısı (PKI) gerektirir. Dijital sertifikalar, ortak anahtarların, kullanılan algoritmalar ile ilgili bilgilerin, sahiplik veya konu verilerinin, konu verilerini doğrulayan Sertifika Yetkilisi'nin dijital imzasının ve sertifikanın geçerli olarak kabul edileceği tarih aralığının yer aldığı bir paket oluşturur.
  
Dijital sertifikalar Internet Explorer gibi bir istemci tarafından doğrulanarak, kullanıcının bağlandığı sunucunun bu ana bilgisayar adı için gerçekten meşru olmasını sağlar. Ancak sertifikalar aynı zamanda kullanıcı kimlik doğrulaması ve yetkilendirmesi için bir güvenlik kimlik bilgisi olarak da kullanılabilir.
  
**Microsoft Güvenli Yuva Katmanı Kitaplığı nedir?**  
Microsoft Güvenli Yuva Katmanı kitaplığı, bir dizi güvenli iletişim protokolü için destek içerir. Aktarım Katmanı Güvenliği 1.0 (TLS 1.0), Güvenli Yuva Katmanı 3.0 (SSL 3.0), daha eski ve nadiren kullanılan Güvenli Yuva Katmanı 2.0 (SSL 2.0) ile Özel İletişim Teknolojisi 1.0 (PCT 1.0) protokolü buna dahildir.
  
Bu protokoller, sunucu ile istemci sistemi arasında şifreli bir bağlantı sağlar. SSL, kullanıcılar Internet gibi ortak ağlar üzerinden bağlandığında bilgilerin korunmasına yardımcı olabilir. SSL desteği, bir sunucuya yüklenmiş olması gereken bir SSL sertifikası gerektirir. SSL hakkında daha fazla bilgi için, bkz: [XADM: Güvenli Yuva Katmanı Nasıl Çalışır](http://support.microsoft.com/default.aspx?scid=kb;tr-tr;q245152).
  
**TLS ek sıkışma nedir?**  
Aktarım Katmanı Güvenliği (TLS) El Sıkışma Protokolü, güvenli oturumlar oluşturmak veya sürdürmek için gereken kimlik doğrulamasından ve anahtar değişiminden sorumludur. El Sıkışma Protokolü güvenli bir oturum oluştururken şunları yönetir:
  
-   Şifre paketi görüşmesi  
-   Sunucunun ve isteğe bağlı olarak istemcinin kimliğinin doğrulanması  
-   Oturum anahtarı bilgilerinin değişimi
  
**Güvenlik Destek Sağlayıcısı (SSP) nedir?**  
Güvenlik Destek Sağlayıcısı, Microsoft Uzaktan Yordam Çağrısı (RPC) gibi aktarım düzeyi uygulamalar ile Windows Dağıtılmış Güvenlik gibi güvenlik sağlayıcıları arasında ortak bir arabirim kullanır. Bunu, bir veya daha fazla güvenlik paketinin uygulamalar tarafından kullanılabilmesini sağlayarak yapar. Her bir güvenlik paketi, bir uygulamanın güvenlik destek sağlayıcısı arabiriminin işlev çağrıları ile asıl güvenlik protokolünün işlevleri arasında eşlemeler sağlar. Güvenlik paketleri, Kerberos kimlik doğrulaması ve Microsoft LAN Yöneticisi gibi güvenlik protokollerini destekler.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Saldırgan bu güvenlik açığından yararlanarak, yetkili kullanıcının özel anahtarına erişimi olmasa da korumalı sunucuda kimliğini doğrulayabilir ve yetkili kullanıcıyı taklit edebilir; oysa ki istemci kimlik doğrulaması gerektirecek biçimde yapılandırılmış bir sunucu söz konusuysa, genellikle özel anahtara erişim gerekmektedir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırganın, kimlik doğrulama için son kullanıcı tarafından kullanılan asıl sertifikanın ortak bileşenine erişim sağlaması gerekir. Daha sonra bir Aktarım Katmanı Güvenliği (TLS) paketi oluşturup SChannel bileşeninin uyguladığı doğrulamayı atlayarak, özel anahtarın kimlik doğrulaması sırasında bulunmasını sağlar.
  
En genel dağıtımlarda, istemci kimlik doğrulaması için kullanılan sertifikalar için bir Windows istemcisinde tek bir amacı bulunmaktadır. Bu sertifikalar yalnızca Web kimlik doğrulaması gibi tek bir amaç için aktarılır. Internet Information Services (IIS) Web sunucusuna gerçekleştirilen bir SSL bağlantısı durumunda, ortak anahtar hiçbir zaman şifresiz metin olarak aktarılmaz. Ancak e-postanın imzalanması gibi, istemci kimlik doğrulaması ve diğer amaçlar için tek bir sertifikanın kullanıldığı ya da birden çok Web sitesinde kimlik doğrulaması amacıyla sertifikanın kullanıldığı durumlarda, ortak anahtar daha yaygın olarak dağıtılabilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Bu güvenlik açığı, sertifikaya dayalı kimlik doğrulaması kullanılan Web sunucuları gibi hizmetleri etkiler ve kimlik doğrulama işlemi sırasında kullanıcının özel anahtara karşılık gelen sertifikaya sahip olduğunu kanıtlamasını gerektirir.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, sunucunun TLS el sıkışma sırasında anahtar değişim verilerini ayrıştırma biçimini değiştirerek güvenlik ortadan kaldırır.

Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının genel olarak duyurulduğuna dair herhangi bir bilgi almamıştı. Bu güvenlik güncelleştirmesi özel olarak duyurulan açığın yanı sıra iç araştırmalar sonucunda keşfedilen ek sorunları da giderir.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.
