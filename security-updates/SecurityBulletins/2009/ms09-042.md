Security Bulletin

Microsoft Güvenlik Bülteni MS09-042 - Önemli
============================================

Telnet'teki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (960859)
----------------------------------------------------------------------------

Yayım Tarihi: 11 Ağustos 2009 Salı | Güncelleştirme Tarihi: 12 Ağustos 2009 Çarşamba

**Sürüm:** 1.1

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Microsoft Telnet hizmetindeki genel olarak duyurulan bir güvenlik açığını giderir. Güvenlik açığı, bir saldırganın kimlik bilgilerini ele geçirmesine ve bunları kullanarak etkilenen sistemde oturum açmasına olanak verebilir. Böylece saldırgan, sistemde oturum açmış kullanıcıyla aynı haklara sahip olabilir. Bu senaryo, etkilenen sistemlerde uzaktan kod yürütülmesine neden olabilir. Bu güvenlik açığından başarıyla yararlanan bir saldırgan program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

Bu güvenlik güncelleştirmesi Microsoft Windows 2000, Windows XP ve Windows Server 2003'ün tüm desteklenen sürümleri için Önemli ve Windows Vista ve Windows Server 2008'in tüm desteklenen sürümleri için Orta olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, kimlik bilgilerinin geçirilmesini önlemek üzere Telnet hizmetinin kimlik doğrulaması yanıtlarını doğrulama biçimini değiştirerek bu güvenlik açığını ortadan kaldırır. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** [Microsoft Bilgi Bankası makalesi 960859](http://support.microsoft.com/kb/960859), bu güvenlik güncelleştirmesini yükleyen müşterilerin karşılaşabileceği bilinen sorunları belgelemektedir. Bu makale ayrıca, bu sorunlar için önerilen çözümleri de içermektedir.

Telnet'te Kimlik Bilgilerini Yansıtma Güvenlik Açığı - CVE-2009-1930  
--------------------------------------------------------------------
  
<span></span>
Microsoft Telnet hizmetinde bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2009-1930](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-1930).
  
#### Telnet'te Kimlik Bilgilerini Yansıtma Güvenlik Açığı - CVE-2009-1930 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   En iyi güvenlik duvarı uygulamaları ve standart varsayılan güvenlik duvarı yapılandırmaları, ağ yapılarının kuruluş dışından gerçekleştirilen saldırılardan korunmasına yardımcı olabilir. En iyi uygulamalar, Internet'e bağlı sistemlerde olabildiğince az sayıda bağlantı noktasının açılmasını önerir. Bu durumda, kimlik bilgilerini geri yansıtmak için en olası hedef olan SMB bağlantı noktasına Internet'ten erişim engellenmelidir.  
-   Bu güvenlik açığından başarıyla yararlanan bir saldırgan, yerel kullanıcı ile aynı haklara sahip olabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.  
-   Telnet varsayılan olarak Windows Vista ve Windows Server 2008 tabanlı sistemlerde yüklü değildir.
  
#### Telnet'te Kimlik Bilgilerini Yansıtma Güvenlik Açığı - CVE-2009-1930 için Geçici Çözümler
  
Microsoft bu güvenlik açığı için herhangi bir geçici çözüm belirleyememiştir.
  
#### Telnet'te Kimlik Bilgilerini Yansıtma Güvenlik Açığı - CVE-2009-1930 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
Telnet protokolü, kullanıcı kimlik bilgilerinin geri yansıtılmamasını ve kullanıcıya karşı kullanılmamasını sağlamak üzere NTLM kimlik bilgilerini yansıtmaya karşı koruma seçeneğini doğru şekilde kullanmaz.
  
**Telnet nedir?**  
Telnet, komut satırına TCP protokolü üzerinden uzaktan erişilmesine olanak veren çift yönlü bir iletişim protokolüdür. Daha fazla bilgi için, [Telnet Sunucusu](http://msdn.microsoft.com/en-us/library/aa919240.aspx) adlı MSDN makalesine bakın.
  
**NTLM nedir?**  
NTLM, sağlanan kimlik bilgilerinin orijinalliğini belirlemek üzere sınama/yanıt mekanizmasına dayalı bir kimlik doğrulama protokolüdür.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bir saldırgan bu güvenlik açığını kullanarak, oturum açmış kullanıcıyla aynı haklara sahip olabilir ve bu kullanıcının ayrıcalığı olan tüm işlemleri gerçekleştirebilir. Kullanıcı yönetimsel haklarla oturum açtıysa, bu güvenlik açığından başarıyla yararlanan bir saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Bu güvenlik açığı, Telnet'in etkilenen bir sürümünün bulunduğu bir kullanıcının özel hazırlanmış bir Telnet sunucusuna erişmesini gerektirir. Saldırganın bu güvenlik açığından yararlanabilmesi için özel hazırlanmış bir Telnet sunucusu barındırması gerekir. Saldırganın, bir kullanıcıyı özel hazırlanmış bir Telnet sunucusuna bağlanmaya zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki özel hazırlanmış bir bağlantıyı tıklatarak onları bu özel hazırlanmış Telnet sunucusuna gitmeye ikna etmesi gerekir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Windows sistemlerinin günümüzde desteklenen tüm sürümleri risk altındadır.
  
**Bu güvenlik açığının önem derecesi neden Windows** **Vista** **ve Windows Server 2008 için Önemli, tüm diğer platformlar için Kritik olarak belirlenmiştir?**  
Telnet istemcisi varsayılan olarak Windows Vista ve Windows Server 2008'in desteklenen sürümlerinde yüklü değildir. Yalnızca Telnet'i el ile yüklediyseniz bilgisayarınız bu bültende açıklanan güvenlik açığından etkilenir. Windows'un tüm diğer desteklenen sürümlerinde Telnet varsayılan olarak yüklüdür.
  
**Güncelleştirme ne yapıyor?**  
Güvenlik güncelleştirmesi, Microsoft Telnet Hizmetleri'nin Windows tarafından sunulan kimlik bilgileri koruma mekanizmalarını doğru şekilde seçmesini sağlayarak bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Evet. Bu güvenlik açığı genel olarak duyurulmuştur. Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE-2009-1930](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-1930) atanmıştır.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.
