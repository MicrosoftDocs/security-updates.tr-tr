Security Bulletin

Microsoft Güvenlik Bülteni MS09-018 - Kritik
============================================

Active Directory'deki Güvenlik Açıkları Uzaktan Kod Yürütülmesine İzin Verebilir (971055)
-----------------------------------------------------------------------------------------

Yayım Tarihi: 9 Haziran 2009 Salı | Güncelleştirme Tarihi: 17 Haziran 2009 Çarşamba

**Sürüm:** 1.1

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi, Microsoft Windows 2000 Server ve Windows Server 2003 üzerinde Active Directory ile Windows XP Professional ve Windows Server 2003 üzerinde Active Directory Uygulama Modu (ADAM) uygulamalarında özel olarak bildirilen iki güvenlik açığını giderir. Daha ciddi olan güvenlik açığı uzaktan kod yürütülmesine olanak verebilir. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, etkilenen sistemin tüm denetimini uzaktan ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. En iyi güvenlik duvarı uygulamaları ve standart varsayılan güvenlik duvarı yapılandırmaları, ağ yapılarının kuruluş dışından gerçekleştirilen saldırılardan korunmasına yardımcı olabilir. En iyi uygulamalar, Internet'e bağlı sistemlerde olabildiğince az sayıda bağlantı noktasının açılmasını önerir.

Güvenlik güncelleştirmesi Microsoft Windows 2000 Server'ın tüm desteklenen sürümleri için Kritik, Windows XP Professional'ın ve Windows Server 2003'ün desteklenen sürümleri için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, LDAP hizmetinin özel hazırlanmış LDAP veya LDAPS isteklerini işlediği sırada bellek ayırma ve belleği bırakma biçimini düzelterek güvenlik açığını giderir.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi hemen uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** Yok

Yazılımınızın sürümünün destek ömrünü belirlemek için [Microsoft Destek Ömrü](http://go.microsoft.com/fwlink/?linkid=21742) Web sitesini ziyaret edin.

-----------------------------------------------------------------

<span></span>
**Dosya bilgileri ayrıntılarını nerede bulabilirim?**  
Dosya bilgileri ayrıntılarının konumu için **Güvenlik Güncelleştirmesi Dağıtımı** bölümündeki başvuru tablolarına bakın.

**Bu güncelleştirme neden bildirilen birkaç güvenlik açığını ele alıyor?**  
Bu güncelleştirme, bazı güvenlik açıklarını ortadan kaldırmaya yönelik değişikliklerin ilgili dosyalarda bulunması nedeniyle bu sorunlar için destek içermektedir. Müşterilerimiz birbiriyle neredeyse aynı birkaç güncelleştirme yerine, yalnızca bu güncelleştirmeyi yüklemelidir.

**Bu güvenlik bülteninde açıklanan yazılımın eski bir sürümünü kullanıyorum. Ne yapmalıyım?**  
Bu bültende listelenen etkilenen yazılımlar, hangi sürümlerinin etkilendiğini belirlemek amacıyla sınanmıştır. Diğer sürümler destek ömrünü tamamlamıştır. Yazılım sürümünüzün destek ömrünü belirlemek için [Microsoft Destek Ömrü](http://go.microsoft.com/fwlink/?linkid=21742) Web sitesini ziyaret edin.

Bu yazılımların eski sürümlerini kullanan müşterilerimiz, ortaya çıkabilecek güvenlik açıklarından etkilenme olasılığını ortadan kaldırmak için desteklenen sürümlere acilen yükseltmelidir. Windows Ürün Destek Ömrü hakkında daha fazla bilgi için [Microsoft Destek Ömrü](http://go.microsoft.com/fwlink/?linkid=21742) Web sitesini ziyaret edin. Bu yazılım sürümleri için genişletilmiş güvenlik güncelleştirmesi destek süreci hakkında daha fazla bilgi için, [Microsoft Ürün Destek Hizmetleri](http://go.microsoft.com/fwlink/?linkid=33328)'ni ziyaret edin.

Başvuru bilgileri için, [Microsoft Worldwide Information](http://go.microsoft.com/fwlink/?linkid=33329) Web sitesini ziyaret edin, bulunduğunuz ülkeyi seçin ve **Go**'yu (Git) tıklatıp telefon numaralarının listesini görüntüleyin. Daha fazla bilgi için, bkz: [Windows İşletim Sistemlerinin Ürün Destek Ömrü Hakkında SSS](http://go.microsoft.com/fwlink/?linkid=33330).

Active Directory'de Geçersiz Boşaltma Güvenlik Açığı - CVE-2009-1138  
--------------------------------------------------------------------
  
<span></span>
Microsoft Windows 2000 Server üzerine yüklendiğinde Active Directory uygulamalarında bir uzaktan kod yürütme güvenlik açığı bulunmaktadır. Güvenlik açığı, özel hazırlanmış LDAP veya LDAPS istekleri işlenirken belleğin hatalı şekilde bırakılması nedeniyle oluşur. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2009-1138](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-1138).
  
#### Active Directory'de Geçersiz Boşaltma Güvenlik Açığı - CVE-2009-1138 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   En iyi güvenlik duvarı uygulamaları ve standart varsayılan güvenlik duvarı yapılandırmaları, ağ yapılarının kuruluş dışından gerçekleştirilen saldırılardan korunmasına yardımcı olabilir. En iyi uygulamalar, Internet'e bağlı sistemlerde olabildiğince az sayıda bağlantı noktasının açılmasını önerir.
  
#### Active Directory'de Geçersiz Boşaltma Güvenlik Açığı - CVE-2009-1138 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Güvenlik duvarında 389, 636, 3268 ve 3269 numaralı TCP bağlantı noktalarını engelleyin**
  
    Güvenlik duvarında aşağıdakileri engelleyin:
  
    389, 636, 3268 ve 3269 numaralı TCP bağlantı noktaları
  
    Bu bağlantı noktaları, etkilenen bileşenle bağlantı başlatmak için kullanılır. Kuruluşun güvenlik duvarında bu bağlantı noktalarının gelen ve giden trafik için engellenmesi, güvenlik duvarının arkasındaki sistemlerin bu açıktan yararlanma girişimlerini engellemeye yardımcı olur. 389 ve 636 numaralı bağlantı noktaları sırasıyla LDAP ve güvenli LDAP hizmetlerini sağlar; 3268 ve 3269 numaralı bağlantı noktaları ise LDAP isteklerini de işleyen Genel Katalog sunucusu tarafından kullanılır.
  
    Microsoft, diğer bağlantı noktalarını kullanabilecek saldırılardan korunmaya yardımcı olmak amacıyla, Internet'ten gelen tüm istenmeyen trafiğin engellenmesini önerir. Bağlantı noktaları hakkında daha fazla bilgi için, bkz. [TCP ve UDP Bağlantı Noktası Atamaları](http://go.microsoft.com/fwlink/?linkid=21312). Windows Güvenlik Duvarı hakkında daha fazla bilgi için, bkz: [Tek Başına Bir Bilgisayarda Windows Güvenlik Duvarı'nı Yapılandırma](http://www.microsoft.com/technet/security/smallbusiness/prodtech/windowsxp/cfgfwall.mspx).
  
-   **Microsoft Windows 2000 sunucularında anonim LDAP erişimini devre dışı bırakın**
  
    Microsoft Windows 2000 sunucularında anonim LDAP erişimi devre dışı bırakıldığında, bu güvenlik açığından yalnızca kimliği doğrulanmış kullanıcılar yararlanabilir. LDAP NULL taban (anonim) bağlantıların devre dışı bırakılmasının etkileri de dahil bu bağlantılar hakkında daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 837964](http://support.microsoft.com/kb/837964).
  
#### Active Directory'de Geçersiz Boşaltma Güvenlik Açığı - CVE-2009-1138 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, etkilenen sistemin tüm denetimini uzaktan ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
LDAP hizmeti özel hazırlanmış LDAP veya LDAPS isteklerini işlerken belleği hatalı şekilde bırakmaktadır.
  
**Active Directory nedir?**  
Active Directory'nin başlıca amacı, Windows tabanlı bilgisayarlar için merkezi kimlik doğrulama ve yetkilendirme hizmetlerini sağlamaktır.
  
**LDAP nedir?**  
Basit Dizin Erişimi Protokolü (LDAP), dağıtılmış dizinlere erişim sağlamak üzere tasarlanmış bir açık ağ protokolü standardıdır.
  
**SSL üzerinden LDAP (LDAPS) nedir?**  
LDAP trafiği varsayılan olarak güvenli olmayan şekilde iletilir. Ancak, Güvenli Yuva Katmanı (SSL) / Aktarım Katmanı Güvenliği (TLS) teknolojisi kullanılarak LDAP trafiği gizlenebilir ve değişikliklere karşı korunabilir. Yöneticiler bir Microsoft sertifika yetkilisinden (CA) veya Microsoft dışındaki bir CA'dan edindikleri düzgün şekilde biçimlendirilmiş bir sertifika yükleyerek SSL üzerinden LDAP (LDAPS) özelliğini etkinleştirebilir. Daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 321051](http://support.microsoft.com/kb/321051).
  
**SSL üzerinden LDAP varsayılan olarak etkinleştirilmiş midir?**  
Hayır. LDAP sunucusunun bir SSL oturumuna katılabilmesi için yöneticinin öncelikle bir dijital sertifika edinmiş ve sunucuya yüklemiş olması gerekmektedir. Bu işlemler yapılmamışsa, SSL üzerinden LDAP kullanılamaz.
  
**Sunucumda LDAP hizmetinin çalışmakta olduğunu nasıl doğrulayabilirim**?    
Bir sunucunun LDAP veya LDAPS sorgularını dinlediğini onaylamak için, aşağıdaki komutu bir yönetici komut isteminden çalıştırın ve LDAP bağlantı noktasını (389), LDAPS bağlantı noktasını (636) ya da Genel Katalog hizmeti bağlantı noktalarından birini (3268 veya 3269) dinleyip dinlemediğine bakın:  
netstat –a  
Sonuç aşağıdakilerden birini içeriyorsa LDAP etkinleştirilmiştir:  
Proto Local Address Foreign Address State  
TCP 0.0.0.0:389 0.0.0.0:0 LISTENING  
TCP 0.0.0.0:636 0.0.0.0:0 LISTENING  
TCP 0.0.0.0:3268 0.0.0.0:0 LISTENING  
TCP 0.0.0.0:3269 0.0.0.0:0 LISTENING
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan yönetici ayrıcalıklarıyla rasgele kod çalıştırabilir. Böylece saldırgan program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir ya da yeni hesaplar oluşturabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırgan, Microsoft Windows 2000 çalıştırılan bir etki alanı denetleyicisine özel hazırlanmış bir LDAP veya LDAPS paketi göndermek yoluyla bu güvenlik açığından yararlanmaya çalışabilir. Hedef ağa erişimi olan herhangi bir anonim kullanıcı etkilenen sisteme özel hazırlanmış bir ağ paketi teslim ederek bu güvenlik açığından yararlanabilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
LDAP, Active Directory için gereklidir. Active Directory Server rolüyle Microsoft Windows 2000 Server çalıştırılan sunucular bu güvenlik açığından etkilenir.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, LDAP hizmetinin özel hazırlanmış istemci LDAP veya LDAPS isteklerini işlediği sırada belleği bırakma biçimini düzelterek güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.
  
Active Directory'de Bellek Sızıntısı Güvenlik Açığı - CVE-2009-1139  
-------------------------------------------------------------------
  
<span></span>
Microsoft Windows 2000 Server ve Windows Server 2003 üzerine yüklendiğinde Active Directory'de bir hizmet reddi güvenlik açığı bulunmaktadır. Güvenlik açığı, Windows XP Professional ve Windows Server 2003 üzerine yüklendiğinde Active Directory Uygulama Modu (ADAM) uygulamalarında da bulunmaktadır. Bu güvenlik açığı, LDAP veya LDAPS isteklerinin belirli türlerinin yürütülmesi sırasında hatalı bellek yönetimi nedeniyle oluşur. Bu güvenlik açığından yararlanan bir saldırgan, etkilenen sunucunun yanıt vermemesine neden olabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2009-1139](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-1139).
  
#### Active Directory'de Bellek Sızıntısı Güvenlik Açığı - CVE-2009-1139 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   En iyi güvenlik duvarı uygulamaları ve standart varsayılan güvenlik duvarı yapılandırmaları, ağ yapılarının kuruluş dışından gerçekleştirilen saldırılardan korunmasına yardımcı olabilir. En iyi uygulamalar, Internet'e bağlı sistemlerde olabildiğince az sayıda bağlantı noktasının açılmasını önerir.
  
#### Active Directory'de Bellek Sızıntısı Güvenlik Açığı - CVE-2009-1139 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Güvenlik duvarında 389, 636, 3268 ve 3269 numaralı TCP bağlantı noktalarını engelleyin**
  
    Güvenlik duvarında aşağıdakileri engelleyin:
  
    389, 636, 3268 ve 3269 numaralı TCP bağlantı noktaları
  
    Bu bağlantı noktaları, etkilenen bileşenle bağlantı başlatmak için kullanılır. Kuruluşun güvenlik duvarında bu bağlantı noktalarının gelen ve giden trafik için engellenmesi, güvenlik duvarının arkasındaki sistemlerin bu açıktan yararlanma girişimlerini engellemeye yardımcı olur. 389 ve 636 numaralı bağlantı noktaları sırasıyla LDAP ve güvenli LDAP hizmetlerini sağlar; 3268 ve 3269 numaralı bağlantı noktaları ise LDAP isteklerini de işleyen Genel Katalog sunucusu tarafından kullanılır.
  
    Microsoft, diğer bağlantı noktalarını kullanabilecek saldırılardan korunmaya yardımcı olmak amacıyla, Internet'ten gelen tüm istenmeyen trafiğin engellenmesini önerir. Bağlantı noktaları hakkında daha fazla bilgi için, bkz. [TCP ve UDP Bağlantı Noktası Atamaları](http://go.microsoft.com/fwlink/?linkid=21312). Windows Güvenlik Duvarı hakkında daha fazla bilgi için, bkz: [Tek Başına Bir Bilgisayarda Windows Güvenlik Duvarı'nı Yapılandırma](http://www.microsoft.com/technet/security/smallbusiness/prodtech/windowsxp/cfgfwall.mspx).
  
-   **Microsoft Windows 2000 sunucularında anonim LDAP erişimini devre dışı bırakın**
  
    Microsoft Windows 2000 sunucularında anonim LDAP erişimi devre dışı bırakıldığında, bu güvenlik açığından yalnızca kimliği doğrulanmış kullanıcılar yararlanabilir. LDAP NULL taban (anonim) bağlantıların devre dışı bırakılmasının etkileri de dahil bu bağlantılar hakkında daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 837964](http://support.microsoft.com/kb/837964).
  
#### Active Directory'de Bellek Sızıntısı Güvenlik Açığı - CVE-2009-1139 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir hizmet reddi güvenlik açığıdır. Bu güvenlik açığından yararlanan bir saldırgan, sistemin yanıt vermemesine ve yeniden başlatılmasının gerekmesine neden olabilir. Hizmet reddi güvenlik açığının, bir saldırganın kod yürütmesine veya kullanıcı haklarını yükseltmesine izin vermeyeceğini, ancak etkilenen sistemin istekleri kabul etmeyi durdurmasına neden olabileceğini unutmayın.
  
**Güvenlik açığına ne neden olmaktadır?**  
LDAP hizmeti belirli OID filtreleri içeren LDAP veya LDAPS isteklerini işlerken belleği hatalı şekilde yönetir.
  
**LDAP nedir?**  
Basit Dizin Erişimi Protokolü (LDAP), dağıtılmış dizinlere erişim sağlamak üzere tasarlanmış bir açık ağ protokolü standardıdır.
  
**SSL üzerinden LDAP (LDAPS) nedir?**  
LDAP trafiği varsayılan olarak güvenli olmayan şekilde iletilir. Ancak, Güvenli Yuva Katmanı (SSL) / Aktarım Katmanı Güvenliği (TLS) teknolojisi kullanılarak LDAP trafiği gizlenebilir ve güvenli olması sağlanabilir. Yöneticiler bir Microsoft sertifika yetkilisinden (CA) veya Microsoft dışındaki bir CA'dan edindikleri düzgün şekilde biçimlendirilmiş bir sertifika yükleyerek SSL üzerinden LDAP (LDAPS) özelliğini etkinleştirebilir. Daha fazla bilgi için, bkz: [Microsoft Bilgi Bankası makalesi 321051](http://support.microsoft.com/kb/321051).
  
**SSL üzerinden LDAP varsayılan olarak etkinleştirilmiş midir?**  
Hayır. LDAP sunucusunun bir SSL oturumuna katılabilmesi için yöneticinin öncelikle bir dijital sertifika edinmiş ve sunucuya yüklemiş olması gerekmektedir. Bu işlemler yapılmamışsa, SSL üzerinden LDAP kullanılamaz.
  
**Sunucumda LDAP hizmetinin çalışmakta olduğunu nasıl doğrulayabilirim?**  
Bir sunucunun LDAP veya LDAPS sorgularını dinlediğini onaylamak için, aşağıdaki komutu bir yönetici komut isteminden çalıştırın ve LDAP bağlantı noktasını (389), LDAPS bağlantı noktasını (636) ya da Genel Katalog hizmeti bağlantı noktalarından birini (3268 veya 3269) dinleyip dinlemediğine bakın:  
netstat –a  
Sonuç aşağıdakilerden birini içeriyorsa LDAP etkinleştirilmiştir:  
Proto Local Address Foreign Address State  
TCP 0.0.0.0:389 0.0.0.0:0 LISTENING  
TCP 0.0.0.0:636 0.0.0.0:0 LISTENING  
TCP 0.0.0.0:3268 0.0.0.0:0 LISTENING  
TCP 0.0.0.0:3269 0.0.0.0:0 LISTENING
  
**OID filtreleri nedir?**  
OID veya Nesne Tanımlayıcısı, bir LDAP nesnesini adlandırmak için kullanılır. LDAP sorgusu bir LDAP ad alanındaki yalnızca belirli bir OID ile eşleşen verileri seçmek üzere bir filtre içerebilir.
  
**Active Directory Uygulama Modu (ADAM) nedir?**  
ADAM bir sistem hizmeti olarak değil, bir kullanıcı hizmeti olarak çalışan bir Basit Dizin Erişim Protokolü (LDAP) dizin hizmetidir. Daha fazla bilgi için, [ADAM Kavramları](http://technet2.microsoft.com/windowsserver/en/library/29fb059e-544c-4577-bf7c-ba4b08df48431033.mspx?mfr=true) adlı TechNet makalesine bakın.
  
**Active Directory Basit Dizin Hizmeti (AD LDS) bu güvenlik açığından etkileniyor mu?**  
Active Directory Basit Dizin Hizmeti, uygulamalar için adanmış dizin hizmetleri sağlayan bağımsız bir Active Directory modudur. AD LDS şu anda yalnızca Microsoft Windows Server 2008 İşletim Sistemi'nde kullanılabilmektedir ve bu güvenlik açığından etkilenmez.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, etkilenen sistemin yanıt vermemesine ve yeniden başlatılması gerekmesine neden olabilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırgan, Active Directory veya ADAM sunucusuna özel hazırlanmış bir LDAP veya LDAPS paketi göndererek bu güvenlik açığından yararlanmaya çalışabilir. Microsoft Windows 2000 Server için, hedef ağa erişimi olan herhangi bir anonim kullanıcı etkilenen sisteme özel hazırlanmış bir ağ paketi teslim ederek bu güvenlik açığından yararlanabilir. Windows Server 2003'te veya ADAM yüklenmiş olan sistemlerde, saldırganın bu güvenlik açığından yararlanabilmesi için geçerli kimlik doğrulama bilgileri olması gerekir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Basit Dizin Erişim Protokolü (LDAP), Active Directory ve ADAM için gerekmektedir. Bu nedenle, Active Directory veya ADAM sunucusu olarak çalışan tüm sistemler bu güvenlik açığından etkilenir.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, LDAP hizmetinin özel hazırlanmış LDAP veya LDAPS isteklerini işlediği sırada bellek ayırma biçimini düzelterek bu güvenlik açığını giderir.

Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı.

Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.

-   **Dosya Sürümü Doğrulaması**

    Microsoft Windows'un birçok sürümü olduğundan, aşağıdaki adımlar sizin sisteminizde farklı olabilir. Böyle bir durumda, bu adımları tamamlamak için ürün belgelerinize bakın.

    1.  **Başlat**'ı ve sonra **Ara**'yı tıklatın.
    2.  **Arama Sonuçları** bölmesinde, **Arama Yardımcısı**'nın altındaki **Tüm dosya ve klasörler**'i tıklatın.
    3.  **Dosya adının tümü veya bir parçası** kutusuna, uygun dosya bilgileri tablosundan bir dosya adı yazın ve **Ara**'yı tıklatın.
    4.  Dosya listesinde, uygun dosya bilgileri tablosundan bir dosya adını sağ tıklatın ve sonra **Özellikler**'i tıklatın.
        **Not** Sisteminizde yüklü işletim sisteminin veya programların sürümüne bağlı olarak, dosya bilgileri tablosunda listelenen dosyalardan bazıları yüklenmeyebilir.
    5.  **Sürüm** sekmesinde, sisteminize yüklenmiş olan dosyanın sürümünü uygun dosya bilgileri tablosunda belgelenen sürümle karşılaştırarak belirleyin.
        **Not** Dosya sürümü dışındaki öznitelikler yükleme sırasında değişebilir. Diğer dosya özniteliklerini dosya bilgileri tablosundaki bilgilerle karşılaştırmak, güncelleştirmenin uygulandığını doğrulama için desteklenen bir yöntem değildir. Ayrıca, belirli durumlarda, dosyalar kurulum sırasında yeniden adlandırılabilir. Dosya veya sürüm bilgileri yoksa, güncelleştirme kurulumunu kullanılabilen diğer yöntemlerden biriyle doğrulayın.
