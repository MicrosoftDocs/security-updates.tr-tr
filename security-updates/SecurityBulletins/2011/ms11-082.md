Security Bulletin

Microsoft Güvenlik Bülteni MS11-082 - Önemli
============================================

Host Integration Server'daki Güvenlik Açıkları Uzaktan Kod Yürütülmesine İzin Verebilir (2607670)
-------------------------------------------------------------------------------------------------

Yayım Tarihi: 11 Ekim 2011 Salı

**Sürüm:** 1.0

### Genel Bilgiler

#### **Yürütmeyle İlgili Özet**

Bu güvenlik güncelleştirmesi, Host Integration Server'daki genel olarak duyurulan iki güvenlik açığını giderir. Bu güvenlik açıkları, uzaktaki bir saldırgan 1478 numaralı UDP bağlantı noktasını veya 1477 ve 1478 numaralı TCP bağlantı noktalarını dinleyen bir Host Integration Server'a özel hazırlanmış ağ paketleri gönderirse hizmet reddine olanak verebilir. En iyi güvenlik duvarı uygulamaları ve standart varsayılan güvenlik duvarı yapılandırmaları, ağ yapılarının kuruluş dışından gerçekleştirilen saldırılardan korunmasına yardımcı olabilir. En iyi uygulamalar, Internet'e bağlı sistemlerde olabildiğince az sayıda bağlantı noktasının açılmasını önerir. Bu durumda, Host Integration Server bağlantı noktalarına Internet'ten erişim engellenmelidir.

Bu güvenlik güncelleştirmesi Microsoft Host Integration Server 2004, Microsoft Host Integration Server 2006, Microsoft Host Integration Server 2009 ve Microsoft Host Integration Server 2010'un tüm desteklenen sürümleri için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, Host Integration Server'ın özel hazırlanmış UDP ve TCP paketlerini işleme biçimini değiştirerek bu güvenlik açıklarını giderir. Bu güvenlik açıkları hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi ilk fırsatta uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** Yok

Snabase.exe Dosyasında Sonsuz Döngü Nedeniyle DoS Güvenlik Açığı - CVE-2011-2007  
--------------------------------------------------------------------------------
  
<span></span>
Host Integration Server'ın bazı UDP ve TCP ağ trafiğini işleme biçiminde kimliği doğrulanmamış bir hizmet reddi güvenlik açığı bulunmaktadır. Bu güvenlik açığı, uzaktaki bir kullanıcı tarafından snabase.exe, snaserver.exe, snalink.exe ya da mngagent.exe dosyasının yeni isteklere yanıt vermemeye başlamasının sağlanmasına olanak verebilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2011-2007](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-2007).
  
#### Snabase.exe Dosyasında Sonsuz Döngü Nedeniyle DoS Güvenlik Açığı - CVE-2011-2007 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   En iyi güvenlik duvarı uygulamaları ve standart varsayılan güvenlik duvarı yapılandırmaları, ağ yapılarının kuruluş dışından gerçekleştirilen saldırılardan korunmasına yardımcı olabilir. En iyi uygulamalar, Internet'e bağlı sistemlerde olabildiğince az sayıda bağlantı noktasının açılmasını önerir. Bu durumda, Host Integration Server bağlantı noktalarına Internet'ten erişim engellenmelidir.
  
#### Snabase.exe Dosyasında Sonsuz Döngü Nedeniyle DoS Güvenlik Açığı - CVE-2011-2007 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Güvenlik duvarınızda1477** **ve** **1478numaralı TCP bağlantı noktaları ile** **1478** **numaralı UDP bağlantı noktasını engelleyin**
  
    Bu bağlantı noktaları, etkilenen bileşenle bağlantı başlatmak için kullanılır. Kuruluşun güvenlik duvarında bu bağlantı noktalarının gelen ve giden trafik için engellenmesi, güvenlik duvarının arkasındaki sistemlerin bu açıktan yararlanma girişimlerine karşı korunmasına yardımcı olur. Diğer bağlantı noktalarını kullanabilecek saldırılardan korunmaya yardımcı olmak amacıyla, Internet'ten gelen tüm istenmeyen trafiğin engellenmesini öneririz. Bağlantı noktaları hakkında daha fazla bilgi için, [TCP ve UDP Bağlantı Noktası Atamaları](http://go.microsoft.com/fwlink/?linkid=21312) adlı TechNet makalesine bakın. Windows Güvenlik Duvarı hakkında daha fazla bilgi için, [Tek Başına Bir Bilgisayarda Windows Güvenlik Duvarı'nı Yapılandırma](http://www.microsoft.com/technet/security/smallbusiness/prodtech/windowsxp/cfgfwall.mspx) adlı TechNet makalesine bakın.
  
#### Snabase.exe Dosyasında Sonsuz Döngü Nedeniyle DoS Güvenlik Açığı - CVE-2011-2007 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir hizmet reddi güvenlik açığıdır. Bu güvenlik açığından uzaktan yararlanan anonim bir saldırgan, etkilenen SNA Server hizmetinin ve tüm bağımlı hizmetlerin yanıt vermemeye başlamasına neden olabilir. Bu güvenlik açığı, bir saldırganın kod yürütmesine veya kullanıcı haklarını yükseltmesine izin vermez.
  
**Güvenlik açığına ne neden olmaktadır?**  
Bu güvenlik açığı, Host Integration Server'ın özel hazırlanmış ağ trafiğini işlerken giriş doğrulamasını hatalı gerçekleştirmesi nedeniyle ortaya çıkar.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından uzaktan yararlanan anonim bir saldırgan, etkilenen sistemin SNA Server hizmetinin yanıt vermemeye başlamasına neden olabilir.
  
**Bir saldırgan bu güvenlik açığından nasıl yararlanabilir?**  
Uzaktaki bir saldırgan, 1478 numaralı UDP bağlantı noktasını veya 1477 ve 1478 numaralı TCP bağlantı noktalarını dinleyen bir Host Integration Server'a özel hazırlanmış ağ paketleri göndererek bu güvenlik açığından yararlanabilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Host Integration Server (HIS) çalıştırılan sunucular.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, Host Integration Server'ın özel hazırlanmış UDP ve TCP paketlerini işleme biçimini değiştirerek bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Evet. Bu güvenlik açığı genel olarak duyurulmuştur. Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE-2011-2007](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-2007) atanmıştır.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının müşterilere saldırmak üzere genel olarak kullanıldığına dair herhangi bir bilgi almamıştı.

Uzaktaki bir saldırgan, 1478 numaralı UDP bağlantı noktasını veya 1477 ve 1478 numaralı TCP bağlantı noktalarını dinleyen bir Host Integration Server'a özel hazırlanmış ağ paketleri göndererek bu güvenlik açığından yararlanabilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Host Integration Server (HIS) çalıştırılan sunucular.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, Host Integration Server'ın özel hazırlanmış UDP ve TCP paketlerini işleme biçimini değiştirerek bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Evet. Bu güvenlik açığı genel olarak duyurulmuştur. Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE-2011-2008](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-2008) atanmıştır.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının müşterilere saldırmak üzere genel olarak kullanıldığına dair herhangi bir bilgi almamıştı.

    3.  **Dosya adının tümü veya bir parçası** kutusuna, uygun dosya bilgileri tablosundan bir dosya adı yazın ve **Ara**'yı tıklatın.
