Security Bulletin

Microsoft Güvenlik Bülteni MS10-075 - Kritik
============================================

Media Player Ağ Paylaşımı Hizmeti'ndeki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (2281679)
---------------------------------------------------------------------------------------------------------

Yayım Tarihi: 12 Ekim 2010 Salı

**Sürüm:** 1.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Microsoft Windows Media Player Ağ Paylaşımı Hizmeti'ndeki özel olarak bildirilen bir güvenlik açığını giderir. Bu güvenlik açığı, bir saldırgan etkilenen bir sisteme özel hazırlanmış bir RTSP paketi gönderirse uzaktan kod yürütülmesine izin verebilir. Ancak ev medyasına Internet üzerinden erişim varsayılan olarak devre dışı bırakılmış durumdadır. Bu varsayılan yapılandırmada, güvenlik açığından yalnızca aynı alt ağ içindeki bir saldırgan tarafından yararlanılabilir.

Bu güvenlik güncelleştirmesi Windows 7'nin desteklenen sürümleri için Kritik ve Windows Vista'nın tüm desteklenen sürümleri için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, Windows Media Player Ağ Paylaşımı Hizmeti'nin özel hazırlanmış RTSP paketlerini işleme biçimini düzelterek bu güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi hemen uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** Yok

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)
-----------------------------------------------------------------

<span></span>
**Dosya bilgileri ayrıntılarını nerede bulabilirim?**  
Dosya bilgileri ayrıntılarının konumu için **Güvenlik Güncelleştirmesi Dağıtımı** bölümündeki başvuru tablolarına bakın.

**Microsoft Windows'un etkilenen bir sürümünü çalıştırıyorum, ancak Windows Media Player Ağ Paylaşımı Hizmeti başlatılmamış durumda. Bu güncelleştirme sunulacak mı?**  
Evet; bu güncelleştirme, Windows Media Player Ağ Paylaşımı Hizmeti'nin başlatılmış olup olmadığına bakılmaksızın, etkilenen bir Windows işletim sistemi sürümünün çalıştırıldığı tüm sistemlere sunulacaktır.

**Neden Windows Vista Önemli, Windows 7 ise Kritik olarak derecelendirilmiş?**  
Windows 7 Home Basic ve Windows 7 Home Premium sürümlerinde Media Player Ağ Paylaşımı Hizmeti varsayılan olarak etkinleştirilmiş durumdadır; bu nedenle, güvenlik açığı Kritik olarak derecelendirilmiştir. Windows 7 Professional, Windows 7 Enterprise ve Windows 7 Ultimate sürümlerinde bu hizmet varsayılan olarak etkinleştirilmemiş durumdadır.

Windows Vista'nın etkilenen sürümlerinde, bu hizmet varsayılan olarak devre dışı bırakılmış durumdadır. Bu azaltıcı etken, Windows Vista'nın desteklenen sürümleri için önem derecesini Önemli düzeyine düşürür.

**Windows 7 Service Pack 1 Beta sürümü bu güvenlik açığından nasıl etkileniyor?**  
Windows 7 Service Pack 1 Beta bu bültende açıklanan güvenlik açığından etkilenir. Bu beta sürümlerini çalıştıran müşterilerin güncelleştirmeyi karşıdan yükleyip sistemlerine uygulamaları önerilir. Güvenlik güncelleştirmeleri [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) ve [Windows Update](http://go.microsoft.com/fwlink/?linkid=21130) sitelerinden edinilebilir. Güvenlik güncelleştirmesi [Microsoft Yükleme Merkezi](http://go.microsoft.com/fwlink/?linkid=21129)'nden de karşıdan yüklenebilir.

**Bu güvenlik bülteninde açıklanan yazılımın eski bir sürümünü kullanıyorum. Ne yapmalıyım?**  
Bu bültende listelenen etkilenen yazılımlar, hangi sürümlerinin etkilendiğini belirlemek amacıyla sınanmıştır. Diğer sürümler destek ömrünü tamamlamıştır. Ürün destek ömrü hakkında daha fazla bilgi için, [Microsoft Destek Ömrü Web sitesini](http://go.microsoft.com/fwlink/?linkid=21742) ziyaret edin.

Bu yazılımların eski sürümlerini kullanan müşterilerimiz, ortaya çıkabilecek güvenlik açıklarından etkilenme olasılığını ortadan kaldırmak için desteklenen sürümlere acilen yükseltmelidir. Yazılım sürümünüzün destek ömrünü belirlemek için, bkz. [Destek Ömrü Bilgileri İçin Bir Ürün Seçin](http://go.microsoft.com/fwlink/?linkid=169555). RTSP'de Serbest Bıraktıktan Sonra Kullanma Güvenlik Açığı - CVE-2010-3225  
-------------------------------------------------------------------------
  
<span></span>
Microsoft Windows Media Player Ağ Paylaşımı Hizmeti'nde, uygulamanın ağ akışı hizmeti örneğine bir uzak kullanıcı tarafından özel hazırlanmış bir ağ paketi gönderilmesine ve geçerli uygulamanın bağlamında uzak kod yürütülmesine olanak verebilecek bir güvenlik açığı bulunmaktadır.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2010-3225](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3225).
  
#### RTSP'de Serbest Bıraktıktan Sonra Kullanma Güvenlik Açığı - CVE-2010-3225 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Ev medyasına Internet üzerinden erişim varsayılan olarak devre dışı bırakılmış durumdadır. Bu varsayılan yapılandırmada, güvenlik açığından yalnızca aynı alt ağ içindeki bilgisayarlar kullanılarak yararlanılabilir.  
-   Windows Vista'nın desteklenen sürümleri, Windows 7 Professional, Windows 7 Enterprise ve Windows 7 Ultimate sürümlerinde, Windows Media Player Ağ Paylaşımı Hizmeti varsayılan olarak devre dışı bırakılmış durumdadır.
  
#### RTSP'de Serbest Bıraktıktan Sonra Kullanma Güvenlik Açığı - CVE-2010-3225 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Windows Media Player Ağ Paylaşımı Hizmeti'ni etkilenen sistemlerde devre dışı bırakın**
  
    **Etkileşimli Yöntem**
  
    **Not** Kayıt Defteri Düzenleyicisi'nin yanlış kullanılması, işletim sisteminizi yeniden yüklemenizi gerektirebilecek ciddi sorunlara neden olabilir. Microsoft, Kayıt Defteri Düzenleyicisi'nin yanlış kullanılmasından kaynaklanan sorunların çözülebileceğini garanti etmemektedir. Kayıt Defteri Düzenleyicisi'ni kullanmak kendi sorumluluğunuzdadır. Kayıt defterini düzenlemeyle ilgili bilgi için, Kayıt Defteri Düzenleyicisi'ndeki (Regedit.exe) "Anahtarları ve Değerleri Değiştirmek " veya Regedt32.exe'deki "Kayıt Defterine Bilgi Eklemek veya Bilgileri Silmek " ve "Kayıt Defteri Verilerini Düzenlemek" adlı Yardım konularına bakın.
  
    1.  **Başlat**'ı tıklatın, **Çalıştır**'ı tıklatın, **Aç** kutusuna **Regedit** yazın ve **Tamam**'ı tıklatın.  
    2.  Aşağıdaki kayıt defteri anahtarını bulun ve seçin:  
        HKEY\_LOCAL\_MACHINE\\SYSTEM\\CurrentControlSet\\services\\WMPNetworkSvc  
    3.  **Dosya** menüsünü ve ardından **Ver**'i tıklatın.  
    4.  **Kayıt Defteri Dosyası Ver** iletişim kutusuna **WMPNetworkSvc\_Backup.reg** yazın ve **Kaydet**'i tıklatın.  
        **Not** Bu işlem, varsayılan olarak Belgelerim klasöründe bu kayıt defteri anahtarının bir yedeğini oluşturur.  
    5.  Kayıt Defteri Düzenleyicisi'nde sağdaki bölmeden **Start** değerini seçin. Anahtarın değerini düzenlemek üzere **Enter** tuşuna basın. **Değer verisi** alanını **4** olarak değiştirin ve **Enter** tuşuna basın.  
    6.  Ayrıcalığı yükseltilmiş bir komut istemine aşağıdaki komutu yazarak hizmeti durdurun:  
        `sc stop WMPNetworkSvc`
  
    **Yönetilen Dağıtım Komut Dosyası**
  
    1.  Aşağıdaki komutu kullanarak kayıt defteri anahtarının yedek bir kopyasını oluşturun:  
        `regedit /e WMPNetworkSvc_Backup.reg HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\WMPNetworkSvc`  
    2.  Aşağıdaki metni .REG uzantılı bir dosyaya kaydedin (ör. Disable\_WMPNetworkSvc.reg):  
        Windows Registry Editor Version 5.00  
        HKEY\_LOCAL\_MACHINE\\System\\CurrentControlSet\\Services\\hizmet\_adı\]  
        Start"=dword:00000004  
    3.  Aşağıdaki komutu kullanarak, 2. adımda oluşturulan kayıt defteri komut dosyasını hedef bilgisayarda çalıştırın:  
        `Regedit /s Disable_WMPNetworkSvc.reg`  
    4.  Ayrıcalığı yükseltilmiş bir komut istemine aşağıdaki komutu yazarak hizmeti durdurun:  
        `sc stop WMPNetworkSvc`
  
    **Geçici çözümün etkisi.** Windows Media Paylaşımı devre dışı bırakılır.
  
    **Geçici çözüm nasıl geri alınır.**
  
    Etkileşimli yöntemi kullanarak:
  
    1.  **Başlat**'ı tıklatın, **Çalıştır**'ı tıklatın, **Aç** kutusuna **Regedit** yazın ve **Tamam**'ı tıklatın.  
    2.  **Dosya** menüsünü ve ardından **Al**'ı tıklatın.  
    3.  **Kayıt Defteri Dosyası Al** iletişim kutusunda, yukarıdaki "Etkileşimli Yöntem" bölümünde oluşturduğunuz **WMPNetworkSvc\_Backup.reg** dosyasını seçin ve **Aç**'ı tıklatın.  
    4.  Kayıt Defteri Düzenleyicisi'nden çıkın ve bilgisayarı yeniden başlatın.
  
    Yönetilen dağıtım komut dosyası kullanarak:
  
    1.  Aşağıdaki komutu kullanarak, 2. adımda oluşturulan kayıt defteri komut dosyasını hedef bilgisayarda çalıştırın:  
        `Regedit /s WMPNetworkSvc_Backup.reg`  
    2.  Bilgisayarı yeniden başlatın.
  
#### RTSP'de Serbest Bıraktıktan Sonra Kullanma Güvenlik Açığı - CVE-2010-3225 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, Ağ Hizmeti hesabının güvenlik bağlamında kod yürütebilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
Güvenlik açığı, Microsoft Windows Media Player Ağ Paylaşımı Hizmeti'nin Gerçek Zamanlı Akış Protokolü (RTSP) paketlerini işleme biçimi nedeniyle ortaya çıkar.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırgan, özel hazırlanmış bir RTSP paketi oluşturup bu paketi etkilenen sisteme göndermek yoluyla bu güvenlik açığından yararlanabilir.
  
**RTSP nedir?**  
Gerçek Zamanlı Akış Protokolü (RTSP), gerçek zamanlı özelliklerle veri teslimi üzerinde denetim sağlamak için kullanılan uygulama düzeyinde bir protokoldür. RTSP, ses ve video gibi gerçek zamanlı verilerin isteğe bağlı olarak denetim altında teslim edilmesine olanak veren genişletilebilir bir altyapı sağlar. Veri kaynağı olarak hem canlı veri akışları hem de depolanan klipler kullanılabilir. Bu protokolün birden çok veri teslimi oturumunu denetlemesi; UDP, çok noktaya yayın UDP ve TCP gibi teslim kanallarını seçmenin bir yolunu sağlaması ve RTP'ye (RFC 1889) dayalı teslim mekanizmaları seçilebilmesini sağlaması amaçlanmıştır. RTSP, [RFC 2326](http://www.ietf.org/rfc/rfc2326.txt)'da tanımlanmıştır.
  
RTSP, dizin oluşturulmuş Windows Media dosyalarını durdurma, duraklatma, geri sarma ve ileri sarma gibi oynatıcı denetimi eylemlerini destekler. RTSP kullanarak, Windows Media Player 9 Series veya sonrası ya da Windows Media Services 9 Series veya sonrası çalıştırılan bilgisayarlara içerik akışı sağlayabilirsiniz.
  
**Ağ Hizmeti hesabı nedir?**  
Ağ Hizmeti hesabı, kaynaklara ve nesnelere Kullanıcılar grubundan daha fazla erişime sahip olan yerleşik bir hesaptır. Ağ Hizmeti hesabı altında çalışan hizmetler, bilgisayar hesabının kimlik bilgilerini kullanarak ağ kaynaklarına erişirler. Hesabın asıl adı NT AUTHORITY\\NETWORK SERVICE şeklindedir. Daha fazla bilgi için, [NetworkService Hesabı](http://msdn.microsoft.com/en-us/library/ms684272(vs.85).aspx) adlı MSDN makalesine bakın.
  
**Windows Media Player Ağ Paylaşımı Hizmeti nedir?**  
Windows Media Player, kitaplığınızdaki dosyaları ev ağınızdaki diğer kullanıcılarla paylaşmanıza ve ağınızdaki diğer kullanıcıların paylaştığı kitaplıklardaki dosyalara gözatmanıza olanak verir. Windows Media Paylaşımı, Windows Media Player Ağ Paylaşımı Hizmeti'ni kullanır.
  
**Windows Media Player Ağ Paylaşımı Hizmeti'nin sistemimde çalışır durumda olup olmadığını nasıl belirleyebilirim?**  
**Başlat**'ı tıklatın, **Çalıştır**'ı tıklatın, **Services.msc** yazın ve **Tamam**'ı tıklatın. Hizmetler denetim masasında, Windows Media Player Ağ Paylaşımı Hizmeti durum bilgisinin "Başladı" olup olmadığına bakın.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, etkilenen sistemin denetimini Ağ Hizmeti hesabının güvenlik bağlamında ele geçirebilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
Medya akışı sağlamak üzere Windows Media Player Ağ Paylaşımı Hizmeti (wmpnetwk.exe) kullanılan sistemler birincil derecede risk altındadır.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, Windows Media Player Ağ Paylaşımı Hizmeti'nin özel hazırlanmış RTSP paketlerini işleme biçimini düzelterek bu güvenlik açığını giderir.
