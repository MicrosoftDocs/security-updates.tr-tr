Security Bulletin

Microsoft Güvenlik Bülteni MS10-030 - Kritik
============================================

Outlook Express ve Windows Mail'deki Güvenlik Açığı Uzaktan Kod Yürütülmesine İzin Verebilir (978542)
-----------------------------------------------------------------------------------------------------

Yayım Tarihi: 11 Mayıs 2010 Salı | Güncelleştirme Tarihi: 19 Mayıs 2010 Çarşamba

**Sürüm:** 1.2

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Outlook Express, Windows Mail ve Windows Live Mail'deki özel olarak bildirilen bir güvenlik açığını giderir. Güvenlik açığı, bir kullanıcı kötü amaçlı bir e-posta sunucusunu ziyaret ederse uzaktan kod yürütülmesine olanak verebilir. Bu güvenlik açığından başarıyla yararlanan bir saldırgan, yerel kullanıcı ile aynı haklara sahip olabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.

Bu güvenlik güncelleştirmesi Microsoft Windows 2000, Windows XP ve Windows Server 2003'ün tüm desteklenen sürümleri üzerinde Microsoft Outlook Express ile Windows Vista ve Windows Server 2008'in tüm desteklenen sürümleri üzerinde Windows Mail için Kritik olarak derecelendirilmiştir. Güvenlik güncelleştirmesi Windows XP, Windows Vista, Windows Server 2008, Windows 7 ve Windows Server 2008 R2'nin tüm desteklenen sürümleri üzerinde Windows Live Mail için Önemli olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, e-posta sunucusunun yanıtlarını düzgün biçimde doğrulayarak bu güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güncelleştirmeyi hemen uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

**Bilinen Sorunlar.** Yok

Bu Güvenlik Güncelleştirmesi İle İlgili Sık Sorulan Sorular (SSS)  
-----------------------------------------------------------------
  
<span></span>
**Bu güvenlik açığı Windows 7 ve Windows Server 2008 R2 için neden Önemli olarak derecelendirilmiş?**  
Windows 7 ve Windows Server 2008 R2 üzerinde Windows Live Mail, varsayılan olarak yüklü olmayan kullanıma hazır bir bileşendir. Kullanıcının bu sorundan etkilenmesi için uygulamayı ayrı olarak yüklemesi gerekir. Ancak güvenlik açığından etkilenen kod, makinedeki inetcomm.dll dosyasının varsayılan yüklemesinde bulunmaktadır ve saldırı vektörünün bulunup bulunmadığına bakılmaksızın, bu güncelleştirme yüklenerek güvenlik açığı giderilebilir.
  
**Windows Live Mail yüklü olmadan Windows 7 veya Windows Server 2008 R2 çalıştırıyorum. Bu güvenlik güncelleştirmesi neden yine de sunuluyor?**  
Güvenlik açığı, işletim sisteminin çekirdek bileşeni olan inetcomm.dll dosyasında bulunmaktadır. Ancak saldırı vektörü olarak yalnızca Windows Live Mail kullanılabilir. Bu güvenlik güncelleştirmesi, sistemdeki güvenlik açığını tümüyle gidermek üzere kapsamlı savunma güncelleştirmesi olarak sunulmaya devam edilir.

Outlook Express ve Windows Mail'de Tamsayı Taşması Güvenlik Açığı - CVE-2010-0816  
---------------------------------------------------------------------------------
  
<span></span>
Windows Mail İstemcisi'nin özel hazırlanmış posta yanıtlarını işleme biçiminde, kimlik doğrulaması olmadan uzaktan kod yürütme güvenlik açığı bulunmaktadır. Güvenlik açığından yararlanma girişimi için kimlik doğrulaması yapılması gerekmez; bu nedenle saldırgan, ortak posta protokolleri olan POP3 ve IMAP kullanarak bir istemciye özel hazırlanmış bir yanıt göndermek yoluyla kendi denetimindeki bir sunucuya bağlantı oluşturup bu güvenlik açığından yararlanabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2010-0816](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0816).
  
#### Outlook Express ve Windows Mail'de Tamsayı Taşması Güvenlik Açığı - CVE-2010-0816 için Azaltıcı Etkenler
  
Azaltıcı etken, varolan bir durumda bulunan ve güvenlik açığından yararlanılmasının önem derecesini azaltabilecek bir ayar, genel yapılandırma ya da en iyi uygulama anlamına gelir. Sizin durumunuzda aşağıdaki azaltıcı etkenler işinize yarayabilir:
  
-   Bu güvenlik açığından başarıyla yararlanan bir saldırgan, yerel kullanıcı ile aynı haklara sahip olabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.  
-   Saldırgan bu güvenlik açığını tetiklemek üzere kötü amaçlı bir e-posta gönderemez.  
-   POP3 veya IMAP protokolleriyle birlikte SSL şifrelemesi kullanan müşteriler artık bu güvenlik açığından yararlanan ortadaki adam saldırılarından etkilenmezler, yalnızca istemci kötü amaçlı veya güvenliği aşılmış bir e-posta sunucusuna bağlanırsa bu açıktan etkilenebilirler.
  
#### Outlook Express ve Windows Mail'de Tamsayı Taşması Güvenlik Açığı - CVE-2010-0816 için Geçici Çözümler
  
Geçici çözüm, arka plandaki güvenlik açığını gidermeyen ama güvenlik açığı uygulanmadan önceki bilinen saldırı vektörlerinin engellenmesine yardımcı olan bir ayar veya yapılandırma değişikliği anlamına gelir. Microsoft aşağıdaki geçici çözümleri sınamıştır ve bir geçici çözümün işlevselliği azaltıp azaltmadığı ilgili açıklamada belirtilmektedir:
  
-   **Mümkünse Web tabanlı e-posta kullanın**
  
    E-postalarınızı denetlemek için IMAP veya POP3 yerine Web tabanlı e-posta kullanın.
  
#### Outlook Express ve Windows Mail'de Tamsayı Taşması Güvenlik Açığı - CVE-2010-0816 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir uzaktan kod yürütme güvenlik açığıdır. Bu açığı başarıyla kullanan bir saldırgan, etkilenen sistemin tüm denetimini ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir.
  
**Güvenlik açığına ne neden olmaktadır?**  
Güvenlik açığı, Outlook Express ve Windows Mail tarafından kullanılan bir ortak kitaplığın gerekli arabellek boyutunu hesaplamak üzere ağ verilerini kullanırken bu verileri yeterli düzeyde doğrulamaması nedeniyle ortaya çıkar.
  
**Windows Mail nedir?**  
Windows Mail (önceden Outlook Express olarak biliniyordu), Windows ile kullanılmak üzere geliştirilmiş bir çevrimiçi iletişim aracıdır.
  
**POP3 ve IMAP posta protokolleri nedir?**  
Postane Protokolü 3 (POP3) sunucuları, siz e-postanızı denetleyinceye kadar gelen e-posta iletilerini saklar ve e-postanızı denetlediğinizde onları bilgisayarınıza aktarır. POP3, kişisel e-postalar için en yaygın hesap türüdür. E-postanızı denetlediğinizde iletiler genelde sunucudan silinir. Internet İleti Erişim Protokolü 4 (IMAP4), uzak sunuculardaki postaları okumak ve ortak klasörlere erişmek için kullanılan bir protokoldür. İstemciler bir posta sunucusunda oturum açıp IMAP4'ü kullanarak ileti üstbilgilerini karşıdan yükleyebilir ve iletileri çevrimiçi olarak tek tek okuyabilirler.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, oturum açan kullanıcı ile aynı haklara sahip olabilir. Kullanıcı yönetimsel haklarla oturum açtıysa, saldırgan etkilenen sistemin denetimini tümüyle ele geçirebilir. Saldırgan, program yükleyebilir; verileri görüntüleyebilir, değiştirebilir veya silebilir; tüm kullanıcı haklarına sahip olan yeni hesaplar oluşturabilir. Hesapları, sistemde az sayıda kullanıcı hakkıyla yapılandırılmış olan kullanıcılar, yönetici haklarıyla çalışan kullanıcılardan daha az etkilenebilir.
  
**Bir saldırgan güvenlik açığından nasıl yararlanabilir?**  
Saldırgan kötü amaçlı bir e-posta sunucusu kurup istemciyi bu makineye bağlanmaya ikna ederek bu güvenlik açığından yararlanabilir. Daha sonra özel hazırlanmış POP3 veya IMAP yanıtı göndererek istemcinin bu güvenlik açığını tetiklemesine yol açabilir. Alternatif olarak, ortadaki adam saldırısıyla ilgili sunucu yanıtları düzenlenerek bu güvenlik açığının tetiklenmesi sağlanabilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
E-posta saklamak ve okumak için kullanılan masaüstü sistemleri bu güvenlik açığına karşı risk altındadır.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, e-posta sunucusu tarafından döndürülen yanıtları işlemeden önce düzgün şekilde doğrulayarak bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Hayır. Microsoft, bu güvenlik açığı hakkında bilgilere sorumlu kişiler aracılığıyla ulaştı. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının genel olarak duyurulduğuna dair herhangi bir bilgi almamıştı.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Microsoft, bu güvenlik açığının müşterilere saldırmak amacıyla açıkça kullanıldığına dair herhangi bir bilgi almadı. Ayrıca, bu güvenlik bülteni ilk yayımlandığında, açığın kanıtı olabilecek kod örneği yayımlandığını görmedi.
