Security Bulletin

Microsoft Güvenlik Bülteni MS12-019 - Orta
==========================================

DirectWrite Güvenlik Açığı Hizmet Reddine Olanak Verebilir (2665364)
--------------------------------------------------------------------

Yayım Tarihi: 13 Mart 2012 Salı

**Sürüm:** 1.0

### Genel Bilgiler

#### Yürütmeyle İlgili Özet

Bu güvenlik güncelleştirmesi Windows DirectWrite'ta genel olarak duyurulan bir güvenlik açığını giderir. Anlık İleti Sistemi tabanlı bir saldırı senaryosunda, bir saldırgan bir Anlık İleti Sistemi istemcisine özel hazırlanmış bir Unicode karakter sırasını doğrudan gönderirse bu güvenlik açığı hizmet reddine olanak verebilir. DirectWrite özel hazırlanmış Unicode karakter sırasını işlerken, hedeflenen uygulama yanıt vermemeye başlayabilir.

Bu güvenlik güncelleştirmesi Windows Vista, Windows Server 2008 (Itanium tabanlı sistemler için Windows Server 2008 dışında), Windows 7 ve Windows Server 2008 R2'nin tüm desteklenen sürümleri için Orta olarak derecelendirilmiştir. Daha fazla bilgi için, bu bölümdeki **Etkilenen ve Etkilenmeyen Yazılımlar** alt bölümüne bakın.

Güvenlik güncelleştirmesi, DirectWrite'ın Unicode karakterleri işleme biçimini değiştirerek bu güvenlik açığını giderir. Bu güvenlik açığı hakkında daha fazla bilgi için, bir sonraki bölüm olan **Güvenlik Açığı Bilgileri** altında, ilgili güvenlik açığı girdisinin Sık Sorulan Sorular (SSS) alt bölümüne bakın.

**Öneri.** Müşterilerin büyük bir bölümü otomatik güncelleştirmeleri etkinleştirmiş durumdadır ve bu güvenlik güncelleştirmesi otomatik olarak karşıdan yüklenip kurulacağı için herhangi bir işlem yapmaları gerekmemektedir. Otomatik güncelleştirmeleri etkinleştirmemiş olan müşterilerin güncelleştirmeleri el ile denetlemeleri ve bu güncelleştirmeyi yüklemeleri gerekir. Otomatik güncelleştirmelerin özel yapılandırma seçenekleri hakkında bilgi için, bkz. [Microsoft Bilgi Bankası makalesi 294871](http://support.microsoft.com/kb/294871).

Yöneticiler ve kurumsal yüklemeler ya da bu güvenlik güncelleştirmesini el ile yüklemek isteyen son kullanıcılar için, Microsoft, müşterilerinin güncelleştirme yönetim yazılımlarını kullanarak veya [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) hizmeti aracılığıyla güncelleştirmeleri denetleyerek güvenlik güncelleştirmesini hemen uygulamalarını önerir.

Bu bültenin sonrasında yer alan **Algılama ve Dağıtım Araçları ve Kılavuzu** bölümüne de bakın.

DirectWrite Uygulamasında Hizmet Reddi Güvenlik Açığı - CVE-2012-0156  
---------------------------------------------------------------------
  
<span></span>
DirectWrite'ın özel hazırlanmış Unicode karakter sırasını işleme biçiminde bir hizmet reddi güvenlik açığı bulunmaktadır. Bu güvenlik açığından başarıyla yararlanan saldırgan, hedeflenen uygulamanın yanıt vermemesine neden olabilir.
  
Bu güvenlik açığını Ortak Güvenlik Açıkları ve Etkilenmeler listesinde standart girdi olarak görüntülemek için, bkz: [CVE-2012-0156](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2012-0156).

Microsoft bu güvenlik açığı için herhangi bir azaltıcı etken belirleyememiştir.
  
#### DirectWrite Uygulamasında Hizmet Reddi Güvenlik Açığı - CVE-2012-0156 için Geçici Çözümler

#### DirectWrite Uygulamasında Hizmet Reddi Güvenlik Açığı - CVE-2012-0156 Hakkında SSS
  
**Bu güvenlik açığının kapsamı nedir?**  
Bu bir hizmet reddi güvenlik açığıdır.
  
**Güvenlik açığına ne neden olmaktadır?**  
Güvenlik açığı, DirectWrite'ın özel hazırlanmış Unicode karakter sırasını bellekte düzgün işlememesi nedeniyle ortaya çıkar.
  
**DirectWrite** **nedir?**  
DirectWrite; metnin ölçülmesini, çizilmesini ve metne tıklatma testi uygulanmasını destekleyen, çözünürlüğe bağlı olmayan, tam Unicode, birden çok biçimde metin işleme API'sidir. DirectWrite hakkında daha fazla bilgi için, [DirectWrite'a Giriş](http://msdn.microsoft.com/en-us/library/windows/desktop/dd371554(v=vs.85).aspx) adlı MSDN makalesine bakın.
  
**Bir saldırgan bu güvenlik açığından yararlanarak neler yapabilir?**  
Bu güvenlik açığından başarıyla yararlanan bir saldırgan, Windows Live Messenger gibi hedeflenen bir uygulamanın yanıt vermemeye başlamasına neden olabilir.
  
**Bir saldırgan bu güvenlik açığından** **nasıl yararlanabilir?**  
Web tabanlı saldırı senaryosunda, kimliği doğrulanmamış bir saldırganın özel hazırlanmış bir Unicode karakter sırasının bulunduğu bir web sitesi barındırması gerekir. Ayrıca, kullanıcı tarafından sağlanan içeriği kabul eden ya da barındıran güvenliği aşılmış web sitelerinde, bu güvenlik açığından yararlanmak için kullanılabilecek özel hazırlanmış içerik bulunabilir. Güvenlik açığından yararlanılabilmesi için, özel hazırlanmış Unicode karakter sırasının, web içeriğini DirectWrite kullanarak işleyen bir uygulama tarafından ayrıştırılması gerekir. DirectWrite özel hazırlanmış Unicode karakter sırasını işlerken, etkilenen uygulama yanıt vermemeye başlayabilir. Ancak bu durumların hiçbirinde, saldırganın kullanıcıları özel hazırlanmış bir Unicode karakter sırasının bulunduğu bir web sitesini ziyaret etmeye zorlama yolu yoktur. Bunun yerine, saldırganın kullanıcıları bir e-posta iletisindeki veya Instant Messenger iletisindeki kendi web sitesine yönlendiren bir bağlantıyı tıklatmalarını sağlayarak onları bu web sitesini ziyaret etmeye ikna etmesi gerekir.
  
Anlık İleti Sistemi tabanlı bir saldırı senaryosunda, kimliği doğrulanmamış bir saldırgan Windows Live Messenger gibi bir Anlık İleti Sistemi istemcisine özel hazırlanmış bir Unicode karakter sırası göndererek bu güvenlik açığından yararlanabilir. DirectWrite özel hazırlanmış Unicode karakter sırasını işlerken, hedeflenen uygulama yanıt vermemeye başlayabilir.
  
**Bu güvenlik açığından birincil derecede hangi sistemler etkileniyor?**  
İş istasyonları ve terminal sunucuları bu güvenlik açığından birincil derecede etkilenir. Yöneticiler kullanıcıların sunucularda oturum açmalarına ve program çalıştırmalarına izin verirse sunucular daha fazla etkilenebilir. Ancak, en iyi uygulamalarda buna izin verilmesi kesinlikle önerilmemektedir.
  
**Güncelleştirme ne yapıyor?**  
Güncelleştirme, DirectWrite'ın Unicode karakterleri işleme biçimini değiştirerek bu güvenlik açığını giderir.
  
**Bu güvenlik bülteni yayımlandığında bu güvenlik açığı genel olarak duyurulmuş muydu?**  
Evet. Bu güvenlik açığı genel olarak duyurulmuştur. Ortak Güvenlik Açığı ve Etkilenme (Common Vulnerability and Exposure) numarası olarak [CVE-2012-0156](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2012-0156) atanmıştır.
  
**Bu güvenlik bülteni yayımlandığında, Microsoft bu güvenlik açığından yararlanıldığına dair herhangi bir rapor almış mıydı?**  
Hayır. Bu güvenlik bülteni ilk yayımlandığında, Microsoft bu güvenlik açığının müşterilere saldırmak üzere genel olarak kullanıldığına dair herhangi bir bilgi almamıştı
